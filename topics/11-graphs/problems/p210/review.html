範例 Trace// DIAMOND DAG

// 輸入 numCourses = 4 prerequisites = [[1,0],[2,0],[3,1],[3,2]] 修 1 前先修 0 → 邊 0→1 修 2 前先修 0 → 邊 0→2 修 3 前先修 1 → 邊 1→3 修 3 前先修 2 → 邊 2→3 圖: 0 →(1, 2)→ 3 (鑽石形 DAG,無環)
// A · DFS 後序(從 0 出發) 0 →1 →3 3 無後代,最先完成 finish=[3] 回到 1,後代 3 已完成 finish=[3,1] 回到 0,走另一支 0→2(→3 已完成) finish=[3,1,2] 0 的後代全完成,最後完成 0 finish=[3,1,2,0] 反轉 → [0,2,1,3] ← 0 在最前、3 在最後
// B · Kahn 入度 indeg = [0,1,1,2] 只有 0 是 0 → queue=[0] pop 0 → order=[0];1,2 各 −1 → 都 0 → queue=[1,2] pop 1 → order=[0,1];3: 2→1(未 0) pop 2 → order=[0,1,2];3: 1→0 → queue=[3] pop 3 → order=[0,1,2,3] size==4 → [0,1,2,3]
兩條答案 [0,2,1,3] 與 [0,1,2,3] 都合法 —— 因為 1 和 2 之間沒有依賴,誰先誰後都行。拓撲序常常不唯一,LeetCode 兩個都收。共同點:0 一定最前、3 一定最後,那才是真正被依賴關係鎖死的部分。

應用題// APPLY

App 1. 若把 210 改成 207(只問能不能修完),DFS 和 Kahn 各要拿掉/改哪一行?
App 2. 若要在有多個合法解時回傳字典序最小的順序,Kahn 要怎麼改?(提示:把 queue 換成什麼?)
App 3. 若題目保證無環,DFS 版可以省掉什麼?還需要三態嗎?
// ANSWERS
A1. 後序是「後代全排完才輪到自己」——所以在 finish 裡,一個節點永遠排在它所有後代之後。但拓撲序要的是「前置(被依賴者)在前」,也就是節點要排在後代之前,方向剛好相反。反轉 finish 後,每個節點就跑到它後代前面,正好是拓撲序。不反轉的話拿到的是反拓撲序(把依賴倒著讀)。
A2. 兩態分不清「正在當前遞迴路徑上」和「早就完成、只是又被別人指到」。DAG 裡一個節點被多條路徑共享很正常(菱形的 3 就被 1、2 都指到),兩態會把「第二次遇到已完成的 3」誤當成環。三態用 1=進行中 專門標「在當前 DFS 堆疊上」,只有再次踩到 1 才是真的繞回來 = 環;踩到 2=完成 只是重複到達,直接 return 不算環。
A3. Kahn 靠「indeg 歸零才入隊」推進。環上的節點互相是彼此的前置——A 等 B、B 等 A(或更長的圈),沒有人能先歸零,於是誰都進不了佇列,永遠留在環裡。佇列空掉時,這些節點都沒被 pop 進 order,所以 order.size() < numCourses,以此判環、回空。
A4. 建 b→a:[a,b] 意思是「修 a 前先修 b」,b 是前置、要先上,所以箭頭從前置指向後修(b 指 a)。建反成 a→b 會讓整個拓撲序顛倒(輸出反過來)。一眼判斷法:箭頭方向 = 上課的時間順序,先上的在箭尾、後上的在箭頭;Kahn 的 indeg[a]++ 也印證 a 有一個前置。
A5. 都對。DAG 的拓撲序本來就常常不唯一:只要兩個節點之間沒有(直接或間接)依賴,它們的相對順序就自由。DFS 反轉和 Kahn 只是用不同規則在這些「自由度」上做了不同選擇,但都不違反任何一條依賴,所以都是合法拓撲序,LeetCode 用特判接受任一組。
App 1. DFS:不需要 order 了,hasCycle 只判有沒有環,主函式 for i: if hasCycle(i) return false; return true;(不必反轉、不必 push)。Kahn:不必收 order 陣列,只要一個計數 cnt,每 pop 一個 cnt++,最後 return cnt == numCourses;。核心的建圖、三態/入度邏輯完全一樣——這正是 207 與 210 的關係:同一套骨架,一個回 bool、一個回順序。
App 2. 把普通 queue 換成最小堆(priority_queue / min-heap)。每次不是取「任意一個 indeg==0」,而是取編號最小的那個入 order,如此在每一步的可選集合裡都挑最小,得到字典序最小的拓撲序。代價是每次取出從 O(1) 變 O(log V),總複雜度 O(V log V + E)。
App 3. 保證無環時可以省掉判環:DFS 不必再區分「進行中」和「完成」——只要一個 visited 布林(兩態)防重複遞迴即可,不會有繞回自己的情況。後序 + 反轉照舊。Kahn 同理可以省掉最後 size() == numCourses 的檢查(必然相等)。三態只是為了判環而存在,無環就用不上。
