題目簡介// THE QUESTION

QUESTION

numCourses 門課(0..n−1)。prerequisites[i] = [a, b] 表示要上 a 得先上 b。 回傳一個能修完所有課的上課順序;若因為互相依賴成環而無解,回空陣列。

把課當節點、依賴 b→a 當有向邊(b 排在 a 前),就是要在 DAG 上輸出一個拓撲序:任一條邊的起點都排在終點之前。 有環就不存在這種順序 → 回空。兩種標準解法:DFS 後序反轉 與 BFS / Kahn(入度)。

2

為什麼用拓撲排序// WHY TOPO SORT

A 隨便輸出一個順序 違反依賴
沒有依照「先修在前」,任一條 b→a 被違反都不合法;也無法偵測環。
× REJECT · 錯
B 反覆掃找「前置都上完」的課 O(V²)
每輪掃全部找可上的課加入、更新。可行但每輪 O(V)、共 V 輪 → 平方,慢。
△ REJECT · 慢
ADOPT
C 拓撲排序:DFS 後序 或 BFS Kahn O(V+E)
① DFS 後序:每個節點的後代都排完才加入 finish,最後反轉 = 拓撲序;用三態 visit 判環。 ② BFS / Kahn:把 indeg==0 的入隊,逐個出隊加入 order、後繼 indeg−1、歸零就入隊。兩者皆 O(V+E)。
● ADOPT · O(V+E)
3

核心策略// STRATEGY

  1. 建圖:邊 b→a(先修 b 指向 a)。DFS 版用三態 visit;BFS 版另記 indeg[a]++。
  2. A · DFS 後序:對每個節點 dfs:遇到「進行中(1)」的節點 → 有環回空;跑完所有後代後把自己 push 進 finish、標「完成(2)」。全部跑完把 finish 反轉。
  3. B · BFS / Kahn:indeg==0 入隊;出隊加入 order、後繼 indeg−1、歸零入隊。最後 order.size()==numCourses 才合法,否則有環回空。

兩種寫法的核心// TWO CORES

LEMMA · 後序反轉 & Kahn 入度

DFS 後序:一個節點在所有後代都完成後才被 push,所以在 finish 裡它排在後代之後。反轉後,它就排在後代之前 —— 正是拓撲序要的「前置在前」。判環靠三態:0 未訪、1 在當前遞迴路徑上、2 完成;若走到 1 代表回到自己路徑上的節點 = 有環。

BFS / Kahn:indeg = 還有幾門「前置」沒上。indeg==0 表示現在就能上。上完一門就把它的後繼各減一個前置;歸零者接著能上。環上的節點入度永遠減不到 0、進不了佇列 → 最後 order 長度 < numCourses → 判定有環。

複雜度// COMPLEXITY

T = O(V + E)　　S = O(V + E)

V = numCourses、E = prerequisites 數。每個節點、每條邊各處理一次。空間為鄰接表 + visit/indeg + 遞迴堆疊或佇列。
