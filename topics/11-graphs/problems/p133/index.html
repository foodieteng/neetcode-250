題目簡介// THE QUESTION

QUESTION

給一個連通無向圖裡某個節點的參考,回傳這張圖的深拷貝(deep copy)。 每個 Node 有 val 和鄰居串列 neighbors。複製後的圖必須是全新的節點,但結構(連線)與原圖一模一樣。

難點是圖有環:直接遞迴複製會在環上無限繞,或把同一個節點複製很多份、連線接錯。 解法是用一張 map(原節點 → 新節點):走訪時每個節點只複製一次,遇到已複製過的節點就從 map 取用,不再新建。

2

為什麼一定要 map// WHY THE MAP

A 只複製 val,共用原鄰居 不是深拷貝
若新節點的 neighbors 還指向原圖的節點,就不是獨立的複本 —— 改動會互相影響。必須連鄰居都是新節點。
× REJECT · 假複製
B 遞迴複製,但不記錄 死圈
圖有環:複製 A 要先複製鄰居 B,B 又要複製 A…… 無限遞迴;就算不爆,也會把同一節點複製成很多份。
× REJECT · 無限迴圈
ADOPT
C DFS/BFS + map(原→新) O(V+E)
走訪圖;每個節點第一次遇到才新建複本並存進 map[原]=新,已在 map 就直接回傳既有複本。 這樣每個節點恰好複製一次,環上的回邊自動接到正確的既有複本,不死圈、不重複。
● ADOPT · O(V+E)
3

核心策略// STRATEGY

  1. node 為空 → 回 nullptr。準備一張 map(原節點 → 新節點)。
  2. dfs(node):若 node 已在 map → 直接回傳 map[node](關鍵:切斷環、避免重複)。
  3. 否則先建複本 copy、立刻存 map[node]=copy,再對每個鄰居遞迴 copy.neighbors.push_back(dfs(nei))。回傳 copy。

Follow-up · 空間能到 O(1) 嗎// CAN SPACE BE O(1)?

注意 · 「O(1) 空間」到底指什麼

真正的 O(1) 總空間,對一般圖是做不到的。 任何圖走訪都得記「哪些點訪過」,那本身就是 O(V)(遞迴堆疊或 BFS 佇列)。這和 138 複製含隨機指標的鏈結串列不同:鏈結串列靠 next 免費列舉所有節點,才能用「穿插複本」做到真 O(1);圖沒有這種免費列舉。

所以這裡的「O(1)」= 省掉 hashmap。 因為題目保證 val 唯一且 1..100,可把 unordered_map(隨 V 長大 → O(V))換成固定 101 格、以 val 為索引的陣列 Node* clone[101]:大小與節點數無關 → O(1)。

結論:map 結構可壓到 O(1),但走訪的堆疊 / 佇列仍是 O(V),無法再省 —— 這正是這題和 138 的關鍵差異。

複雜度// COMPLEXITY

T = O(V + E)　　S = O(V)

每個節點複製一次、每條邊看一次,O(V+E)。空間:map O(V) + 走訪堆疊 O(V)。Follow-up 把 map 降到 O(1),但走訪仍 O(V)。

一句洞察// INSIGHT

複製有環的圖,靠一張「原→新」的 map:每點只建一次、回邊重用既有複本。 map 可用 val 索引壓到 O(1),但圖走訪的 O(V) 省不掉。
