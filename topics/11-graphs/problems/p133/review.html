重點回顧// RECAP

面向要點
問題本質有環的圖的深拷貝(全新節點、結構相同)
核心一張 map 原→新:每點只建一次、回邊重用
關鍵順序先 map[node]=copy 再遞迴鄰居(切斷環)
Follow-upval 唯一≤100 → map 換固定 101 陣列 → O(1)
注意真 O(1) 總空間做不到;走訪必須 O(V)
複雜度時間 O(V+E),空間 O(V)(A)/ map O(1)(B)

範例 Trace// [[2,4],[1,3],[2,4],[1,3]] → 4-環

// 原圖 · 4 節點成環 1—2 1 鄰居 {2,4} 2 鄰居 {1,3} | | 3 鄰居 {2,4} 4 鄰居 {1,3} 4—3 (正方形四邊,有環)
// DFS(1) · 先存 map 再遞迴 dfs(1):新建 1',map[1]=1'。走鄰居: dfs(2):新建 2',map[2]=2'。走鄰居: dfs(1):1 已在 map → 回傳 1'(不新建、切斷環) dfs(3):新建 3',map[3]=3'。走鄰居: dfs(2):2 已在 map → 回傳 2' dfs(4):新建 4',map[4]=4'。走鄰居 1、3 皆在 map → 回傳既有 dfs(4):4 已在 map → 回傳 4'
// RESULT map = {1→1', 2→2', 3→3', 4→4'};每個原節點恰好一個新節點 新圖連線:1'-2'-3'-4'-1'(與原圖相同,全是新節點) // OUTPUT: 1'(新圖的起點)
整個過程每個節點只「新建」一次(共 4 次),其餘遇到都從 map 取用。 map 就是「切斷環、避免重複」的那把鑰匙 —— 拿掉它,環會讓遞迴永遠回不了頭。

應用題// APPLY

App 1. 把 DFS 換成 BFS(queue) 怎麼寫?先做什麼、迴圈裡做什麼?
App 2. 若 val 不唯一或範圍很大(例如 10⁹),follow-up 的「val 陣列」還能用嗎?退回什麼做法?
App 3. 若圖不保證連通(可能有多個分量,給你所有節點的列表),複製流程要改哪裡?
// ANSWERS
A1. map 做兩件事:① 去重 —— 每個原節點只對應一個新節點,不會複製出很多份;② 切斷環 —— 遇到已複製的節點直接回傳既有複本,不再往下遞迴。沒有 map,圖有環時複製 A 要先複製 B、複製 B 又要複製 A……無限遞迴爆堆疊;就算僥倖不爆,也會產生重複節點、連線接錯。
A2. 因為要在遞迴鄰居之前就讓 node 在 map 裡「掛號」。假設環 A→B→A:若先遞迴 B、還沒存 A,則複製 B 時走到鄰居 A,發現 A 不在 map → 又新建一個 A → 再遞迴 B → 死圈。先存後遞:B 遞迴回來看到 A 已在 map,直接取用,環被切斷。這一行順序是正確性的關鍵。
A3. 深拷貝 = 複製出的圖是完全獨立的一份,改動它不影響原圖。若新節點的 neighbors 還指向原圖的節點,那新舊兩圖共用了部分結構 —— 改新圖會牽動原圖,這只是「淺拷貝」。所以每個鄰居也必須換成 dfs(nei) 產生的新節點,整張圖的節點與邊都是全新的。
A4. 省掉的是 map 這個資料結構:從隨節點數長大的 unordered_map(O(V))換成固定 101 格、以 val 為索引的陣列(O(1),靠 val 唯一且 ≤100)。但走訪本身要記錄「訪過哪些節點」(遞迴堆疊 / BFS 佇列 / visited),這至少 O(V),省不掉。所以總空間仍是 O(V);「O(1)」只針對 map 結構。
A5. 138 是鏈結串列,有 next 指標可以免費、線性地列舉所有節點,所以能用「把複本穿插在原節點之後(A→A'→B→B'…)」的技巧,不需額外 map、不需 visited → 真 O(1) 額外空間。133 是一般圖,沒有這種天然的線性列舉,任何走訪都得靠堆疊/佇列/visited 記錄進度,至少 O(V)。這就是「圖不能像鏈結串列那樣真 O(1)」的根本原因。
App 1. 先處理起點:map[node]=new Node(node->val),把 node 入佇列。迴圈:取出 cur,對每個鄰居 nei —— 若 nei 不在 map,新建其複本並入佇列;然後把 map[nei] 接到 map[cur]->neighbors。佇列空了就完成。BFS 用佇列取代遞迴堆疊,適合超深的圖避免 stack overflow。
App 2. 不能用。val 陣列的前提是「val 唯一、範圍小(當索引)」。若 val 重複,不同節點會撞同一格;若範圍到 10⁹,陣列開不下。這時退回用 unordered_map<Node*, Node*>(以節點指標為鍵,永遠唯一、不受 val 影響),空間回到 O(V) —— 也就是主解法 A。
App 3. 單一起點的 DFS/BFS 只能走遍一個連通分量。若不保證連通、且給你所有節點的列表,就外層再包一層迴圈:對每個節點,若它還沒在 map 裡,就從它啟動一次複製(共用同一張 map)。這樣每個分量各被複製一次,和「島嶼數量」外層掃描的思路相同。
