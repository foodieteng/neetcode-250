題目簡介// THE QUESTION

QUESTION

n 個節點(0..n−1)、一批無向邊 edges。 問這張圖是不是一棵合法的樹?回傳 true / false。

一棵樹的充要條件:恰好 n−1 條邊 且 全連通(等價於無環) —— 兩者缺一不可。 用 並查集(Union-Find):逐條邊合併兩端;若一條邊的兩端已在同一組,代表成環 → 不是樹。

2

為什麼是這兩個條件// TWO CONDITIONS

A 只檢查「連通」 不夠
連通但邊太多 → 有環,不是樹(例:三角形)。少了邊數條件會漏判。
× REJECT · 漏環
B 只檢查「n−1 條邊」 不夠
邊數對但可能「一邊成環、另一邊斷開」→ 不連通,不是樹。
× REJECT · 漏斷
ADOPT
C n−1 條邊 且 union 全部成功(無環) α(n)≈O(1)
先擋 edges.size() != n−1。再逐條 union:每次成功代表接上「新的一組」。若某條邊兩端已同根 → 成環 → 非樹。 成功次數 == n−1 ⟺ 無環且連通 = 樹。
● ADOPT · 近乎 O(1)/邊
3

核心策略// STRATEGY

  1. 前置檢查:edges.size() != n−1 直接回 false(邊數不對必不是樹)。
  2. 並查集:parent[x] < 0 代表 x 是根,−parent[x] = 這棵樹的 size。
  3. Find(路徑壓縮):parent[x] = Find(parent[x]),把沿途節點直接掛到根。
  4. Union(按 size):小樹掛到大樹下,回傳是否真的合併(同根則 false)。
  5. 判定:成功合併次數 add == n−1 ⟺ 合法樹。

兩個關鍵優化// TWO OPTIMIZATIONS

LEMMA · 路徑壓縮 + 按 size 合併

路徑壓縮(Find):parent[x] = Find(parent[x]) —— 找根的同時把沿途每個節點直接改指向根,下次查詢一步到位,樹被壓扁。

按 size 合併(Union):用 parent[root] 存負的 size(−size)。合併時把小樹掛到大樹下,避免樹長歪、變高。兩者合用後,每次操作攤還近乎 O(1)(反阿克曼 α(n),實務上 ≤ 4)。

小技巧:用一個陣列 parent 同時當「父指標」和「size」——負值表示自己是根、絕對值是 size;非負值表示指向的父節點。省一個 size[] 陣列。

6

複雜度// COMPLEXITY

T = O(E · α(n)) ≈ O(E)　　S = O(n)

α(n) 是反阿克曼函數,實務上 ≤ 4,視為常數。每條邊一次近乎 O(1) 的 union。空間只需一個 parent 陣列。

7

一句洞察// INSIGHT

樹 = n−1 條邊 + 無環。並查集逐邊合併:成功 n−1 次 ⟺ 無環且連通。 「兩端已同根的邊」就是環的訊號。路徑壓縮 + 按 size 讓每步近乎 O(1)。
