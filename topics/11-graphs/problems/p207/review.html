範例 Trace// CYCLE CASE

// 輸入(有環) numCourses = 4 prerequisites = 讓圖成為 3→0, 0→1, 1→2, 2→0 0,1,2 互相依賴繞成一個環;3 指向 0 預期輸出:false(0,1,2 永遠修不完)
// A · DFS 三態(從 0 出發) dfs(0): visit[0]=1 stack=[0] dfs(1): visit[1]=1 stack=[0,1] dfs(2): visit[2]=1 stack=[0,1,2] 2 的鄰居是 0,visit[0]==1 ← 還在堆疊上! → 繞回自己路徑 = 環 → return true canFinish = false
// B · Kahn 入度 indeg = [2,1,1,0] 只有 3 是 0 → queue=[3] pop 3 → done=1;0: indeg 2→1(未 0) queue 空 0,1,2 互為前置:0 等 2、2 等 1、1 等 0 → indeg 減不到 0、進不了佇列 done=1 ≠ 4 → false
兩種解法從不同角度撞上同一個環:DFS 是「沿路徑走回自己」,Kahn 是「剝到剝不動、還有剩」。 節點 3 雖然能被處理(它不在環上),但環裡的 0,1,2 誰都動不了 —— 這正是「修不完」的具體樣子。

應用題// APPLY

App 1. 若把 207 改成 210(回傳一個合法上課順序),DFS 和 Kahn 各要多做哪一步?
App 2. 若還要回傳環上到底有哪些課(不只是有沒有環),Kahn 版怎麼撈出來?
App 3. 若題目改成問「兩門課 u、v 之間有沒有先後依賴關係(可傳遞)」,這還是判環嗎?該用什麼?(提示:傳遞閉包 / 課程表 IV)
// ANSWERS
A1. 上課有「先修」的偏序:要上 A 得先上它的所有前置。若依賴形成環(A 要先上 B、B 又要先上 A,或更長的圈),那圈上每一門都在等別人先上,誰都無法成為第一門,永遠開不了工。反之沒有環時,依賴關係是一個 DAG,一定存在合法順序、全部能上完。所以「能修完」與「無環」完全等價。
A2. 兩態分不清「正在當前遞迴路徑上」和「早就完成、只是又被別人指到」。在 DAG 裡一個節點被多條路徑共享是正常的(例如菱形的底點被兩邊都指到),兩態會把「第二次到達已完成節點」誤當成環而回錯答案。三態用 1 專門標「在當前 DFS 堆疊上」,只有再次踩到 1 才是真的沿路徑繞回來 = 環;踩到 2 只是重複到達,直接 return 不算。
A3. Kahn 每輪只能剝掉 indeg==0(前置全上完)的課。環上的節點互相是彼此的前置 —— A 等 B、B 等 A —— 沒有人能先歸零,於是全都卡在環裡、永遠進不了佇列。佇列空掉時這些節點都沒被剝出,所以 done < numCourses,以此判環。
A4. 因為圖可能不連通,或環藏在從 0 走不到的地方。如果只從 0 出發 DFS,走不到的節點永遠不會被檢查,那裡的環就漏判了。對每個 i 都當起點能保證每個節點至少被進入一次;已完成(state 2)的會立刻 return,不會重複做功,所以總複雜度仍是 O(V+E)。
A5. 骨架完全一樣,差別只在「要不要收集順序」。DFS:210 在完成節點時 order.push_back(u),最後 reverse;207 不必收、只判環回 bool。Kahn:210 在出隊時 order.push_back(u),最後檢查 order.size()==numCourses 並回傳 order;207 只用一個 done 計數、回 done==numCourses。學會一題就等於學會兩題。
App 1. DFS:在 visit[u]=2 之後加 order.push_back(u)(後序),主函式最後 reverse(order) 回傳。Kahn:把 done 換成 order 陣列,出隊時 order.push_back(u),最後 return order.size()==numCourses ? order : {}。兩者都只是「順手把出場順序記下來」。
App 2. 跑完 Kahn 後,凡是 indeg[i] > 0 的節點就在某個環上(或被環擋住上不了)。掃一遍 indeg 把還大於 0 的收集起來即可。若要精確找出環本身,可在 DFS 版用「當前路徑堆疊」在偵測到 visit==1 時把堆疊上從該點到頂的區段切出來,那就是環。
App 3. 那就不是判環了,而是求可達性 / 傳遞閉包:u 是否能透過依賴鏈到達 v。做法是對每個節點跑 BFS/DFS 標記可達集合,或用 Kahn 拓撲順序 + 沿途合併前驅集合(bitset)一次算完所有配對 —— 這正是 1462 課程表 IV 的做法,把「有沒有環」升級成「誰能到誰」。
