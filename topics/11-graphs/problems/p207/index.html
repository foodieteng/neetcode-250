題目簡介// THE QUESTION

QUESTION

numCourses 門課(0..n−1)。prerequisites[i] = [a, b] 表示要上 a 得先上 b。 問能不能把所有課都修完?回傳 true / false。

把課當節點、依賴當有向邊,「能修完所有課」就等價於「圖裡沒有環」 —— 有環代表 A 要先修 B、B 又要先修 A,永遠卡死。 所以本題核心是有向圖判環:用 DFS 三態 或 BFS / Kahn(入度)。這正是 210 課程表 II 的「只問能不能、不用回順序」版本。

2

為什麼是判環// WHY CYCLE DETECTION

A 只看「有沒有課沒前置」 不充分
有起點不代表能全部上完;環可能藏在圖的深處,單看入口判不出來。
× REJECT · 錯
B 兩態(訪過/沒訪)DFS 判環 誤判
DAG 裡一個節點被多條路徑共享很正常,兩態會把「重複到達」誤當成環。
△ REJECT · 誤判
ADOPT
C DFS 三態判環 或 BFS Kahn 計數 O(V+E)
① DFS 三態:0 未訪 / 1 在當前路徑上 / 2 完成;走回 1 = 環。 ② BFS / Kahn:indeg==0 入隊逐個剝,若剝出的數量 < numCourses 就有環。兩者皆 O(V+E)。
● ADOPT · O(V+E)
3

核心策略// STRATEGY

  1. 建圖:邊 b→a(先修 b 指向 a)。DFS 版用三態 visit;BFS 版另記 indeg[a]++。
  2. A · DFS 三態:對每個節點 dfs:走回「進行中(1)」的節點 → 有環回 false;跑完所有後代標「完成(2)」。全部沒環 → true。
  3. B · BFS / Kahn:indeg==0 入隊,逐個出隊、done++、後繼 indeg−1、歸零入隊。最後 done==numCourses 才無環。

兩種判環的核心// TWO CORES

LEMMA · 三態 & 入度剩料

DFS 三態:關鍵在區分「還在當前遞迴路徑上(1)」和「早就完成、只是又被指到(2)」。DAG 裡節點被多條路徑共享很正常,只有沿當前路徑繞回一個 state 1 的節點才是真環。少了這個區分(只用兩態)就會把普通的重複到達誤判成環。

BFS / Kahn:每輪只剝掉 indeg==0(沒有未上前置)的課。環上的節點互為前置 —— 你等我、我等你 —— indeg 永遠減不到 0,誰都進不了佇列。所以剝完後 done < numCourses 就代表有一群課卡在環裡,修不完。

複雜度// COMPLEXITY

T = O(V + E)　　S = O(V + E)

V = numCourses、E = prerequisites 數。每個節點、每條邊各處理一次。空間為鄰接表 + visit/indeg + 遞迴堆疊或佇列。
