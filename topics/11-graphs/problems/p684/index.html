題目簡介// THE QUESTION

QUESTION

一棵有 n 個節點的樹,被多加了一條邊,變成一張有 n 個點、n 條邊的圖(恰好一個環)。 找出那條可以刪掉、讓它變回樹的邊;若有多解,回傳在輸入中最後出現的那條。

n 點 + n 邊 → 剛好一個環。用 並查集照順序逐邊 union: 第一條「兩端已經連通」的邊,就是把環閉合的那條 = 答案。因為照順序掃,第一條閉環的邊自然就是最後出現的多餘邊。

2

核心觀察// KEY IDEA

A 每加一條邊就 DFS 判環 O(n²)
對每條邊重跑一次找環,可行但重複遍歷整圖,慢。
△ REJECT · 慢
觀察 邊連的兩端「本來就連通」 = 閉環
若 u、v 已在同一連通塊,它們之間已有路徑;再加 (u,v) = 多一條 = 環。
◆ 洞察
ADOPT
C Union-Find:第一條 union 失敗的邊 O(n·α)
照輸入順序逐邊 Union(u,v)。若某條回 false(find(u)==find(v),已同組)→ 它就是把環閉合的邊 → 直接回傳。
● ADOPT · O(n·α)
3

核心策略// STRATEGY

  1. 並查集初始化:節點 1..n,parent 全 −1(各自為根)。
  2. 依序逐邊:對每條 (u,v) 呼叫 Union(u,v)。
  3. 抓第一條失敗:Union 回 false(兩端已同根)→ 立刻 return {u,v}。
  4. 因為照順序處理,第一條閉環的邊就是題目要的「最後出現的多餘邊」。

5

為什麼「第一條失敗」就是答案// WHY

LEMMA · 順序掃描 + 唯一環

圖有 n 點 n 邊,恰好一個環。並查集逐邊合併時,只要兩端還在不同塊,這條邊就是把兩塊接起來的「有效樹邊」;直到某條邊的兩端已在同一塊 —— 那代表它們間已存在路徑,這條邊閉合了環。

由於我們照輸入順序掃,遇到的第一條閉環邊,正是「在環上、且輸入位置最後」的那條(環上較早的邊都還在當有效樹邊被接上)。所以直接回傳它,既是能刪的邊、也滿足「多解取最後出現」的要求。

6

複雜度// COMPLEXITY

T = O(n · α(n)) ≈ O(n)　　S = O(n)

每條邊一次近乎 O(1) 的 union。空間只需一個 parent 陣列。

7

一句洞察// INSIGHT

多餘邊 = 讓已連通兩點又多一條路的那條。並查集照順序 union, 第一條失敗(同根)的邊就是答案。這正是 261 判環 的「回傳那條環邊」版。
