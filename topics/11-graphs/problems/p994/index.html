題目簡介// THE QUESTION

QUESTION

m×n 網格:0 = 空格、1 = 新鮮橘子、2 = 腐爛橘子。 每一分鐘,任何與腐爛橘子上下左右相鄰的新鮮橘子也會腐爛。回傳讓所有橘子腐爛所需的最少分鐘數;若有橘子永遠爛不到,回 -1。

腐爛從多個腐爛橘子同時往外一圈一圈擴散 —— 這正是多源 BFS。把所有腐爛橘子一起入隊當起點, 一波 = 一分鐘;每擴一層就是過了一分鐘。用一個 fresh 計數追蹤還剩幾顆新鮮,歸零那刻的分鐘就是答案;若佇列空了 fresh 仍 > 0 → 有橘子爛不到 → -1。

2

為什麼用多源 BFS// WHY MULTI-SOURCE

A 對每顆新鮮橘子 BFS 找最近腐爛 O((mn)²)
每顆新鮮各自 BFS 找最近的腐爛源,答案取最大。平方級,TLE,也不直觀。
× REJECT · TLE
B 逐分鐘掃全網格模擬腐爛 易錯
每分鐘掃整格找相鄰新鮮變腐爛。可行,但要小心「這分鐘新爛的不能立刻再傳染」,得用標記分批,容易寫錯。
△ REJECT · 繁瑣
ADOPT
C 多源 BFS(所有腐爛一起入隊) O(mn)
先把所有腐爛橘子入隊,再一層一層 BFS:每處理完一整波就是過一分鐘(用 sz = q.size() 分批)。 fresh 歸零那刻回傳分鐘;佇列空了還有 fresh → -1。一趟 O(mn)。
● ADOPT · ONE PASS
3

核心策略// STRATEGY

  1. 掃一遍:所有 2(腐爛)入隊當源;數 1(新鮮)的總數 fresh。若 fresh==0 直接回 0。
  2. 逐層 BFS:while 佇列非空,每輪先取 sz = q.size(),用內層迴圈一次清完這一波;把相鄰新鮮 1 變 2、fresh--、入隊。
  3. 計分鐘:每清完一波 time++;fresh 歸零時回傳當下分鐘(time+1)。佇列空仍 fresh>0 → 回 -1。

和 286 的關鍵差別// VS 286

LEMMA · 為什麼這題要「數波次」

994 和 286 牆與門是同一個引擎(多源 BFS),差別在要回傳什麼: 286 要每格的距離值,把距離寫回格子(rooms[r][c]+1)就好,不必數層;994 要的是總共幾分鐘,是「BFS 擴散了幾層」,所以需要 sz = q.size() 把同一波一次清完、每清一波 time++。

fresh 計數是判 -1 的關鍵。 BFS 只會擴散到「與腐爛連通」的新鮮橘子;若有橘子被牆/空格隔開,佇列空時它仍是 1,fresh>0 → 永遠爛不到 → 回 -1。少了 fresh 就無法區分「全爛」和「有孤島」。

複雜度// COMPLEXITY

T = O(m·n)　　S = O(m·n)

每格最多入隊、出隊一次(腐爛時),鄰居檢查是常數。總工作量與格數成正比。空間為 BFS 佇列,最壞 O(mn)。

面向量級說明
TimeO(m·n)每格常數次造訪
SpaceO(m·n)BFS 佇列最壞大小

一句洞察// INSIGHT

「同時擴散、問幾層(分鐘)」 = 多源 BFS + 波次計數。所有源一起入隊、一波一分鐘; 用 fresh 計數判斷有沒有爛不到的橘子(-1)。
