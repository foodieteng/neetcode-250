重點回顧// RECAP

面向要點
問題本質最大連通分量的大小(格數)
核心DFS 沉島時 curArea++,maxArea=max(maxArea,curArea)
與 200同骨架;「數幾座」→「量多大、取最大」
注意①grid 是 int,== 0/= 0 不帶引號
注意②curArea 傳參考 int&(或 dfs 回傳面積)
複雜度時間 O(mn),空間 O(mn)

範例 Trace// 3×4 grid → 3

// GRID · 1=陸地 0=水 1 1 0 1 1 0 0 1 0 0 0 0 兩座島:左上藍島、右上棕島。maxArea 初值 = 0
// 掃描 + DFS 量面積 (0,0)=1 → curArea=0,dfs 沉 (0,0)(1,0)(0,1),curArea=3 maxArea = max(0, 3) = 3 → 藍島 3 格 繼續掃,已沉的是 0 → 跳過 (0,3)=1 → curArea=0,dfs 沉 (0,3)(1,3),curArea=2 maxArea = max(3, 2) = 3 → 棕島 2 格(較小,不更新)
// RESULT 掃完網格,最大的一座是藍島 // OUTPUT: 3
每座島各開一個全新的 curArea(歸零),量完才和 maxArea 比。max 只增不減 —— 所以就算後面的島較小(棕島 2),答案仍停在最大的藍島 3。

應用題// APPLY

App 1. 把 dfs 改成回傳面積(不用傳參考)要怎麼寫?base case 與遞迴式各是什麼?
App 2. 若要同時回傳「島的數量」與「最大面積」,能不能一趟掃描做完?怎麼做?
App 3. 若要回傳每座島的面積列表(由大到小),主流程與收集方式怎麼改?
// ANSWERS
A1. 主流程把「count++」換成「開 curArea=0、呼叫 dfs(..., curArea)、maxArea=max(maxArea,curArea)」;dfs 內多一行 curArea++(沉掉當前格後)。掃描、沉島、base case 全都一樣。本質從「發現一座就 +1」變成「量出每座多大、取最大」。
A2. 因為整座島的格數要累加到同一個計數器。DFS 會往下遞迴很多層,每層沉一格都 curArea++;用 int& 傳參考,所有層改的是同一個 curArea,回到主流程就是整座島的面積。若傳值,每層加的是自己的副本,函式返回就丟掉,主流程拿到的還是 0,面積算不出來。替代法:讓 dfs 回傳 int,面積 = 1 + 四鄰 dfs 的回傳和。
A3. 這題 grid 是 vector<vector<int>>,元素是整數 0/1,所以要 == 0、= 0(不帶引號)。若照抄 200 寫成 == '0',是把 int 和字元 '0'(值 48)比較 —— 這裡永遠不等於任何陸地格,base case 失效、島沉不掉,邏輯全錯。200 是 char、695 是 int,別搬錯引號。
A4. 兩個原因:① 防止相鄰互遞迴 —— 先沉當前格,鄰居踩回來看到 0 就 return,不會無限來回爆堆疊;② 保證每格只算一次面積 —— 若晚點才沉,同一格可能被不同鄰居重複進入、curArea++ 多次,面積偏大。所以一進 dfs 就 grid[r][c]=0; curArea++。
A5. 回傳 0。maxArea 初值設為 0;若整張圖都是水,外層雙迴圈每格都 grid[i][j]==0 而 continue,從不啟動 DFS,maxArea 保持 0。這也符合題目「沒有島回 0」的要求。
App 1. 改簽名為 int dfs(grid, r, c):base case「出界 或 grid[r][c]==0」→ return 0;否則 grid[r][c]=0 後 return 1 + dfs(上) + dfs(下) + dfs(左) + dfs(右)。主流程 maxArea = max(maxArea, dfs(i,j))。不需傳參考,靠回傳值往上累加。
App 2. 可以,一趟就好:主流程維護 count 與 maxArea 兩個變數。每次在未訪陸地啟動 DFS 時 count++、量出 curArea 後更新 maxArea。兩者共用同一輪掃描與同一次沉島,互不衝突,仍是 O(mn)。
App 3. 主流程改用一個 vector<int> areas:每座島量完把 curArea push_back 進去。掃完後 sort(areas.rbegin(), areas.rend()) 由大到小。DFS 沉島與計面積的部分完全不變 —— 只是把「取 max」換成「收集所有面積」。
