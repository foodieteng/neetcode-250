島嶼的最大面積

題目簡介// THE QUESTION

QUESTION

m×n 的 0/1 矩陣,1 是陸地、0 是水。上下左右相連的陸地組成一座島。 一座島的面積 = 島內陸地格的數量。求所有島中最大的面積(沒有島回傳 0)。

這是 200 島嶼數量的孿生題:同樣掃描 + DFS 沉島,只是把「數幾座島」換成「量每座多大、取最大」。 DFS 沉一座島時順便數走過幾格(面積),主流程用 maxArea = max(maxArea, curArea) 記住最大的一座。

2

為什麼 DFS 回面積// WHY DFS-AREA

A 先數島數、再回頭量每座 多掃一趟
分兩階段沒必要 —— 沉一座島的同一趟 DFS 就能順手把格子數出來,不用重掃。
× REJECT · 冗餘
B Union-Find + 每根計 size 可行
DSU 併相鄰陸地、每個根記 size,最後取最大 size。正確,但要維護 size 陣列;DFS 回面積更短。
△ OK · 較繁
ADOPT
C DFS 沉島時累加面積 O(m·n)
掃到未訪陸地 1 就啟動 DFS:每沉一格 curArea++,整座島沉完 curArea = 面積。 回到主流程 maxArea = max(maxArea, curArea)。一趟掃描搞定,O(mn)。
● ADOPT · ONE PASS
3

核心策略// STRATEGY

  1. 雙迴圈掃描每格;遇到 grid[i][j] == 0 直接跳過。
  2. 遇到陸地 1:開一個 curArea = 0,呼叫 dfs(i, j, curArea)。
  3. dfs 沉島 + 計數:出界或踩到 0 就 return;否則 grid[r][c]=0(立刻沉)、curArea++,再遞迴四方向。回主流程後 maxArea = max(maxArea, curArea)。

動畫演示 · DFS 量面積// 藍島 3 · 棕島 2 → 3

注意 · 和 200 的關鍵差別// CAUTION

注意 · 這題最容易踩的坑

① 這題 grid 是 int,不是 char! 和 200 的 char '1'/'0' 不同,這裡用整數 0/1,所以 grid[r][c] == 0、grid[r][c] = 0 不帶引號才對。搬 200 的寫法過來、誤加引號會型別錯。

② curArea 要「傳參考」(int&)。 讓所有遞迴共用同一個計數器;若傳值(int),各層自己加自己的,回不到上層,面積算不對。或改成 DFS 回傳面積 1 + Σdfs(鄰)。

③ 進 dfs 就立刻沉掉並計數。 先標 0、curArea++,再遞迴鄰居;否則相鄰互遞迴 → 無限迴圈,且同一格重複計數面積偏大。

④ 沒有島要回 0。 maxArea 初值設 0,全水時不會啟動任何 DFS,自然回 0。

複雜度// COMPLEXITY

T = O(m·n)　　S = O(m·n)

每格最多被掃一次、被 DFS 沉一次(順便計面積)。空間為遞迴堆疊深度,最壞(整張圖一座蛇形大島)可達 O(mn)。

面向量級說明
TimeO(m·n)每格常數次造訪
SpaceO(m·n)遞迴堆疊最壞深度
面積計數O(1)curArea 沉一格 +1

一句洞察// INSIGHT

和 200 同一個 DFS 沉島骨架,只多做一件事:沉一格就 +1 數面積,每座島量完取最大。 「數幾座」→「量多大」,換湯不換藥。
