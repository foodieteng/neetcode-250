完整實作// IN / OUT DEGREE

p997.cpp C++17 
class Solution {
public:
    int findJudge(int n, vector<vector<int>>& trust) {
        vector<int> in(n + 1, 0), out(n + 1, 0);   // 1-indexed:1..n

        for (auto t : trust) {
            int a = t[0], b = t[1];          // a 信任 b
            out[a]++; in[b]++;                    // a 的出度、b 的入度各 +1
        }

        for (int i = 1; i <= n; i++) {
            if (out[i] == 0 && in[i] == n - 1) return i;  // 不信任人 + 被所有人信任
        }

        return -1;                                // 沒有法官
    }
};

複雜度分析// COMPLEXITY

T = O(n + m)　　S = O(n)

m = trust.size()。第一個迴圈掃過所有邊 O(m);第二個掃 1..n 找法官 O(n)。空間是兩個長度 n+1 的計數陣列。

步驟成本說明
累加度數O(m)每條 trust 邊各動兩個計數
掃描找法官O(n)檢查每人 out==0 && in==n−1
額外空間O(n)in[] 與 out[]

邊界與陷阱// PITFALLS

陷阱後果正解
方向搞反 in[a]++ / out[b]++法官條件全錯a→b:out[a]++、in[b]++
寫成 in[i] == n永遠找不到法官不信任自己 → 被 n−1 人信任
陣列開 n 而非 n+1越界(編號 1..n)vector<int>(n + 1, 0) 用 1-indexed
漏掉 n==1、trust 空誤判此時 in[1]=0=n−1、out[1]=0 → 回傳 1(正確)

細節走讀// WALKTHROUGH

// 方向就是一切:a→b 邊 [a, b] = 「a 信任 b」 a 主動信任了別人 → a 的「出度」out[a]++ b 被別人信任了 → b 的「入度」in[b]++ 法官:out 必須 0(誰都不信),in 必須 n−1(被其他所有人信)
// 為什麼是 n−1 不是 n 鎮上共 n 人。法官不會信任自己,所以「信任法官的人」最多 n−1 個 若某人 in == n−1,代表「除了他自己」的每個人都信任他 寫成 in == n 就要求連自己都信任自己,永遠不成立 → 找不到法官
// 邊界 · n==1、trust 空 只有 1 人、沒有任何信任關係 in[1]=0, out[1]=0;n−1 = 0 out[1]==0 且 in[1]==0==n−1 → 條件成立 → return 1 單獨一人時,他自己就是法官。
PREV概念與圖示 NEXT複習與 Trace
// KEY LESSON
a→b 記 out[a]++、in[b]++;法官是 out==0 && in==n−1 的唯一節點(注意 n−1 與 1-indexed)。
