找出城鎮法官

// CONCEPT · 入度 n−1、出度 0 的唯一節點

題目簡介// THE QUESTION

QUESTION

鎮上有 n 人(編號 1..n)。若存在法官,則:① 法官誰都不信任;② 其他每個人都信任法官;③ 恰好一人同時滿足 ①②。 給 trust[i] = [a, b] 表示 a 信任 b,回傳法官編號,不存在回傳 -1。

把信任看成有向圖:邊 a→b = 「a 信任 b」。法官就是唯一「入度 in = n−1(被其他所有人指向)、且出度 out = 0(自己不指向任何人)」的節點。 只要數每個人的 in / out 度,再掃一遍找符合者即可。

2

為什麼用度數計數// WHY DEGREE COUNT

A 對每個候選人掃整個 trust 驗證 O(n·m)
對每個人都掃一遍 trust 確認「被所有人信任、且不信任別人」。n 個候選 × m 條邊,重複掃太慢。
× REJECT · 慢
B 建鄰接表 + BFS/DFS 找入度 殺雞牛刀
建整張圖再走訪,可行但這題根本不用走訪 —— 答案只跟「每個點的 in / out 度」有關,不需要路徑或連通性。
△ REJECT · 過度
ADOPT
C in[] / out[] 度數計數 O(n + m)
掃一次 trust 累加 out[a]++, in[b]++。再掃 1..n 找 out[i]==0 && in[i]==n−1 的人 —— 這正是法官,唯一且直接。 不需要建圖、不需要走訪,兩個計數陣列搞定。
● ADOPT · LINEAR
3

核心策略// STRATEGY

  1. 兩個計數陣列:in[1..n]、out[1..n] 全設 0。
  2. 累加度數:對每條 [a, b],out[a]++(a 信任了人)、in[b]++(b 被信任)。
  3. 找法官:掃 1..n,第一個 out[i]==0 && in[i]==n−1 就回傳 i;都不符合回傳 -1。
4

圖示 · 一次全顯示// n=3 · trust=[[1,3],[2,3]] → 3

整張圖一次呈現:P1、P2 都指向 P3(信任),P3 不指向任何人。P3 的 in=2=n−1、out=0 → 唯一符合法官條件 → 回傳 3。

▶ 信任關係圖 · in / out 度 · 判定
法官 = 被所有人信任(in=n−1)且自己不信任任何人(out=0);唯一符合者即答案,否則 −1。
5

為什麼「一個條件」就夠// WHY IT'S UNIQUE

LEMMA · in==n−1 且 out==0 → 唯一法官

out[i]==0 保證 i 不信任任何人(條件 ①);in[i]==n−1 保證其他 n−1 人全都信任 i(條件 ②)。兩者合起來正是法官定義。

為什麼唯一?若有兩人 x, y 都滿足,則 x 要被所有人信任、y 也得信任 x → out[y]≥1,和 out[y]==0 矛盾。所以最多一人符合,掃到就是答案。

6

複雜度// COMPLEXITY

T = O(n + m)　　S = O(n)

m = trust.size()。掃一次邊累加度數 O(m),再掃 1..n 找法官 O(n)。空間為兩個長度 n+1 的計數陣列。

面向量級說明
TimeO(n + m)一次累加 + 一次掃描
SpaceO(n)in[] 與 out[] 兩陣列
可優化O(n)合成一個 net = in − out,找 net==n−1
