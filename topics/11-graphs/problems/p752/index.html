題目簡介// THE QUESTION

QUESTION

4 個轉輪的鎖,每輪 0..9 可環繞(9→0、0→9)。每次轉一個輪一格。從 "0000" 出發, deadends 是不能停留的死鎖狀態。求轉到 target 的最少轉數,無法到達回 -1。

把每個 4 位密碼看成圖上一個節點,一次轉輪 ±1 連到 8 個鄰居(每輪上/下)。「最少轉數」=「起點到終點的最短路徑長度」,而且每條邊權重都是 1(轉一次算一步)—— 這正是 BFS 的主場:一層一層往外擴,第一次碰到 target 的層數就是答案。deadends 當作不能進的牆、seen 避免重複。

2

為什麼用 BFS// WHY BFS

A DFS 找路徑 非最短
DFS 一頭鑽到底,先找到的不保證最短,要窮舉所有路徑再取最短 → 指數爆炸。最短路不該用 DFS。
× REJECT · 非最短
B Dijkstra 殺雞牛刀
Dijkstra 處理不同邊權;這題每條邊都是 1(轉一次),用 heap 是多餘。BFS 就是邊權 1 的 Dijkstra。
△ REJECT · 過度
ADOPT
C BFS(逐層 = 逐次轉) O(狀態數)
從 "0000" 逐層擴展:每層把當前狀態的 8 個鄰居(未死、未訪)入隊。一層 = 轉一次, 第一次讓 target 出隊時的層數 = 最少轉數。狀態最多 10⁴ 個,每個常數鄰居 → O(10⁴)。
● ADOPT · SHORTEST
3

核心策略// STRATEGY

  1. 特判:"0000" 在 deadends → -1;target == "0000" → 0。
  2. 逐層 BFS:佇列放狀態、seen 去重。每輪用 sz = q.size() 清完這一層;每個狀態產生 8 個鄰居(4 輪 × 上下),不在 dead、不在 seen 才入隊。
  3. 計層數:每清完一層 step++;u == target 時回傳當下 step;佇列空還沒到 → -1。

錯誤筆記 · char 運算的坑// MY BUG NOTE

注意 · 「數字值 +1」不等於「下一個數字字元」

我原本把「輪 +1」寫成 u[i] = char(cache[i] - '0' + 1) —— 錯。cache[i] 是字元(如 '5' = ASCII 53)。cache[i] - '0' = 5(轉成數字值),+ 1 = 6,而 char(6) 是 ASCII 6 的控制字元,不是字元 '6'(ASCII 54)。少了把數字轉回字元的 + '0'。

後果:產生的鄰居字串塞進了亂碼控制字元,根本不是合法的鎖狀態。BFS 於是在一堆垃圾狀態裡亂走(它們也不在 dead/seen),不但答案錯,狀態空間還爆開、幾乎跑不完。

正解:直接用 ASCII 運算 cache[i] + 1('5'+1 = '6'),或補回轉換 char(cache[i] - '0' + 1 + '0')。另外別忘 u = cache; 還原 —— 每個輪產生完鄰居要把 u 復原,否則下一個輪會建在「已被轉過」的字串上,變成多輪同時轉的錯鄰居。

複雜度// COMPLEXITY

T = O(10⁴ · 8 · 4)　　S = O(10⁴)

狀態最多 10⁴ 個,每個產生 8 個鄰居、每個鄰居比較 4 個字元。都是常數倍,整體 O(10⁴)。空間為 seen + 佇列。
