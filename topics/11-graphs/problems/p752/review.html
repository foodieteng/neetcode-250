觀念題// QUIZ

Q1. 為什麼這題用 BFS 而不是 DFS 或 Dijkstra?從「最短路 + 邊權」兩個角度說。
Q2.(錯誤筆記)u[i] = char(cache[i] - '0' + 1) 錯在哪?以 cache[i] = '5' 為例,它算出什麼、正確應該是什麼?
Q3. 為什麼每個輪產生完鄰居要 u = cache; 還原?如果漏掉,下一個輪的鄰居會變成什麼?
Q4. int sz = q.size() 的「一次清完一層」有什麼用?這和 994 腐爛橘子的 sz 是同一招嗎?
Q5. 開頭兩個特判 dead.count("0000")、target == "0000" 分別在處理什麼邊界?少了會怎樣?

範例 Trace// target 0202 → 6

// 輸入 deadends = [0201, 0101, 0102, 1212, 2002] target = 0202 起點 0000,一次轉一輪 ±1(9↔0 環繞)
// BFS 逐層(一層 = 轉一次) step 0: 0000 step 1: 1000 (wheel0 ↑) step 2: 1100 (wheel1 ↑) step 3: 1200 (wheel1 ↑) step 4: 1201 (wheel3 ↑) step 5: 1202 (wheel3 ↑) step 6: 0202 = target (wheel0 ↓:1→0)
// RESULT 第一次讓 0202 出隊時 step = 6 全程避開 deadends(如 0102、0201) // OUTPUT: 6
BFS 一層一層像水波往外擴,先到的必定是最短的。這條 6 步的路只是眾多同長最短路之一; 題目要的是「幾步」,BFS 第一次觸及 target 那刻就給出答案。

應用題// APPLY

App 1. 用雙向 BFS(同時從 0000 和 target 往中間搜)能怎麼加速?為什麼能減少擴展的狀態數?
App 2. 若輪子不是 0-9,而是每輪有 k 個符號、共 L 個輪,狀態數與複雜度變成多少?BFS 還可行嗎?
App 3. 若每個輪「轉一格的代價不同」(例如某些輪較緊、轉一格算 2 步),還能用純 BFS 嗎?該換成什麼?
// ANSWERS
A1. 這題要「最少轉數」= 最短路徑長度,而且每轉一次的代價都相同(邊權 = 1)。DFS 先找到的路不保證最短,要窮舉再取最小、指數爆炸;Dijkstra 是為「不同邊權」設計的,用 min-heap 是殺雞牛刀。BFS 正好是「邊權都 1 的最短路」:逐層擴展,第一次到達即最短,O(狀態數)。
A2. cache[i] 是字元,'5' 的 ASCII 是 53。cache[i] - '0' 把它變成數字值 5;+ 1 = 6;而 char(6) 是 ASCII 6 的控制字元,不是字元 '6'(ASCII 54)。少了把數字轉回字元的 + '0'。正解:直接 ASCII 運算 cache[i] + 1('5'+1='6'),或 char(cache[i] - '0' + 1 + '0')。
A3. 因為一個狀態的 8 個鄰居都要從同一個原始 u 產生。處理完輪 i 後,u[i] 已被改成「下轉值」;若不 u = cache; 還原,下一輪 i+1 就會在「輪 i 已被轉過」的字串上再改輪 i+1 → 產生的是「兩個輪同時轉」的非法鄰居(一步只能轉一個輪)。還原確保每個鄰居都是「原狀態只動一個輪」。
A4. sz = q.size() 先鎖住當前這一層的狀態數,內層只處理這 sz 個(它們與起點的距離都是 step);清完才 step++,確保「一層 = 轉一次」。是,和 994 腐爛橘子完全同一招 —— 凡是「BFS 要回傳層數/步數」都用 sz 把每一層切開來數。
A5. dead.count("0000"):起點本身就是死鎖 → 一步都動不了,直接 -1(否則會從一個非法狀態開始 BFS)。target == "0000":目標就是起點 → 0 步,直接 0(否則要等 BFS 出隊才回,雖然多數寫法也對,但先擋掉最乾淨、也避免把 0000 當「已走一層」誤加)。兩個特判把退化情形先清掉。
App 1. 雙向 BFS:同時從 0000 和 target 各開一個 frontier 往中間擴,哪邊小就擴哪邊,兩邊相遇時距離相加即答案。因為 BFS 擴展的狀態數隨層數指數級成長,兩個「半程」的總量遠小於一個「全程」(2·b^(d/2) ≪ b^d),所以能大幅減少擴展的狀態、加速。
App 2. 狀態數 = k^L(每輪 k 選擇、L 個輪),每個狀態有 2L 個鄰居。BFS 時間 O(k^L · L)。k,L 小時(如本題 10^4)完全可行;但 k^L 一大就爆炸,這時要靠雙向 BFS、或 A*(有好的啟發式)、或問題本身的結構來剪枝。
App 3. 不行。純 BFS 的正確性建立在「每條邊權重相同」——先進佇列就等於距離最短。一旦轉不同輪代價不同(邊權不一),就變成加權最短路,要改用 Dijkstra(以累計代價為 key、min-heap 取最小);若代價只有 0/1 兩種,也可用 0-1 BFS(deque)。BFS 的「一層一次數」前提被打破。
