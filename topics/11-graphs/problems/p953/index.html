題目簡介// THE QUESTION

QUESTION

外星語用同樣 26 個小寫字母,但字母順序是給定的 order(一個 26 字母的排列)。 給一組 words,判斷它們是否已依這個外星字母序排好(由小到大)。是回傳 true,否則 false。

關鍵:order 已經給了,不用去「推導」字母順序。只要把 order 轉成一張 rank[26](每個字母排第幾), 再檢查每一對相鄰單字是否遞增即可 —— 這其實不是圖走訪,而是rank 映射 + 相鄰對比較。

2

為什麼不用拓撲/排序// WHY THIS

A 跑拓撲排序推字母順序 搞錯題
那是 269 Alien Dictionary(要你推出 order)。這題 order 已給,不必推導,拓撲是殺雞用牛刀還答非所問。
× REJECT · 想錯題
B 用 comparator 排完再比對原序 多餘
自訂 comparator 把 words 排一次,看是否等於原順序。可行但 O(n log n · L),而且「是否已排好」只要線性掃相鄰對就知道。
△ REJECT · 過度
ADOPT
C rank[] + 相鄰對逐欄比較 O(總字元)
先建 rank[c] = 字母 c 在 order 的位置(O(26))。再對每對相鄰單字逐欄比較: 第一個不同的字元由 rank 決定大小(前者 rank 較大就違規);若一路相同、前者較長(是後者的前綴超出)也違規。全程 O(所有字元總長)。
● ADOPT · LINEAR
3

核心策略// STRATEGY

  1. 建 rank:for i in 0..25: rank[order[i]-'a'] = i。之後比任意兩字母大小 = 比它們的 rank。
  2. 掃相鄰對:對每組 (words[i], words[i+1]) 逐欄比較,取到 minLen 為止:
    • 字元相同 → 看下一欄。
    • 字元不同 → 若 rank[前] > rank[後] → return false;否則這對 OK,break 去看下一對。
  3. 前綴規則:若比到 minLen 都相同(沒 break),而前者比後者長 → return false(如 "apple" 排在 "app" 前是錯的)。

為什麼「只檢查相鄰對」就夠// WHY ADJACENT ONLY

LEMMA · 相鄰皆遞增 ⇒ 整體有序

「已排序」在任何全序下都有遞移性:若 w0 ≤ w1 且 w1 ≤ w2,則 w0 ≤ w2。 所以只要每一對相鄰都成立 words[i] ≤ words[i+1],整個 list 就是有序的 —— 不必比較所有 O(n²) 對。

兩字比較就是字典序,只是字母大小改看 rank:第一個不同的字元定勝負;若一方是另一方的前綴,短的較小。這兩條規則涵蓋所有情況。

複雜度// COMPLEXITY

T = O(Σ|words|)　　S = O(1)

建 rank 是 O(26)。每對相鄰只比到第一個不同的字元;所有比較的字元加起來不超過全部字元總長 S = Σ|words[i]|。空間只有固定的 rank[26]。

面向量級說明
TimeO(Σ|words|)掃過所有字元一次(最壞)
SpaceO(1)固定 26 格 rank 表
