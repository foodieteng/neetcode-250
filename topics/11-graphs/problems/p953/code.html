完整實作// RANK + ADJACENT COMPARE

p953.cpp C++17 
class Solution {
public:
    bool isAlienSorted(vector<string>& words, string order) {
        vector<int> rank(26, 0);
        for (int i = 0; i < 26; i++) {
            rank[order[i] - 'a'] = i;          // 字母 → 它在 order 的位置
        }

        int n = words.size();

        for (int i = 0; i < n - 1; i++) {          // 只看每對相鄰
            string w1 = words[i], w2 = words[i + 1];
            int minL = min(w1.size(), w2.size());
            int j = 0;

            for (; j < minL; j++) {
                if (w1[j] == w2[j]) continue;            // 相同 → 看下一欄
                if (rank[w1[j] - 'a'] > rank[w2[j] - 'a']) return false;  // 前者較大 → 違規
                break;                                // 不同但合法 → 這對 OK,停
            }

            if (j == minL && w1.size() > w2.size()) return false;  // 前綴規則:前者較長 → 違規
        }

        return true;
    }
};

複雜度分析// COMPLEXITY

T = O(Σ|words|)　　S = O(1)

建 rank 是 O(26)。外層掃 n−1 對,每對內層只比到第一個不同的字元;所有內層比較的字元加總 ≤ 全部字元總長。空間只有固定 rank[26]。

步驟成本說明
建 rank 表O(26)order 是 26 字母排列
掃相鄰對比較O(Σ|words|)逐欄比,最壞掃過所有字元
額外空間O(1)rank[26] 固定大小

邊界與陷阱// PITFALLS

陷阱後果正解
直接比字元 w1[j] > w2[j]用了正常字母序比 rank[w1[j]-'a'] vs rank[w2[j]-'a']
忘了漏寫 break合法的不同也繼續比後面不同但合法 → break,換下一對
忘了前綴規則["apple","app"] 誤判 true一路相同時,w1 較長 → false
rank 建反 rank[i]=order[i]映射錯rank[order[i]-'a'] = i(字母→位置)

細節走讀// WALKTHROUGH

// rank[] 是什麼 order = "worldabc…" → rank[w]=0, rank[o]=1, rank[r]=2, rank[l]=3, rank[d]=4, … 要比 x 和 y 誰在外星序較小 → 比 rank[x] 和 rank[y] 於是「外星字典序」= 把每個字母的大小改看 rank 的「正常字典序」
// 內層迴圈三種結局 相同 : w1[j]==w2[j] → continue,看下一欄 不同違規 : rank[w1[j]] > rank[w2[j]] → return false(前者較大,排錯) 不同合法 : rank[w1[j]] < rank[w2[j]] → break,這對已定序、換下一對 關鍵:一旦第一個不同的字元決定了大小,後面的字元不用再看。
// 前綴規則(跑完內層沒 break) 若 j 一路加到 minL 都沒 break → 前 minL 個字元完全相同 此時 w1 是 w2 的前綴、或反之 字典序:短的較小。若 w1 較長(w1.size() > w2.size())→ 前者較大 → false 例:["apple","app"] → 前 3 字元同,apple 較長 → return false

// KEY LESSON
rank 換掉字母大小;逐欄比相鄰對,第一個不同定勝負,別忘前綴規則(短的較小)。
