重點回顧// RECAP

面向要點
問題本質把「不碰邊界」的 O 區域翻成 X
核心翻轉不找被包圍的,反過來從邊界標安全
安全定義與邊界連通 = DFS 從邊界 O 能到 = 標 '#'
翻面'#' → 'O' 保留;其餘 'O' → 'X' 捕獲
與 417同「從邊界反向 DFS」家族
複雜度時間 O(mn),空間 O(mn)

觀念題// QUIZ

Q1. 為什麼「從邊界標安全、剩下的捕獲」比「直接找被包圍的區域再翻」好?後者難在哪?
Q2. 為什麼中間要用 '#' 這個第三種狀態?如果只用 'O'/'X' 兩種,最後翻面會遇到什麼問題?
Q3. DFS 裡 board[nr][nc] != 'O' 就 continue,這一條同時擋掉了哪三種情況?
Q4. 為什麼四條邊(左、右、上、下)都要掃過、當 DFS 起點?只掃一兩條會漏什麼?
Q5. 這題和 417 太平洋大西洋有什麼共同的解題心法?

範例 Trace// 5×5

// 初始棋盤 X X X X X O O X X X X O X X X X X X O X X X X X X 邊界上的 O:只有 (1,0)(左緣)
// ① 從邊界 O 標安全(#) 從 (1,0) DFS:(1,0)→(1,1)→(2,1) 連通,整團標 '#' X X X X X # # X X X X # X X X X X X O X ← (3,3) 碰不到邊界,沒被標到 X X X X X
// ② 翻面:'#'→O,其餘 O→X X X X X X O O X X X X O X X X X X X X X ← (3,3) 被捕獲成 X X X X X X // OUTPUT: 邊界連通區保留,內部區被捕獲
關鍵一眼:(3,3) 是一團「孤島 O」,四周全是 X 又不碰邊界 → 被捕獲。 而 (1,0) 那團因為有一格踩在左緣,整團跟著逃過捕獲。安全與否是「整團」的性質,由邊界那格決定。

應用題// APPLY

App 1. 把 DFS 換成 BFS 怎麼寫?把所有邊界 O 一起入隊,和「多源 BFS」有什麼關係?什麼情況下更該用 BFS?
App 2. 若不准引入 '#' 這種暫存符號(只能用 'O'/'X'),還能一趟解嗎?可以怎麼繞?
App 3. 若要額外回傳被捕獲的格子數,程式怎麼改最省(不必再多掃一遍)?
// ANSWERS
A1. 「被包圍」是整團的性質 —— 要捕獲一團 O,得先確認它每一格都不在邊界,這不能只看單格,得追完整團、暫存所有座標再判斷,邏輯繞、容易漏。反過來「安全 = 碰得到邊界」則有一個天然的起點:邊界。從邊界 O 出發 DFS,能到的就是安全的;掃完後沒被標到的一律是被包圍的。判斷從「證明全團不碰邊」變成「有沒有被邊界標到」,單純又快。
A2. 因為翻面那一刻,盤上需要同時分辨三種格:安全的 O、被包圍的 O、本來的 X。若只用兩態,DFS 把安全 O 標成……標成什麼都會和另一種撞:標成 X 會和牆混、留成 O 又和被包圍的混。用 '#' 當暫時的第三態把「已確認安全」獨立出來,最後 '#'→'O'、殘留 'O'→'X' 就能乾淨分開。
A3. 擋掉三種:① 出界(nr/nc 越界);② '#'(已訪的安全格 → 避免重複遞迴、死圈);③ 'X'(牆 → 不可穿越)。只有「還是 'O'」的鄰居才往下走。所以一條 != 'O' 同時處理了 visited、障礙、邊界。
A4. 因為任何一條邊上的 O 都可能是某個安全區的「出口」。一團 O 只要在四條邊的任一格露頭就安全。只掃左右兩緣會漏掉「只從上緣或下緣接觸邊界」的區域,那些區域會被誤判成被包圍而錯翻。所以左、右、上、下四條邊都得當起點種進去(角落被掃兩次沒關係,'#' 去重)。
A5. 共同心法是「直接算目標難,就從邊界反向標記」。417 要「能流到海的格」,就從海邊逆流往高處淹、標出可達;130 要「被包圍(不碰邊界)的格」,就從邊界順著相連 O 淹、標出安全,剩下的即是答案。兩題都把「對每個內部格各自判斷」換成「從邊界一次淹沒」,O((mn)²) → O(mn)。
App 1. BFS 版:把所有邊界上的 O 一次全入隊(這正是多源 BFS),再逐格出隊、往還是 'O' 的鄰居擴散並標 '#'、入隊。翻面邏輯不變。BFS 用佇列取代遞迴堆疊,超大且幾乎全是 O 的盤面用 BFS 可避免遞迴爆堆疊,同樣 O(mn)。
App 2. 可以。既然邊界那趟 DFS本來就會走遍所有安全 O,可以在 DFS 裡把安全 O 暫時改成 'X'——不,那會和牆混。乾淨的兩態做法是:先把所有 'O' 都翻成 'X'(全部先當被捕獲),再從邊界對「原本是 O 的位置」BFS/DFS 把安全區改回 'O'。但這需要另存一份原盤或先記邊界起點座標。用 '#' 暫存仍是最省最直觀的;硬要兩態就得付出額外空間或兩次掃描。
App 3. 在翻面那一次掃描裡順手數:每當一格不是 '#' 且原本是 'O'(即將被翻成 'X')時 captured++。因為翻面本來就要掃全盤一遍,計數是零額外成本,不必再多掃。回傳 captured 即可。
