重點回顧// RECAP

面向要點
問題本質隱式圖上求最短路徑(無權)
節點 / 邊節點 = 單字;邊 = 差一字母且在字典
隱式圖鄰居即時生成(每位試 a..z),不建圖
演算法BFS 分層,層數 = 接龍長度
命中碰到 endWord 回 step+1
複雜度時間 O(N·L·26),空間 O(N·L)

觀念題// QUIZ

Q1. 為什麼用 BFS 而不是 DFS?BFS 的什麼性質保證找到的是最短接龍?
Q2. 「隱式圖」是什麼意思?為什麼即時生鄰居(每位置試 26 字母)比先建整張圖快?
Q3. 換字母試鄰居後,為什麼一定要把 w 還原成 cache?不還原會怎樣?
Q4. 為什麼要入隊當下就標 seen,而不是出隊時才標?
Q5. 命中 endWord 時回 step + 1,那個 +1 是怎麼來的?

範例 Trace// hit → cog

// 輸入 begin = hit end = cog dict = {hot, dot, dog, lot, log, cog}
// BFS 分層 step1 queue={hit} → 生 hot step2 queue={hot} → 生 dot, lot step3 queue={dot,lot} → 生 dog, log step4 queue={dog,log} → dog 生 cog == end → return 4+1
// 結果 最短接龍:hit → hot → dot → dog → cog(也可 →lot→log→) 單字數 = 5 (若 dict 沒有 cog,BFS 擴散完仍碰不到 → return 0)
同一層可能有多條路併行(dot/lot、dog/log),但 BFS一視同仁一起擴散, 誰先碰到 cog 都在同一層,所以答案唯一是 5。這就是無權圖「按層 = 按距離」的威力。

應用題// APPLY

App 1. 若字典非常大、字又長,怎麼用雙向 BFS加速?為什麼能把搜尋量從 b^d 降到 2b^(d/2)?
App 2. 若還要回傳其中一條最短接龍的完整路徑(不只長度),程式怎麼改?
App 3. 126 Word Ladder II 要回傳所有最短路徑,為什麼比本題難很多?大方向怎麼做?
// ANSWERS
A1. BFS 按「離起點的距離」一層一層擴散:第 k 層的所有節點,與起點的最短距離都恰好是 k。所以第一次碰到 endWord 時,它所在的層數就是最短距離,不可能有更短的路徑先於它被更晚的層找到。DFS 會沿一條路一路走到底,先找到的往往不是最短、還得回溯比較所有路徑,對「無權最短路」既慢又不直接。無權圖求最短 → BFS 是標準答案。
A2. 「隱式圖」= 圖的節點與邊沒有被實際存下來,而是需要時現算。這題只要能「從一個字快速列出它的鄰居」BFS 就能跑。即時生鄰居:一個長 L 的字只有 L×26 個「差一字母」候選,逐一 hashset 查是否在字典,是常數量級(相對字典大小)。相比「兩兩比對建圖」的 O(N²·L),當字典大時 26L 遠小於 N,省掉整個建圖成本與空間。
A3. 因為我們在原地改 w[i] 來試鄰居。試完第 i 位的 26 個字母後,要換去試第 i+1 位,此時 w 必須是沒被改壞的原字,否則會在「已經改過一個位置」的錯字上再改一位,生出差兩個字母的假鄰居(甚至完全無關的字)。用 cache = w 存原字、內圈結束 w = cache 還原,保證每次只試「和原字差一位」的候選。
A4. 為了防止同一個字被重複入隊。若等出隊才標 seen,那麼在它還在佇列裡、尚未出隊的期間,別的節點可能又生出同一個字、再次入隊,造成佇列膨脹、甚至指數級重複。入隊當下就 seen.insert,等於「一旦排進隊伍就鎖定」,每個字最多入隊一次,總量受控、複雜度才成立。(這是 BFS 的通則:visited 在推入時標記。)
A5. step 代表目前正在擴散的層的接龍長度:step=1 時佇列裡是 beginWord(接龍第 1 個字)。當我們在第 step 層生出 endWord,它是下一層的節點 —— 接龍的第 step+1 個字。因為題目要的長度是「含頭尾的單字數」,所以回 step+1。舉例:在 step=4(佇列是 dog/log)生出 cog,cog 是第 5 個字 → 回 5。
App 1. 雙向 BFS:同時從 beginWord 和 endWord 兩邊往中間擴,維護兩個 frontier(前沿集合),每輪擴展較小的那一側;當某一側生出的字出現在另一側的 frontier,兩條路接上,總長 = 兩側層數和。單向 BFS 最壞要探 b^d 個節點(b 分支度、d 深度);雙向兩邊各只探到 d/2 深,是 2·b^(d/2),指數的一半,通常快好幾個數量級。
App 2. 只回一條路徑:BFS 時對每個入隊的字記一個 parent[字] = 從哪個字變來的(前驅)。碰到 endWord 後,從它沿 parent 一路回溯到 beginWord、再反轉,就是一條最短接龍。額外空間 O(N·L) 存前驅,時間不變。關鍵:因為每個字只入隊一次、只記第一個發現它的前驅,回溯得到的必是一條最短路。
App 3. 126 要所有最短路徑,難在:① 一個字可能被同一層的多個前驅同時到達,不能只記一個 parent,要記所有最短前驅(parents[字] = {前驅們});② 不能一發現就標死 seen 而擋掉同層其他到達方式,得改成「分層處理、層內允許多前驅、層末才把整層標訪」。BFS 建好「最短前驅 DAG」後,再從 endWord 用 DFS 回溯枚舉所有路徑。既要 BFS 定層、又要 DFS 收集,還要小心去重與剪枝,所以比只求長度的 127 難不少。
