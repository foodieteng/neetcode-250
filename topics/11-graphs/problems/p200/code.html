完整實作 · DFS 沉島// SINK THE ISLAND

p200.cpp C++17 
class Solution {
private:
    int m, n;
    const vector<pair<int, int>> dirs = {{-1, 0}, {1, 0}, {0, -1}, {0, 1}};

public:
    int numIslands(vector<vector<char>>& grid) {
        m = grid.size(); n = grid[0].size();
        int cnt = 0;

        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                if (grid[i][j] == '1') {       // 未訪陸地 = 新島
                    cnt++;
                    dfs(grid, i, j);            // 沉掉整座相連的島
                }
            }
        }
        return cnt;
    }

    void dfs(vector<vector<char>>& grid, int r, int c) {
        if (r < 0 || r >= m || c < 0 || c >= n || grid[r][c] == '0') return;
        grid[r][c] = '0';                     // 立刻沉掉(標記已訪)再遞迴
        for (auto [dr, dc] : dirs) {
            int nr = r + dr, nc = c + dc;
            dfs(grid, nr, nc);
        }
    }
};

複雜度分析// COMPLEXITY

T = O(m·n)　　S = O(m·n)

每個陸地格只被 DFS 沉一次、每格被掃描一次。總造訪量 O(mn)。空間為遞迴堆疊深度,最壞(蛇形大島)可達 O(mn)。

步驟成本說明
雙迴圈掃描O(mn)每格看一次
DFS 沉島(全部)O(mn)每格最多被沉一次
遞迴堆疊O(mn)最壞深度

注意 · 邊界與陷阱// CAUTION · PITFALLS

陷阱後果正解
字元 vs 整數:寫 == 1 / = 0比到 ASCII,永遠不對用單引號:== '1'、= '0'(grid 是 char)
遞迴鄰居前沒先標記相鄰互遞迴 → 爆堆疊進 dfs 立刻 grid[r][c] = '0'
沉島改動了輸入 grid破壞原資料不許改動 → 另開 visited 陣列
超大全陸地網格用遞迴 DFS堆疊溢位改 BFS(queue)或迭代 DFS
base case 少了出界檢查越界存取 · UB先驗界再讀 grid[r][c](|| 短路)

// 為什麼「啟動 DFS 次數 = 島數」 外層掃到未訪陸地才 count++ 並啟動 dfs dfs 會把「這一格能走到的所有相連陸地」全沉成 '0' → 同一座島只會觸發「一次」count++,其餘格都已是 '0' 被跳過 所以 count 累加幾次,就代表發現了幾個獨立連通塊 = 幾座島
// 為什麼要「立刻」沉掉當前格 若先遞迴鄰居、後才標記自己: A 進 dfs → 遞迴到鄰居 B → B 又遞迴回 A(A 還是 '1')→ A 再遞迴 B… → 無限來回、堆疊爆掉 正解:一進 dfs 就 grid[r][c]='0',鄰居再踩回來時看到 '0' 直接 return
