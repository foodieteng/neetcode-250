題目簡介// THE QUESTION

QUESTION

m×n 網格,'1' 是陸地、'0' 是水。上下左右相連的陸地組成一座島(斜角不算)。 求島的數量。

把網格看成圖:每個陸地格是節點,和上下左右的陸地格相鄰。島 = 一個連通分量。 逐格掃描,每碰到一個「還沒被訪過的陸地」,就代表發現一座新島 → count +1,並用 DFS 把整座相連的島沉掉(標記已訪),避免同一座島被重複計數。

2

為什麼用「掃描 + 沉島」// WHY FLOOD FILL

A 只數陸地格數 / 只找 1 答錯
島數不是陸地格數,也不是 1 的個數。一座島可能有很多格,關鍵是「有幾個連通塊」。
× REJECT · 想錯
B Union-Find 併相鄰陸地 可行
把相鄰陸地 union,最後數不同的根。正確,但要寫 DSU;這題用 DFS/BFS 沉島更直接、程式碼更短。
△ OK · 較繁
ADOPT
C 掃描 + DFS 沉島(flood fill) O(m·n)
雙迴圈掃每一格;遇到未訪的陸地 '1' 就 count++,然後 DFS 把整座相連的島全部標成 '0'(沉掉)。 啟動 DFS 的次數 = 島的數量。每格最多被訪一次,O(mn)。
● ADOPT · LINEAR
3

核心策略// STRATEGY

  1. 雙迴圈掃描每一格 (i, j)。
  2. 遇到 grid[i][j] == '1':count++(發現新島),呼叫 dfs(i, j)。
  3. dfs 沉島:出界或踩到 '0' 就 return;否則把 grid[r][c] = '0'(立刻標記),再對上下左右各遞迴一次。整座島沉完才回到掃描。

注意 · 小心的地方// CAUTION

注意 · 這題最容易踩的四個坑

① 是字元 '1'/'0',不是整數。 網格型別是 char,比較與賦值都要用單引號:grid[r][c] == '0'、grid[r][c] = '0'。寫成 == 0 會比到 ASCII 0(空字元),永遠不成立。

② 進 dfs 就立刻標記 '0'。 必須在遞迴鄰居之前把當前格沉掉;否則相鄰兩格會互相遞迴呼叫 → 無限迴圈 / 爆堆疊。

③ 這個寫法會改動輸入 grid。 沉島 = 把陸地覆寫成水,原網格被破壞。若題目要求不可修改輸入,改用額外的 visited 陣列。

④ 遞迴深度風險。 最壞情況整張圖都是陸地(300×300 = 90000 格),DFS 遞迴可能堆疊溢位。超大網格改用 BFS(queue) 或迭代式 DFS 較安全。

複雜度// COMPLEXITY

T = O(m·n)　　S = O(m·n)

每格最多被掃一次、被 DFS 沉一次,總工作量與格數成正比。空間是 DFS 遞迴堆疊深度,最壞(整張圖是一座蛇形島)可達 O(mn)。
