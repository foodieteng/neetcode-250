題目簡介// THE QUESTION

QUESTION

給一批等式:equations[i] = [A, B] 搭配 values[i] 表示 A / B = values[i]。 再給一批查詢 [C, D],求 C / D = ?。若無法從已知等式推出(變數沒出現、或兩者不連通),回 -1.0。

把變數當節點、等式當加權有向邊:A/B=v 建 A→B 權重 v、反向 B→A 權重 1/v。 那麼 C/D 就是沿任一條 C 到 D 的路徑,把邊權連乘 —— 因為中間項會像電話接龍一樣相消:a/b · b/c = a/c。用 BFS/DFS 帶著累積權重走即可。

2

核心觀察// KEY IDEA

建圖 A/B=v → 兩條有向邊 v 與 1/v
graph[A][B]=v、graph[B][A]=1/v。除法可逆,反向就是倒數。
◆ 加權邊
查詢 路徑邊權連乘 中間項相消
C/D = 沿路徑 C/x₁ · x₁/x₂ · … · xₖ/D,中間全消 = C/D。
◆ 連乘
ADOPT
C BFS 帶累積權重找 C→D O(q·(V+E))
每個查詢從 C 出發 BFS,佇列存 (節點, 累積權重);每走一條邊就乘上邊權。碰到 D 回累積值;走不到 / 變數未定義回 -1。
● ADOPT · 加權 BFS
3

核心策略// STRATEGY

  1. 建圖:對每個 A/B=v,graph[A][B]=v、graph[B][A]=1/v(巢狀 hashmap)。
  2. 邊界:查詢 C/D 時,若 C 或 D 不在圖裡 → -1;若 C==D → 1.0。
  3. BFS:佇列存 (節點, 累積權重),起點 (C, 1.0);用 visit 防重複。
  4. 命中:遇到鄰居 == D → 回 累積 × 邊權;佇列空還沒到 → -1。

為什麼「連乘」就對// TELESCOPING

LEMMA · 中間項相消

一條從 C 到 D 的路徑 C → x₁ → x₂ → … → D,每條邊的權重就是頭除以尾:C/x₁、x₁/x₂、…、xₖ/D。把它們全部相乘:

(C/x₁) · (x₁/x₂) · … · (xₖ/D) = C/D

中間每個變數都出現一次分子、一次分母,剛好相消,只留下 C/D。所以「沿路徑連乘邊權」不管走哪條路徑、經過幾個中間變數,結果都是 C/D(題目保證無矛盾,所以每條路徑給同一個值)。這就是為什麼 BFS 找到任一條路徑即可回答。

6

複雜度// COMPLEXITY

T = O(q · (V + E))　　S = O(V + E)

V = 變數數、E = 等式數、q = 查詢數。每個查詢一次 BFS 走遍圖。空間為鄰接表 + visit + 佇列。

進階:若查詢非常多,可先用 Floyd-Warshall 預算所有配對(O(V³) 建表、每查 O(1)),或用並查集帶權(把每點壓到根、存「到根的比值」)。這裡 V、q 都 ≤ 20,直接 BFS 最省事。

7

一句洞察// INSIGHT

除法查詢 = 加權圖上的路徑連乘。A/B=v 建 v 與 1/v 兩條邊;BFS 帶累積權重走, 到終點回乘積、走不到回 −1。中間項相消是它成立的關鍵。
