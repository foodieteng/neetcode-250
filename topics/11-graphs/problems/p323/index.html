題目簡介// THE QUESTION

QUESTION

n 個節點(0..n−1)、一批無向邊 edges。 求圖裡有幾個連通分量(彼此相連的一團團)。

兩種經典解都要會:① DFS/BFS 洪水填充 —— 對每個沒訪過的點開一趟 DFS、cnt++,一趟把整片染掉; ② Union-Find —— 逐邊合併,最後數「還是根」的數量。這題是 261 合法樹 的近親(同一套並查集)。

2

兩種解法// TWO WAYS

A DFS / BFS 洪水填充 O(V+E)
建鄰接表。外層掃每個點,遇到未訪就 cnt++ 並 DFS 把整片標訪。DFS 次數 = 分量數。
● 直觀
B Union-Find(並查集) O(E·α)
逐邊 union。最後數 parent[i] < 0(還是根)的數量 = 分量數。或 cnt = n − 合併成功次數。
● 增量友善

兩者都 O(V+E) 等級。DFS 直觀好寫;Union-Find 在「邊會一條條進來、要隨時回答分量數」的動態場景更省。

3

核心策略// STRATEGY

  1. A · DFS:建無向鄰接表;for i in 0..n−1,若 i 未訪 → cnt++、dfs(i) 染整片。
  2. B · Union-Find:parent 全 −1;逐邊 Union(u,v);最後統計 parent[i] < 0 的個數。
  3. 兩者本質一樣:把「彼此相連」的點歸成一組,數共有幾組。

兩種寫法的核心// TWO CORES

LEMMA · 數 DFS 次數 & 數根

DFS:外層 for 只有在碰到「還沒被任何一趟 DFS 染到」的點時才 cnt++。那一趟 DFS 會把整個相連的塊全部標訪,所以同一塊不會被數第二次。cnt = 啟動 DFS 的次數 = 塊數。

Union-Find:每個連通塊在並查集裡剛好對應一個根。初始 n 個孤點 = n 個根 = n 塊;每次成功合併把兩個根併成一個 → 少一塊。所以最後「根的數量」= 塊數,也 = n − 成功合併次數。

6

複雜度// COMPLEXITY

寫法TimeSpace
A · DFS/BFSO(V+E)O(V+E)
B · Union-FindO(V+E·α(n))O(V)

Union-Find 空間只需一個 parent 陣列(不必存整張圖),α(n) 視為常數。

7

一句洞察// INSIGHT

數連通塊 = 數「有幾團」。DFS:每碰到未訪點就開一趟、cnt++;Union-Find:合併完數根。 兩者都 O(V+E) 等級,和 261 共用同一套並查集。
