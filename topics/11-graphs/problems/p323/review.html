觀念題// QUIZ

Q1. DFS 版為什麼「啟動 DFS 的次數」剛好等於連通塊數?為什麼同一塊不會被重複數?
Q2. Union-Find 版為什麼「根的數量」= 連通塊數?cnt = n − 合併次數 又是怎麼來的?
Q3. 無向圖建鄰接表時只 push 單向會怎樣?為什麼分量數會偏大?
Q4. 一個完全沒有邊的孤立點算不算一個連通塊?兩種寫法會不會自動處理?
Q5. 什麼情境下你會選 Union-Find 而不是 DFS?反過來呢?

範例 Trace// n=5

// 輸入 n = 5 edges = [[0,1],[1,2],[3,4]] 圖: 0—1—2 (一團) 3—4 (一團) 預期:2 個連通塊
// A · DFS i=0 未訪 → cnt=1;dfs 標 {0,1,2} i=1,2 已訪 → skip i=3 未訪 → cnt=2;dfs 標 {3,4} i=4 已訪 → skip return 2
// B · Union-Find parent 初始 [-1,-1,-1,-1,-1] (5 根) union(0,1) union(1,2) → 0 這棵含 {0,1,2} union(3,4) → 3 這棵含 {3,4} parent = [-3,0,0,-2,3] 數 <0 的:index 0、3 → cnt=2
兩種寫法從不同角度數同一件事:DFS 數「我啟動了幾趟」,Union-Find 數「合併後剩幾個根」。 每個連通塊在 DFS 裡對應一趟起點、在並查集裡對應一個根 —— 一一對應,所以答案一致。

應用題// APPLY

App 1. 若邊會一條一條動態加入,每加一條就要回報「現在剩幾塊」,哪種寫法幾乎不用改?怎麼即時維護?
App 2. 若還要回報最大連通塊的大小,兩種寫法各怎麼加?(提示:UF 的 −parent[root])
App 3. 把這題搬到二維網格(島嶼數量,200),DFS 版幾乎照搬;此時還需要顯式建鄰接表嗎?
// ANSWERS
A1. 外層 for 只有在碰到「還沒被任何一趟 DFS 標訪」的點時才 cnt++ 並啟動 DFS。而那一趟 DFS 會沿著邊把整個和它相連的塊全部標成 visited。所以塊內其他點之後在外層迴圈都會被 visit[i]==true 跳過,不會再啟動新的一趟。於是「啟動次數」和「塊數」一一對應。
A2. 並查集裡每個連通塊恰好一個根(該塊所有點最終都指向它)。所以掃一遍 parent、數 parent[i] < 0 的個數就是塊數。至於 n − 合併次數:一開始 n 個孤點 = n 塊;每次成功的 Union(兩個不同根)會把兩塊併成一塊、少一個根。合併 k 次就剩 n − k 塊。兩種算法給同一個數。
A3. 無向邊 (u,v) 代表 u、v 互通,DFS 從任一端都要能走到另一端。若只 graph[u].push_back(v) 而漏了反向,從 v 出發時看不到 u,本來連通的塊可能被拆開數兩次 → 分量數偏大。所以無向圖一定要 graph[u].push_back(v) 且 graph[v].push_back(u)。(Union-Find 沒這問題,Union(u,v) 本身對稱。)
A4. 算,一個孤立點自成一個連通塊。兩種寫法都會自動處理:DFS 版外層掃到那個沒被訪過的孤點時 cnt++、跑一趟只標它自己;Union-Find 版那個點從沒被 Union 過,parent[i] 仍是 −1(自己是根),數根時被算進去。所以只要外層掃每個 i(而非只掃邊),孤立點就不會漏。
A5. 選 Union-Find:邊動態進來、要邊加邊回答連通性 / 分量數(online);或只想省空間(不必存整張圖)。選 DFS/BFS:圖是一次性給定、還要順便做別的走訪(找路徑、算塊大小、標記);或圖以網格 / 隱式形式出現(鄰居用座標推,不必建表)。兩者都 O(V+E) 等級,選順手、貼合場景的。
App 1. Union-Find 幾乎不用改:維護一個計數 comp,初始 n;每次加邊呼叫 Union(u,v),若這次真的合併了兩個不同根就 comp−−。任何時刻直接回報 comp 即可,每次 O(α(n))。DFS 版每加一條邊都要重掃整圖,成本高得多 —— 這正是動態場景 UF 完勝的地方。
App 2. Union-Find:因為 −parent[root] 就是該塊 size,合併後順手 maxSize = max(maxSize, −parent[newRoot]) 即可,O(1) 維護。DFS:每趟 DFS 回傳它染了幾個點(遞迴累加),外層取最大。兩種都只是在原本的骨架上加一個「大小」的統計,不影響複雜度。
App 3. 對,島嶼數量(200)本質就是「網格上的連通分量數」。DFS 版照搬:外層雙重迴圈掃每格,遇到陸地且未訪 → cnt++、DFS 把整座島標掉。不需要顯式鄰接表:鄰居直接用座標 (r±1,c)、(r,c±1) 推出來(隱式圖),省下建表的空間。這也是為什麼「網格連通塊」常比一般圖更省 —— 結構規則、鄰居可算。
