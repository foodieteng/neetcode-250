完整實作 · 多源 BFS// SEED ALL GATES

p286.cpp C++17 
class Solution {
public:
    void wallsAndGates(vector<vector<int>>& rooms) {
        int m = rooms.size(), n = rooms[0].size();
        const int INF = INT_MAX;

        queue<pair<int, int>> q;
        for (int i = 0; i < m; i++) {             // 種下所有的「源」
            for (int j = 0; j < n; j++) {
                if (rooms[i][j] == 0) q.push({i, j}); // 所有門一起入隊
            }
        }

        const vector<pair<int, int>> dirs = {{-1, 0}, {1, 0}, {0, -1}, {0, 1}};
        while (!q.empty()) {
            int sz = q.size();
            for (int _ = 0; _ < sz; _++) {          // 一次處理「同一波」
                auto [r, c] = q.front(); q.pop();
                for (auto [dr, dc] : dirs) {
                    int nr = r + dr, nc = c + dc;
                    if (nr < 0 || nr >= m || nc < 0 || nc >= n || rooms[nr][nc] != INF) continue;
                    rooms[nr][nc] = rooms[r][c] + 1;   // 只填仍是 INF 的空房
                    q.push({nr, nc});
                }
            }
        }
    }
};

複雜度分析// COMPLEXITY

T = O(m·n)　　S = O(m·n)

每格最多入隊、出隊一次(第一次被填時);四個鄰居檢查是常數。所以總工作量 O(mn)。空間是佇列,最壞裝下大半個網格 O(mn)。

步驟成本說明
種下所有門O(mn)掃一遍找 0
逐層 BFS 擴散O(mn)每格進出佇列一次
額外空間O(mn)佇列(距離就地寫回 rooms)

注意 · 邊界與陷阱// CAUTION · PITFALLS

陷阱後果正解
只種一個門就 BFS只算到那扇門,非最近先把所有 0 一起入隊(多源)
沒有 != INF 的守衛重複填 / 覆蓋 / 死圈!= INF 當 visited,每格只填一次
另建 visited 陣列多餘牆(−1)、門(0)、已填都 != INF,自動略過
距離寫死成波次固定值算錯距離 = rooms[r][c] + 1(從當前格 +1)
先讀 rooms[nr][nc] 才驗界越界 · UB先驗界(|| 短路)再讀格值

細節走讀// WALKTHROUGH

// 為什麼多源 BFS 給的是「最近」門 把所有門放進第 0 波 = 當成一個「超級源」 BFS 一層一層擴,第 k 波的格離超級源恰好 k 步 = 離「最近那扇門」k 步 → 就是最短距離 所以不必對每個空房、或每個門各跑一次
// != INF 一行,身兼三種守衛 rooms[nr][nc] != INF 時 continue,擋掉三種格: 牆 (-1) → != INF,跳過(不可走) 門 (0) → != INF,跳過(它是源,不用再填) 已填的空房 (1,2,3…) → != INF,跳過(最近的門已先到) 所以「第一次碰到才填」自動成立,不需另開 visited
// sz 那層迴圈其實可省 int sz = q.size(); for(_ ...) 是「一次處理一整波」的寫法 但這題距離已存在格子裡(rooms[r][c]+1),不靠波次計數 所以就算拿掉 sz、直接 while 逐格處理,結果一樣正確 (保留 sz 只是讓「一波一波」的意象更清楚)
