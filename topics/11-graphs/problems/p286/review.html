重點回顧// RECAP

面向要點
問題本質每格到最近一個源(門)的最短距離
核心所有門一起入隊做多源 BFS,一波一波擴
最短性所有門同時擴 → 第一次碰到 = 最近門
!= INF一行同時擋牆(−1)、門(0)、已填 → 免 visited
距離rooms[r][c] + 1,就地寫回網格
複雜度時間 O(mn),空間 O(mn)

觀念題// QUIZ

Q1. 為什麼把所有門一起入隊的多源 BFS,填出來的就是「最近」門的距離?一句話說出關鍵。
Q2. rooms[nr][nc] != INF 這一行擋掉了哪三種格子?為什麼有它就不必另開 visited 陣列?
Q3. 某個空房被最近的門填了值之後,更遠的門也走到它時,為什麼不會把它覆蓋成更大的距離?
Q4. 新填的距離為什麼是 rooms[r][c] + 1,而不是「目前第幾波」的固定數字?
Q5. 程式裡 int sz = q.size() 那層「一次處理一整波」的迴圈,能不能省?為什麼這題距離不靠波次計數?

範例 Trace// 4×4 · 2 gates

// 初始(-1=牆 0=門 ∞=空房) ∞ -1 0 ∞ ∞ ∞ ∞ -1 ∞ -1 ∞ -1 0 -1 ∞ ∞ 門:(0,2) 與 (3,0) → 兩個都先入隊
// 一波一波擴散(距離) 第1波:(1,2)=1 (0,3)=1 (2,0)=1 第2波:(2,2)=2 (1,1)=2 (1,0)=2 第3波:(3,2)=3 (0,0)=3 第4波:(3,3)=4 每格只在「還是 ∞」時才填 → 最近的門先到、先填
// RESULT 3 -1 0 1 2 2 1 -1 1 -1 2 -1 0 -1 3 4 // OUTPUT: 每個空房 = 到最近門的距離
看 (0,0):離 (0,2) 那扇門要繞牆走 3 步、離 (3,0) 也是往下再往上… BFS 自動幫它挑最短的那條(3) —— 因為第 3 波才第一次碰到它,而那一波來自最近的門。

應用題// APPLY

App 1. 這題和 994 腐爛橘子本質相同,都是多源 BFS。994 問的是什麼?它為什麼需要「波次計數」而這題不用?
App 2. 若不填每格、只要回傳「離門最遠的空房距離」(可達的),程式怎麼改最省?
App 3. 若改用 DFS 從每個門填距離,行不行?為什麼容易算出非最短的值、要怎麼補救?
// ANSWERS
A1. 把所有門當成一個「超級源」放進第 0 波,BFS 一層一層擴,第 k 波的格離超級源恰好 k 步 = 離最近那扇門 k 步。所以每格第一次被碰到時填的,就是它到最近門的最短距離。
A2. 擋掉三種:牆 -1(不可走)、門 0(它是源、不用再填)、已填的空房 1,2,3…(最近的門已先到)。因為這三種都 != INF,一個條件就全過濾掉;唯一會被填的只有「還是 INF 的空房」。所以 != INF 本身就是「visited / 可填」判斷,不必另開陣列。
A3. 因為它一旦被填就 != INF,之後任何路徑(包含更遠的門)再走到它時,都會在 != INF 那行被 continue 跳過,不會重新賦值。而 BFS 的擴散順序保證最近的門最先到達 —— 先到先填、後到不改,自然就是最小值。
A4. 因為距離是相對於把它填進來的那一格:當前格 (r,c) 已經是「到最近門的距離」,它的鄰居再多走一步就是 +1。用 rooms[r][c]+1 讓每一格都從「上一格的正確距離」延伸,不需要另外追蹤現在是第幾波 —— 距離資訊已經藏在網格值裡。
A5. 可以省。 sz 那層是「一次清空一整波」的寫法,常用在需要知道經過幾層的題(如 994 要回傳分鐘數)。但這題距離已存在格子裡(rooms[r][c]+1),不靠波次計數,所以直接 while 逐格 pop、處理鄰居,結果完全一樣。保留 sz 只是讓「一波一波」的語意更清楚。
App 1. 994 問「所有新鮮橘子都腐爛需要幾分鐘(或 −1)」。它把所有腐爛橘子當多源、BFS 往外擴,每一波 = 一分鐘,答案是波次數,所以需要 sz 那層來數層數;286 只要每格的距離值、不需總層數,故可不數。兩題骨架都是「所有源一起入隊的多源 BFS」。
App 2. 照樣跑多源 BFS,但邊填邊記 ans = max(ans, rooms[nr][nc]);或用 sz 波次寫法、每處理完一整波 ans++,最後一波的層數就是最遠距離。兩種都 O(mn),只是把「填每格」順手收斂成一個最大值。
App 3. DFS 從門往外填會算錯:DFS 沿一條路徑深入,可能繞遠路先到某格、填了較大的值;之後短路徑到達時,若沒有「新值更小才更新」的判斷,就覆蓋不回來。要補救得改成「if (newDist < rooms[nr][nc]) 才更新並繼續」,而且同一格可能被多次重訪,最壞退化。BFS 天生按距離分層、先到即最短,才是這題的自然解。
