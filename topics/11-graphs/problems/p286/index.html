題目簡介// THE QUESTION

QUESTION

m×n 網格三種格子:-1 = 牆(不可走)、0 = 門、INF(2³¹−1)= 空房。 把每個空房填上「到最近一扇門的距離」(上下左右移動);到不了任何門就維持 INF。

對每個空房各自找最近的門會重複計算。反過來想:讓所有門「同時」往外擴散 —— 這正是多源 BFS。把所有門一起丟進佇列當起點,一層一層往外走,某個空房第一次被碰到的那一波,就是離它最近的門 → 最短距離。

每個空房都從自己出發 BFS 一次,格數 mn 個房間 × 每次 O(mn),平方級,TLE。

從每個門各做一次 BFS、每格取最小距離。正確,但門很多時要掃很多遍,重複工作。
△ REJECT · 重複掃
ADOPT
C 多源 BFS(所有門一起入隊) O(mn)
先把所有門一起放進佇列,再一層一層 BFS。所有門同時往外擴,某空房第一次被碰到 = 離它最近的門。 用 != INF 當「已訪」判斷,每格只填一次。一趟 O(mn) 全部搞定。
● ADOPT · ONE PASS
3

核心策略// STRATEGY

  1. 種下所有源:掃一遍網格,把每個 0(門)的座標都 push 進佇列。
  2. 逐層 BFS:每次取出佇列裡的一格 (r,c),看四個鄰居 (nr,nc)。
  3. 只填 INF:若鄰居出界、是牆/門/已填(!= INF)就跳過;否則 rooms[nr][nc] = rooms[r][c] + 1 並入隊。佇列空即完成。

為什麼「第一次碰到」= 最近// WHY SHORTEST

LEMMA · 多源 BFS 的最短性

BFS 按「與源的距離」一層一層擴展:第 k 波處理的所有格,離最近的源恰好 k 步。把所有門都放進第 0 波,等於把它們看成一個超級源 —— 每格到「超級源」的距離,就是到最近那扇門的距離。

!= INF 就是 visited。 某格第一次被填時,是最近的門先到;之後更遠的門再走到它,因為它已 != INF 而被跳過,不會被覆蓋成更大的值。牆(-1)、門(0)也都 != INF,自動被略過。

複雜度// COMPLEXITY

T = O(m·n)　　S = O(m·n)

每格最多入隊、出隊一次(第一次被填時),四個鄰居檢查是常數。總工作量與格數成正比。空間為 BFS 佇列,最壞 O(mn)。
