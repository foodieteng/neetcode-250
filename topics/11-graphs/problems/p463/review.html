重點回顧// RECAP

面向要點
問題本質周長 = 每個陸地格朝向水/出界的邊總數
DFS 機制踩出界 / 踩到水 → res += 1;踩到新陸地 → 標 -1 續走
錯誤筆記判斷要當 base case,不能寫在 dfs 呼叫前(pre-guard 會漏算)
去重grid[r][c] == -1 直接 return,避免重複與爆堆疊
複雜度時間 O(mn),空間 O(mn)(遞迴堆疊)

觀念題// QUIZ

Q1. 為什麼「周長」等於每個陸地格朝外邊的總和?內部邊為什麼不算?
Q2.(錯誤筆記)為什麼邊界判斷要放進 dfs 的 base case,而不是在鄰居迴圈裡呼叫前就 continue 掉?會怎麼錯?
Q3. grid[r][c] = -1 這行的作用是什麼?如果拿掉,程式會發生什麼?
Q4. 外層雙迴圈掃到第一個陸地格就 DFS —— 為什麼一次 DFS 就能算完整座島?後面的陸地格會被重複處理嗎?
Q5. base case 裡「出界檢查」和「grid[r][c] == 0」的順序能不能對調?為什麼?

範例 Trace// [[1,1],[1,0]] → 8

// GRID · 1=陸地 0=水 1 1 1 0 從 (0,0) 開始 DFS。dirs 順序 = 上、下、左、右。
// DFS 展開(每條「踩出島」記 +1) dfs(0,0) 標 -1 上(-1,0) 出界 res=1 下(1,0) 陸地 → dfs(1,0) 標 -1 上(0,0)=-1 已訪 return 下(2,0) 出界 res=2 左(1,-1) 出界 res=3 右(1,1) 水 res=4 左(0,-1) 出界 res=5 右(0,1) 陸地 → dfs(0,1) 標 -1 上(-1,1) 出界 res=6 下(1,1) 水 res=7 左(0,0)=-1 已訪 return 右(0,2) 出界 res=8
// RESULT 三個陸地格各自朝外的邊:(0,0)→2、(1,0)→3、(0,1)→3 // OUTPUT: 2 + 3 + 3 = 8
注意 dfs(1,0) 往上踩回 (0,0) 時,因為已標 -1 而 return(不加也不重走);而往「水」和「出界」的每一步都確實走進 base case、各 +1。這就是為什麼判斷要放在 dfs 裡面。

應用題// APPLY

App 1. 若網格有多座島,要求所有島的周長總和,這份程式要改嗎?為什麼?
App 2. 若面試官不准你改動輸入 grid(它是 const),你會怎麼調整去重的做法?
App 3. 把 DFS 換成不遞迴的「直接數」:雙迴圈掃每個陸地格、看右鄰與下鄰。怎麼寫、為什麼只看兩個方向就夠?
// ANSWERS
A1. 每格陸地有 4 條邊。一條邊若兩側都是陸地,它被兩個相鄰陸地格共用,是島的內部,不在外圍;一條邊若另一側是水或網格外,就沒有鄰居共用它,它就是外圍的一段周長。把每個陸地格「另一側非陸地」的邊加起來,正好數到、且只數到所有外圍邊 → 等於周長。
A2. 因為要數的東西正是「從陸地踩進水/出界」這個事件——它必須真的被執行到才數得到。如果在呼叫前就 if (出界 || grid[nr][nc]==0) continue;,那一步被攔在遞迴之外,res += 1 永遠不會跑,朝外的邊全部漏算,周長偏小。正確做法是照樣 dfs(nr,nc),把判斷放進 dfs 開頭的 base case,讓「踩出島」真的發生一次、加一次。
A3. grid[r][c] = -1 把走過的陸地標成「已訪」。少了它,兩個相鄰陸地格會互相無限遞迴(A 走到 B、B 又走回 A…),堆疊爆掉;就算不爆,同一格也會被重複展開、把內部邊誤當外邊數,答案錯。-1 讓每個陸地格只被展開一次。
A4. 因為題目保證只有一座島、且陸地四連通,從任一陸地格 DFS 都能走遍整座島,並在過程把所有陸地格標成 -1。等外層雙迴圈繼續掃到後面的陸地格時,它們已經是 -1(!= 1)而被 continue 跳過,不會重複 DFS。(即使有多島,這寫法也仍正確——見 App 1。)
A5. 不能對調。C++ 的 || 是短路求值:必須先確認 r,c 在界內,才能安全讀 grid[r][c]。若把 grid[r][c] == 0 放前面,對一個出界的 (r,c) 就會先做越界存取 → UB / crash。所以出界檢查一定在讀陣列之前。
App 1. 不用改。外層雙迴圈會對「還沒被標 -1 的每個陸地格」各起一次 DFS;不同島彼此不連通,各自被獨立走遍,res 一路累加就是所有島的周長總和。這份程式天生就支援多島。
App 2. 用一個獨立的 vector<vector<bool>> seen 取代就地 -1:base case 判斷 出界 || grid==0 照舊(讀不寫),進入陸地格時檢查/設定 seen[r][c] 而不是改 grid。代價是多 O(mn) 空間,但輸入保持不變。
App 3. 掃每個陸地格,先 res += 4;若右鄰是陸地 res -= 2,若下鄰是陸地 res -= 2。只看右和下,是因為每一對相鄰陸地只會被「左上那個」數到一次(共用邊被扣 2:雙方各少一條外邊),不重不漏。無遞迴、O(mn) 時間、O(1) 額外空間,是最省的寫法;DFS 版則更貼近「圖走訪」的通用模板。
