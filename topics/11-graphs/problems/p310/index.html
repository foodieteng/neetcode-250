題目簡介// THE QUESTION

QUESTION

n 個節點、n−1 條邊構成一棵樹。可以選任一節點當根,不同的根會得到不同的樹高。 找出所有能讓樹高最小的根,回傳它們(可能有 1 或 2 個)。

讓樹最矮的根,一定落在樹的中心(centroid) —— 一棵樹的中心永遠只有 1 或 2 個。 找它的巧招:像拓撲排序的反面,把葉子(度數 1)一層層往內剝,剝到只剩 ≤ 2 個節點,那就是中心 = 答案。

為什麼剝葉子// WHY PEEL LEAVES

A 對每個點當根各算一次樹高 O(n²)
每個根跑一次 BFS 求高,取最小。可行但 n 大時平方級,慢。
△ REJECT · 慢
觀察 最矮根 = 樹的中心 1 或 2 個
最遠葉子決定樹高;站在「離最遠葉子最近」的中心當根,高度才最小。中心必為 1~2 個。
◆ 中心
ADOPT
C BFS 從葉子一層層往內剝 O(n)
所有 deg==1 的葉子入隊,一次剝掉一整層,n 減去該層數量;鄰居 deg−1 若降到 1 就是下一層的葉。剩 ≤ 2 個時停,那就是答案。
● ADOPT · O(n)

核心策略// STRATEGY

  1. 特例:n==1 回 {0};n==2 回 {0,1}。
  2. 建圖 + 度數:無向鄰接表,記每點 deg。
  3. 初始葉入隊:所有 deg==1 的點入佇列。
  4. 逐層剝:當剩餘 n > 2,一次處理整層(sz = q.size()),n −= sz;每剝一葉,鄰居 deg−1,降到 1 就入隊。
  5. 收尾:佇列裡剩下的 1~2 個就是答案。

5

為什麼中心最多 2 個// AT MOST TWO

LEMMA · 每層對稱剝掉,收斂到中央

樹高由「從根到最遠葉子的距離」決定。每剝掉一層葉子,相當於把樹的「最長路徑」兩端各縮短一格。持續縮,最後停在最長路徑的正中間:若最長路徑的邊數是偶數,正中間是一個點;若是奇數,正中間是一條邊的兩個端點 = 兩個點。所以中心只可能是 1 或 2 個,不會更多。

站在中心當根,到最遠葉子的距離最短,樹高因此最小。這也是為什麼「一次剝一整層、剝到剩 ≤ 2」剛好停在答案上 —— 用 sz = q.size() 把「同一層」的葉子綁在一起處理,正是這個「對稱往內」的關鍵。

複雜度// COMPLEXITY

T = O(V + E) = O(n)　　S = O(n)

每個節點、每條邊各處理一次(剝一次)。樹的 E = n−1,所以線性。空間為鄰接表 + deg + 佇列。
