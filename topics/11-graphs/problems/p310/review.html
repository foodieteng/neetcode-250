重點回顧// RECAP

面向要點
問題本質找讓樹高最小的根 = 樹的中心
中心數量永遠 1 或 2 個
做法從葉子(deg==1)一層層往內剝
層剝sz = q.size() 綁定同一層
停止剩餘 n ≤ 2,佇列裡即答案
複雜度時間 O(n),空間 O(n)

觀念題// QUIZ

Q1. 為什麼「讓樹高最小的根」一定是樹的中心?從「最遠葉子」的角度說明。
Q2. 為什麼中心最多只有 2 個,不會是 3 個或更多?
Q3. while 迴圈的條件為什麼是剩餘 n > 2 而不是 !q.empty()?後者會怎樣?
Q4. sz = q.size() 這行為什麼是關鍵?若拿掉、逐個剝會出什麼問題?
Q5. 為什麼鄰居度數降到 1(而非 0)就入隊?

範例 Trace// two shapes

// 單中心 path 0-1-2-3-4(n=5) deg=[1,2,2,2,1] q={0,4} n=5 剝 0,4 → n=3;q={1,3} n=3>2,剝 1,3 → n=1;q={2} n=1,停 → {2}(一個中心)
// 雙中心 path 0-1-2-3-4-5(n=6) deg=[1,2,2,2,2,1] q={0,5} n=6 剝 0,5 → n=4;q={1,4} n=4>2,剝 1,4 → n=2;q={2,3} n=2,停 → {2,3}(兩個中心)
// 有分支 星形 + 尾巴 edges = [[3,0],[3,1],[3,2],[3,4],[5,4]] deg[3]=4, deg[4]=2, 其餘=1 q={0,1,2,5} 剝這一整層(sz=4)→ n=2;deg[3]→1、deg[4]→1;q={3,4} n=2,停 → {3,4}
不論樹長怎樣,「剝葉子」都在做同一件事:把最長路徑從兩端往中間縮。 最長路徑邊數偶數 → 停在一個點,奇數 → 停在一條邊的兩端(兩個點)。這就是為什麼答案恰好是 1 或 2 個。

應用題// APPLY

App 1. 若要順便回傳最小樹高是多少,程式怎麼改?(提示:數剝了幾層)
App 2. 這個「剝葉子」和 207/210 拓撲排序的「剝入度 0」有什麼異同?為什麼一個用 deg==1、一個用 indeg==0?
App 3. 若輸入不保證是樹(可能有環、或不連通),這套解法還對嗎?會發生什麼?
// ANSWERS
A1. 以某點當根時,樹高 = 從根到最遠葉子的距離。要讓這個「最遠距離」最小,根就得站在「離所有葉子都盡量近」的位置 —— 也就是樹裡最長路徑(直徑)的中點。這個中點就是樹的中心。站得離中心越遠,某一側的最遠葉子就離你越遠、樹高越大。所以最矮根一定是中心。
A2. 中心是樹直徑(最長路徑)的中點。一條路徑的中點:若它有偶數條邊(奇數個節點),正中間是單一節點;若有奇數條邊(偶數個節點),正中間落在一條邊上,對應兩個端點。中點不可能是三個以上的點,所以中心最多 2 個。剝葉子的過程正是每輪把直徑兩端各砍一格,必然收斂到這 1~2 個中點。
A3. 因為中心的度數最後也會降到 1、跟著進佇列。如果用 !q.empty() 當條件,迴圈會連中心一起剝掉,佇列清空、答案變空,錯。正確是用剩餘節點數 n:每剝一層 n −= sz,當 n ≤ 2 就收手,此時還沒被剝、留在佇列裡的 1~2 個正是中心。「用 n 控制何時停」是這題的骨架。
A4. sz = q.size() 把「當前這一整層的葉子」框出來,一次剝完再進下一層。這對應「直徑兩端同步往內縮一格」的對稱性。若不綁層、逐個 pop 到 n≤2,新加入的下一層葉子會混進來被當本層剝掉,破壞對稱、可能剝過頭、留下錯的中心。綁層是保證「一層一層、對稱收斂」的關鍵(和 BFS 分層計數同一招)。
A5. 剝掉一個葉子等於拿掉它那條唯一的邊,鄰居因此少一條邊、度數 −1。若鄰居度數從 2 掉到 1,代表它現在只剩一條邊 = 變成新的葉子,該進下一層待剝,所以「降到 1」就入隊。降到 0 表示被完全孤立(它是最後的中心、周圍都被剝光),那不該再剝 —— 這種情況由 n ≤ 2 的停止條件擋住,不會靠「降到 0 入隊」處理。
App 1. 加一個層數計數器:每進一次 while 迴圈(剝掉一層)就 height++。剝完停下時,height 就是「從中心到最遠葉子的層數」= 最小樹高。因為剝的層數正好等於中心到最外圈葉子的距離。(若中心有 2 個,兩者樹高相同,取任一即可。)零額外成本,順手就能回傳。
App 2. 同:都是「反覆移除邊界節點、更新鄰居度數、用佇列分層」的 BFS 剝皮。異:拓撲排序在有向圖上,邊界是入度 0(沒有前置)的點,目標是排出順序、剝到全空;本題在無向樹上,邊界是度數 1的葉,目標是找中心、剝到剩 ≤ 2 就停。方向不同(有向看入度、無向看總度)、停止點不同(全空 vs 剩 2),但「從邊界往內剝」的骨架一模一樣。
App 3. 這套解法假設輸入是樹(n 點 n−1 邊、連通、無環)。若有環:環上的點度數都 ≥ 2,永遠不會降到 1、剝不掉,迴圈會在 n > 2 卡住(q 提早空掉但 n 還 > 2)甚至無法終止/回錯。若不連通:每個連通塊各有自己的中心,單一「剩 ≤ 2」的邏輯會混亂。所以要先確保是樹;否則得改題意(例如各連通塊分別找中心,或先判環)。本題的 constraints 已保證是樹,不必處理這些。
