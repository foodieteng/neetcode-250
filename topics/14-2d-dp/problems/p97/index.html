題目簡介// THE QUESTION

QUESTION

給 s1、s2、s3,問 s3 能否由 s1 和 s2 交錯組成 —— 兩者各自的字元保持原順序,交替穿插即可。例 s1="ab"、s2="cd" → s3="acbd" ✓。

雙字串 DP:dp[i][j] = 「s1 前 i 字 + s2 前 j 字」能否交錯成「s3 前 i+j 字」。 最漂亮的地方:s3 的位置不需要第三個維度 —— 用掉 i+j 個字就剛好填滿 s3 前 i+j 位,所以待配字元固定是 s3[i+j−1]。

2

兩種來源// FROM s1 or s2

核心 · s3 的最後一字從誰來?

要湊出 s3 的第 k = i+j−1 個字,它只可能來自 s1 的最後一字或 s2 的最後一字 —— 兩種情況 OR:

● 從 s1 拿(上):dp[i−1][j] && s1[i−1] == s3[k]
前面用 i−1 個 s1 字已經湊好,現在這個 s1 字剛好對上 s3[k]。
● 從 s2 拿(左):dp[i][j−1] && s2[j−1] == s3[k]
前面用 j−1 個 s2 字已經湊好,現在這個 s2 字剛好對上 s3[k]。

兩者任一成立即可(OR)。注意:不只是字元對上就好 —— 還要「前一格為 true」(前面真的能交錯出來),兩個條件必須同時成立。

3

核心策略 · 三步驟// STRATEGY

  1. 先擋長度:m + n != s3.size() → 直接 false(字數對不上,免談)。
  2. base:dp[0][0]=true;第一行/列 = 只用單一字串時,逐字對 s3 前綴(一旦不符,之後全 false)。
  3. 轉移:dp[i][j] = (dp[i−1][j] && s1[i−1]==s3[i+j−1]) || (dp[i][j−1] && s2[j−1]==s3[i+j−1]);答案 dp[m][n]。

為什麼兩維就夠// k = i+j−1

LEMMA · s3 的位置被 i+j 綁死

直覺上狀態好像要三個維度:(用了幾個 s1 字, 幾個 s2 字, 填到 s3 哪裡)。但交錯的定義決定了 —— 每個字都必須用掉、且都填進 s3,所以「用了 i 個 s1 字 + j 個 s2 字」必然剛好填滿 s3 的前 i+j 位。

於是第三維是前兩維的函數(k = i+j),可以消掉 —— 二維就夠,待配字元固定是 s3[i+j−1]。這是 DP 設計裡很常見的一招:發現某個維度可由其他維度算出來,就把它消掉,狀態立刻降一維。

6

複雜度// COMPLEXITY

T = O(mn)　　S = O(mn) → 壓成 O(n)

每格 O(1)。m,n ≤ 100。因為只依賴上與左(沒有斜對角!),可直接壓成一維 O(s2.length) —— 正好回答題目的 follow-up。

7

一句洞察// INSIGHT

s3 的位置 = i+j−1(第三維被綁死,可消掉)→ 二維就夠。 每格問「s3 這個字從 s1(上)還是 s2(左)拿」,兩者 OR;只依賴上/左 → 壓一維 O(n)。
