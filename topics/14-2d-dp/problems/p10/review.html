重點回顧// RECAP

面向要點
狀態dp[i][j] = s 前 i 字能否被 p 前 j 字完整匹配
字母 / .對上 → 左上 dp[i−1][j−1]
* 零次dp[i][j−2](丟掉 x*,無條件,左跳 2)
* 一次以上(p[j−2]配 s[i−1]) && dp[i−1][j](吃輸入,上跳 1)
第一列空 s 只有 x*(零次)存活,單獨初始化
兩箭頭零次動 p(j−2)· 一次動 s(i−1)
複雜度O(mn);純遞迴無 memo 會指數爆

觀念題// QUIZ

Q1. '*' 的兩條分支「零次」和「一次以上」分別對應 dp 表的哪個方向(j−2?i−1?)?為什麼是這兩個方向?
Q2. 「一次以上」為什麼要先檢查 p[j−2] == s[i−1],「零次」卻不用檢查任何字元?
Q3. a* 要配兩個 a,dp 表是怎麼「連吃兩次」的?哪個索引在動、哪個不動?
Q4. 為什麼第一列 dp[0][j] 要單獨初始化?不設會漏掉哪種匹配?dp[0][j] 為什麼從 j=2 開始?
Q5. 這題和 97 交錯字串 都是兩字串 DP。為什麼 97 只看「上 / 左」、10 卻要看「左上 / 左跳 2 / 上跳 1」這麼多方向?根源差在哪?

範例 Trace// s="ab", p=".*"

// .* = 「任意字元的零或多個」= 配任何字串 '' . * '' T F T a F T T b F F T dp[0][2]=T : .* 取零次 → dp[0][0]=T dp[1][1]=T : '.'配 s[0]='a' → 左上 dp[0][0]=T dp[1][2]=T : .* 吃第一個('.' 配 a) → 上 dp[0][2]=T dp[2][2]=T : .* 再吃第二個('.' 配 b) → 上 dp[1][2]=T return dp[2][2] = true —— .* 靠 '.' 通配 + 往上連吃,配掉整個 "ab"
.* 是最強的模式:'.' 讓「一次以上」的字元檢查永遠通過(通配),'*' 讓它能往上無限吃。 兩者合體 = 配任意字串。理解 .* 怎麼在表上「一路往上吃到底」,就抓到了 '*' 機制的精髓。

應用題// APPLY

App 1. LC 44 Wildcard Matching 的 '*' 是「配任意長度任意字元」(不綁前一字元),'?' 配任意單一字元。轉移和本題差在哪?哪個其實更簡單?
App 2. 若要把空間壓成 O(n)(一維滾動),要注意什麼?'*' 的「上跳 1」dp[i−1][j] 會不會有麻煩?
App 3. 若改成記憶化遞迴寫,函式怎麼設?為什麼有人覺得遞迴版比填表版更好想?
// ANSWERS
A1. ● 零次 → dp[i][j−2](左跳 2):把 "x*" 當作不存在,等於刪掉 p 的最後兩個字元(x 和 *),而 s 完全沒動 → j 減 2、i 不變。
● 一次以上 → dp[i−1][j](上跳 1):"x*" 吃掉一個 s 字 s[i−1] → i 減 1;但 "x*" 還沒用完、留在原地繼續(j 不變)。
為什麼是這兩個方向:'*' 的語意是「前一元素的零或多個」。「零個」= 不用它 = 動模式;「多個」= 反覆用它吃字元 = 動輸入、模式不變。零次橫跳(丟模式)、一次直跳(吃輸入),兩個方向 OR 起來,剛好覆蓋「零 or 一 or 二 or …」所有可能。
A2. 因為「一次以上」的動作是真的拿 x* 去配一個 s 字 —— 那當然要求 x(即 p[j−2])配得上這個字 s[i−1](相等或 x 是 '.')。少了這個檢查,a* 會去吃 b、c 任意字元,退化成 .*,亂配一通。反例 s="b"、p="a*":正確是 false(a* 只配 a 或空),漏檢查會錯判 true。
而「零次」的動作是把 x* 整組丟掉、根本不碰 s —— 既然不拿它配任何字,就沒有「配得上嗎」的問題,無條件看 dp[i][j−2] 即可。
一句話:吃字元才驗字元,丟模式不驗。兩條分支的字元檢查有無,正好反映它們「碰不碰 s」的本質差異。
A3. 靠「上跳 1、j 不動」連續兩次。以 p="c*a*b"、s="aab" 為例,a* 在第 j=4 欄:
● dp[1][4](配第一個 a):'a' 配 s[0]='a' && dp[0][4]=T → T
● dp[2][4](配第二個 a):'a' 配 s[1]='a' && dp[1][4]=T → T
每往下一列(i 加 1 = 多吃一個 s 字),都指回同一欄的上一格 dp[i−1][4]。i 一直在動(吃輸入)、j 釘死在 4(a* 沒用完)。想吃幾個就往上連幾格 —— 這就是 '*' 用「上跳 1」表達「零或多個」的機制。模式不動、輸入前進,是「重複使用同一個 x*」的 DP 化身。
A4. 第一列 dp[0][j] = 「空的 s 能否被 p 前 j 字配上」。空 s 要被配,p 就只能是一串全取零次的 x*(任何實體字元都會多出東西)。這條路不會被主迴圈算到 —— 主迴圈從 i=1 起(至少一個 s 字),所以「p 開頭一串 x* 全取零」這個起點必須手動初始化。
不設會漏:例 p="a*b" 配 s="b" —— 需要 a* 取零、再用 b 配 s 的 b。而 b 那格會回頭讀 dp[0][2](a* 取零配空);若 dp[0][2] 沒設成 T,這條就斷了。
為什麼從 j=2:x* 至少要 2 個字元(x + *),所以 j=1 不可能是完整的 x*(dp[0][1] 恆 false:單一實體字元配不了空 s)。從 j=2 起才有 p[j−1]=='*' 且能讀 dp[0][j−2]。
A5. 根源是「一個模式字元消耗多少輸入」不同。
● 97 交錯字串:每一步固定「拿一個字配一個字」—— 要嘛從 s1 拿(上)、要嘛從 s2 拿(左),一對一、長度固定。所以只有上 / 左兩個方向,而且沒有左上(它不比較兩字串的同一字元)。
● 10 正則:模式字元和輸入不是一對一 —— 字母/. 是一對一(左上),但 '*' 可以配零個(模式吃 2 格、輸入吃 0 → 左跳 2)或多個(模式吃 0、輸入吃 1 → 上跳 1)。「一個模式能對應變長的輸入」逼出了多個方向。
通則:轉移方向的數量 = 一步能有幾種「消耗組合」。一對一 → 方向少(97);變長匹配(*)→ 方向多(10)。這也是 10 是 Hard、97 是 Medium 的本質原因 —— '*' 的變長性是全部難度所在。
App 1. LC 44 的 '*' 不綁前一字元,自己就是「任意長度任意字元」。轉移:遇 '*' → dp[i][j] = dp[i][j−1](配零個) || dp[i−1][j](多吃一個 s 字) —— 「多吃」不用檢查字元(* 通配任何字),這反而比本題簡單;'?' 或字母相等 → 左上 dp[i−1][j−1]。
關鍵差異:本題 '*' 是「前一字元的零或多個」(綁定、要驗字元、左跳 2 取零);44 的 '*' 是「任意字元的零或多個」(獨立、不驗字元、左跳 1 取零)。10 的 * 更受限,反而更難寫對(那個字元檢查和 j−2 都是坑)。
App 2. 可以壓成 O(n),但要小心 '*' 的「上跳 1」dp[i−1][j]。一維滾動時 dp[j] 同時是「本列新值」和「上列舊值」:
● 左上 dp[i−1][j−1] 和 左跳 2 dp[i][j−2] 用一般手法(存 prev 對角、左往右掃)可處理。
● 上跳 1 dp[i−1][j] = 「同一欄、上一列」= 一維裡 dp[j] 更新前的舊值 —— 剛好還沒被本列覆蓋,直接讀就對(和 97 讀「上」一樣)。
所以真正要小心的是左上(會被本列左鄰覆蓋,需 prev 暫存)和更新順序。實務上 m,n≤20,二維表最清楚,面試講得出「只依賴 i−1 列 → 可壓一維」即可,不必硬寫。
App 3. 記憶化遞迴:bool dp(i, j) = 「s[i..] 能否配 p[j..]」(用後綴視角最順)。
if (j == n) return i == m;（模式用完,輸入也要剛好用完)
bool first = i < m && (p[j]==s[i] || p[j]=='.');
if (j+1 < n && p[j+1]=='*') return dp(i, j+2) || (first && dp(i+1, j));（零次 OR 一次以上）
return first && dp(i+1, j+1);（字母/.）
用 memo[i][j] 快取。
為什麼更好想:遞迴直接照著 '*' 的定義寫 ——「試零次或試吃一個」几乎是題目語意的翻譯,不用先想「依賴方向對不對、要不要單獨初始化第一列」。填表版的威力是常數小、無遞迴開銷,但要自己保證填表順序；記憶化把順序交給遞迴、memo 去重,心智負擔更低。兩者同 O(mn),是同一個 DP 的兩種驅動(和 329 的 memo DFS vs Kahn 同理)。
