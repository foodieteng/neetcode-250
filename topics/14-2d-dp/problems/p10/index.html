題目簡介// THE QUESTION

QUESTION

給字串 s 和模式 p,判斷 p 能否完整匹配 s。'.' 配任意單一字元;'*' 配前一個元素的零或多個。 例 s="aab"、p="c*a*b" → true(c*=零個 c、a*=兩個 a、b=b)。

同屬兩字串 DP(和 115、1143、97 一家),但 '*' 讓它變 Hard —— 因為 '*' 不是配「一個字元」,而是配可長可短(含零)的一段。難點全在怎麼拆 '*'。

2

核心 · 看 p 的當前字元是什麼// TWO CASES

dp[i][j] = 「s 前 i 字」能否被「p 前 j 字」完整匹配。轉移只看 p[j−1] 是哪一種:

情況 A · p[j−1] 是字母或 '.'(配一個字元)

若 s[i−1] == p[j−1] 或 p[j−1] == '.' —— 這個字元對上了,兩邊各消一個字,退到左上:

dp[i][j] = dp[i−1][j−1]

情況 B · p[j−1] 是 '*'(配「前一元素 p[j−2]」的零或多個)

'*' 和它前面那個字元 p[j−2] 綁成一組 "x*"。有兩條路,任一成立即可(OR):

● 零次:"x*" 當作不存在,跳過這兩個模式字元 → dp[i][j−2]
例 c* 配零個 c —— 直接無視 c*,看 p 更前面。

● 一次以上:若 p[j−2] 能配 s[i−1](相等或 '.'),就吃掉一個 s 字、"x*" 留著繼續 → dp[i−1][j]
例 a* 配第二個 a —— 消一個 s 的 a,a* 還在(j 不動),下一步還能再吃。

3

為什麼「零次」看 j−2、「一次」看 i−1// THE ARROWS

兩個箭頭,方向完全不同

● 零次 → dp[i][j−2](往左跳 2 格)
「不用 x*」= 把 p 的最後兩個字元(x 和 *)整組刪掉,s 一個字都沒動(i 不變),所以 j 減 2、i 不變。

● 一次以上 → dp[i−1][j](往上跳 1 格)
「x* 再吃一個 s 字」= s 消掉 s[i−1](i 減 1),但 x* 還沒用完、留在原地繼續(j 不變)。這就是 '*' 能配「多個」的來源 —— j 不動,可以反覆往上吃。

記憶法:零次是「丟掉模式」(動 p、j−2),一次是「吃掉輸入」(動 s、i−1)。 前者橫跳、後者直跳,兩個方向合起來就把 '*' 的「零或多個」完整覆蓋。

第一列 · 空 s 的 base// EMPTY s

dp[0][j]:什麼樣的 p 能配「空字串」?

dp[0][0] = true(空配空)。空的 s 要被非空 p 配上,p 就只能全是「零次」的 x* —— 任何實體字元都會多出東西、配不了空。

dp[0][j] = (p[j−1] == '*' && dp[0][j−2])

例 p="c*a*" 配空:c* 零個、a* 零個 → dp[0][4]=true。這一列必須單獨初始化 —— 它是所有「s 前綴用零次 x* 開頭」的種子,漏了它,整張表會少掉「模式開頭一串 x* 全取零次」的路徑。

複雜度// COMPLEXITY

T = O(mn)　　S = O(mn) → 可壓 O(n)

m,n ≤ 20,填 (m+1)(n+1) 格、每格 O(1)。純遞迴(不 memo)會因 '*' 的分支指數爆炸 —— DP / 記憶化把它壓成多項式。
