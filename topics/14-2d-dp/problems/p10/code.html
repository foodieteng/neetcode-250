完整實作// O(mn) TIME · O(mn) SPACE

p10.cpp C++17 
class Solution {
public:
    bool isMatch(string s, string p) {
        int m = s.size(), n = p.size();
        vector<vector<bool>> dp(m + 1, vector<bool>(n + 1, false));
        dp[0][0] = true;                     // 空配空

        // 第一列:空 s 只能被 "x*"(零次)配上
        for (int j = 2; j <= n; j++)
            dp[0][j] = (p[j - 1] == '*' && dp[0][j - 2]);

        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                if (s[i-1] == p[j-1] || p[j-1] == '.') {
                    dp[i][j] = dp[i][j] || dp[i-1][j-1];   // 字母/. → 左上
                } else if (p[j-1] == '*') {
                    bool zero = (j-2 >= 0 && dp[i][j-2]);          // x* 取零次
                    bool oneMore = (p[j-2] == s[i-1] || p[j-2] == '.')
                                   && dp[i-1][j];                         // x* 再吃一個
                    dp[i][j] = dp[i][j] || zero || oneMore;
                }
            }
        }
        return dp[m][n];
    }
};

整段邏輯就是「看 p[j−1] 是字母/. 還是 '*'」兩大分支。 '*' 那支的 zero(左跳 2)和 oneMore(上跳 1)用 || 併起來 —— 只要一條路通,這格就是 true。

關鍵 · oneMore 前面那個字元檢查// THE GUARD

為什麼「再吃一個」前要先看 p[j−2] == s[i−1]

"x*" 想「再吃掉一個 s 字」s[i−1],前提是 x(也就是 p[j−2])真的配得上這個 s 字 —— 相等,或 x 是萬用的 '.'。

少了這個檢查會出大錯:a* 會去吃 b、c… 任何字元,等於把 a* 當成 .*。例 s="b"、p="a*" 應是 false(a* 只能配 a 或空,配不了 b),漏檢查就會誤判 true。

對比「零次」不用檢查字元:取零次是「當 x* 不存在」,根本不碰 s,所以只看 dp[i][j−2],無條件可嘗試。吃字元才要驗字元,丟模式不用。

3

為什麼要 DP / 記憶化// WHY NOT PLAIN RECURSION

LeetCode 官方的樸素解是遞迴:遇到 '*' 就「試零次 OR 試吃一個再遞迴」。它對、但慢 —— '*' 的兩條分支會讓同一個 (i,j) 子問題被重複計算指數多次。

// 樸素遞迴(正確但指數) bool rec(i, j): if j 到底: return i 到底 first = (i 沒到底) && (p[j]==s[i] || p[j]=='.') if p[j+1]=='*': return rec(i, j+2) // 零次 || (first && rec(i+1, j)) // 一次以上 return first && rec(i+1, j+1) // 字母/. 問題:rec(i, j+2) 和 rec(i+1, j) 會不斷觸發重疊子問題 → 最壞指數。加 memo[i][j] 或改填表 → O(mn)
填表版和記憶化遞迴是同一個 DP 的兩種寫法(bottom-up vs top-down),都 O(mn)。 填表要自己想清楚依賴方向(左上 / 左跳 2 / 上跳 1,都指向已算好的格子),記憶化則照遞迴定義寫、memo 去重就好。

邊界與陷阱// PITFALLS

陷阱症狀正解
oneMore 漏檢字元 a* 當成 .* 亂吃。"b"/"a*" 誤判 true 先驗 p[j−2]==s[i−1] 或 '.'
第一列沒初始化 漏掉「模式開頭一串 x* 全取零」的路徑 → 少匹配 dp[0][j] = (p[j−1]=='*' && dp[0][j−2])
零次也去檢字元 把 dp[i][j−2] 加上字元條件 → 該通的路被擋 零次無條件,只看 dp[i][j−2]
j−2 越界 '*' 在最前面時 j−2 < 0(題目保證不會,但防禦性) 判 j−2 >= 0(題目已保證 * 前有字元)
把 '*' 當「配任意」 誤以為像 shell 萬用字元。這裡 * 是「前一字元的零或多個」 * 綁 p[j−2],不是獨立萬用
只判部分匹配 回傳「p 是 s 子串」而非完整匹配 答案是 dp[m][n](整條配完)

細節走讀// s="aab", p="c*a*b"

// 完整 dp 表(實跑) '' c * a * b '' T F T F T F a F F F T T F a F F F F T F b F F F F F T return dp[3][5] = true
// 幾個關鍵格子 dp[0][2]=T : p="c*" 取零次 c → dp[0][0]=T (第一列,零次) dp[1][3]=T : p[2]='a'==s[0]='a' → 左上 dp[0][2]=T (字母匹配) dp[1][4]=T : a* 吃第一個 a:'a'配 s[0] && dp[0][4]=T (一次以上,往上) dp[2][4]=T : a* 再吃第二個 a:'a'配 s[1] && dp[1][4]=T (一次以上,又往上) dp[3][5]=T : p[4]='b'==s[2]='b' → 左上 dp[2][4]=T (字母匹配) a* 靠「往上吃」連吃兩個 a(dp[0][4]→dp[1][4]→dp[2][4]),這就是 * 配「多個」

看 a* 怎麼配兩個 a:dp[1][4] 靠 dp[0][4](吃第一個)、dp[2][4] 靠 dp[1][4](吃第二個)—— 每次 i−1 往上一格、j 不動,a* 原地反覆吃。這就是「上跳 1」表達「零或多個」的方式。
