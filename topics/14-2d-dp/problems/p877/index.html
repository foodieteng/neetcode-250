這題的「淨勝分」框架是一整類博弈 DP 的模板,兩題很值得一起練:
● LC 486 · Predict the Winner —— 幾乎同一題,只是堆數不限奇偶、值不限,dp[0][n−1] ≥ 0 即先手不輸。
● LC 1690 · Stone Game VII —— 得分改成「拿走後剩下的總和」,轉移要配合前綴和,是淨勝分框架的進階變形。

1

題目簡介// THE QUESTION

QUESTION

一排石頭堆 piles(偶數堆、總和為奇)。Alice、Bob 輪流從最左或最右拿走一整堆,Alice 先手。 兩人都最優,問 Alice 能不能拿到比較多石頭(回傳 true/false)。

「輪流從兩端拿、拿完換人」是區間博弈的招牌。這題有兩個層次值得學: 標準的 區間 DP(以及它漂亮的「淨勝分」改寫),還有一個一行解的數學捷徑。

2

核心 · 「淨勝分」改寫// SCORE DIFFERENCE

關鍵 · 別分開記兩人分數,只記「差」

直覺會想開 dp[i][j][誰的回合] 記兩人各自的分。但有個更省的視角: dp[i][j] = 輪到某人、面對 piles[i..j] 這段時,他能拿到的「淨勝分」(自己 − 對手)。

為什麼可以不管「是誰」?因為博弈是對稱的 —— 不論輪到 Alice 還是 Bob,面對同一段石頭,最優策略和能拿到的淨勝分都一樣。所以狀態不需要記玩家,只跟區間 [i,j] 有關。

先手拿左端 piles[i] 後,換對手在 piles[i+1..j] 當先手 —— 對手的淨勝分是 dp[i+1][j],對我而言是負的。所以:

dp[i][j] = max( piles[i] − dp[i+1][j],　piles[j] − dp[i][j−1] )

那個減號是整題的靈魂:它把「對手也會最優反擊」這件事,優雅地折進同一個 dp —— 不用第二張表、不用記玩家。最後 dp[0][n−1] > 0 ⇔ Alice 淨勝 ⇔ true。

3

核心策略 · 依「區間長度」填// STRATEGY

  1. base:dp[i][i] = piles[i] —— 只剩一堆,先手全拿,淨勝 = 該堆值。
  2. 填表順序:按區間長度 L 由小到大(L=2,3,…,n)。因為 dp[i][j] 依賴更短的 dp[i+1][j]、dp[i][j−1],短的必須先算好。
  3. 轉移:dp[i][j] = max(piles[i] − dp[i+1][j], piles[j] − dp[i][j−1]);答案 dp[0][n−1] > 0。

這就是區間 DP 的標準骨架:狀態是一個區間 [i,j],轉移看更短的子區間,按長度遞增填。 填出來的是一張上三角表。

數學捷徑 · 其實 Alice 永遠贏// return true;

LEMMA · 偶數堆 + 總和為奇 ⇒ 先手必勝

把堆依 index 分成兩組:奇數位(1,3,5…)和偶數位(0,2,4…)。因為堆數是偶數,兩端剛好一個奇位、一個偶位。

Alice 先手可以「鎖定」其中一組:如果她決定只拿偶位,每次對手拿掉一端後,暴露出來的另一端永遠是偶位(反之亦然)。所以她能整組拿到「全偶位」或「全奇位」——由她挑。

兩組總和相加 = 全部石頭 = 奇數,所以兩組不可能相等,必有一組較大。Alice 挑大的那組 → 穩贏。 所以在本題約束下,答案恆為 true,一行 return true; 就對。實測 9976 組(偶堆、奇和)全部 true。

但別只寫捷徑。面試想看的是區間 DP —— 因為換成 486(堆數任意、可平手)或 1690(改變計分)時,捷徑就失效,只有 DP 撐得住。捷徑當彩蛋、DP 當正解。

6

複雜度// COMPLEXITY

DP:T = O(n²)　S = O(n²)　　捷徑:T = O(1)

區間 DP 填上三角 ~n²/2 格,每格 O(1) → O(n²),n ≤ 500 輕鬆。 空間可壓成 O(n)(滾動一維,只留上一條對角線)。數學捷徑則是 O(1) 一行解。
