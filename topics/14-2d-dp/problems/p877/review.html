重點回顧// RECAP

面向要點
狀態dp[i][j] = 先手面對 piles[i..j] 的淨勝分(自己 − 對手)
不記玩家博弈對稱,淨勝分只跟區間有關
轉移max(piles[i] − dp[i+1][j], piles[j] − dp[i][j−1])
減號換對手當先手 → 對手淨勝分對我為負
填法對角線 base,依區間長度 L 遞增填上三角
答案dp[0][n−1] > 0;本題約束下恆真
捷徑偶堆 + 奇和 ⇒ 先手鎖定全奇/全偶位 ⇒ 必勝

範例 Trace// [3,7,2,3] → true

// dp 上三角(實跑,piles=[3,7,2,3]) j=0 j=1 j=2 j=3 i=0 3 4 −2 5 i=1 . 7 5 6 i=2 . . 2 1 i=3 . . . 3 L=2: dp[0][1]=max(3−7, 7−3)=4 dp[1][2]=max(7−2, 2−7)=5 dp[2][3]=max(2−3, 3−2)=1 L=3: dp[0][2]=max(3−dp[1][2], 2−dp[0][1])=max(3−5, 2−4)=−2 ← 負的! dp[1][3]=max(7−dp[2][3], 3−dp[1][2])=max(7−1, 3−5)=6 L=4: dp[0][3]=max(3−dp[1][3], 3−dp[0][2])=max(3−6, 3−(−2))=max(−3, 5)=5 dp[0][3] = 5 > 0 → Alice 淨勝 5 分 → return true
// 對照捷徑:全奇位 vs 全偶位 piles = [3, 7, 2, 3] index: 0 1 2 3 偶位(0,2): 3 + 2 = 5 奇位(1,3): 7 + 3 = 10 10 ≠ 5(總和 15 為奇)→ Alice 鎖定奇位,拿 10 > 5 → 淨勝 10 − 5 = 5 = dp[0][3] = 5,DP 與捷徑指向同一策略
注意 dp[0][2] = −2:單看 piles[0..2]=[3,7,2] 這段,先手其實會輸 2 分(拿不到中間的 7)。 負的淨勝分完全合法 —— 它是「這個子局面對先手不利」的誠實記錄。dp[0][3] 用到它時 3−(−2)=5,負負得正反而幫了 Alice:把對手在那段的劣勢轉成自己的優勢。這正是減號框架的精妙處。

應用題// APPLY

App 1. 486 Predict the Winner:堆數任意、值任意、可平手,先手不輸算贏。程式要動哪裡?捷徑還能用嗎?
App 2. 若要回傳先手實際多拿幾分(不只 true/false),程式怎麼改?若還要印出先手每一步拿左還拿右呢?
App 3. 1690 Stone Game VII:每次拿一端,得分 = 拿完後剩下堆的總和。淨勝分框架要怎麼配合前綴和改寫?
// ANSWERS
A1. 靠博弈的對稱性:不論輪到 Alice 還是 Bob,面對同一段 piles[i..j],可用的動作(拿左 / 拿右)和最優策略完全一樣,能拿到的「淨勝分」也一樣。玩家的身分不影響這段的最優值,所以狀態只需要區間 [i,j],不需要「誰的回合」這一維。
這是博弈 DP 的關鍵簡化:只要把分數記成「當前行動者相對於對手」的差,玩家維度就消失了,三維降二維。反之,如果你記的是「Alice 的絕對分數」,那就非記玩家不可 —— 因為同一段對 Alice 是進攻、對 Bob 是防守,值不同。選對「相對」的視角,是省掉一維的前提。
A2. 因為拿完左端 piles[i] 後,換對手在 piles[i+1..j] 當先手。dp[i+1][j] 是「對手在那段的淨勝分」——對手贏多少 = 我輸多少,所以從我的角度是 −dp[i+1][j]。我的總淨勝 = 剛拿的 piles[i] 減去對手接下來的優勢 = piles[i] − dp[i+1][j]。
寫成加會怎樣:piles[i] + dp[i+1][j] 等於假設「對手也幫我加分」——那是兩人合作把總分最大化,不是對抗。算出來的是「兩人聯手能湊多少」,和誰贏無關,完全錯題。減號 = 對抗;加號 = 合作。這個符號就是「minimax」的化身。
A3. 因為 dp[i][j] 依賴更短的兩個子區間 dp[i+1][j](去掉左端)和 dp[i][j−1](去掉右端)。這兩者的長度都是 L−1,必須先於 L 算好。
直接 for i = 0..n, for j = i..n 的話,算 dp[0][3] 時會用到 dp[1][3] —— 但 i=1 那列還沒開始跑,dp[1][3] 還是 0,用到未初始化的值,結果錯。
正確順序:外層跑區間長度 L=2..n,內層跑左端 i。這保證「所有長度 < L 的區間都已填好」。這是區間 DP 的通用填法 —— 凡是「狀態是區間、轉移看子區間」的題(戳氣球 312、矩陣鏈乘…)都這樣按長度填。
A4. 把 n 堆(n 偶)依 index 分兩組:偶位 {0,2,4,…} 和奇位 {1,3,5,…},各 n/2 堆。
先手能鎖定任一組:開局兩端是 index 0(偶)和 n−1(奇,因 n 偶)。假設 Alice 想要全偶位 —— 她先拿 index 0。現在剩 [1, n−1],兩端 index 1、n−1(都奇)。Bob 不管拿哪端,拿的都是奇位,而且會再暴露出一個偶位在某一端。Alice 下一步又能拿偶位。如此每一輪,Alice 恆拿偶位、Bob 恆拿奇位。她也可以反過來全拿奇位。
因為總和是奇數,偶位和 + 奇位和 = 奇數 → 兩組不可能相等,必有一組嚴格較大。Alice 開局先算哪組大就鎖哪組 → 拿到嚴格過半 → 必勝。所以本題答案恆 true。「偶堆」給了鎖定的機制,「奇和」排除了平手。
A5. 改成 dp[0][n−1] >= 0。
差別在「平手算誰贏」。877 保證總和為奇 → 淨勝分永不為 0,>0 和 >=0 結果一樣。但 486 可以平手(淨勝分 = 0),而題目定義「先手不輸就算 Player 1 贏」(平手也回 true)。所以要放寬成 >= 0。
其餘完全不用改 —— 淨勝分 DP 的轉移、填法一模一樣,486 只是拿掉了「偶堆奇和」約束(所以捷徑失效,只能靠 DP),並把判定邊界從 >0 調成 >=0。這就是為什麼面試要寫 DP 不寫捷徑:同一份 DP 改一個比較符號就解了 486,捷徑卻整個報廢。
App 1. 程式幾乎不動:轉移 dp[i][j]=max(piles[i]−dp[i+1][j], piles[j]−dp[i][j−1]) 和填法完全共用,只把回傳從 dp[0][n−1] > 0 改成 >= 0(平手算先手贏)。
捷徑不能用:486 沒有「偶堆 + 奇和」約束 —— 堆數可奇可偶、可能平手,「鎖定一組」的論證垮掉。例 [1,5,2](3 堆)先手其實會輸。所以 486 只有 DP 一條路。這正是 877 該練 DP 而非背捷徑的理由:DP 是可遷移的能力,捷徑是一次性的巧合。
App 2. 回傳分差:dp[0][n−1] 本身就是先手比對手多拿的淨分,直接回傳那個 int 即可(不用改任何轉移)。想要各自實際得分:先手 = (total + dp[0][n−1]) / 2、對手 = (total − dp[0][n−1]) / 2(因為兩人和 = total、差 = dp)。
印出每步決策:要保留完整 dp 表,從 (0, n−1) 正向回放 —— 在 (i,j) 比較 piles[i]−dp[i+1][j] 和 piles[j]−dp[i][j−1] 哪個等於 dp[i][j],相等的那邊就是這步的選擇(拿左 → 走 (i+1,j);拿右 → 走 (i,j−1)),交替印出雙方動作直到區間空。O(n) 回放。DP 表本身就記錄了最優決策,回放只是讀它。
App 3. 1690 的計分不是「拿走那堆的值」,而是「拿完後剩下堆的總和」。用前綴和 pre 表示區間和 sum(i,j)=pre[j+1]−pre[i]。
在 piles[i..j] 拿左端 i:得分 = 剩下 [i+1, j] 的和 = sum(i+1, j),然後對手在 [i+1,j] 當先手,淨勝 = sum(i+1,j) − dp[i+1][j]。拿右端同理 = sum(i,j−1) − dp[i][j−1]。取 max:
dp[i][j] = max(sum(i+1,j) − dp[i+1][j], sum(i,j−1) − dp[i][j−1]),base dp[i][i]=0(只剩一堆,拿完剩 0 分)。
骨架和 877 一模一樣 —— 一樣的「淨勝分 + 減號 + 按長度填」,只是「當前這步的得分」從 piles[端] 換成 sum(剩下),靠前綴和 O(1) 取得。這說明淨勝分不是一題的技巧,是一整類輪流博弈的框架 —— 換計分規則,只換「這步得幾分」那一項。
