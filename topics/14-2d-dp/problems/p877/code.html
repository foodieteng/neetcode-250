class Solution {
public:
    bool stoneGame(vector<int>& piles) {
        int n = piles.size();
        // dp[i][j] = 先手面對 piles[i..j] 能拿到的「淨勝分」(自己 − 對手)
        vector<vector<int>> dp(n, vector<int>(n, 0));

        for (int i = 0; i < n; i++) {
            dp[i][i] = piles[i];              // 只剩一堆 → 全拿
        }

        for (int L = 2; L <= n; L++) {            // 區間長度由小到大
            for (int i = 0; i + L - 1 < n; i++) {
                int j = i + L - 1;
                dp[i][j] = max(piles[i] - dp[i + 1][j],    // 拿左端
                                piles[j] - dp[i][j - 1]);   // 拿右端
            }
        }
        return dp[0][n - 1] > 0;            // 先手淨勝 > 0 ⇔ Alice 贏
    }
};

整份程式沒有「玩家」這個維度 —— 淨勝分把兩人的最優對抗折進同一張表。 填表依 L(區間長度)由小到大,保證 dp[i+1][j]、dp[i][j−1] 這兩個更短的子區間已經算好。

那個「減號」到底在幹嘛// THE MINUS

為什麼是 piles[i] − dp[i+1][j] 而不是 +

dp[i][j] 定義成「當前先手的淨勝分」。當我拿走左端 piles[i],剩下 piles[i+1..j] —— 但接下來是對手當先手。dp[i+1][j] 算的是「對手在那段的淨勝分」。

對手贏多少,就是我輸多少。所以從我的角度,這一段的貢獻是 −dp[i+1][j]。加上我剛拿的 piles[i]:

我的淨勝 = piles[i] − dp[i+1][j]

這一招叫「score difference / relative score」 —— 把「兩人分別多少」壓縮成「一人領先多少」, 再靠正負號翻轉表達「換人」。它讓博弈 DP 從三維(含玩家)降到二維,是這類題的核心手法。

// piles = [5,3,4,5] —— dp 上三角(實跑) j=0 j=1 j=2 j=3 i=0 5 2 4 1 i=1 . 3 1 4 i=2 . . 4 1 i=3 . . . 5 L=1(對角線): dp[i][i] = piles[i] = 5,3,4,5 L=2: dp[0][1] = max(5−3, 3−5) = 2 dp[1][2] = max(3−4, 4−3) = 1 dp[2][3] = max(4−5, 5−4) = 1 L=3: dp[0][2] = max(5−dp[1][2], 4−dp[0][1]) = max(5−1, 4−2) = 4 L=4: dp[0][3] = max(5−dp[1][3], 5−dp[0][2]) = max(5−4, 5−4) = 1 dp[0][3] = 1 > 0 → Alice 淨勝 1 分 → return true
// 對照捷徑:全奇位 vs 全偶位 piles = [5, 3, 4, 5] index: 0 1 2 3 偶位(0,2): 5 + 4 = 9 奇位(1,3): 3 + 5 = 8 9 ≠ 8(因為總和 17 是奇數)→ Alice 鎖定偶位,拿 9 > 8 → 贏 和 DP 結論一致:Alice 淨勝 9 − 8 = 1,正是 dp[0][3]

注意最後一行:捷徑的「偶位和 − 奇位和 = 9 − 8 = 1」剛好等於 dp[0][3] = 1。 這不是巧合 —— 當 Alice 用「鎖定一組」策略時,淨勝分就是兩組之差。DP 找到的最優,和捷徑指出的策略是同一個。
