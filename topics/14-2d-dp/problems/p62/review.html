重點回顧// RECAP

面向要點
狀態dp[i][j] = 走到 (i,j) 的路徑數
轉移dp[i][j] = dp[i−1][j] + dp[i][j−1](上 + 左)
base第一列、第一行都是 1
答案dp[m−1][n−1]
壓維dp[j] += dp[j−1](由左而右)→ O(n)
數學解C(m+n−2, m−1),O(min(m,n))

觀念題// QUIZ

Q1. 為什麼 dp[i][j] = dp[i−1][j] + dp[i][j−1]?這個「加」對應到什麼移動?
Q2. 為什麼第一列、第一行的路徑數都是 1?
Q3. 一維版 dp[j] += dp[j-1] 裡,dp[j] 和 dp[j-1] 分別代表二維的哪一格?為什麼掃描要由左而右?
Q4. 組合數解 C(m+n−2, m−1) 是怎麼來的?為什麼它不適合推廣到「有障礙」的 63?
Q5. 這題和 64 最小路徑和 的 DP 骨架幾乎一樣,差在哪個運算?那反映了什麼共性?

範例 Trace// m=3, n=3 → 6

// 二維表 i\j 0 1 2 0 1 1 1 1 1 2 3 2 1 3 6 dp[2][2] = dp[1][2] + dp[2][1] = 3 + 3 = 6 return 6
每一格的數字,就是「從左上角走到這一格」的路徑數。你會發現這張表正是楊輝三角(帕斯卡三角)沿對角線的樣子 —— 這也解釋了為什麼答案剛好是組合數 C(m+n−2, m−1):每條路是「在 m+n−2 步裡選哪幾步往下」。

應用題// APPLY

App 1. 63 Unique Paths II:格子裡有障礙(不能走)。轉移要怎麼改?障礙格的 dp 該設成什麼?
App 2. 64 Minimum Path Sum:每格有成本,求左上到右下的最小成本和。轉移從 加 換成什麼?
App 3. 若機器人可以往右、下、右下(斜)三種移動,轉移要加哪一項?一維壓維還做得到嗎?
// ANSWERS
A1. 因為機器人只能往右或往下,所以要走到 (i,j),它的前一步一定是「從上面 (i−1,j) 往下」或「從左邊 (i,j−1) 往右」——沒有別的來源。這兩類路徑不重疊、且窮盡所有到達 (i,j) 的走法,所以路徑數相加。這正是計數 DP 的通則:把「到這裡的所有方式」按「最後一步從哪來」分類,各類相加。
A2. 因為到第一列的任何格子,只能一路往右(不能往上、也還沒機會往下轉);到第一行任何格子只能一路往下。兩者都只有唯一一條路徑,所以路徑數 = 1。它們是遞迴的邊界:內部格全靠這圈 1 一層層累加出來。少了它們(設成 0),整張表就全是 0。
A3. 一維 dp[] 在處理第 i 列、掃到第 j 欄的當下:dp[j] 還沒被本列更新,所以它仍是上一列同欄 dp[i−1][j](=「上」);dp[j-1] 是本列剛更新過的左鄰 dp[i][j−1](=「左」)。所以 dp[j] += dp[j-1] 恰好是 上 + 左。必須由左而右,才能保證用到 dp[j-1] 時它已是本列新值;若由右而左,dp[j-1] 還是舊(上一列)的值,「左」就抓錯了。
A4. 從左上到右下,總共要走 m−1 步往下、n−1 步往右,合計 m+n−2 步。一條路徑就是「在這 m+n−2 步中,決定哪 m−1 步是往下」——所以路徑數 = C(m+n−2, m−1)。它不適合有障礙的 63:組合數假設「任何格都能走、每條路等價」,一旦某些格被擋,就沒有乾淨的封閉公式了 —— 必須回到逐格 DP,把障礙格的 dp 設 0(此路不通),讓它自然不貢獻給後續。這就是為什麼 DP 才是通用骨架、組合數只是本題特例。
A5. 差在把 +(計數)換成 min(...)+cost(最佳化)。64 的轉移是 dp[i][j] = grid[i][j] + min(dp[i−1][j], dp[i][j−1]) —— 一樣「從上或左來」,但要取最小再加上本格成本。這反映了網格 DP 的共性:骨架(依賴上、左兩鄰)不變,換掉聚合運算就切換題型 —— 計數用 +、最小成本用 min、可行性用 OR。62、63、64 是同一個模板的三種收尾。
App 1. 障礙格代表「走不到」,把它的 dp 設為 0(不論上下左右都不可能停在這、也不能經過它繼續)。轉移仍是 dp[i][j] = dp[i−1][j] + dp[i][j−1],但若 (i,j) 是障礙就直接 dp[i][j]=0,不做累加。base 也要小心:第一列/行一旦遇到障礙,其後的格子全變 0(被擋住,過不去),不能無腦填 1。這就是 63,無法用組合公式、必須逐格 DP。
App 2. 把 + 換成 min:dp[i][j] = grid[i][j] + min(dp[i−1][j], dp[i][j−1])。base 第一列/行是前綴和(只有一條路,成本累加),不是 1。答案 dp[m−1][n−1] 就是最小路徑和。骨架與 62 完全一樣,只換聚合(加法計數 → 取最小),是「同模板換運算」的典型。
App 3. 多一個來源:dp[i][j] = dp[i−1][j] + dp[i][j−1] + dp[i−1][j−1](從上、左、左上斜三個方向來)。一維壓維還做得到,但要小心:dp[i−1][j−1](左上)在一維滾動裡是「上一列的左鄰」,而 dp[j-1] 在你更新它之前是上一列值、更新之後是本列值 —— 需要一個暫存變數先存住「更新前的 dp[j-1]」(即左上),再更新,否則會拿錯。這和「需要左上角」的編輯距離、LCS 壓維時的處理同源。
