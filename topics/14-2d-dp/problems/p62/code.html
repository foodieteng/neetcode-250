寫法 A · 二維網格(最直覺)// 2-D GRID

p62.cpp C++17 
class Solution {
public:
    int uniquePaths(int m, int n) {
        vector<vector<int>> dp(m, vector<int>(n, 1));  // 第一列/行預設 1

        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                dp[i][j] = dp[i - 1][j] + dp[i][j - 1]; // 上 + 左
            }
        }

        return dp[m - 1][n - 1];
    }
};

整張表初始化成 1,巧妙地一次搞定「第一列、第一行都是 1」的 base。內部格 dp[i][j] = 上 + 左。這是二維 DP 最好懂的原型 —— 面試先寫這個,再談壓維。

寫法 B · 一維滾動(你的版本)// O(n) SPACE

p62_1d.cpp C++17 
class Solution {
public:
    int uniquePaths(int m, int n) {
        vector<int> dp(n, 0);
        dp[0] = 1;                       // 起點

        for (int i = 1; i <= m; i++) {       // 逐列滾動
            for (int j = 1; j < n; j++) {
                dp[j] += dp[j - 1];       // dp[j](上) += dp[j-1](左)
            }
        }

        return dp[n - 1];
    }
};

關鍵洞察:更新 dp[j] 前,它還是「上一列」的值(=「上」);dp[j-1] 是本列剛更新的左鄰(=「左」)。所以 dp[j] += dp[j-1] 就等於 dp[i][j]=dp[i-1][j]+dp[i][j-1]。空間 O(mn) → O(n)。就地覆蓋、由左而右是這招成立的前提。

3

複雜度分析// COMPLEXITY

T = O(mn)　　S = O(mn)(A) / O(n)(B)

兩者都填 mn 格,每格 O(1)。A 存整張表;B 只留一列。組合數解另有 O(min(m,n))(見複習)。

4

邊界與陷阱// PITFALLS

陷阱後果正解
忘記第一列/行 = 1邊界格為 0 → 全錯初始化整張表為 1(或先鋪 base)
1D 版由右而左掃dp[j-1] 已被覆蓋 → 錯本題要由左而右(左鄰須是本列新值)
組合數先算階乘(m+n)! 溢位邊乘邊除:r = r*(a−b+i)/i
把 m、n 弄反列數行數互換(本題對稱不影響)看清 m=列、n=行即可

註:本題結果對稱(uniquePaths(m,n)==uniquePaths(n,m)),把哪個當外層都行。但 1D 滾動的掃描方向(由左而右)不能反 —— 這與背包壓維的方向規則同源。
