題目簡介// THE QUESTION

QUESTION

一個 m × n 網格,機器人從左上角出發,每步只能往右或往下,問走到右下角共有幾條不同路徑?

這是二維 DP 的入門題。狀態自然是二維的:dp[i][j] = 走到格子 (i,j) 的路徑數。因為只能右/下移,到 (i,j) 只可能從「上一格」或「左一格」走來 → dp[i][j] = dp[i−1][j] + dp[i][j−1]。

為什麼是二維 DP// WHY 2-D

A 遞迴枚舉每條路 O(2^(m+n))
每步右/下二選一,分支爆炸;同一格被重複走到無數次。
× REJECT · TLE
B 組合數 C(m+n−2, m−1) O(min(m,n))
共走 m+n−2 步、選 m−1 步往下 → 純數學最快;但不通用(有障礙 63、要最小和 64 就失效)、且易溢位。
○ OK · 不通用
ADOPT
C 二維 DP:每格 = 上 + 左 O(mn) / O(n)
dp[i][j] = dp[i−1][j] + dp[i][j−1],第一列/行皆 1。每格算一次;是整個 2-D DP 章節的骨架,可無痛推廣到障礙、最小路徑和等變體。
● ADOPT · O(mn)

核心策略 · 三步驟// STRATEGY

  1. 狀態:dp[i][j] = 走到 (i,j) 的路徑數。
  2. base:第一列 dp[0][j]=1、第一行 dp[i][0]=1(只能一路右 / 一路下)。
  3. 轉移:其餘 dp[i][j] = dp[i−1][j] + dp[i][j−1];答案 dp[m−1][n−1]。

從 2D 壓到 1D// YOUR 1-D VERSION

LEMMA · 只看「上」和「左」→ 一列就夠

你貼的版本 dp[j] += dp[j-1] 是滾動陣列優化。訣竅:一維 dp[] 存「上一列」的值;逐列由左而右就地更新時 ——

● 更新前的 dp[j] = 上一列同欄的值 = dp[i−1][j](「上」)
● dp[j-1] = 本列已更新的左鄰 = dp[i][j−1](「左」)

所以 dp[j] += dp[j-1] 正好是 dp[i][j] = dp[i−1][j] + dp[i][j−1],一個式子同時當「上」與「左」。空間從 O(mn) 降到 O(n)。這是二維網格 DP 最常見的壓維技巧,務必看懂。

6

複雜度// COMPLEXITY

DP:T = O(mn)　　S = O(mn) → 壓成 O(n)

每格算一次、常數運算。滾動陣列把空間降到 O(n)(甚至只留一列)。若用組合數 C(m+n−2, m−1) 可 O(min(m,n)),但要小心溢位、且不通用。

一句洞察// INSIGHT

二維網格 DP 的原型:每格 = 能走來的鄰格之和(這裡是上 + 左)。 因為只依賴上一列 → 可壓成一維 dp[j] += dp[j-1]。障礙(63)、最小和(64)都是它的變體。
