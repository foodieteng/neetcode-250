完整實作 · 開區間 DP// O(n³)

p312.cpp C++17 
class Solution {
public:
    int maxCoins(vector<int>& nums) {
        int n = nums.size();
        // dp[i][j] = 開區間 (i,j) 全爆的最大金幣(i、j 不爆,當邊界)
        vector<vector<int>> dp(n + 2, vector<int>(n + 2, 0));

        nums.insert(nums.begin(), 1);    // 兩端補 1 當永不越界的邊界
        nums.push_back(1);              // index 變 0..n+1

        for (int L = 1; L <= n; L++) {              // L = 開區間內氣球數量
            for (int i = 0; i + L + 1 <= n + 1; i++) {
                int j = i + L + 1;              // (i,j) 之間剛好 L 顆
                for (int k = i + 1; k < j; k++) {   // k = 最後爆的那顆
                    dp[i][j] = max(dp[i][j],
                        dp[i][k] + dp[k][j] + nums[i] * nums[k] * nums[j]);
                }
            }
        }
        return dp[0][n + 1];                   // 整個開區間全爆
    }
};

三層迴圈:L(區間大小)→ i(左邊界)→ k(最後爆的那顆)。 nums[i]·nums[k]·nums[j] 是「k 最後爆」的得分 —— 那時它的鄰居正好是固定邊界 i, j。

索引拆解 · j = i + L + 1// INDEXING

片段意思
L開區間 (i,j) 裡面的氣球數量(要爆的顆數)
j = i + L + 1左邊界 i + L 顆 + 右邊界 → j。兩端各佔 1 個 index,故 +1
k = i+1 .. j−1「最後爆」只能是區間內的球,不含邊界
nums[i]·nums[k]·nums[j]k 最後爆的得分:左右鄰居是還沒爆的 i, j
dp[i][k] + dp[k][j]左段(以 i,k 為牆)+ 右段(以 k,j 為牆),獨立
dp[0][n+1]補了 1 之後,整個開區間就是頭尾兩個 1 之間
關鍵在 dp[i][k] 和 dp[k][j] 讀的都是更短的區間(裡面球數 < L),所以按 L 由小到大填,用到時一定已算好 —— 標準區間 DP 填法,和 877 同源。

補 1 與開區間的配合// PADDING

// 補 1 前後 原始 nums = [3, 1, 5, 8] index 0..3 補 1 nums = [1, 3, 1, 5, 8, 1] index 0..5 兩端的 1 = 「越界鄰居當作 1」的規則,直接寫進陣列 → 邊界氣球爆時不用特判「有沒有左/右鄰居」 → dp[0][5] 就是「頭尾兩個 1 之間,全爆」的答案
// 開區間為什麼拆得乾淨 dp[i][j] = 開區間 (i,j),i、j 不爆(是牆) k 最後爆时,拆成: 左段 dp[i][k] → 開區間 (i,k),牆是 i 和 k 右段 dp[k][j] → 開區間 (k,j),牆是 k 和 j k 同時是「左段的右牆」和「右段的左牆」 → 兩段語意一致、拼起來剛好 = (i,j),不重不漏 閉區間會讓「k 算左邊還算右邊」打架,開區間天生沒這問題

邊界與陷阱// PITFALLS

陷阱症狀正解
枚舉「第一個爆」 兩側合併、子問題糾纏,拆不開 → 想不出 DP 枚舉「最後爆的 k」
忘了補 1 邊界氣球得分要特判鄰居,又易越界 兩端 insert(1) / push_back(1)
得分用 nums[k−1]·nums[k]·nums[k+1] 那是「當下鄰居」,但 k 最後爆時鄰居是邊界,不是原位置的鄰居 nums[i]·nums[k]·nums[j]
不按 L 填 dp[i][k]、dp[k][j] 還沒算就用到 外層跑區間大小 L
閉區間定義 邊界歸屬打架,拆分重疊或遺漏 用開區間 (i,j),i,j 不爆
nums[i]==0 特判 以為 0 要跳過 不用,乘出來就是 0,max 自然處理

細節走讀// [3,1,5,8]

// padded nums = [1,3,1,5,8,1] (index 0..5),求 dp[0][5] dp[0][5]:枚舉「最後爆的 k = 1..4」 k=1 (值3): dp[0][1](0) + dp[1][5](159) + 1×3×1 = 0 + 159 + 3 = 162 k=2 (值1): dp[0][2](3) + dp[2][5](48) + 1×1×1 = 3 + 48 + 1 = 52 k=3 (值5): dp[0][3](30)+ dp[3][5](40) + 1×5×1 = 30 + 40 + 5 = 75 k=4 (值8): dp[0][4](159)+ dp[4][5](0) + 1×8×1 = 159 + 0 + 8 = 167 max(162,52,75,167) = 167 → 最後爆值 8(先爆光左邊 3,1,5,再爆 8)
// 逐長度填表(dp[i][j],上三角,實跑) L=1: dp[0][2]=3 dp[1][3]=15 dp[2][4]=40 dp[3][5]=40 (單顆:1·x·1 兩邊界) L=2: dp[0][3]=30 dp[1][4]=135 dp[2][5]=48 L=3: dp[0][4]=159 dp[1][5]=159 L=4: dp[0][5]=167 依 L 遞增,每格用到的都是更短區間 → 答案 dp[0][5] = 167

看 k=4 那條:左段 dp[0][4]=159 是開區間 (0,4) —— 把值 8 左邊的 [3,1,5] 全部最優爆光(牆是邊界 1 和 8)。 8 撐到最後才爆,那時左右鄰居正好是兩個邊界 1(nums[0] 與 nums[5])→ 得 1×8×1=8。右段 dp[4][5]=0(8 和右邊界之間沒有球)。合計 159+0+8 = 167。
