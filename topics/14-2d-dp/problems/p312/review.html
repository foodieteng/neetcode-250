重點回顧// RECAP

面向要點
反轉枚舉「最後爆的 k」而非「第一個」
為何有效k 最後爆 → 鄰居固定是邊界 i,j → 兩側獨立
狀態dp[i][j] = 開區間 (i,j) 全爆的最大金幣(i,j 不爆)
轉移max(dp[i][k] + dp[k][j] + nums[i]·nums[k]·nums[j])
補邊界兩端補 1,把「越界當 1」寫進陣列
填法按區間大小 L 遞增;答案 dp[0][n+1]
複雜度O(n³) / O(n²)(對比暴力 O(n!))

觀念題// QUIZ

Q1. 為什麼枚舉「第一個戳的氣球」會失敗,而「最後戳的」就成立?具體差在哪一步讓子問題能不能獨立?
Q2. k 最後爆時,得分為什麼是 nums[i]·nums[k]·nums[j] 而不是 nums[k−1]·nums[k]·nums[k+1]?
Q3. 為什麼狀態要定成開區間(i,j 不爆)?若用閉區間會遇到什麼麻煩?
Q4. 兩端補 1 的作用是什麼?不補會怎樣?
Q5. 這題和 877 石頭遊戲 都是「按區間長度填的區間 DP」。兩題枚舉的「分割點」意義有何不同?為什麼一個是加法拆分、一個是減法對抗?

範例 Trace// [1,5] → 10

// 小例 nums=[1,5],補 1 → [1,1,5,1] (index 0..3),求 dp[0][3] 枚舉最後爆的 k = 1,2: k=1 (值1): dp[0][1](0) + dp[1][3](?) + nums[0]·nums[1]·nums[3] = 0 + dp[1][3] + 1·1·1 dp[1][3]: 開區間(1,3)只剩 index 2(值5),最後爆它 = nums[1]·nums[2]·nums[3] = 1·5·1 = 5 → k=1 得 0 + 5 + 1 = 6 k=2 (值5): dp[0][2](?) + dp[2][3](0) + nums[0]·nums[2]·nums[3] = dp[0][2] + 0 + 1·5·1 dp[0][2]: 開區間(0,2)只剩 index 1(值1),= nums[0]·nums[1]·nums[2] = 1·1·5 = 5 → k=2 得 5 + 0 + 5 = 10 max(6, 10) = 10 → 先戳 1(得 1·1·5=5),再戳 5(得 1·5·1=5)
驗一下直覺:先戳 1(鄰居 1,5)得 1·1·5=5,剩 [5];再戳 5(鄰居都是邊界 1)得 1·5·1=5,共 10。 反過來先戳 5 得 1·5·1=5… 一樣?其實 [1,5] 兩種順序都 10。但一般情況順序極度重要 —— 這也是為什麼要 DP 而非貪心。

應用題// APPLY

App 1. 若改成記憶化遞迴(自頂向下)而非填表,函式簽名和邊界怎麼設?兩者複雜度一樣嗎?
App 2. 若要還原最優的戳破順序(不只金幣數),要多存什麼?怎麼回溯?
App 3. 若題目反過來 —— 戳破 i 得 nums[i−1] + nums[i] + nums[i+1](加法而非乘法),最優順序還重要嗎?這題會不會塌掉?
// ANSWERS
A1. 差別在子問題能不能獨立。
「第一個戳 k」:戳掉 k 後,它左右兩邊立刻變相鄰 —— 左半的最右球和右半的最左球接在一起,之後誰爆會影響對方的鄰居、得分互相牽動。左右合併成一坨,無法拆成兩個獨立區間,DP 失敗。
「最後戳 k」:k 撐到這段區間的最後才爆,代表在此之前 k 一直站著,像一道牆把左段 (i,k) 和右段 (k,j) 隔開。兩段各自爆光的過程中,中間永遠隔著沒爆的 k,互不影響 → 完美拆成 dp[i][k] + dp[k][j]。
一句話:「先戳」讓兩側連通、糾纏;「最後戳」讓兩側被 k 隔開、獨立。能不能獨立,就是能不能 DP 的分水嶺。
A2. 因為「k 最後爆」意味著爆它的那一刻,這段區間 (i,j) 內其他球都已經沒了。此時 k 左右緊鄰的,不是它在原陣列的鄰居 k−1, k+1(那些早爆光了),而是這段區間還沒爆的兩個邊界 i 和 j。所以得分是 nums[i]·nums[k]·nums[j]。
寫成 nums[k−1]·nums[k]·nums[k+1] 是典型錯誤 —— 那是「當下鄰居」的思維,但整題的反轉正是要跳出「當下」,改用「最後爆時的固定邊界」。這個得分之所以確定(不隨戳法變),就是因為 i, j 在這段全程沒爆,是穩定的錨點。得分能確定 → 才能當 DP 的轉移項。
A3. 開區間讓「一分為二」接縫剛好。dp[i][j] 定義成「爆光 (i,j) 內、但 i,j 不爆」。k 最後爆時拆成:左段 dp[i][k](牆是 i,k)、右段 dp[k][j](牆是 k,j)。
k 同時當左段的右牆、右段的左牆 —— 它在兩個子問題裡都是「沒爆的邊界」,語意一致。兩段拼起來 = (i,k) ∪ {k} ∪ (k,j) 剛好覆蓋 (i,j) 內每顆球一次,不重不漏。
閉區間的麻煩:若 dp[i][j] 含 i,j 端點,那 k 到底算左段還右段?端點歸屬會重複計算或遺漏,轉移得加一堆 ±1 修正,又容易錯。開區間把邊界的角色講死(永遠是牆、永遠不爆),拆分才乾淨。
A4. 補 1 是把「越界的鄰居當作 1」這條規則直接寫進資料。原本爆最左 / 最右的球要特判「有沒有左 / 右鄰居」,補了 1 之後,每顆球都保證有左右鄰居(最外面是那兩個 1),轉移不用任何邊界特判。
而且答案 dp[0][n+1] 的意思就變得很自然:「頭尾兩個 1 之間的所有球全爆」—— 那兩個 1 是虛擬邊界,永遠站著。
不補會怎樣:轉移裡的 nums[i]·nums[k]·nums[j] 在 i 或 j 落到真實邊界外時要寫成 1,程式碼充斥 i<0?1:nums[i] 之類的三元判斷,又醜又易錯。補 1 = 用 O(1) 空間換掉所有邊界特判,是 sentinel(哨兵)技巧的經典用法。
A5. 兩題都「按區間長度填、枚舉一個分割點」,但分割點的意義和聚合完全不同:
● 312 是「合作 / 加法拆分」:k = 最後爆的球,把區間切成兩段獨立子問題,兩段的金幣相加(自己跟自己合作,最大化總和)。轉移是 dp[i][k] + dp[k][j] + 得分,聚合是 max + 加法。
● 877 是「對抗 / 減法」:分割點 = 拿左端還右端,對手接手另一段,聚合是 piles[…] − dp[子區間] —— 減號表達「對手的優勢是我的劣勢」。
根源:312 是單人最佳化(所有金幣都歸我,只是求最優順序),所以子問題結果相加;877 是雙人零和,所以子問題結果要相減。「加還是減」取決於子問題的收益是同一人的、還是對手的。這是區間 DP 兩大分支的分水嶺。
App 1. 記憶化遞迴:int solve(i, j) = 開區間 (i,j) 的最大金幣。邊界:if (i + 1 == j) return 0;(中間沒有球可爆)。主體:for k = i+1..j-1: best = max(best, solve(i,k) + solve(k,j) + nums[i]*nums[k]*nums[j]),用 memo[i][j] 快取,回傳前寫回。呼叫 solve(0, n+1)。
複雜度一樣:O(n²) 個狀態、每個 O(n) 枚舉 k → O(n³),和填表相同。差別只在「填表按 L 保證順序」vs「遞迴按需求、memo 去重」—— 兩者是同一個 DP 的兩種驅動方式(bottom-up vs top-down),就像 329 的 memo DFS 與 Kahn。記憶化版更好想(直接照「最後爆」的定義寫遞迴),填表版常數更小。
App 2. 要多存一個 choice[i][j] = 「區間 (i,j) 取到最優時,最後爆的是哪個 k」。轉移時,每當 dp[i][j] 被更新,就記下當前的 k。
回溯:從 (0, n+1) 出發,遞迴地 reconstruct(i, j):讀出 k = choice[i][j],先遞迴左段 reconstruct(i,k) 和右段 reconstruct(k,j),最後才輸出 k —— 因為 k 是這段最後爆的,順序上要排在兩側之後。這樣輸出的序列就是一個合法的最優戳破順序。O(n) 回溯(每顆球輸出一次)。「最後爆」的定義天然決定了回溯的後序輸出。
App 3. 改成加法後,順序完全不重要 —— 題目塌成 O(n) 的簡單求和,DP 直接消失。
因為每顆球 i 被戳時貢獻 nums[i−1] + nums[i] + nums[i+1],你可以逐項拆開想:nums[i] 自己一定被算一次(戳它時);而它作為鄰居被加的次數,取決於它左右的球被戳時它還在不在 —— 但無論什麼順序,總貢獻可以直接用組合論算出固定值,不受順序影響(加法沒有「鄰居相乘放大」的耦合)。
關鍵對比:312 之所以難、順序之所以重要,是因為乘法把「大球留到最後、靠好鄰居放大」變成有利可圖 —— 大數乘大數的非線性耦合讓貪心失效、逼出 DP。換成線性的加法,耦合消失,最優 = 隨便戳。是「乘法的非線性」造就了這題,不是「戳氣球」這個包裝。
