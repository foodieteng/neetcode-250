範例 Trace// 4×4 有兩個障礙

// grid（X = 障礙 1） 0 0 0 0 dp: 1 1 1 1 0 X 0 0 1 0 1 2 0 0 0 X 1 1 2 0 ← (2,3) 障礙 → 0 0 0 0 0 1 2 4 4 dp[2][2]=上1+左1=2;(2,3)障礙→0;dp[3][3]=上0+左4=4 return dp[3][3] = 4
注意 (2,3) 的障礙讓 dp[2][3]=0,於是 dp[3][3] 的「上」變成 0 —— 右下角只能靠「左」dp[3][2]=4接上。 障礙就像在網格上挖洞:洞格貢獻 0,逼所有路徑改道。DP 逐格累加,自然把改道算進去。

應用題// APPLY

App 1. 64 Minimum Path Sum:每格有成本,求最小成本路徑。若同時有障礙,轉移要怎麼合併「取 min」與「障礙 = ∞」?
App 2. 若允許機器人往右、下、右下(斜)三向移動,且有障礙,轉移要加哪一項?邊界哨兵還管用嗎?
App 3. 若要列出一條實際可行路徑(不只數量),怎麼從 dp 回溯?障礙會讓回溯出什麼問題?
// ANSWERS
A1. 只多一句:「若 (i,j) 是障礙就把 dp[i][j] 設 0(或 continue 保持 0)」。它把障礙格標成「走不到」,於是它給下、右鄰的貢獻是 0,路徑自動繞開。轉移主體 dp[i][j]=上+左 與 62 一字不差 —— 障礙只是把某些格「關掉」。
A2. 因為第一列只能一路往右、第一行只能一路往下,沒有別條路可繞。一旦中間卡了一個障礙,後面的格子就再也到不了(唯一的直線通道被切斷),路徑數是 0。62 沒有障礙,直線通道永遠暢通,所以邊界全是 1。差別就在「直線唯一通道會不會被擋」 —— 62 不會、63 會。
A3. padding 解決「第一列/行沒有『上』或『左』鄰居」的邊界特判。加一圈全 0 的哨兵(第 0 列、第 0 行)後,真正的第一列/行(索引 1)在算 dp[i][j]=dp[i−1][j]+dp[i][j−1] 時,它們的「上」或「左」自動落在哨兵 0 上,不必寫 if (i==0) / if (j==0) 這類判斷。程式因此更短、更不易出錯。代價只是多一圈記憶體。
A4. 因為 C(m+n−2, m−1) 的推導假設每一格都可走、每條「下/右序列」都是合法路徑,答案才等於「在 m+n−2 步裡選 m−1 步往下」的組合數。一旦有障礙,某些「下/右序列」會穿過障礙 → 不合法,但組合數把它們照算不誤,無法扣除;而障礙的分布沒有簡潔封閉式能描述「有多少序列避開所有障礙」。所以只能回到逐格 DP,讓障礙格自然貢獻 0。這就是 DP 通用、公式只是無障礙特例的原因。
A5. 在一維滾動裡,dp[j] 進入第 i 列時還帶著上一列(第 i−1 列)的值。若 (i,j) 是障礙,這格本身走不到,它對第 i 列的貢獻必須是 0 —— 所以要主動把 dp[j]=0,清掉那個「上一列殘留值」。若只 continue(不清零),dp[j] 會保留上一列的舊值,等於把障礙格誤當成可走、還把上一列的路徑數帶下去,答案偏大。二維版可以 continue(因為 dp[i][j] 本來就是新的一格、初值 0);一維版共用同一格,所以必須顯式歸零。
App 1. 64 求最小成本:dp[i][j] = grid[i][j] + min(dp[i−1][j], dp[i][j−1])。若再加障礙,把障礙格的 dp 設成 ∞(極大值),代表「不可能經過」;取 min 時它自然不會被選,且會把「經過它的路徑」推成 ∞ 而作廢。要小心用足夠大的哨兵(避免 ∞ + cost 溢位,或加 if 跳過)。核心:計數版障礙用 0(加法單位)、最小化版障礙用 ∞(min 的單位)—— 障礙的「屏蔽值」隨聚合運算而不同。
App 2. 加左上斜向來源:dp[i][j] = dp[i−1][j] + dp[i][j−1] + dp[i−1][j−1](障礙格仍設 0)。邊界哨兵照樣管用 —— 第 0 列/行的 0 讓斜向 dp[i−1][j−1] 在邊界也能安全取到 0。二維版直接加項即可;一維滾動要多留一個變數存「更新前的左上角(舊 dp[j−1])」,因為它會在更新 dp[j−1] 後被覆蓋 —— 這和編輯距離/LCS 壓維的處理同源。
App 3. 從終點 (m−1,n−1) 往回走:在 (i,j) 看它是從「上」還是「左」來 —— 哪個鄰格的 dp > 0 且能構成 dp[i][j] 就往那邊退(計數版任選一個非零來源即可;若要字典序最小可固定優先順序)。障礙的問題:回溯時要跳過 dp=0 的格子(它們不可能在路徑上);若某步兩個來源都是 0,代表根本無路(應在一開始就判掉)。另外 dp 一維滾動版丟了逐格歷史,回溯需要保留二維表或額外記每格的來源方向。
