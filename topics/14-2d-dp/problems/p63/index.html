題目簡介// THE QUESTION

QUESTION

和 62 不同路徑 一樣,機器人只能往右/下走。但這次網格裡有障礙(值 1),不能經過。求左上到右下的路徑數。

就是 62 的障礙版。骨架不變 —— dp[i][j] = dp[i−1][j] + dp[i][j−1](上 + 左),只多一條規則:障礙格 dp=0(此路不通),讓它自然貢獻 0、路徑繞開。這正是「DP 通用、組合公式失效」的分水嶺。

2

和 62 的唯一差別// OBSTACLE = 0

核心 · 障礙格 dp = 0

轉移完全照搬 62,只加一句:若 (i,j) 是障礙,dp[i][j]=0,不做累加。因為走不到那格,它給下、右鄰的貢獻自然是 0,路徑就繞開了。

要小心 base(第一列/行):62 是全 1,但這裡一旦遇到障礙,其後全變 0 —— 第一列只能一路往右,被擋住就過不去,後面全到不了。所以不能無腦把邊界填 1;必須「遇障礙即停,之後保持 0」。用整格從 0 起算 + 障礙設 0 的寫法可自動處理這件事。

這也是為什麼 62 的組合公式 C(m+n−2, m−1) 在這裡失效:公式假設每格都能走、每條路等價,一有障礙就沒有封閉解 —— 只能回到逐格 DP。DP 才是網格路徑問題的通用骨架。

3

你的 padding 技巧// 1-INDEXED PAD

你的版本開 (m+1)×(n+1) 的 dp、從 1 開始索引,還設 dp[1][1]=1。這是加一圈 0 的哨兵邊界技巧:

為什麼 padding 省事

多一列 dp[0][*]、一行 dp[*][0] 全 0,當作「界外」哨兵。這樣算 dp[i][j]=dp[i−1][j]+dp[i][j−1] 時,第一列/行也能用同一條式子 —— 它們的「上」或「左」自動是哨兵 0,不必特別寫邊界判斷。

起點 dp[1][1]=1 當種子(對應真正格子 (0,0))。之後每格 dp[i][j] += dp[i−1][j] + dp[i][j−1],障礙格 continue(保持 0)。padding 把邊界特判消掉,程式更乾淨 —— 這是網格 DP 常見的實作小技巧。答案在 dp[m][n]。

4

核心策略 · 三步驟// STRATEGY

  1. 特判:起點或終點是障礙 → 直接回傳 0。
  2. 轉移:非障礙格 dp[i][j] = dp[i−1][j] + dp[i][j−1];障礙格 dp=0(跳過)。
  3. base:起點 = 1;邊界遇障礙後保持 0(或用 padding 哨兵處理)。答案右下角。

複雜度// COMPLEXITY

T = O(mn)　　S = O(mn) → 可壓成 O(n)

每格算一次。可用一維滾動壓到 O(n)(障礙格把 dp[j] 設 0 即可)。答案 ≤ 2×10⁹,略大 —— 用 int 剛好(<2³¹),保險可用 long。
