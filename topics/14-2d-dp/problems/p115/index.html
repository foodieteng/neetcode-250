題目簡介// THE QUESTION

QUESTION

給兩字串 s、t,問 t 在 s 裡以子序列出現幾次(可以跳過字元,但保持順序)。 例 s="rabbbit"、t="rabbit" → 3(三個 b 挑兩個,有 3 種挑法)。

關鍵字是「幾次」—— 這是計數題,不是「能不能」也不是「最長多少」。 跟 1143 LCS(求長度)、97 交錯字串(求可行性)同屬兩字串 DP 家族, 但這題數的是方法數,所以聚合是加法。

2

核心 · 每個 s 字元「拿或不拿」// TAKE or SKIP

狀態 · dp[i][j] = s 前 i 字裡,有幾個子序列 = t 前 j 字

站在 s 的第 i 個字元 s[i−1],問它要不要拿來配 t[j−1]:

● 字元相等 s[i−1] == t[j−1] → 兩條路都可以,相加:
　　dp[i][j] = dp[i−1][j]　(不拿:跳過這個 s 字元,用前面的湊)
　　　　　　+ dp[i−1][j−1]　(拿:用它配掉 t[j−1],兩邊各退一格)

● 字元不等 s[i−1] != t[j−1] → 只能跳過:
　　dp[i][j] = dp[i−1][j]　(這個 s 字元派不上用場,直接丟掉)

為什麼相等時「不拿」也要算?因為就算 s[i−1] 能配 t[j−1],你也可以選擇不用它、 改用 s 前面某個一樣的字元 —— 那是不同的子序列,得分開數。rabbbit 的三個 b 就是這樣各算一種。

3

核心策略 · 三步驟// STRATEGY

  1. base:dp[i][0] = 1 —— 空的 t 永遠有一種湊法(什麼都不選)。第一行 dp[0][j]=0(j≥1):空的 s 湊不出非空 t。
  2. 轉移:相等 → dp[i−1][j] + dp[i−1][j−1];不等 → dp[i−1][j]。
  3. 答案:dp[m][n]。只依賴上(i−1,j)與左上(i−1,j−1)—— 沒有「左」。

和 LCS 的一線之隔// vs 1143

同骨架,差在「相等時做什麼」與「不等時看幾個來源」

115(本題,計數)
　相等:dp[i−1][j] + dp[i−1][j−1]　　不等:dp[i−1][j]　(只看「上」)

1143 LCS(長度)
　相等:dp[i−1][j−1] + 1　　不等:max(dp[i−1][j], dp[i][j−1])　(看「上」和「左」取大)

差別的本質:LCS 兩個字串對等(誰跳過都行),所以不等時要比較「跳 s」和「跳 t」; 115 是把 t 當成固定目標去 s 裡數 —— 只有 s 能跳過,t 的每個字都必須被配掉, 所以不等時只有「跳 s」一條路(只看「上」)。留意 115 沒有「左」dp[i][j−1]。

6

複雜度// COMPLEXITY

T = O(mn)　　S = O(mn) → 可壓成 O(n)

m,n ≤ 1000 → 最壞 10⁶ 格。因為只讀上與左上,一維滾動時要像 1143 一樣留一個 prev 暫存左上 (和 97 不同 —— 97 沒有左上,不用暫存)。

7

一句洞察// INSIGHT

每個 s 字元問「拿去配 t、還是跳過」。相等時兩條都算(相加),不等時只能跳過。 t 是固定目標、只有 s 能跳 → 沒有「左」,只有上 + 左上。
PREV← 含冷凍期 309 NEXT程式碼與複雜度
// ONE-LINE TAKEAWAY
dp[i][j]=s前i字裡=t前j字的子序列數。相等:上+左上(不拿+拿)。不等:只有上。答案 dp[m][n]。
