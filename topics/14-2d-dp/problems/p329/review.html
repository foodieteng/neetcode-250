矩陣中最長遞增路徑

重點回顧// RECAP

面向要點
核心洞察嚴格遞增 ⇒ 網格是 DAG(無環)⇒ 最長路徑可多項式求
A 狀態dp[r][c] = 從此格出發的最長遞增路徑
A 轉移1 + max(更大鄰居的 dp),memo 快取
B 做法Kahn:indeg=更小鄰居數,剝層,層數=答案
不用 visited嚴格 > 天生防環
複雜度兩者 O(mn) / O(mn)

觀念題// QUIZ

Q1. 為什麼這題可以 O(mn) 解,而「一般圖的最長路徑」是 NP-hard?關鍵性質是什麼?
Q2. 為什麼不需要 visited 陣列?如果把嚴格 > 改成 >= 會發生什麼?
Q3. memo 的哨兵用 0 代表「還沒算」為什麼安全?什麼情況下這個技巧會出錯?
Q4. Kahn 版裡,為什麼「層數」剛好等於最長路徑長?一個格子憑什麼要等到它的第幾層才被剝?
Q5. A 的 dp 表和 B 的層數表數字不一樣(A 是 3 4 2 那格最大,B 是 4 在左上),為什麼答案卻相同?它們各自算的是什麼?

範例 Trace// [[3,4,5],[3,2,6],[2,2,1]] → 4

// 另一個官方例 —— DFS memo 表(實跑) matrix memo(dp = 從此格出發最長) 3 4 5 4 3 2 3 2 6 → 1 4 1 2 2 1 2 1 2 最大 dp = 4,出現在左上角 3 這格(也在中間 2 那格) dp[3(左上)] = 1 + dp[4] = 1 + (1 + dp[5]) = 1 + 1 + (1 + dp[6]) = 4 答案 = max(memo) = 4
// 驗證:最長遞增路徑之一 3 → 4 → 5 → 6 (第一列 3,4,5 再到 (1,2)=6) 座標 (0,0)→(0,1)→(0,2)→(1,2) 值 3 < 4 < 5 < 6 ✓ 長度 4 確實 4 格,和 dp 表最大值一致
這題「答案」只是一個數字(最長長度),不用還原路徑。 但 memo 表其實藏了完整資訊:從最大 dp 的格子出發,每步挑「dp = 當前 −1 且更大」的鄰居,就能還原出實際路徑。

應用題// APPLY

App 1. 若要印出其中一條最長路徑(不只長度),用 memo 表怎麼回溯?
App 2. 若允許走八方向(含斜角),或改成「非遞減」(≥ 也能走),解法要怎麼改?哪個改動會破壞正確性?
App 3. 若矩陣是 10⁵ × 10⁵ 但大部分格子相等(遞增路徑很短),兩種解法誰更省?為什麼?能不能只用 O(1) 額外空間(不算輸入)?
// ANSWERS
A1. 關鍵是「嚴格遞增」讓整張圖無環(DAG)。把每格連「小→大」的邊,走一步值必變大,不可能回到起點 —— 沒有環。
一般圖的最長(簡單)路徑要求「不重複經過節點」,環的存在讓這變成 NP-hard(等價於 Hamiltonian path 之類)。但 DAG 沒有環,就沒有「不重複」的顧慮 —— 任何路徑本來就不會重訪節點,於是可以用 DP / 拓撲排序在 O(V+E) 解決。這題 V=mn、E≤4mn,所以 O(mn)。「無環」不是細節,是這題可解的全部理由。
A2. 因為嚴格遞增本身就防環 —— DFS 只會往「更大」走,絕不可能沿路走回一個已在當前路徑上的格子(那要求某值 > 自己)。傳統圖 DFS 需要 visited 來擋環,這裡環根本不存在,visited 是多餘的;更糟的是,加了它可能擋掉合法的重訪 —— 不同起點的路徑會共用同一段尾巴,那段該被重複走過(靠 memo 快取,不是靠 visited 封鎖)。
改成 >=:相等的相鄰格會互相連邊(A≥B 且 B≥A)→ 出現環 → DFS 無窮遞迴(或 memo 邏輯崩壞)。這也是為什麼題目強調「strictly increasing」—— 那個 strict 是正確性的命脈。
A3. 安全,因為真實答案永遠 ≥ 1(任何格子自己就是長度 1 的路徑)。所以 memo[r][c]==0 不可能是一個「算完的合法值」,拿它當「還沒算」的哨兵不會誤判。
什麼時候會出錯:如果某題的 dp 合法值可能是 0(例如「最多能拿幾個」而答案可以是 0),用 0 當哨兵就會把「算出 0」誤認成「還沒算」,每次都重算甚至邏輯錯誤。那時要改用 -1 當哨兵,或另開一個 bool computed[][]。哨兵值必須落在合法答案的範圍之外 —— 這題剛好 0 就在範圍外,是幸運也是要想清楚的點。
A4. 在 Kahn 裡,一個格子只有當它所有「更小的前驅」都被剝掉之後才會 indeg 歸零、進入佇列。所以它被剝的層數 = 從某個源點到它的最長鏈長度 —— 它必須等到最深的那條前驅鏈整條剝完。
形式上:層數滿足 layer[v] = 1 + max(layer[u]),對所有邊 u→v(u 更小)。這正是 DAG 最長路徑的遞推式,只是用 BFS 分層「自動」算出來。所以剝到最後一層時的總層數 = 整張圖最長鏈的格數 = 答案。「等最深前驅」這件事,就是最長路徑的定義。
A5. 因為兩者算的是最長鏈的兩端:
● A 的 dp[r][c] = 「從這格往後還能走多長」(往更大的方向)。所以最小值的格子 dp 最大(它是整條鏈的起點),例 1 裡值 1 那格 dp=4。
● B 的層數 = 「到這格為止已經走了多長」(從最小值數過來)。所以最大值的格子層數最大(它是鏈的終點),例 1 裡值 9 那格層=4。
一個從尾巴倒數、一個從頭正數,但同一條最長鏈的總長度是同一個數。A 取 max(dp)、B 取總層數,都命中那條鏈的長度 4。兩個表是同一座山的兩側等高線 —— 峰頂高度(答案)當然一樣。
App 1. memo 表足以回溯。步驟:先找 dp 最大的格子當起點(它的 dp = 答案 = 鏈長)。然後在當前格 (r,c),找一個更大且 dp[鄰居] == dp[r][c] − 1 的鄰居,走過去、記錄;重複直到 dp==1(鏈尾,沒有更大鄰居)。
正確性:dp[r][c] = 1 + dp[下一步] 保證了「dp − 1」的鄰居一定存在且在最長鏈上。O(鏈長) 回溯,O(1) 額外空間。DP 表天然是一張「決策圖」,回溯只是順著它走。
App 2. 八方向:只要把 DIR 從 4 個改成 8 個(加四個斜角),其餘完全不變 —— 邊多了但仍是「小→大」的 DAG,無環性質不受影響,O(mn) 照舊(常數變 2 倍)。正確性不破壞。
改成「非遞減」(≥ 也能走):會破壞正確性 —— 相等的相鄰格互相連邊 → 成環 → 不再是 DAG，memo/Kahn 都失效(DFS 無窮迴圈、Kahn 有格子永遠 indeg>0 剝不掉)。若題目真要「非遞減最長路徑」,得先把相等的連通塊縮成一個超級節點(縮點),再在縮點後的 DAG 上跑 —— 難度整個升一級。strict 與否，決定了它還是不是 DAG。
App 3. 兩者都是 O(mn) 時間,和路徑長短無關(每格都要碰一次)。但論空間與穩健性:
● 記憶化 DFS 的隱憂是遞迴深度 = 最長路徑長。這題「路徑很短」的前提下,遞迴很淺,DFS 反而安全又省。但若換成「一條長蛇」矩陣,深度 O(mn) 會爆 stack —— 那時 Kahn 更穩(無遞迴)。
● 額外空間:兩者都要一張 O(mn) 輔助表(memo / indeg)。想壓到 O(1) 額外空間,可以把 dp 值直接編碼進原矩陣(例如用高位 bit 存 dp、低位存原值),省掉 memo 表 —— 但這是 hacky 技巧,面試講得出思路即可,不必真寫。
判斷準則:路徑淺 → DFS 簡潔夠用;可能很深 / 怕爆 stack → Kahn。時間都一樣,選的是穩健性和可讀性。
