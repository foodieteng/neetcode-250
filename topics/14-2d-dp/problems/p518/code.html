完整實作// O(n·amount) TIME · O(amount) SPACE

p518.cpp C++17 
class Solution {
public:
    int change(int amount, vector<int>& coins) {
        vector<unsigned int> dp(amount + 1, 0);
        dp[0] = 1;                        // 不拿任何硬幣 = 湊出 0 的一種

        for (int coin : coins) {              // 外層=硬幣 → 組合(不管順序)
            for(int i = coin; i <= amount; i++) {  // 內層遞增 → 完全背包(可重複拿)
                dp[i] += dp[i - coin];
            }
        }

        return (int)dp[amount];
    }
};

整題只有兩個決定,而且都藏在迴圈裡:外層是硬幣(所以是組合)、內層遞增(所以可重複拿)。 沒有任何去重程式碼 —— 去重是迴圈順序免費送的。

四種寫法並排 · 每個都跑過// MEASURED

同樣 coins=[1,2,5]、amount=5,四種迴圈寫法給出四個不同的答案:

// A · 硬幣外層 + 遞增 —— 518 正解 for (int coin : coins) for (int i = coin; i <= amount; i++) dp[i] += dp[i - coin]; → 4 組合數:{5} {2,2,1} {2,1,1,1} {1×5} ✓ 這題要的
// B · 金額外層 + 遞增 —— 你問的「交換」,變成 377 for (int i = 1; i <= amount; i++) for (int coin : coins) if (i >= coin) dp[i] += dp[i - coin]; → 9 排列數:多算了 1+2+2 / 2+1+2 / 2+2+1 等順序 ✗ 這題會 WA
// C · 硬幣外層 + 遞減 —— 變成 0/1 背包(416/494 的形狀) for (int coin : coins) for (int i = amount; i >= coin; i--) dp[i] += dp[i - coin]; → 1 每種硬幣只准拿一枚 → 只剩 {5} ✗ 忽略了「無限多枚」
// D · 金額外層 + 遞減 —— 沒有意義 金額當外層時,每個 dp[i] 只被算一次、且只讀「比自己小」的格子 → 內層方向根本無關,遞減不會產生任何「防重複」效果 → 9 和 B 一樣。這格在概念上是空的
4 / 9 / 1 —— 三個答案,同一份資料,只差迴圈。 這就是為什麼背包題「看起來會寫」和「真的懂」差很多: 能講出你選的每一行為什麼是那個方向,才算真的會。
3

這題的 unsigned:是好判斷// 377 vs 518 vs 494

你在 494 用 unsigned 是多餘(答案 ≤ 2²⁰)。 但這題不一樣 —— 這裡它真的有用,理由很細,值得記住。

關鍵 · 題目只保證「最終答案」放得下,沒保證中間值

題目寫「The final answer is guaranteed to fit into a signed 32-bit integer」—— 注意是 final answer。中間的 dp[i](i < amount)不在保證範圍內。

實測的反例(完全合法的輸入):coins = [2,4,6,…,300](150 種)、amount = 4999。 金額是奇數、硬幣全是偶數 → 湊不出來,dp[4999] = 0,答案完美符合保證。 但過程中:

　dp[4998] = 天文數字(遠超 2⁶⁴)
　5000 格裡有 2378 格超過 INT_MAX

用 int 寫,這些格子的 += 就是有號溢位 = UB。 拿 -fsanitize=signed-integer-overflow 一跑當場抓到: runtime error: signed integer overflow: 27131803 + 2123074792 cannot be represented in type 'int'。 換成 unsigned int,溢位是定義良好的 mod 2³²,不是 UB —— 而且因為答案那格是 0,結果照樣正確。

題目答案本身會溢位?中間值會溢位?unsigned 的角色
377 會(可達 3.8×10⁹) 會 必要 —— 題目就是要 mod 2³² 的截斷值
518(本題) 不會(題目保證) 會(實測 2378 格) 好判斷 —— 避開合法輸入上的 UB
494 不會(≤ 2²⁰) 不會(≤ 2²⁰) 多餘 —— 但無害
三題三種情況,不要一招打天下。問法固定是兩句: 「答案的上界是多少?」→ 決定回傳型別夠不夠; 「中間值的上界是多少?」→ 決定計算過程會不會 UB。 518 是第二句才踩到的那種 —— 也是最容易被忽略的那種。

複雜度分析// COMPLEXITY

步驟成本說明
開 dpO(amount)amount ≤ 5000
外層硬幣n 輪n ≤ 300
內層金額每輪 ≤ amount一次加法,O(1)
T = O(n · amount) ≤ 300 × 5000 = 1.5×10⁶　　S = O(amount)

原本的完整狀態是二維 dp[k][i](用前 k 種硬幣湊出 i)。 因為每輪只讀自己這一列的左邊(dp[k][i−coin],遞增時已更新),第一維可以直接滾掉 → 一維。 「內層遞增」和「能壓成一維」是同一件事的兩面。

邊界與陷阱// PITFALLS

陷阱症狀正解
迴圈交換(你問的) 答案偏大 —— 算成排列。[1,2,5]/5 → 9 而非 4 硬幣必須在外層
內層寫成遞減 答案偏小 —— 變 0/1 背包。[1,2,5]/5 → 1 而非 4 硬幣無限多枚 → 遞增
忘了 dp[0]=1 整張表全 0,答案永遠 0 空組合是計數的種子
amount = 0 以為要特判 不用。dp 長度 1、dp[0]=1,內層一次都不跑 → 直接回 1 ✓
用 int 合法輸入上有號溢位 UB(UBSan 可抓到) unsigned int(見上一節)
內層從 0 起跑 dp[i − coin] 索引負數 → 爆 i = coin 起跑,天然擋掉

細節走讀// WALKTHROUGH

// A · 正解逐輪:coins=[1,2,5], amount=5 金額: 0 1 2 3 4 5 初始 [ 1, 0, 0, 0, 0, 0 ] coin=1 (i=1→5) [ 1, 1, 1, 1, 1, 1 ] 只用 1:每個金額都只有 1 種 coin=2 (i=2→5) [ 1, 1, 2, 2, 3, 3 ] 用 {1,2} coin=5 (i=5→5) [ 1, 1, 2, 2, 3, 4 ] 用 {1,2,5} return dp[5] = 4
// 每一輪 dp[i] 的意思(這是理解的關鍵) 跑完 coin=1 那輪 → dp[i] = 「只用硬幣 {1}」湊出 i 的組合數 跑完 coin=2 那輪 → dp[i] = 「只用硬幣 {1,2}」湊出 i 的組合數 跑完 coin=5 那輪 → dp[i] = 「只用硬幣 {1,2,5}」湊出 i 的組合數 = 二維 dp[k][i]「用前 k 種硬幣湊出 i」滾掉第一維的結果 → 每個組合被綁定成唯一的標準順序(硬幣索引由小到大)→ 天生不重複
// B · 交換後逐輪:同樣資料,答案變 9 金額: 0 1 2 3 4 5 [ 1, 1, 2, 3, 5, 9 ] dp[3] = dp[2] + dp[1] = 2 + 1 = 3 ← 組合數其實只有 2({1,1,1}、{1,2}) 多出來的第 3 個是「2+1」= 和「1+2」重複 dp[5] = dp[4] + dp[3] + dp[0] = 5 + 3 + 1 = 9 9 = 排列數 —— 這是 LC 377 的正解,不是 518 的

驗算小技巧:coins=[1,2]、amount=4 是最小的能看出差別的例子 —— 組合 3 種({1,1,1,1}、{1,1,2}、{2,2}), 排列 5 種(多了 1+1+2 / 1+2+1 / 2+1+1 算三種)。 寫完之後拿這組跑一次,馬上知道自己寫的是哪一題。
