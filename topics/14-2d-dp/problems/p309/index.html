買賣股票含冷凍期

題目簡介// THE QUESTION

QUESTION

給每日股價 prices。可以買賣任意多次(但同時最多持有一股,要先賣才能再買), 限制是賣出後隔天不能買(冷凍期一天)。求最大獲利。 例 [1,2,3,0,2] → 3(買 1 賣 2 → 冷凍 → 買 0 賣 2)。

前面幾題(198 打家劫舍之類)的狀態是「位置」。 這題的關鍵是察覺:每天結束時,你的處境不只由「第幾天」決定,還要知道手上有沒有股票、昨天有沒有剛賣。 這就是狀態機 DP —— 一天不只一個 dp 值,而是好幾個並排的狀態。

2

三個狀態 · 為什麼是三個// THE STATE MACHINE

核心 · 每天結束時,你一定在這三種之一

● hold[i] —— 第 i 天結束時手上有股票的最大獲利
● sold[i] —— 第 i 天剛剛賣掉(→ 明天是冷凍期,不能買)
● rest[i] —— 第 i 天結束時空手,而且不是剛賣(→ 明天可以買)

為什麼「空手」要拆成 sold 和 rest 兩個? 因為光知道「空手」不夠 —— 明天能不能買,取決於「是不是今天才賣的」。 兩種都是空手,但能做的事不同,所以必須是不同狀態。

這就是狀態設計的通則:如果兩個處境「接下來能做的事」不一樣,它們就不能合併成同一個狀態。 冷凍期的全部資訊,就靠 sold 和 rest 的區分承載。

3

三條轉移 · 冷凍期藏在哪// TRANSITIONS

狀態轉移白話
hold[i] max(hold[i−1], rest[i−1] − p[i]) 續抱,或今天買 —— 只能從 rest 買
sold[i] hold[i−1] + p[i] 今天賣掉 —— 昨天必然持有,沒有 max
rest[i] max(rest[i−1], sold[i−1]) 繼續空手,或昨天賣的今天冷凍完
整個冷凍期,就濃縮在第一條的一個字上:hold 讀的是 rest[i−1], 不是 sold[i−1]。昨天剛賣的錢碰不到今天的買進 —— 它得先花一天走到 rest。 把 rest[i−1] 改成 max(rest[i−1], sold[i−1]),冷凍期就沒了,這題直接退化成 LC 122。

起手式與收尾// BASE & ANSWER

邊界 · 為什麼 base 和答案長這樣

Base(第 0 天):hold[0] = −prices[0](第一天就買,現金變負)、sold[0] = 0、rest[0] = 0。
sold[0]=0 其實是個「不可能發生」的狀態(第一天不可能賣掉沒買的股票),但設 0 是安全的 —— 它只會流進 rest[1] = max(rest[0], sold[0]) = 0,而 rest[1] 本來就該是 0。

答案:max(sold[n−1], rest[n−1]) —— 不看 hold。 因為最後一天手上還抱著股票,那筆錢沒兌現(hold 是扣掉成本的負值), 永遠不可能比賣掉更好。留著股票不賣沒有任何好處。

6

複雜度// COMPLEXITY

T = O(n)　　S = O(n) → 可壓成 O(1)

每天只做三次 max/加法。因為每個狀態只看昨天,三條陣列可以直接換成三個變數 → O(1) 空間 (就是你的 follow-up 寫法,見下一頁)。

7

一句洞察// INSIGHT

「空手」要拆成 sold(剛賣,不能買)和 rest(可以買) —— 因為兩者接下來能做的事不同。冷凍期就藏在 hold 只讀 rest[i−1] 這一個字裡。
