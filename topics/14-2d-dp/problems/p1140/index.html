題目簡介// THE QUESTION

QUESTION

一排石頭堆 piles。Alice、Bob 輪流,Alice 先手。當前參數 M(初始 1),每回合可拿最前面 X 堆,1 ≤ X ≤ 2M,拿完後 M 更新為 max(M, X)。兩人最優,求 Alice 最多拿多少石頭。

和 877 石頭遊戲同屬博弈 DP,但這裡有兩個新東西: 狀態多了一個參數 M(能拿幾堆會變),而且拿的是前綴(只從一端拿、可拿多堆)。 這兩點讓狀態變成 dp[i][m],並且很自然地用上後綴和。

2

核心 · dp[i][m] 與那個恆等式// THE STATE

狀態 · dp[i][m] = 我從 piles[i..] 能拿的最多石頭

dp[i][m] = 「輪到我、面對 piles[i..n−1]、當前參數 M=m 時,我能拿到的最多石頭」。 和 877 一樣不記玩家(誰面對這個局面都一樣),但多了 m 這一維,因為「能拿幾堆」會隨遊戲變。

關鍵恆等式(整題的靈魂):我拿走前 x 堆後,對手在 piles[i+x..] 玩同一個遊戲,拿到 dp[i+x][max(m,x)]。 而從第 i 堆之後,不是對手拿的,全是我的。所以:

我的 = suffixSum[i] − dp[i+x][max(m,x)]

suffixSum[i] = piles[i..] 的總和。這一步把「我拿的 + 我從剩下再拿的」合併成 「剩下總共 減 對手拿的」—— 和 877 的減號是同一個把戲,只是這裡用後綴和表達「剩下總共」。

3

兩個分支 · base 與 general// BASE + GENERAL

分支 · 拿得到底就全拿,否則枚舉拿幾堆

● BASE:i + 2m ≥ n
當前能拿的堆數 2m 已經覆蓋到最後一堆 —— 直接一次全拿,對手沒東西可拿:
　dp[i][m] = suffixSum[i]

● GENERAL:否則枚舉 x = 1..2m
拿前 x 堆,對手接手 max(m,x) 的參數,取讓我最多的 x:
　dp[i][m] = max( suffixSum[i] − dp[i+x][max(m,x)] )

為什麼 base 是必要的、不能只寫 general?因為 general 的迴圈 x=1..2m 會存取 dp[i+x][…], 當 i+x 超出範圍就爆。base 正是「x 能一路拿到底」的情形 —— 那時答案直接是 suffixSum[i], 連遞迴都不用。這就是你標的「base + general」:一個收尾、一個推進。

為什麼「多拿」不一定好// THE TRADE-OFF

拿越多 → 對手的 M 越大 → 對手能搶越多

動畫裡 dp[0][1]:拿 1 堆得 10、拿 2 堆只得 9。拿少的反而多!

因為拿 x 堆會把對手的參數推到 max(m, x) —— 你拿越多,對手下一步能拿的上限 2·max(m,x) 就越大, 可能一口氣把後面的大堆全掃走。所以這不是貪心「能拿多少拿多少」,而是權衡:眼前多吃一點 vs 給對手更大的胃口。 這正是要 DP、不能貪心的原因。

6

複雜度// COMPLEXITY

T = O(n³)　　S = O(n²)

狀態 dp[i][m] 有 O(n²) 個(i,m ≤ n),每個 general 分支枚舉 x 最多 2m = O(n) → O(n³)。 n ≤ 100,10⁶ 級,穩過。後綴和讓「剩下總共多少」變 O(1) 查詢,是能壓到這個複雜度的關鍵。

7

一句洞察// INSIGHT

我的 = suffixSum[i] − 對手的(和 877 減號同源,用後綴和表達「剩下總共」)。 狀態多一維 m(能拿幾堆);base(i+2m≥n)全拿收尾、general 枚舉 x 推進。多拿會養大對手 → 要 DP 權衡。
