完整實作 · dp[i][m] + 後綴和// O(n³)

p1140.cpp C++17 
class Solution {
public:
    int stoneGameII(vector<int>& piles) {
        int n = piles.size();

        // dp[i][m] = 從 piles[i..] 開始、當前 M=m,先手能拿的最多石頭
        vector<vector<int>> dp(n, vector<int>(n + 1, 0));

        // suffixSum[i] = piles[i..] 的總和
        vector<int> suffixSum(n, 0);
        suffixSum[n - 1] = piles[n - 1];
        for (int i = n - 2; i >= 0; i--)
            suffixSum[i] = suffixSum[i + 1] + piles[i];

        for (int i = n - 1; i >= 0; i--) {       // i 由後往前(依賴更右邊)
            for (int m = 1; m <= n; m++) {
                if (i + 2 * m >= n) {            // BASE:拿得到底 → 全拿
                    dp[i][m] = suffixSum[i];
                } else {                          // GENERAL:枚舉拿 x 堆
                    for (int x = 1; x <= 2 * m; x++) {
                        dp[i][m] = max(dp[i][m],
                            suffixSum[i] - dp[i + x][max(m, x)]);
                    }
                }
            }
        }
        return dp[0][1];                     // 從頭、M=1、Alice 先手
    }
};

i 由後往前填,因為 dp[i][m] 依賴 dp[i+x][…](更右邊的、更短的局面)。 dp 開 n+1 欄是為了讓 m 最大到 n 不越界。

2

拆解 · suffixSum[i] − dp[i+x][max(m,x)]// THE LINE

片段意思
suffixSum[i]piles[i..] 全部石頭 —— 我和對手加起來會拿光這些
i + x我拿走前 x 堆後,對手從第 i+x 堆開始
max(m, x)對手接手時的參數:M 更新為 max(舊 M, 我拿的 x)
dp[i+x][max(m,x)]對手在剩下那段能拿的最多
suffixSum[i] − …總共 − 對手拿的 = 我拿的(我剛拿的 x 堆 + 我從剩下再搶回的)
這一行把「我拿的 x 堆」和「我之後從剩下再拿的」合併成一個減法。 不用分開追蹤「我已經拿了多少」—— 後綴和 + 一個減號就把整條帳算清。這是博弈 DP 的招牌手法(對照 877)。
3

base 與 general 各自的職責// WHY BOTH

// BASE: i + 2m >= n 當前能拿 2m 堆,而從 i 到底只剩 n − i 堆,且 2m >= n − i → 我「拿得到最後一堆」→ 直接把 piles[i..] 全部收走,對手 0 分 → dp[i][m] = suffixSum[i] 它同時是「終止條件」:general 的 dp[i+x][…] 遞迴到某點,x 一定能拿到底, 在那裡 base 回傳確定值,遞迴才收得住。少了 base: 1) x 可能讓 i+x 越界 → RE 2) 沒有終止的確定值 → 邏輯不收斂
// GENERAL: 否則枚舉 x = 1..2m 還沒拿到底 → 必須「選拿幾堆」。拿 x 堆的代價是把對手參數推到 max(m,x)。 逐一試 x = 1,2,…,2m,每個算「我的 = suffixSum[i] − dp[i+x][max(m,x)]」,取最大。 base 收尾給確定值,general 在中途做選擇 —— 缺一不可

這就是你標注的「base + general」:base 是「不用選、直接全拿」的終局,general 是「還要權衡拿幾堆」的中局。 所有博弈 / 區間 DP 都是這個骨架 —— 先寫清楚 base(什麼時候答案是確定的),再寫 general(否則怎麼遞推)。

邊界與陷阱// PITFALLS

陷阱症狀正解
漏 base dp[i+x][…] 在 i+x = n 越界 / 遞迴不收斂 i + 2m ≥ n 時直接 suffixSum[i]
max(m, x) 忘了取 對手參數沒更新 → 少算對手能拿的 → 答案錯 對手接手參數 = max(m, x)
轉移用 + 沒把「對手拿的」減掉,變成兩人合作 suffixSum[i] − dp[i+x][…]
i 由前往後填 dp[i+x][…] 還沒算就用到 i 從 n−1 往 0 填
回貪心「能拿多少拿多少」 多拿養大對手 → WA(dp[0][1] 拿 1 堆才最優) 枚舉 x 取 max,DP 權衡
沒開 n+1 欄 m 最大到 n,dp[i][n] 越界 vector<int>(n + 1, 0)
