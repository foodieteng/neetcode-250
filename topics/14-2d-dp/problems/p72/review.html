觀念題// QUIZ

Q1. 三個方向(上 / 左 / 左上)各對應哪種操作?為什麼「操作消耗哪個字串,index 就往哪退」?
Q2. 字元相同時為什麼只取左上、而且不 +1?可不可以還是走上或左?
Q3. base 為什麼是 dp[i][0]=i 和 dp[0][j]=j?這兩條的操作各是什麼?
Q4. 「插入」和「刪除」為什麼是對稱的?這對稱性讓 dp 表呈現什麼性質?
Q5. 這題和 1143 LCS 都有「相同 → 左上」的對角線。兩題的相同分支和不同分支各差在哪?為什麼一個求 min、一個求 max?

應用題// APPLY

App 1. 若三種操作成本不同(刪=1、插=2、替換=3),程式怎麼改?轉移式長什麼樣?
App 2. 若不准替換(只能插入 / 刪除),這題會退化成哪一題?答案和 word1、word2 的什麼量有關?
App 3. 若要還原具體的操作序列(不只距離),要怎麼從 dp 表回溯?為什麼一維滾動版做不到?
// ANSWERS
A1. 上 = 刪除、左 = 插入、左上 = 替換(相同時免費)。
● 刪除 只丟掉 word1 的字 → i 退一步、j 不動 → 來自上 dp[i−1][j]。
● 插入 在 word1 補一個字去配 word2 → 消化掉 word2 的一個字、word1 索引不變 → j 退一步、i 不動 → 來自左 dp[i][j−1]。
● 替換 把 word1 的字改成 word2 的字 → 兩個字對成一對,各消一個 → i、j 都退 → 來自左上 dp[i−1][j−1]。
「操作消耗哪個字串,index 就往那個方向退」:刪除吃 word1(動 i=垂直=上)、插入吃 word2(動 j=水平=左)、替換兩個都吃(斜著退=左上)。三種消耗組合窮盡了「怎麼對齊兩字串最後一個字」的所有方式,所以恰好三個方向、沒有第四個。
A2. 相同時,word1[i−1] 和 word2[j−1] 這對字元本來就對得上,不需要任何操作就能對齊 → 直接繼承「兩邊都少一個字」的子問題 dp[i−1][j−1],成本不變(不 +1)。這是整個 DP 唯一的零成本轉移,也是「距離」能小於 max(m,n) 的原因。
可以走上或左嗎:可以「嘗試」,但那代表「明明能免費對齊,卻硬要刪一個再插一個」—— 一定不會更省(至少多 1 步)。所以相同時直接鎖定左上,連 min 都不用比;硬寫 min(上,左,左上) 也不會錯(左上必然最小),但白算。相同 = 免費對齊,是貪心安全的一步。
A3. base 是「把一個字串變成空字串」或「從空字串生出一個字串」的距離:
● dp[i][0] = i:word2 是空的,要把 word1 前 i 個字全部清掉 → i 次刪除。
● dp[0][j] = j:word1 是空的,要從無到有打出 word2 前 j 個字 → j 次插入。
● dp[0][0] = 0:空到空,零操作。
這兩條邊是所有內部格子遞迴的終點 —— 任何 dp[i][j] 一路退 i、j,最後都會落到某條邊界。漏設(留 0)= 告訴程式「把 5 個字的字串變成空字串只要 0 步」,整張表的刪/插成本全錯。
A4. 因為「在 word1 插入一個字」和「在 word2 刪除一個字」是同一件事的兩種說法 —— 都是讓兩字串在該位置「錯開一格」。所以編輯距離是對稱的:editDist(A, B) == editDist(B, A),把兩個字串互換,答案不變。
在 dp 表上,這對稱性表現為:交換 word1、word2 相當於把整張表沿主對角線轉置,而「上(刪)」和「左(插)」剛好互換位置 —— 兩個方向對稱地站在對角線兩側。
實用價值:因為對稱,你可以讓較短的字串當內層(壓一維時維度取 min(m,n)),省空間。也是驗算的好工具:minDistance(a,b) 和 minDistance(b,a) 應該相等。
A5. 兩題共用「相同 → 左上對角線」的骨架,但做的事相反:
● 相同分支:LCS 是 dp[i−1][j−1] + 1(找到一個共同字元,長度加一);72 是 dp[i−1][j−1](對齊成功,成本不加)。一個「+1 獎勵」、一個「+0 免費」。
● 不同分支:LCS 是 max(上, 左)(跳過某字串一個字,只有兩個方向、求最長);72 是 1 + min(上, 左, 左上)(三種操作、求最少,多了左上「替換」)。
為什麼一 max 一 min:LCS 要「最長的共同子序列」→ 求大;72 要「最少的操作」→ 求小。目標是最大化還是最小化,決定聚合是 max 還是 min —— 同一張表、同一條對角線,換個聚合方向就是另一題。這是兩字串 DP 家族的統一觀。
App 1. 把「+1」換成各操作各自的成本:
dp[i][j] = min( dp[i−1][j] + costDel,　　// 刪
          dp[i][j−1] + costIns,　　// 插
          dp[i−1][j−1] + costRep )// 換,相同時仍是 dp[i−1][j−1](免費)。base 也要跟著改:dp[i][0] = i × costDel、dp[0][j] = j × costIns。
這叫加權編輯距離,是拼寫糾正、DNA 比對的實際模型(不同突變成本不同)。骨架不變,只是每個方向帶自己的價碼 —— 說明這題的結構比「+1」更通用。
App 2. 不准替換 → 只剩插入 / 刪除,退化成 LCS 的變形。
此時 dp[i][j] = 相同 ? dp[i−1][j−1] : 1 + min(上, 左)(拿掉左上那支)。答案 = m + n − 2 × LCS(word1, word2) —— 因為共同子序列的部分不用動,剩下的 word1 獨有字元要刪(m − LCS 個)、word2 獨有字元要插(n − LCS 個)。
直覺:沒有替換這個「一步搞定不同字元」的捷徑,你只能靠一刪一插(兩步)處理每個不匹配。所以答案和 LCS 直接掛鉤 —— 這兩題本就是一家人,替換操作是 72 比 LCS 多出來的那把利器。
App 3. 要保留完整二維表,從 (m, n) 回溯:
在 (i, j),看 dp[i][j] 是從哪個鄰居來的 —— 若 word1[i−1]==word2[j−1] 且 dp[i][j]==dp[i−1][j−1],是「保留(免費)」,退到 (i−1,j−1);否則比對三個來源,哪個 +1 等於 dp[i][j] 就是這步的操作(上→刪、左→插、左上→換),記下並退到對應格。走到邊界為止,反轉就是操作序列。O(m+n) 回溯。
為什麼一維版做不到:回溯需要每一格「當初從哪來」的資訊,而一維滾動只留最後一列、把歷史覆蓋掉了 —— 求「距離」夠,但「怎麼走到的」沒了。這是「算最優值只要 O(n) 空間、還原最優解通常要 O(mn)」的經典取捨,和 LCS 印出子序列、312 印出戳法完全同型。
