完整實作// O(mn) TIME · O(mn) SPACE

p72.cpp C++17 
class Solution {
public:
    int minDistance(string word1, string word2) {
        int m = word1.size(), n = word2.size();
        vector<vector<int>> dp(m + 1, vector<int>(n + 1, 0));

        for (int i = 1; i <= m; i++) dp[i][0] = i;   // word2 空 → 全刪
        for (int j = 1; j <= n; j++) dp[0][j] = j;   // word1 空 → 全插

        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                if (word1[i - 1] == word2[j - 1]) {
                    dp[i][j] = dp[i - 1][j - 1];              // 相同 → 免費對角線
                } else {
                    dp[i][j] = min({dp[i - 1][j],       // 上   → 刪除
                                    dp[i][j - 1],       // 左   → 插入
                                    dp[i - 1][j - 1]})  // 左上 → 替換
                               + 1;
                }
            }
        }
        return dp[m][n];
    }
};

min({a, b, c}) 是 C++11 的 initializer_list 寫法,一次取三個最小 —— 對應三種操作。 相同字元那支不 +1(免費),是整個 DP 唯一「零成本」的轉移。

三方向 → 三操作 對照表// THE MAP

來源方向操作直覺
dp[i−1][j]上刪除 word1[i−1]word1 丟掉一個字,word2 不動
dp[i][j−1]左插入 word2[j−1]補一個字配上 word2,word1 不動
dp[i−1][j−1]左上替換(不同時)/ 免費(相同時)兩字對成一對,各消一個
記法:刪動列(上)、插動行(左)、換動對角(左上)。 「插入」和「刪除」是對稱的 —— 在 word1 插入一個字,等價於在 word2 刪除一個字,所以一個往左、一個往上,方向相反。
3

base 的意義// BASE

// 第一行 / 第一列 = 對「空字串」的距離 dp[i][0] = i : word2 是空的 把 word1 前 i 個字「全部刪掉」才能變成空 → i 次刪除 dp[0][j] = j : word1 是空的 要從空字串生出 word2 前 j 個字 → j 次插入 dp[0][0] = 0 : 空到空,不用動

這兩條邊界是所有轉移的地基 —— 每個內部格子最終都會遞迴回邊界。 漏設(留 0)會讓「把長字串改成短字串」的刪除成本算不出來,整張表崩掉。

邊界與陷阱// PITFALLS

陷阱症狀正解
相同字元也 +1 把「不用動」當成一次操作 → 距離偏大 相同 → dp[i−1][j−1],不 +1
忘了 base dp[i][0]/dp[0][j] 留 0 → 刪/插成本算錯 dp[i][0]=i、dp[0][j]=j
min 只取兩個 漏了替換(左上)→ 有些情況多算 三個都要:min({上, 左, 左上})
空字串 m=0 或 n=0 以為要特判 不用,base 迴圈自然處理(答案 = 另一字串長度)
壓一維左往右覆蓋左上 dp[j−1] 被本列新值蓋掉 → 左上丟失 存 prev 暫存左上(同 1143)
方向記反 把「上」當插入、「左」當刪除 → 對稱題答案剛好也對,但概念錯 上=刪 word1、左=插 word2

細節走讀// horse → ros

// 完整 dp 表(實跑) '' r o s '' 0 1 2 3 h 1 1 2 3 o 2 2 1 2 r 3 2 2 2 s 4 3 3 2 e 5 4 4 3 return dp[5][3] = 3
// 幾個關鍵格子 dp[2][2] : 'o'=='o' → 免費對角線 dp[1][1]=1 (相同) dp[1][1] : 'h'≠'r' → 1 + min(上1, 左1, 左上0) = 1 (替換 h→r) dp[5][3] : 'e'≠'s' → 1 + min(上 dp[4][3]=2, 左 dp[5][2]=3, 左上 dp[4][2]=3) = 1 + 2 = 3 (刪除 e) 最優路徑:horse →(替換 h→r) rorse →(刪 r) rose →(刪 e) ros,共 3 步

看 dp[5][3]=3 怎麼來:三個鄰居 上=2、左=3、左上=3,最小是上=2(刪除),2+1=3。 「上最小 → 這步用刪除」—— 從哪個方向取到 min,回溯時就知道用了哪個操作,這也是還原編輯序列的依據。
