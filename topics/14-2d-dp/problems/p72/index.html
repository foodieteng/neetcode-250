題目簡介// THE QUESTION

QUESTION

給兩字串 word1、word2,問把 word1 改成 word2 的最少操作數。 允許三種操作:插入一個字元、刪除一個字元、替換一個字元。 例 horse → ros → 3。

經典的 Levenshtein 編輯距離,兩字串 DP 家族的代表作(和 1143 LCS、10 正則 同源)。 最漂亮的地方:三種操作剛好對應 dp 表的三個方向 —— 上、左、左上。

2

核心 · 三個方向 = 三種操作// 3 DIRECTIONS

狀態 · dp[i][j] = word1 前 i 字 → word2 前 j 字的最少操作

當 word1[i−1] != word2[j−1],看三個鄰居、各代表一種操作,取最省的 + 1:

● 上 dp[i−1][j] → 刪除
刪掉 word1[i−1],word1 少一個字(i−1)、word2 不變。

● 左 dp[i][j−1] → 插入
在 word1 插入 word2[j−1] 配上 word2 這個字,word2 少一個要配(j−1)、word1 不變。

● 左上 dp[i−1][j−1] → 替換
把 word1[i−1] 換成 word2[j−1],兩邊各消一個字。

字元相同時 · 免費對角線

當 word1[i−1] == word2[j−1],這個字不用動 —— 直接沿用左上 dp[i−1][j−1],不 +1。 相同字元是「白撿的對齊」,零成本走對角線。這是編輯距離和 LCS 都有的核心對角線。

3

核心策略 · 三步驟// STRATEGY

  1. base:dp[i][0] = i(word2 空 → 把 word1 的 i 個字全刪);dp[0][j] = j(word1 空 → 插入 j 個字)。
  2. 相同:dp[i][j] = dp[i−1][j−1](免費對角線)。
  3. 不同:dp[i][j] = 1 + min(上, 左, 左上)(刪 / 插 / 換,挑最省);答案 dp[m][n]。

為什麼恰好是這三個方向// WHY THESE 3

每種操作「消耗」哪個字串,決定 index 往哪走

三種操作的本質是「對齊 word1[i−1] 和 word2[j−1] 的最後一個字」的三種方式:

刪除:word1 這個字不要了 → i 退一步、j 不動 → 上
插入:word2 這個字用「插」補上 → j 退一步、i 不動 → 左
替換:兩個字對成一對 → i、j 都退 → 左上

「操作消耗哪個字串,index 就往哪個方向退。」刪除只動 word1、插入只動 word2、替換兩個都動 —— 三種消耗組合,剛好是上、左、左上三個來源。這也解釋了為什麼沒有第四個方向:三種操作窮盡了所有對齊方式。

6

複雜度// COMPLEXITY

T = O(mn)　　S = O(mn) → 可壓 O(n)

填 (m+1)(n+1) 格、每格 O(1)。m,n ≤ 500 → 25 萬格,穩過。 因為只依賴上一列(上 / 左上)和本列左鄰(左),可壓成一維 O(n),但要用 prev 暫存左上(同 1143)。

7

一句洞察// INSIGHT

三方向 = 三操作:上=刪、左=插、左上=換。相同字元走免費對角線,不同則 1 + min(三者)。 base 是「對空字串全刪 / 全插」。操作消耗哪個字串,index 就往哪退。
