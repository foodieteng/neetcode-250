題目簡介// THE QUESTION

QUESTION

一堆石頭,每次挑兩顆相撞:重的減輕的,差值留下(相等則都消失)。最後至多剩一顆,求它最小可能的重量。

看似模擬遊戲,其實是偽裝的 0/1 背包。關鍵轉化:每顆石頭最終被算成 + 或 −,剩下的重量 = |帶號總和|。要最小 → 把石頭分兩堆、讓兩堆和盡量接近 → 找「不超過 total/2 的最大子集和」。這正是 416 分割等和子集 的同一招。

2

關鍵轉化 · 砸石頭 = 分兩堆// THE REDUCTION

LEMMA · 每次相減 ⟺ 給每顆一個 ± 號

每次「y − x」的相撞,展開到最後,等於把每顆石頭前面配一個 + 或 −,最終剩下 |Σ ± stone|(可證明任意 ± 組合都能由某種相撞順序達成)。

設 正號那堆和 = P、負號那堆和 = N,則 P + N = total、剩重 = |P − N| = |total − 2N|。要最小化,就讓 N(某一堆)盡量接近 total/2。

於是問題變成:用 0/1 背包找「≤ total/2 的最大子集和 best」,答案 = total − 2·best。best 越接近一半,兩堆越平衡,剩重越小。

3

核心策略 · 三步驟// STRATEGY

  1. 目標:total = Σstones、target = total / 2。
  2. 0/1 背包:dp[s] = 子集和 s 是否可達;每顆石頭內層由大到小掃(各用一次),同步記最大可達 best。
  3. 回傳:total − 2·best。
4

動畫演示// [2,3,4] → 1

total=9、target=4。看 0/1 背包把「可達子集和」逐步點亮,找到 ≤ 4 的最大 best=4,答案 = 9 − 2×4 = 1。

和 416 的關係// SAME TABLE

題問用同一張子集和 dp
416能否分成兩等堆看 dp[total/2] 是否可達
1049兩堆最接近時的差找 ≤ total/2 的最大可達 best,答 total−2·best
494± 湊出 target 的方法數子集和 計數(用 + 不是 OR)

三題共用「把 ± 分堆化成子集和」的框架 —— 416 問存在性(能不能剛好一半)、1049 問最接近(差多少)、494 問計數(幾種)。認得這個轉化,一次通吃。

複雜度// COMPLEXITY

T = O(n × total)　　S = O(total)

n ≤ 30、stone ≤ 100 → total ≤ 3000、target ≤ 1500。約 30×1500 = 4.5×10⁴,瞬殺。bitset 可再快約 64 倍。

一句洞察// INSIGHT

砸石頭 = 給每顆 ± 號 = 分兩堆,剩重 = 兩堆差。 讓一堆盡量接近 total/2 → 0/1 背包子集和(同 416),答案 total − 2·best。
