寫法 A · 0/1 背包(你的版本)// SUBSET SUM

p1049.cpp C++17 
class Solution {
public:
    int lastStoneWeightII(vector<int>& stones) {
        int total = accumulate(stones.begin(), stones.end(), 0);
        int target = total / 2;              // 想讓一堆接近一半

        vector<bool> dp(target + 1, false);
        dp[0] = true;
        int best = 0;

        for (int stone : stones) {
            for (int i = target; i >= stone; i--) {  // 由大到小 → 每顆用一次
                dp[i] = dp[i] || dp[i - stone];
                best = dp[i] ? max(i, best) : best; // 記最大可達子集和
            }
        }

        return total - 2 * best;            // 兩堆差
    }
};

這就是 416 的 0/1 背包子集和,只是不要求剛好一半,而是邊填邊記「能湊到、且 ≤ target 的最大和 best」。內層由大到小掃保證每顆石頭只用一次。答案 total − 2·best:best 是較小那堆,另一堆是 total − best,差 = total − 2·best。

寫法 B · bitset 位元平移// ~64× FASTER

p1049_bitset.cpp C++17 
class Solution {
public:
    int lastStoneWeightII(vector<int>& stones) {
        int total = accumulate(stones.begin(), stones.end(), 0);
        int target = total / 2;

        bitset<1501> dp; dp[0] = 1;       // 30×100/2 = 1500 上限
        for (int s : stones) dp |= dp << s;  // 每個可達和 → +s 也可達

        for (int i = target; i >= 0; i--)
            if (dp[i]) return total - 2 * i;       // 由大往小,第一個可達即 best
        return total;
    }
};

dp << s 把「所有可達的子集和」整體往上平移 s、|= 併回去 —— 一行完成「每個可達和 x → x+s」(和 416 同款)。再從 target 往下找第一個為 1 的位,就是 best。用 64-bit 字並行,常數快約 64 倍。

邊界與陷阱// PITFALLS

陷阱後果正解
當成貪心模擬遊戲每次撞最大兩顆 → 不一定最優轉成子集和 DP
內層由小到大掃石頭被重複用(變完全背包)由大到小(0/1,各一次)
回傳 best 本身那是一堆的和,不是差差 = total − 2·best
target = total/2 取整以為漏了 total 奇數的情況沒問題:找 ≤ ⌊total/2⌋ 最大即可

註:total 為奇數時,兩堆最接近也差 1;best ≤ ⌊total/2⌋,total − 2·best ≥ 1,公式自動涵蓋,不必特判。
