範例 Trace// [31,26,33,21,40] → 5

// 子集和最接近一半 total = 151, target = 75 找 ≤ 75 的最大可達子集和: 33 + 40 = 73; 26 + 21 + ... ; 最大能湊到的是 73 best = 73 answer = 151 − 2×73 = 5
兩堆分別是 73 與 78(151−73),差 5。best=73 是「不超過一半、又最接近一半」的那堆和。 這題的難不在 DP(就是 416),而在看穿「砸石頭遊戲」其實是子集和 —— 這種「把模擬題翻譯成經典 DP」的轉化,是進階題的核心能力。

應用題// APPLY

App 1. 494 目標和:每個數前放 + 或 −,問湊出 target 的方法數。怎麼把它也化成子集和?聚合從 OR 換成什麼?
App 2. 若把「差最小」改成「差恰好等於某個 k 是否可能」,DP 要怎麼查?
App 3. 若石頭重量可以到 10⁹(但顆數仍少),O(n·total) 的背包爆掉。這時還有別的路嗎?為什麼「值域大、顆數小」會改變策略?
// ANSWERS
A1. 每次「重的減輕的」y − x,把整個過程展開,等於最後每顆石頭前面掛一個 + 或 −,剩下 |Σ ± stone|(可證明:任何一種 ± 分配,都存在一種相撞順序達成它;反之相撞結果一定是某種 ± 分配)。而「+ 號一堆、− 號一堆」正是把石頭分成兩堆 —— 剩重就是兩堆和的差 |P − N|。所以砸石頭、± 號、分兩堆是同一件事的三種說法。
A2. 設較小那堆的和是 best,另一堆就是 total − best,兩堆差 = (total − best) − best = total − 2·best。要最小化差就要最大化 best(讓小堆盡量大、逼近一半)。best = 「用子集湊得出、且不超過 total/2 的最大和」。所以答案就是 total − 2·best。
A3. 因為兩堆是對稱的:若一堆 > total/2,另一堆必 < total/2,兩者的「差」一樣。所以只需盯著「較小(≤ 一半)那堆」把它做到最接近一半就好 —— 找 ≤ total/2 的最大可達和即可,超過一半的情形由對稱自動涵蓋。這也讓 dp 陣列只要開到 total/2,省一半空間。
A4. 同一張 dp[s](子集和 s 是否可達),差在讀法:416 問「能不能剛好平分」→ 直接看 dp[total/2] 是不是 true(且 total 要偶數);1049 問「最接近時差多少」→ 從 total/2 往下找第一個 true 當 best,回 total − 2·best。416 是 1049 的特例:當 best 剛好 = total/2(答案 0)時,就是能平分。一張表、兩種提問。
A5. 因為「差最小」是全域最佳化,貪心的局部選擇會漏掉更好的全域組合。反例直覺:[3, 9, 7, 3] —— 貪心先撞 9 和 7 得 2,再和 3、3… 不一定得到最小;實際最優是把它分成 {9,3}=12 與 {7,3}=10,差 2,需要「看整體怎麼分兩堆」而非逐次撞最大。貪心看不到「為了整體平衡,現在該撞誰」,所以要 DP 枚舉所有子集和。
App 1. 494 也化成子集和。設正號子集和 P、負號 N,P + N = total、P − N = target → P = (total + target) / 2。於是「有幾種 ± 湊出 target」= 「有幾個子集其和 = (total+target)/2」—— 變成子集和計數。聚合從 dp[s] |= dp[s−num](存在性 OR)換成 dp[s] += dp[s−num](計數用加法),base dp[0]=1。前提 (total+target) 為偶且 |target| ≤ total。同一個「± 分堆 → 子集和」框架的計數版。
App 2. 「差恰好 = k」代表兩堆和為 (total−k)/2 與 (total+k)/2(需 total−k 為偶、非負)。所以查 dp[(total−k)/2] 是否可達即可(較小那堆)。這又回到 416 式的「存在性」查詢:同一張子集和 dp,把目標從「一半」換成「(total−k)/2」。若那格 true 就可能、否則不行。
App 3. 值域大(重量到 10⁹)會讓 O(n·total) 的背包陣列開不下、時間也爆 —— 因為複雜度綁在「和的大小 total」上,不是顆數。但若顆數 n 很小(如 ≤ 30~40),可改用折半枚舉(meet in the middle):把石頭分兩半各 O(2^(n/2)) 枚舉所有子集和,再排序 + 雙指針/二分找「兩半湊起來最接近 total/2」。複雜度 O(2^(n/2) · n),和「值域」無關。「值域大、顆數小」→ 折半枚舉;「值域小、顆數大」→ 背包,是子集和問題選型的關鍵。
