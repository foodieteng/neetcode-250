題目簡介// THE QUESTION

QUESTION

給 nums 和 target。在每個數字前面加上 + 或 −,問有幾種加法組合會等於 target。 例 nums=[1,1,1,1,1]、target=3 → 5 種。

n ≤ 20、sum(nums) ≤ 1000。表面上是「枚舉 2ⁿ 種符號」, 但只要做一個代數改寫,它會塌成一題你已經做過的 —— 416 分割等和子集的計數版。

2

核心改寫 · 這題的全部// THE REFRAME

LEMMA · ± 號 = 分兩堆 ⇒ 正號堆的和是「固定值」

給每個數字一個符號 = 把它們分成兩堆:正號堆(和記為 P)與負號堆(和記為 N)。於是:

　P − N = target　← 算式的值
　P + N = total　← 所有數字的總和(固定)
─────────────────── 兩式相加,N 消掉
　2P = target + total　⇒　P = (target + total) / 2

關鍵在於:P 是一個算得出來的常數 —— 它只跟輸入有關,不是變數。 所以「有幾種 ± 組合湊出 target」完全等價於「nums 有幾個子集的和 = P」。 後者就是標準 0/1 背包計數。原本 2²⁰ 的符號枚舉,變成 O(n·P) 的填表。

和 1049 石頭重量 II 是同一個改寫,只差最後要什麼: 1049 要「兩堆差最小」→ 找最接近 total/2 的可達和; 494 要「湊出指定 target 的方法數」→ P 直接算出來,數方法數。

核心策略 · 三步驟// STRATEGY

  1. 算 P:total = sum(nums),P = (target + total) / 2。
  2. 兩道守門:total < |target| → 湊不到,回 0;(target+total) 是奇數 → P 不是整數,回 0。
  3. 0/1 背包計數:dp[0]=1;每個 num 由大到小跑 dp[i] += dp[i−num];答案 dp[P]。

兩道守門為什麼必要// THE TWO GUARDS

GUARD · 沒擋的話會拿到負數索引或錯答案

● total < |target| → 回 0
全部給正號最多也才 total、全給負號最少 −total。目標超出這個區間就不可能。 不擋的話 P = (target+total)/2 會是負數,vector 開負長度直接炸。

● (target + total) 是奇數 → 回 0
因為 P = (target+total)/2 必須是整數(它是一堆整數的和)。 奇數代表根本湊不出來 —— 例 nums=[2,2]、target=1:±2±2 永遠是偶數。 不擋的話整數除法會無聲地捨去,算出一個錯的 P,回傳錯答案。

兩道守門的順序也有意義:先擋 total < |target|,才保證 target+total ≥ 0, 後面 sum & 1 的位元判斷才不會碰到負數。

複雜度// COMPLEXITY

T = O(n · P)　　S = O(P)　　P = (target+total)/2 ≤ 1000

最壞 20 × 1000 = 2 萬次更新。對比暴力的 2²⁰ ≈ 100 萬 —— 快 50 倍,而且不隨 n 指數成長。 這是偽多項式(pseudo-polynomial):成本跟數值大小 P 走,不是跟位元數走。

面向成本說明
TimeO(n·P)n 個數字 × 每個掃過 P 格
SpaceO(P)一維滾動,不需二維表
答案上界2²⁰≤ 1,048,576 —— 遠小於 INT_MAX,不會溢位
常數極小只有一次加法;dp[i−num]==0 時可跳過
