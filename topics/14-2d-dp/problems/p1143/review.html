重點回顧// RECAP

面向要點
狀態dp[i][j] = 前 i、前 j 字的 LCS 長度
相同dp[i][j] = dp[i−1][j−1] + 1(左上斜)
不同dp[i][j] = max(dp[i−1][j], dp[i][j−1])
base第一列/行 = 0(和空字串)
答案dp[m][n](右下角)
1D 壓維需 prev 留住左上斜

Q1. 字元相同時為什麼看左上斜 dp[i−1][j−1],而不是 max(上, 左)+1?

Q5. LCS 和「最長公共子字串(要連續)」的 DP 差在哪?為什麼一個答案在右下角、一個要取全表最大?

範例 Trace// "bl" × "yby" → 1

// 填表 ∅ y b y ∅ 0 0 0 0 b 0 0 1 1 b==b → dp[0][1]+1=1 l 0 0 1 1 l≠… → max(上,左) return dp[2][3] = 1(LCS = "b")
只有一個公共字元 b,所以 LCS = 1。注意最後一列的 l 和誰都不同,整列靠 max(上, 左) 把前面的 1 沿用下來 —— 這就是「不同時不新增、但把已有的最好結果帶過來」的機制,確保右下角是全域最優。

應用題// APPLY

App 1. 72 編輯距離:把 a 改成 b 的最少「插入/刪除/替換」次數。轉移和 LCS 有多像?相同、不同時各是什麼?
App 2. 583 兩個字串的刪除操作:只能刪字,問讓兩字串相同的最少刪除數。怎麼用 LCS 一步得到答案?
App 3. 若要還原出一條 LCS 字串(不只長度),怎麼從 dp 表回溯?回溯時「相同」和「不同」各往哪走?
// ANSWERS
A1. 因為相同的這兩個字元配成一對加進 LCS 後,它們都被用掉了,剩下的子問題是「text1 前 i−1 字 與 text2 前 j−1 字」的 LCS —— 那正是左上斜 dp[i−1][j−1],再 +1 算上這對。若寫 max(上,左)+1,上=dp[i−1][j] 只退了 text1、text2 的第 j 字還在,可能把同一個字元重複配對,算出過大的長度。所以相同必用斜對角。
A2. 字元不同,代表這兩個字不可能同時出現在 LCS 的結尾(它們不相等),所以至少要捨棄其中一個。dp[i−1][j](上)= 「丟掉 text1 的第 i 字、看剩下的 LCS」;dp[i][j−1](左)= 「丟掉 text2 的第 j 字」。兩種捨棄取較好的(max),就是不同時的最優。不新增長度(這步沒配到對),只是繼承。
A3. 因為 dp[0][j] 代表「text1 取空字串 與 text2 前 j 字」的 LCS,空字串和任何東西都沒有共同子序列 → 長度 0;同理 dp[i][0]=0。這圈 0 是遞迴的錨點:所有 dp[i][j] 最終都回溯到某個和空字串比較的 base。padding 成 (m+1)×(n+1) 讓這圈 0 自然存在,內部格也能無邊界特判地套轉移。
A4. prev 存的是左上斜 dp[i−1][j−1]。在一維滾動裡,掃到第 j 格時,dp[j−1] 已經被本列(第 i 列)更新成新值了,不再是我們要的「上一列的 j−1」。所以要在更新 dp[j−1] 之前先把它(其實是上一輪存下的 dp[j])記在 prev。作法:每格先 tmp = dp[j](這是這一格的舊值 = 下一格的左上斜),算完再 prev = tmp。不留 prev 就會用到被覆蓋的新值,相同情形算錯。這是「需要斜對角」的 DP 壓維通用技巧。
A5. 差在「不同 / 不匹配時怎麼辦」與「答案在哪」。LCS(子序列,可不連續):不同時 max(上,左) —— 把已累積的最好結果繼承下去,所以 dp 單調不減,最終最優落在右下角 dp[m][n]。最長公共子字串(要連續):一旦字元不同就斷開歸 0(連續性中斷,不能繼承),dp[i][j] 只表示「以 i、j 結尾的公共後綴長度」,所以最長可能出現在任何一格,要取全表最大。同一張表、同樣「相同 → 斜 +1」,但不同時的處理(繼承 vs 歸零)決定了兩種語意。
App 1. 72 幾乎同構。dp[i][j] = 把 a 前 i 字改成 b 前 j 字的最少操作。相同:dp[i][j]=dp[i−1][j−1](不用動,直接繼承斜對角);不同:dp[i][j]=1+min(dp[i−1][j−1] 替換, dp[i−1][j] 刪除, dp[i][j−1] 插入)。base 是 dp[i][0]=i、dp[0][j]=j(全刪/全插)。骨架和 LCS 一樣(依賴上、左、左上斜),只是相同時「繼承斜對角」、不同時「三選一 +1」——LCS 是它的近親。
App 2. 直接用 LCS:兩字串各自要刪到只剩「公共部分」,而能保留的最長公共部分就是 LCS。所以最少刪除數 = (m − LCS) + (n − LCS) = m + n − 2·LCS。先算 L = longestCommonSubsequence(a,b),答案就是 m + n − 2L。這是 LCS 的經典應用:凡是「刪字使兩串相等 / 相似度」的題,常先求 LCS 再換算。
App 3. 從右下角 (m,n) 往回走:① 若 text1[i−1]==text2[j−1](相同),這個字元在 LCS 裡 → 記下它,往左上斜 (i−1,j−1) 退;② 若不同,往 dp 較大的那個鄰格走(dp[i−1][j] vs dp[i][j−1],相等任選)—— 因為當初就是從那裡繼承來的。一路退到邊界,把記下的字元反轉就是一條 LCS。要回溯需保留二維 dp 表(一維滾動丟了歷史)。
