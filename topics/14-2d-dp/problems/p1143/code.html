寫法 A · 二維(你的版本)// 2-D

p1143.cpp C++17 
class Solution {
public:
    int longestCommonSubsequence(string text1, string text2) {
        int m = text1.size(), n = text2.size();
        vector<vector<int>> dp(m + 1, vector<int>(n + 1, 0)); // 第一列/行 = 0

        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                if (text1[i - 1] == text2[j - 1]) {       // 相同 → 配對
                    dp[i][j] = dp[i - 1][j - 1] + 1; // 左上斜 +1
                } else {                                // 不同 → 丟一字
                    dp[i][j] = max(dp[i - 1][j], dp[i][j - 1]); // 上、左取大
                }
            }
        }

        return dp[m][n];
    }
};

兩種情況一目了然:相同就從左上斜 dp[i−1][j−1] 接上、+1;不同就在「丟 text1 這字(上)」與「丟 text2 這字(左)」之間取大。(m+1)×(n+1) 的 0 邊界對應「和空字串的 LCS = 0」,答案在 dp[m][n]。

寫法 B · 一維(留住左上斜)// O(n) SPACE

p1143_1d.cpp C++17 
class Solution {
public:
    int longestCommonSubsequence(string a, string b) {
        int m = a.size(), n = b.size();
        vector<int> dp(n + 1, 0);

        for (int i = 1; i <= m; i++) {
            int prev = 0;                     // prev = dp[i-1][j-1](左上斜)
            for (int j = 1; j <= n; j++) {
                int tmp = dp[j];              // 先存 dp[i-1][j](下一輪的左上斜)
                if (a[i - 1] == b[j - 1])
                    dp[j] = prev + 1;          // 用留住的左上斜
                else
                    dp[j] = max(dp[j], dp[j - 1]); // dp[j]=上, dp[j-1]=左
                prev = tmp;                    // 更新左上斜給下一格
            }
        }
        return dp[n];
    }
};

壓維的唯一難點:相同時要用 dp[i−1][j−1](左上斜),但一維陣列裡 dp[j−1] 已經被本列更新成新值了。所以用一個 prev 變數先存住上一輪的 dp[j](那正是下一格的左上斜),更新完再把它傳給 prev。這個「留住斜對角」是 LCS/編輯距離壓維的招牌手法。

複雜度分析// COMPLEXITY

T = O(mn)　　S = O(mn)(A) / O(n)(B)

每格 O(1)。m,n ≤ 1000 → 約 10⁶。B 只留一列 + 一個 prev。

4

邊界與陷阱// PITFALLS

陷阱後果正解
相同時用 max(上,左)+1重複配對 → 算過大相同用左上斜 dp[i−1][j−1]+1
索引沒對齊(i vs i−1)字元取錯 → 全錯padding 版:字元用 text[i−1]
1D 版沒留 prevdp[j−1] 已被覆蓋 → 斜對角錯先存 tmp=dp[j],再 prev=tmp
忘記 base = 0空字串 LCS 應為 0第一列/行全 0

註:LCS 是「子序列」(可不連續)。若改成「最長公共子字串(要連續)」,相同時仍是 dp[i−1][j−1]+1,但不同時要歸 0(斷開),且答案取全表最大 —— 別搞混。
