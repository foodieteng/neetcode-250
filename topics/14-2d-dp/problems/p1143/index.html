題目簡介// THE QUESTION

QUESTION

給兩個字串,回傳它們最長公共子序列(LCS)的長度。子序列可不連續、但要保持原順序;公共 = 在兩個字串裡都出現。例 "abcde" 與 "ace" → LCS "ace",長度 3。

這是雙字串網格 DP 的原型(編輯距離、通配符比對都同款)。dp[i][j] = text1 前 i 個字元 與 text2 前 j 個字元 的 LCS 長度。它和 62 的差別:轉移要用到左上斜對角 dp[i−1][j−1],不只上與左。

2

兩種情況// MATCH vs MISMATCH

核心 · 看這兩個字元相不相同

比較 text1[i−1] 與 text2[j−1] 這兩個字元:

● 相同 → 這兩字可配成一對加進 LCS:dp[i][j] = dp[i−1][j−1] + 1。兩字都用掉了,所以看「兩字都還沒用」的左上斜 dp[i−1][j−1] 再 +1。
● 不同 → 至少丟掉其中一個字:dp[i][j] = max(dp[i−1][j], dp[i][j−1])。丟 text1 的第 i 字(看「上」)或丟 text2 的第 j 字(看「左」),取較好。

base:第一列/行 = 0(任何字串和空字串的 LCS 是 0)。答案在右下角 dp[m][n]。

3

核心策略 · 三步驟// STRATEGY

  1. 狀態:dp[i][j] = 兩字串各取前 i、前 j 字的 LCS 長度;開 (m+1)×(n+1)、第一列/行 = 0。
  2. 相同:text1[i−1]==text2[j−1] → dp[i][j] = dp[i−1][j−1] + 1(左上斜 +1)。
  3. 不同:dp[i][j] = max(dp[i−1][j], dp[i][j−1]);回傳 dp[m][n]。

為什麼相同時看「左上斜」// THE DIAGONAL

LEMMA · 配成一對 = 兩字串都往前退一格

當 text1[i−1] == text2[j−1],把這兩個相同字元配成 LCS 的一對是最優的(可證明不會更差)。配對後,這兩個字都被用掉 —— 於是剩下的問題是「text1 前 i−1 字 與 text2 前 j−1 字」的 LCS,也就是左上斜 dp[i−1][j−1],再 +1(這對貢獻 1)。

這就是雙字串 DP 和 62 網格 的關鍵差異:62 只依賴「上、左」,LCS 相同情形要依賴「左上斜」。這也讓它的一維壓維需要一個暫存變數存住斜對角的舊值(見程式頁)。

6

複雜度// COMPLEXITY

T = O(mn)　　S = O(mn) → 可壓成 O(n)

表格 O(mn) 格,每格 O(1)。可用一維滾動壓到 O(n)(但要多留一個變數存左上斜)。m,n ≤ 1000 → 至多 10⁶,很快。

一句洞察// INSIGHT

雙字串 DP:相同就配對(左上斜 +1)、不同就丟一字(上、左取大)。 比 62 多依賴「左上斜」—— 編輯距離(72)、通配符比對都是這家族的變體。
