題目簡介// THE QUESTION

QUESTION

m × n 網格,每格有非負成本。從左上走到右下(只能右/下),求路徑上成本總和最小是多少。

這是 62 不同路徑 的最佳化版:骨架仍是「到 (i,j) 只能從上或左來」,但要的不是「數路徑」而是「取最小成本」。所以把 62 的相加,換成 min(上, 左) + 本格成本。這三題 62 / 63 / 64 是同一個模板,差別只在聚合運算。

2

62 / 63 / 64 同模板// SWAP THE AGGREGATE

核心 · 骨架不變,換掉「上、左怎麼合併」

三題都依賴同兩個鄰格(上、左),只是合併方式不同:

● 62(數路徑)：dp[i][j] = dp[i−1][j] + dp[i][j−1]
● 63(有障礙)：同 62,但障礙格 = 0
● 64(最小成本)：dp[i][j] = grid[i][j] + min(dp[i−1][j], dp[i][j−1])

題要什麼合併(上、左)
62路徑數+(相加)
63路徑數(避障)+,障礙 = 0
64最小成本min(...) + 成本

這就是 1-D DP 家族「同骨架、換聚合」在二維網格上的翻版:計數用 +、最佳化用 min/max、可行性用 OR。學會一題就通三題。

3

核心策略 · 三步驟// STRATEGY

  1. 狀態:dp[i][j] = 走到 (i,j) 的最小成本和。
  2. base:第一列/行是前綴和(只有一條路,成本一路累加),不是 1。
  3. 轉移:dp[i][j] = grid[i][j] + min(dp[i−1][j], dp[i][j−1]);答案右下角。

複雜度// COMPLEXITY

T = O(mn)　　S = O(mn) → 可壓成 O(n)

每格算一次、常數運算。可用一維滾動壓到 O(n)(dp[j]=min(dp[j], dp[j−1])+grid)。m,n ≤ 200。
