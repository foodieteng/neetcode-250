狀態dp[i][j] = 走到 (i,j) 的最小成本和
轉移dp[i][j] = grid[i][j] + min(dp[i−1][j], dp[i][j−1])

答案dp[m−1][n−1]

範例 Trace// [[1,2,3],[4,5,6]] → 12

// 填表 grid dp 1 2 3 1 3 6 ← 第一列前綴和 4 5 6 5 8 12 dp[1][0]=1+4=5;dp[1][1]=5+min(上3,左5)=5+3=8;dp[1][2]=6+min(上6,左8)=6+6=12 return 12(路徑 1→2→3→6)
最省的路是 1→2→3→6(沿第一列走到底再下)= 12,而不是 1→4→5→6=16。 min 在每個交叉口都選「到這裡最省的前路」,一路累加就得全域最小 —— 局部最優能拼出全域最優,正是 DP 成立的條件。

應用題// APPLY

App 1. 120 三角形最小路徑和:三角形裡每步走到下一列的相鄰兩格。轉移和 64 有多像?差在「鄰格」怎麼取?
App 2. 174 地下城遊戲:騎士從左上走到右下救公主,格子有正負,要求全程血量 > 0,求最少初始血量。為什麼這題要從右下往左上反著 DP?
App 3. 若要回報那條最小成本路徑本身(不只總和),怎麼從 dp 回溯?
// ANSWERS
A1. 只差把 + 換成 min(...)(再加本格成本)。62 是 dp[i][j]=dp[i−1][j]+dp[i][j−1](把兩條前路的路徑數相加);64 是 dp[i][j]=grid[i][j]+min(dp[i−1][j],dp[i][j−1])(取兩條前路裡最省的)。這反映網格 DP 的共性:骨架(依賴上、左)不變,換聚合就換題型 —— 計數用 +、最佳化用 min/max。62/63/64 是同一模板的三種收尾。
A2. 因為到第一列/行的格子只有唯一一條路(一路右或一路下),但 64 在意的是成本累加,不是路徑數。走到第一列第 j 格,成本 = 前面所有格的成本和 = 前綴和。62 在意「有幾條路」,唯一路 → 路徑數是 1。同樣「唯一一條路」,計數版記 1、成本版記累加和 —— 邊界的意義隨題目要的量而變。
A3. 因為「到這一格最省」要看的是整條前路的累計成本,不是眼前鄰格的單格值。min(dp[上], dp[左]) 比較的是「已經算好的、到上/左那格的最小總成本」,不是 grid[上] vs grid[左]。單純每步挑較小的鄰格(貪心)會被後段的大成本反超:眼前省一點,可能繞進一條後面很貴的路。DP 用 min 累加整條路的最優,才保證全域最小。
A4. 因為第一行(j==0)的格子只有「上」一個來源(左邊界外),不能取 min。所以每進入新的一列 i,要先把 dp[0](它此刻還是上一列第 0 欄的值 = 「上」)加上本格成本 grid[i][0],更新成第 i 列第 0 欄的值。它對應二維的 dp[i][0] = dp[i−1][0] + grid[i][0]。漏了這步,第一行成本就沒累加,整列全錯。
A5. 改「最大成本」:把 min 換成 max 即可(dp[i][j]=grid+max(上,左)),邊界前綴和不變。再要「有幾條路達到最小值」:多維護一個 cnt[i][j] —— 先算 best=min(上,左),若上、左都等於 best 則 cnt=cnt[上]+cnt[左],只有一邊等於就繼承那邊的 cnt。這是「最優值 + 計數」型 DP:一個陣列記最優、一個記達到最優的方法數,同步維護 —— 和 300 LIS 計數(673) 同套路。
App 1. 幾乎一樣。120 的 dp[i][j] = 從頂點走到三角形 (i,j) 的最小路徑和,轉移 dp[i][j] = triangle[i][j] + min(dp[i−1][j−1], dp[i−1][j]) —— 差別只在「上一列的鄰格」是正上方與左上方(三角形結構),不是 64 的「上與左」。骨架、min、邊界(每列兩端只有一個來源)全同理。常見還會從底往上做,答案直接落在頂點、省去最後掃一列取 min。
App 2. 因為 174 的限制是「全程血量都要 > 0」,而某格需要多少初始血量,取決於它之後(往右下)還會扣多少 —— 未來影響現在。若從左上往右下做,你不知道後面的路,無法決定當前該留多少血。從右下往左上反著 DP,dp[i][j] = 「站上 (i,j) 前至少要有的血量」,由已知的右、下鄰推得:dp[i][j]=max(1, min(dp[右],dp[下]) − grid[i][j])。方向反轉是因為「需求由終點回推」,這是 min-path 家族一個重要變形。
App 3. 從終點 (m−1,n−1) 往回走:在 (i,j) 比較「上」dp[i−1][j] 與「左」dp[i][j−1],往較小的那個退(因為 dp[i][j]=grid+min(那兩者),最小來源就是路徑的前一步);相等時任選(或按固定優先序求字典序路徑)。一路退到 (0,0),把經過的格子反轉就是最小成本路徑。要能回溯,需保留二維 dp 表(一維滾動版丟了歷史);或在算 dp 時額外記每格「從上還是從左來」的方向指標。
