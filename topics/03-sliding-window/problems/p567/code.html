完整實作 · 定長視窗計數// O(n)

p567.cpp C++17 
class Solution {
public:
    bool checkInclusion(string s1, string s2) {
        if (s1.size() > s2.size()) return false;

        vector<int> s1Map(26, 0), cmp(26, 0);

        for (int i = 0; i < s1.size(); i++) {
            s1Map[s1[i] - 'a']++; cmp[s2[i] - 'a']++;   // 目標 + 首窗計數
        }

        if (s1Map == cmp) return true;                       // 首窗就是排列

        int l = 0;
        for (int r = s1.size(); r < s2.size(); r++) {
            cmp[s2[r] - 'a']++; cmp[s2[l++] - 'a']--;   // 加右 + 減左(鎖步)

            if (cmp == s1Map) return true;                   // 計數相等 → 排列
        }

        return false;
    }
};
cmp[s2[r]-'a']++; cmp[s2[l++]-'a']--; 一行做完「加右、減左」—— 視窗鎖步平移一格,大小恆為 |s1|,不需要 while 收縮。 vector<int> == 會逐元素比對 26 個計數,O(26)=O(1)。開頭的 s1.size() > s2.size() 提前擋掉不可能的情況。

優化 · 每步嚴格 O(1)// MATCHES COUNTER

// 不每步比 26 個,改維護「有幾個字母的計數已相等」matches bool checkInclusion(string s1, string s2) { if (s1.size() > s2.size()) return false; vector<int> need(26, 0), win(26, 0); for (char c : s1) need[c - 'a']++; int m = s1.size(), matches = 0; for (int i = 0; i < 26; i++) if (need[i] == win[i]) matches++; // 起始都 0，先算一次 for (int r = 0; r < (int)s2.size(); r++) { int in = s2[r] - 'a'; win[in]++; // 加右 if (win[in] == need[in]) matches++; else if (win[in] == need[in] + 1) matches--; // 剛從相等變不等 if (r >= m) { // 視窗超過 m → 減左 int out = s2[r - m] - 'a'; win[out]--; if (win[out] == need[out]) matches++; else if (win[out] == need[out] - 1) matches--; } if (matches == 26) return true; // 26 個字母全相等 } return false; } 每次加/減只影響「一個字母」的相等狀態 → 用 matches 增量維護,每步 O(1)

原版每步 cmp == s1Map 掃 26 個(常數但有係數)。這版維護 matches = 「已相等的字母數」:每次只有被 ++/-- 的那個字母的相等狀態會變,O(1) 更新。matches == 26 就是全部相等。兩者都是 O(n),這版係數更小 —— 對超長字串或 438 很實用。

陷阱症狀正解
沒先擋 |s1|>|s2| 首窗建立時 s2[i] 越界 開頭 if (s1.size()>s2.size()) return false;
忘了比首窗 答案剛好在 s2 開頭時漏掉 建完首窗先比一次 s1Map==cmp
加右忘了減左 視窗越變越大,不再是定長 每步 ++ 右、-- 左成對出現
matches 版邊界更新錯 == need+1 / == need-1 條件寫反 加後看「剛超過」、減後看「剛低於」
誤以為要找子序列 排列是連續子字串,不是子序列 視窗天然是連續段

細節走讀// WALKTHROUGH

// s1="ab"(目標 a:1 b:1), s2="eidbaooo" 首窗 [0,1]="ei": cmp{e:1,i:1} ≠ 目標 r=2 '+d −e': [1,2]="id" cmp{i:1,d:1} ≠ 目標 r=3 '+b −i': [2,3]="db" cmp{d:1,b:1} ≠ 目標(缺 a) r=4 '+a −d': [3,4]="ba" cmp{b:1,a:1} == 目標 {a:1,b:1} → return true true(視窗 "ba" 是 "ab" 的排列)
每次滑動只動兩個計數(進的 +1、出的 -1),其餘 24 個字母的計數不變 —— 這就是為什麼相鄰視窗能 O(1) 更新、不必重數。 比對時整個 cmp 要和 s1Map 完全相等(每個字母都對),缺一個 a 或多一個 o 都不算排列。
