題目簡介// THE QUESTION

QUESTION

給 s1、s2,判斷 s2 是否含有 s1 的某個排列當作子字串。也就是:s2 裡有沒有一段連續子字串,是 s1 重排後的樣子。 例:s1="ab", s2="eidbaooo" → true("ba" 是 "ab" 的排列)。

關鍵改寫:「s1 的排列」= 長度等於 |s1| 且每個字母出現次數與 s1 相同的子字串。排列不管順序、只管「哪些字母、各幾個」。 於是滑一個恰好 |s1| 長的定長視窗過 s2,比對 26 字母的計數即可。

2

為什麼「定長視窗 + 計數」// WHY

× REJECT · O(n·m log m)
每個視窗排序再比

對每個長度 m 的視窗排序和 sort(s1) 比。每次 O(m log m) × n 個視窗 → 慢。

△ 每次重數
每個視窗重新數 26 計數

對每個視窗從頭數一遍計數再比。O(m) × n = O(n·m),浪費了「相鄰視窗只差一進一出」。

FIXED WINDOW · ADOPT ↓
● ADOPT · O(n) / O(1)
定長視窗鎖步滑動,增量更新計數

先數第一個 |s1| 長視窗的計數;之後每步視窗鎖步右移一格：+ 新進的右字元、− 移出的左字元(大小永遠 = |s1|)。 每步只做 2 次計數更新 + 一次比對(26 個字母,O(1))。整條掃一趟 → O(n),計數表固定 26 → O(1)。

3

核心策略 · 三步驟// STRATEGY

  1. 建目標與首窗 —— 數 s1 的計數 s1Map,同時數 s2 前 |s1| 個字元的計數 cmp。相等就已成立。
  2. 鎖步滑動 —— r 從 |s1| 開始:cmp[s2[r]]++(加右)、cmp[s2[l++]]--(減左),視窗大小不變。
  3. 每步比對 —— cmp == s1Map 即找到排列,回傳 true。掃完沒有 → false。

滑動視窗的三種面貌// THE THREE SHAPES

恰好定長 · 上限定長 · 可變長

同樣「右擴、左縮」,視窗大小的控制方式有三種:

‧ 恰好定長(本題):大小永遠 = |s1|,每步加右同時減左、鎖步平移 —— 用來「找固定長度的匹配」。
‧ 上限定長(219):大小 ≤ k+1,只在超過上限時才縮左。
‧ 可變長(3):大小隨資料浮動,違反合法條件時才縮。

本題屬第一種 —— 它和另兩種最大的差別是:左右指針同進同退、間距恆定,不需要 while 收縮,只是一格格平移。判斷「視窗是不是排列」則靠計數向量相等。這也是 438 找所有字母異位詞 的同一招(那題收集所有命中的起點)。

複雜度// COMPLEXITY

T = O(n)　　S = O(1)

建首窗 O(m);之後滑 n-m 步,每步 2 次計數更新 + 一次 26 元素比對 → O(26·n)=O(n)。計數表固定 26 個 int → O(1)。若嫌每步比 26 慢,可維護「有幾個字母計數已相等」的計數器做到嚴格 O(1)/步(見程式頁)。

面向成本說明
建首窗O(m)數前 m 個字元
每步滑動O(1)加右 + 減左
每步比對O(26)兩向量相等,視為常數
空間O(1)兩個 26-長陣列
7

一句洞察// INSIGHT

排列 = 同長度 + 同字元計數。滑一個恰好 |s1| 長的定長視窗,每步加右減左增量更新計數,和目標比相等。 這是滑動視窗的第三種面貌(恰好定長),和 219、3 並列;同招也解 438。O(n)。
