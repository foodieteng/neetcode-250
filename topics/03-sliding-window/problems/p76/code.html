class Solution {
public:
    string minWindow(string s, string t) {
        unordered_map<char, int> freq;
        for (char c : t) freq[c]++;                // freq = t 每字元需求(正數)

        int required = freq.size();              // t 的相異字元數
        int formed = 0, l = 0;
        int bestLen = INT_MAX, bestStart = 0;

        for (int r = 0; r < (int)s.size(); r++) {
            if (--freq[s[r]] == 0) formed++;      // 湊滿一種 → formed++

            while (formed == required) {            // 覆蓋 t → 縮左求最短
                if (r - l + 1 < bestLen) {
                    bestLen = r - l + 1;
                    bestStart = l;
                }
                if (++freq[s[l]] > 0) formed--;   // 移出致不足 → formed--
                l++;
            }
        }

        return bestLen == INT_MAX ? "" : s.substr(bestStart, bestLen);
    }
};
freq 一開始存 t 的需求(正數);--freq[s[r]] 減到 恰好 0 = 該字元湊滿 → formed++。多餘的字元讓 freq 變負(surplus),不再觸發。縮左 ++freq[s[l]] 若加到 > 0 = 移出後不足 → formed--。只在跨過「剛好滿足」界線時動 formed,所以每步 O(1)。

為什麼 freq「可正可負」是關鍵// SIGNED COUNTS

// freq[c] 的符號就是「還缺 / 剛好 / 多餘」的三態 freq[c] > 0 : 視窗還「缺」c(需求未滿) freq[c] == 0: c「剛好」滿足需求 freq[c] < 0 : 視窗有「多餘」的 c(surplus) 擴張 --freq[s[r]]: 正 → 0 這一刻「湊滿」→ formed++ 縮左 ++freq[s[l]]: 0 → 正 這一刻「變不足」→ formed-- 用同一個 freq 的正負,同時表達「還缺多少」和「多了多少」,不必兩張表

這段程式最精巧的地方,是用一個 freq 的正負號承載「缺 / 剛好 / 多餘」三種狀態。因為初值是需求(正),減到 0 剛好、再減成負就是多餘;而 formed 只在「正→0」(湊滿)和「0→正」(變不足)這兩個跨零瞬間變動。 也可以用兩張表(need 與 window)寫得更直白 —— 對每個字元比 window[c] >= need[c] 維護 formed,語意一樣,只是多一張表。核心都是「覆蓋 = formed==required」。

邊界與陷阱// PITFALLS

陷阱症狀正解
required 用 t.size() t 有重複字元時 required 偏大 → 永不達標 required = freq.size()(相異數)
湊滿判斷用 <= 0 surplus 時又 formed++ → 重複計數 只在 == 0(剛好)才 formed++
不足判斷用 >= 0 移出 surplus 也 formed-- → 誤判 只在 > 0(變不足)才 formed--
縮完才記最短 當前覆蓋視窗可能就是最短,漏掉 進 while 先記,再縮
沒處理「不存在」 回傳垃圾 bestLen==INT_MAX ? "" : ...

// s = "AABC", t = "ABC"(need A:1 B:1 C:1, required=3) r=0 A: freq[A] 1→0 湊滿 → formed=1 窗"A" [0,0] r=1 A: freq[A] 0→-1 surplus → formed=1 窗"AA" [0,1] r=2 B: freq[B] 1→0 → formed=2 窗"AAB" [0,2] r=3 C: freq[C] 1→0 → formed=3==required → 覆蓋! 記最短 "AABC"(4); 縮左: ++freq[A] -1→0 (仍 ≤0,不足未觸發) formed=3, l=1 仍覆蓋 → 記最短 "ABC"(3,更短); 縮左: ++freq[A] 0→1 (>0 變不足) formed=2, l=2 → 停 回傳 "ABC"
看縮左那兩步:第一次移出的是多餘的 A(freq[A] 從 -1 加到 0,仍 ≤0,不觸發 formed--)—— 視窗仍覆蓋,所以能安全縮短到 "ABC"。 第二次移出的是必要的 A(freq[A] 從 0 加到 1,>0 觸發 formed--)—— 覆蓋被破壞,停止。surplus 可丟、必要不可丟,全靠 freq 的正負自動分辨。
