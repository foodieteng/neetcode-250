完整實作 · 兩端收縮// O(n−k)

p658.cpp C++17 
class Solution {
public:
    vector<int> findClosestElements(vector<int>& arr, int k, int x) {
        int l = 0, r = arr.size() - 1;

        while (r - l + 1 > k) {                     // 收縮到剩 k 個
            if (abs(arr[r] - x) >= abs(arr[l] - x)) {
                r--;                                // 右端較遠(或平手)→ 砍右
            } else {
                l++;                                // 左端較遠 → 砍左
            }
        }

        return vector<int> (arr.begin() + l, arr.begin() + r + 1);  // 連續 k 個,已升序
    }
};
>= 是 tie-break 的靈魂:當 |arr[r]-x| == |arr[l]-x|,含等號的 >= 會走 r-- 砍右端,保留較小的左端 —— 正好符合「同距取較小 a」。 回傳的 arr[l..r] 本來就在原陣列裡連續、已排序,直接是升序答案,不必再排。

更快 · 二分搜尋找左界// O(log(n−k)+k)

// 答案是長度 k 的連續視窗,左界 lo ∈ [0, n-k],對 lo 二分 vector<int> findClosestElements(vector<int>& arr, int k, int x) { int lo = 0, hi = arr.size() - k; // 左界的可能範圍 while (lo < hi) { int mid = (lo + hi) / 2; // 比「視窗左端 arr[mid]」與「視窗右邊界外一格 arr[mid+k]」誰離 x 遠 if (x - arr[mid] > arr[mid + k] - x) // 左端較遠 → 視窗該右移 lo = mid + 1; else hi = mid; } return vector<int>(arr.begin() + lo, arr.begin() + lo + k); } 比較 arr[mid](納入則為左端)與 arr[mid+k](排除的下一個)到 x 的距離,決定左界往哪半移

當 k 遠小於 n 時,兩端收縮要砍 n-k 次(慢);二分直接對「視窗左界」做二分,O(log(n-k)) 找到起點,再 O(k) 複製。 比較式 x - arr[mid] > arr[mid+k] - x 的巧思:arr[mid] 是「若左界取 mid 則納入的最左元素」,arr[mid+k] 是「被排除的下一個」;誰離 x 遠就往另一邊收。注意這裡用有號差(不是 abs),因為在有序陣列上 arr[mid] ≤ x ≤ arr[mid+k] 不一定成立,但這個式子在該題範圍已被證明正確(等號情況自然偏向較小的左界)。

邊界與陷阱// PITFALLS

陷阱症狀正解
tie 用 > 而非 >= 平手時砍左,保留較大者 → 違反「同距取較小」 用 >=,平手砍右
想挑不連續的最近元素 額外複雜、還可能錯 已排序 → 答案必連續,只找視窗
最後還去排序 多此一舉 arr[l..r] 本已升序
二分 hi 設成 n-1 左界最多到 n-k,設 n-1 會越界 hi = n - k
k == n 擔心空迴圈 r-l+1==n 直接回整個陣列

// arr=[1,2,3,4,5,6], k=3, x=4 l=0,r=5 (size6): |1-4|=3, |6-4|=2 → 6>=? 2>=3 否 → 砍左 l++ [2..6] l=1,r=5 (size5): |2-4|=2, |6-4|=2 → 2>=2 是 → 砍右 r-- [2..5] l=1,r=4 (size4): |2-4|=2, |5-4|=1 → 1>=2 否 → 砍左 l++ [3..5] l=2,r=4 (size3 == k) → 停 回傳 arr[2..4] = [3, 4, 5]
第二步是 tie:|2-4| == |6-4| == 2,>= 含等號 → 走 r-- 砍右端(丟 6)、保留較小的左端 2。 關鍵情境是「最後一步收縮時兩端都還是候選、距離又相等」:此時 >= 保證留下較小的那一端,正是題目 |a-x|==|b-x| 取 a<b 的要求。把 >= 改成 > 就會在平手時砍左、保留較大者 → 錯。
