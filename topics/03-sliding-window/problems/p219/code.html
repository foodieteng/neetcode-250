完整實作 · 視窗計數// O(n)

p219.cpp C++17 
class Solution {
public:
    bool containsNearbyDuplicate(vector<int>& nums, int k) {
        int n = nums.size(), l = 0;
        unordered_map<int, int> freq;          // 視窗內每個值的計數

        for (int r = 0; r < n; r++) {
            while (r - l > k) {                   // 視窗超過 k+1 長 → 收縮
                freq[nums[l++]]--;              // 移出最左值,l 前進
            }

            if (++freq[nums[r]] == 2) return true;  // 加入後某值變 2 個 → 命中
        }

        return false;
    }
};
freq 記「目前視窗 [l, r] 內每個值出現幾次」。收縮時 freq[nums[l]]-- 把移出的值計數減回。 加入 nums[r] 後,++freq[nums[r]] == 2 代表視窗內同一個值有兩份 —— 因為視窗長 ≤ k+1,這兩份索引差必 ≤ k,直接回傳 true。 因為一發現 2 就回傳,任何計數其實不會超過 2,freq 等同一個「最近 k+1 個值」的集合。

同一招 · 直接用集合// SET VARIANT

// 既然計數只會是 0/1/2,乾脆用 set 存「最近 k+1 個值」 bool containsNearbyDuplicate(vector<int>& nums, int k) { unordered_set<int> window; for (int r = 0; r < (int)nums.size(); r++) { if (r > k) window.erase(nums[r - k - 1]); // 把離開視窗的最左值移出 if (!window.insert(nums[r]).second) return true; // insert 失敗 = 已存在 → 命中 // window 始終只裝 index (r-k .. r] 這 ≤ k+1 個值 } return false; } insert().second == false 代表值已在集合 → 找到距離 ≤ k 的重複

兩版完全等價:計數版用 while 顯式維護左指針 l;集合版直接算出離開視窗的索引 r-k-1 並 erase。 集合版更能凸顯本質 —— 視窗就是「最近 k+1 個值的集合」,加入撞名即命中。面試講這版最清楚。

邊界與陷阱// PITFALLS

陷阱症狀正解
視窗長度算成 k 條件寫 r-l >= k → 視窗只剩 k,漏掉距離剛好 k 的一對 r - l > k(允許長度 k+1)
先檢查再收縮 沒先把太遠的值移出就比對 → 誤判遠處重複 先 while 收縮,再加入/檢查
集合版忘了 erase 集合累積全部值 → 退化成「有沒有重複」(LC 217) r > k 時 erase nums[r-k-1]
k = 0 要求 i≠j 但距離 ≤ 0 不可能 視窗長 1,永遠找不到 → 自然回 false
暴力 O(n·k) n、k 皆 1e5 → 1e10 逾時 滑動視窗 O(n)

// nums = [1,0,1,1], k = 1(視窗最長 2) l=0 r=0: r-l=0≤1 不收縮; ++freq[1]=1 視窗{1} r=1: r-l=1≤1 不收縮; ++freq[0]=1 視窗{1,0} r=2: r-l=2>1 → 收縮 freq[nums[0]=1]-- =0, l=1 ++freq[1]=1(剛歸零又回1) ≠2 視窗{0,1} 早先的 1 已移出 r=3: r-l=3-1=2>1 → 收縮 freq[nums[1]=0]-- =0, l=2 ++freq[1]=2 == 2 → return true 視窗{1,1} idx2、idx3 距離 1 ≤ k true(存在距離 ≤ 1 的相等元素:idx2 與 idx3)
注意 r=2 那步:idx0 的 1 在加入 idx2 的 1 之前就先被收縮出去(freq[1] 歸零),所以不會把「距離 3 的一對 1」誤判成命中。 先收縮、再加入的順序,正是保證「只在同一個 ≤ k+1 視窗內比較」的關鍵。
