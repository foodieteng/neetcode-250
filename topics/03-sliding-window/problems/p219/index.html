1

題目簡介// THE QUESTION

QUESTION

給整數陣列 nums 和整數 k,判斷是否存在兩個不同索引 i、j,使得 nums[i] == nums[j] 且 abs(i - j) ≤ k。存在回傳 true,否則 false。 例:[1,0,1,1], k=1 → true(idx2、idx3 兩個 1,距離 1);[1,2,3,1], k=2 → false(兩個 1 距離 3 > 2)。

這是 第 03 章滑動視窗的入門題。關鍵改寫:「存在距離 ≤ k 的相等元素」⇔「存在一個長度 ≤ k+1 的視窗,裡面有重複值」。 於是問題變成:維持一個「最近 k+1 個元素」的滑動視窗,只要視窗內冒出重複就成立。

2

為什麼「定長滑動視窗 + 集合」// WHY

× REJECT · TLE
暴力雙迴圈 O(n·k)

對每個 i 往後看 k 格找相等。n、k 都可到 10⁵ → 最壞 10¹⁰ 次,超時。

△ 想太多
值 → 所有索引清單,再兩兩比

用 map<值, vector<idx>> 存每個值的所有位置,檢查相鄰索引差 ≤ k。可行,但存了全部索引,空間到 O(n) 且較繞。

SLIDING WINDOW · ADOPT ↓
● ADOPT · O(n) / O(min(n,k))
維持「最近 ≤ k+1 個值」的視窗集合

右指針 r 掃過去,視窗 [l, r] 只要超過 k+1 長就從左邊收縮。視窗內用雜湊集合記住出現過的值。 加入 nums[r] 時集合裡已經有它 → 命中:視窗長 ≤ k+1 保證這兩個相等值的索引差 ≤ k。掃一趟 O(n),集合最多裝 k+1 個 → O(min(n,k))。

3

核心策略 · 三步驟// STRATEGY

  1. 右指針擴張 —— r 從左到右掃,把 nums[r] 準備加入視窗。
  2. 左指針收縮 —— 若 r - l > k(視窗超過 k+1 長),把 nums[l] 移出視窗、l++,直到長度 ≤ k+1。
  3. 檢查重複 —— 加入 nums[r] 時若視窗集合已有此值 → 回傳 true。掃完都沒有 → false。

為什麼視窗長度是 k+1// THE INVARIANT

視窗長 ≤ k+1 ⟺ 內部任兩點索引差 ≤ k

視窗 [l, r] 的長度是 r - l + 1。我們維持 r - l ≤ k,也就是長度 ≤ k+1。 在這樣的視窗裡,最左 l 到最右 r 的索引差最多 k —— 所以只要視窗內出現兩個相等的值,它們的索引差必定 ≤ k,直接滿足題目。

反過來:任何一對「距離 ≤ k 的相等元素」,一定會在某個時刻同時落在同一個視窗內(當 r 走到後者時,前者還沒被收縮出去)。所以「掃遍所有 k+1 視窗找重複」不會漏、也不會多 —— 這就是把「距離條件」翻譯成「定長視窗」的等價性。動畫裡 idx0 的 1 距離 idx3 是 3 > k,早在 r=2 就被收縮出去,正確地不被誤判。

6

複雜度// COMPLEXITY

T = O(n)　　S = O(min(n, k))

r 掃一趟共 n 步,l 全程最多也前進 n 步(攤還 O(1)/步);集合的插入/查詢/刪除平均 O(1)。集合最多同時裝 k+1 個值 → 空間 O(min(n, k))。

面向成本說明
右指針掃描O(n)每個元素進視窗一次
左指針收縮攤還 O(1)全程最多移動 n 次
集合操作O(1)雜湊插入/查詢/刪除
空間O(min(n,k))視窗最多 k+1 個值
7

一句洞察// INSIGHT

「距離 ≤ k 的相等元素」= 「長度 ≤ k+1 的視窗內有重複」。維持一個只裝最近 k+1 個值的集合,加入時撞名就命中。 這就是滑動視窗的第一課:把「範圍條件」翻譯成一個定長視窗,再用集合/計數摘要視窗內容。
