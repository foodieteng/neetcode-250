重點回顧// RECAP

關鍵一句話
核心改寫距離 ≤ k 的相等元素 ⟺ 長度 ≤ k+1 的視窗有重複
視窗不變式r - l ≤ k(長度 ≤ k+1)
視窗摘要集合/計數記「最近 k+1 個值」
命中條件加入 nums[r] 時集合已有此值
順序先收縮(移出太遠的)再加入檢查
複雜度O(n) 時間 / O(min(n,k)) 空間
兄弟題LC 217 有沒有重複(不限距離)· LC 220 值也允許誤差

觀念題// CONCEPT Q

Q1

為什麼視窗長度要允許到 k+1 而不是 k?條件寫成 r-l >= k 會漏掉哪種情形?

Q2

為什麼一定要先收縮、再加入 nums[r] 檢查?順序反過來(先檢查再收縮)會出什麼錯?用動畫的 r=2 那步說明。

Q3

計數版用 unordered_map<int,int> freq,但作者說「計數其實不會超過 2」。為什麼?這代表它本質上等於什麼資料結構?

Q4

集合版用 window.erase(nums[r-k-1]) 移出離開視窗的值。為什麼移出的索引是 r-k-1 而不是 r-k?

Q5

如果把「移出視窗左值」這步拿掉、集合只增不減,程式會變成解哪一題?(提示:LeetCode 217)

範例 Trace// 距離剛好 > k 的反例

// nums = [1,2,3,1], k = 2:兩個 1 距離 3 > k → 應回 false l=0 r=0: 不收縮; ++freq[1]=1 視窗{1} r=1: 不收縮; ++freq[2]=1 視窗{1,2} r=2: r-l=2≤2 不收縮; ++freq[3]=1 視窗{1,2,3}(長度 3 = k+1) r=3: r-l=3>2 → 收縮 freq[nums[0]=1]-- =0, l=1 ++freq[1]=1(剛歸零又回1) ≠2 視窗{2,3,1} 早先的 1 已移出 掃完 → return false false(兩個 1 索引差 3 > k=2,不算鄰近)
這正是「定長視窗」的價值:idx0 的 1 在 r 走到 idx3 前就因為超出 k+1 範圍被收縮出去,所以加入第二個 1 時集合裡沒有它 → 正確回 false。 視窗長度 k+1 就是「距離 ≤ k」這個條件的幾何化身。

應用題// APPLY

App 1

LC 217 存在重複:只問「陣列有沒有任何重複」(不限距離)。用本題的骨架怎麼退化?視窗要多大?

App 2

LC 220 存在鄰近且值相近:距離 ≤ k 且 abs(nums[i]-nums[j]) ≤ t。集合換成什麼結構,才能在視窗內快速找「值落在 [x-t, x+t]」的元素?

App 3

定長 vs 可變長視窗:本題視窗長度上限固定(k+1),屬「定長」;而「最長不重複子字串」(LC 3)視窗會伸縮。兩者在「什麼時候收縮左指針」上判斷依據差在哪?

// ANSWERS
A1

題目要的是 abs(i-j) ≤ k —— 距離剛好 k 也算。索引差為 k 的兩個元素,落在同一視窗時視窗長度是 k+1(含兩端共 k+1 格)。若限制長度只到 k(即 r-l >= k 就收縮),距離恰為 k 的那一對永遠無法同時在窗內,會漏判。所以不變式必須是 r - l ≤ k,允許長度 k+1。

A2

必須先收縮再加入檢查。以動畫 r=2([1,0,1,1], k=1)為例:此時 idx0 的舊 1 距離 idx2 是 2 > k,不該算命中。先收縮會把 idx0 移出(freq[1] 歸零),再加入 idx2 的 1 只得 freq[1]=1,正確不命中。若先檢查再收縮,加入 idx2 時 idx0 的 1 還在集合 → freq[1] 變 2 → 誤判成 true。收縮的作用就是「把超出距離的舊值先清掉」,順序錯了距離條件就失效。

A3

因為程式一旦某值計數到 2 就立刻 return true,所以任何時刻視窗內每個值最多只有 1 份(計數 0 或 1),永遠不會停在 2 之後繼續。這代表 freq 本質上就是一個集合(存在性):值在不在窗內。用 map<int,int> 只是順手用收縮時的 -- 維護「離開就移除」,換成 unordered_set 加 erase(nums[r-k-1]) 完全等價、更直白。

A4

當右指針在 r、視窗要維持長度 k+1 時,窗內合法索引是 [r-k, r]。所以「剛好離開視窗」的是索引 r-k-1(比窗左端再左一格)。在把 nums[r] 加入之前,先把 nums[r-k-1] 移出,窗內就恰好保留 (r-k .. r] 這 ≤ k+1 個。寫成 r-k 會多移一格,把還在窗內的值誤刪。r > k 的判斷確保 r-k-1 ≥ 0 不越界。

A5

拿掉收縮、集合只增不減,就變成 LC 217「存在重複元素」:只要整個陣列出現過同一個值兩次就回 true,不管距離。等於把視窗放大到整個陣列(k = n-1)。換句話說,本題是 217 加上「只看最近 k+1 個」的距離限制 —— 用一個定長視窗把「全局重複」收窄成「鄰近重複」。

App 1

LC 217:視窗放到最大(不收縮),集合裝全部見過的值,insert 撞名即回 true。等價於本題 k = n-1。所以 217 是 219 的「無距離限制」特例;219 只是多維護「最近 k+1 個」這件事,骨架一模一樣。

App 2

LC 220:除了距離 ≤ k,還要值差 ≤ t。集合換成有序結構 set<long>(平衡樹):對每個 nums[r],用 lower_bound(nums[r]-t) 找視窗內第一個 ≥ x-t 的值,檢查它是否 ≤ x+t;同樣維護視窗長 ≤ k+1(超出就 erase 最左值)。時間 O(n log k)。或用桶(bucket,桶寬 t+1)做到 O(n)。核心仍是「定長視窗 + 能查範圍的結構」。

App 3

定長(本題):收縮的依據是視窗大小 —— 只要 r-l > k 就縮,和窗內內容無關,窗長上限固定。可變長(LC 3 最長不重複子字串):收縮的依據是視窗內容是否違反條件 —— 當窗內出現重複字元才縮,縮到條件恢復為止,窗長隨資料浮動。一句話:定長看「幾個」,可變長看「合不合法」。兩者都是右擴左縮,只是收縮的觸發條件不同。
