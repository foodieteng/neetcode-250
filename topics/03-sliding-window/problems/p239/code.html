完整實作 · 單調隊列// O(n)

p239.cpp C++17 
class Solution {
public:
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {
        deque<int> dq;                      // 存索引,對應值單調遞減
        int n = nums.size();
        vector<int> ans;

        for (int i = 0; i < n; i++) {
            while (!dq.empty() && nums[i] >= nums[dq.back()]) dq.pop_back();  // 彈掉較小的 back
            while (!dq.empty() && i - k >= dq.front()) dq.pop_front();    // 彈掉過期的 front
            dq.push_back(i);

            if (i >= k - 1) ans.push_back(nums[dq.front()]);       // 滿窗 → front 是最大
        }

        return ans;
    }
};
隊列存索引(不是值)—— 這樣才能用 i - k >= dq.front() 判斷「front 是否滑出視窗」。彈 back 用 >=(相等也彈,留較晚的、索引更大更晚過期);彈 front 只需一個 if 也可(隊列按索引遞增,只有 front 會過期,至多彈一次),寫 while 更保險。i >= k-1 才代表視窗湊滿 k 個,開始輸出。

為什麼存索引、彈 back 用 ≥// INDEX & TIES

// 存索引 → 才能判斷「過期」;彈 back 用 >= → 相等時留較晚的 存「索引」而非「值」: 過期判斷 i - k >= dq.front() 需要索引;只存值就不知道它在哪、何時滑出。 彈 back 用 nums[i] >= nums[back](含等號): 遇到相等值,彈掉舊的、留新的(索引更大)→ 新的更晚過期,更能撐住 front。 用 > (不含等號)也對(相等的都留著),但隊列更長、常數更大。 存索引是為了判過期;>= 是為了讓相等值只留最晚的那個

兩個實作要點。存索引:因為要判斷 front「是否已滑出視窗」,必須知道它的位置,所以隊列裝索引,取值時再 nums[dq.front()]。彈 back 用 >=:當新值和隊尾相等,彈掉舊的、留索引較大的新值 —— 因為它更晚過期,能讓最大值在隊列裡待更久,隊列也更短。用嚴格 > 也正確(保留相等的重複),只是常數略大。

陷阱症狀正解
隊列存值不存索引 無法判斷 front 是否過期滑出 存索引,取值 nums[front]
忘了彈過期 front front 已滑出視窗仍當最大 → 錯 i-k >= dq.front() 就 pop_front
彈 back 用 > 卻沒差意識到 其實也對,但隊列變長 >= 更精簡(留最晚的)
輸出時機 i >= k 差一,第一個視窗漏輸出 視窗滿是 i >= k-1
用堆但沒懶刪除 過期最大值沒清 → 錯 堆存 (值,索引),取頂時丟棄過期的

deque 存索引。for i：while nums[i]>=nums[back] pop_back；while i-k>=front pop_front；push i；i>=k-1 輸出 nums[front]。存索引為判過期、>= 留最晚。攤還 O(n)/O(k)。
