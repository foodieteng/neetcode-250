題目簡介// THE QUESTION

QUESTION

給陣列 nums 和視窗大小 k,視窗從最左滑到最右、每次右移一格。回傳每個視窗的最大值組成的陣列。 例:nums=[1,3,-1,-3,5,3,6,7], k=3 → [3,3,5,5,6,7]。

這是定長視窗(大小固定 k)問「每個視窗的最大值」。難點:視窗滑動時,舊的最大值可能滑出、新元素可能成為最大 —— 如何不必每個視窗重掃 k 個就取得最大? 答案是單調遞減雙端隊列(monotonic deque):隊列存索引、對應值遞減,front 永遠是當前視窗的最大值,每個元素進出隊列各一次 → 攤還 O(n)。

2

為什麼「單調遞減雙端隊列」// WHY

× REJECT · O(n·k)
每個視窗重掃 k 個

對每個視窗掃一遍取最大。n·k 最壞 10¹⁰,超時。沒利用「相鄰視窗只差一進一出」。

△ O(n log k)
大小 k 的多重集/堆

用 multiset 或懶刪除堆維護視窗,取最大 O(log k)。可行,但比單調隊列多一個 log。

MONOTONIC DEQUE · ADOPT ↓
● ADOPT · O(n) / O(k)
遞減隊列存索引,front 即最大

維持一個索引的雙端隊列,對應值單調遞減。加入 i 前,把 back 端所有 ≤ nums[i] 的彈掉(被 i 蓋過、永無出頭);front 端若已滑出視窗就彈掉(過期)。 於是 front 恆為視窗最大值,滿視窗就輸出 nums[front]。每個元素最多進出隊列各一次 → 攤還 O(n)。

3

核心策略 · 四步驟// STRATEGY

  1. 彈 back(維持遞減) —— 加入 i 前,while 隊列非空且 nums[i] ≥ nums[dq.back()],pop_back。
  2. 彈 front(去過期) —— 若 dq.front() 已滑出視窗(i - k ≥ dq.front()),pop_front。
  3. 推入 i —— dq.push_back(i)。
  4. 輸出 —— 視窗滿(i ≥ k-1)時,nums[dq.front()] 就是最大值。

為什麼彈掉 back 的較小值永不後悔// THE INVARIANT

被更晚、更大的元素蓋過 → 永遠不可能再當最大

加入 nums[i] 時,若隊尾某元素 nums[j] ≤ nums[i](j < i),就把 j 彈掉。理由:j 比 i 早、又比 i 小。今後任何包含 j 的視窗一定也包含 i(i 在 j 右邊、更晚才滑出),而 nums[i] ≥ nums[j] —— 所以只要 i 還在,j 永遠當不了最大。既然它再也沒有出頭之日,丟掉不後悔。

彈完後隊列從 front 到 back 值嚴格遞減(索引遞增)。這保證兩件事:front 是最大(值最大),而後面的候選依序備位 —— 一旦 front 過期滑出,次大的自然遞補到 front。

front 過期(front ≤ i-k,滑出視窗左界)則從 front 彈掉。因為隊列按索引遞增,只有 front 可能過期(它索引最小),所以每步至多彈一次 front。 攤還 O(n):每個索引被 push 一次、被 pop 至多一次(從 back 或 front),總操作 ≤ 2n。

6

複雜度// COMPLEXITY

T = O(n)　　S = O(k)

看似有內層 while(彈 back),但每個索引一生只被 push 一次、pop 至多一次,所有 pop 加總 ≤ n → 攤還 O(n)。隊列最多裝一個視窗的候選 → O(k) 空間。勝過 O(n·k) 暴力與 O(n log k) 堆。

一句洞察// INSIGHT

單調遞減雙端隊列:存索引、值遞減,front 恆為視窗最大。彈掉「被更晚更大者蓋過」的 back、彈掉「過期」的 front。 每個元素進出各一次 → 攤還 O(n)。這是「維護滑動視窗極值」的萬用結構,也用在單調隊列優化 DP。
