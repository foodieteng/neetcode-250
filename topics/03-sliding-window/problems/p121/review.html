重點回顧// RECAP

關鍵一句話
核心一問今天賣,最好買在「左側最低價」
兩個變數minPrice(跑動最低)· maxProfit(答案)
順序先算利潤(舊 min)→ 再更新 min
初值minPrice=prices[0],maxProfit=0
不獲利maxProfit 初值 0 → 一路跌回 0
另一視角差分陣列的最大子陣列和(Kadane)
複雜度O(n) 時間 / O(1) 空間

觀念題// CONCEPT Q

Q1

為什麼「固定賣日,買點取左側最低」能保證找到全域最佳解?用「枚舉賣日」的角度說明為何不漏解。

Q2

兩行為什麼要先算利潤、再更新 minPrice?交換順序在什麼輸入下會出錯?交換後利潤會變負嗎?

Q3

maxProfit 初值為什麼是 0 而不是 INT_MIN?這個 0 對應題目哪個合法選擇?

Q4

有人說「答案就是 max(prices) - min(prices)」。給一個反例說明為什麼錯,錯在哪個約束?

Q5

把 prices 換成每日漲跌差分,本題就變成「最大連續子陣列和」。為什麼「一段連續持有的利潤」等於「那段差分之和」?

範例 Trace// 一路下跌 → 0

// prices = [7,6,4,3,1]:全程下跌,應回 0 init: minPrice=7, maxProfit=0 i=1 (價6): profit=6-7=-1 → maxProfit=max(0,-1)=0; minPrice=min(7,6)=6 i=2 (價4): profit=4-6=-2 → maxProfit=0; minPrice=4 i=3 (價3): profit=3-4=-1 → maxProfit=0; minPrice=3 i=4 (價1): profit=1-3=-2 → maxProfit=0; minPrice=1 回傳 maxProfit = 0(任何買賣都虧 → 選擇不交易)
每天的利潤都是負的,maxProfit 全程被初值 0 守住 → 回傳 0,正確表達「不交易」。 minPrice 雖然一路壓到 1,但它右邊已經沒有更高的價可賣,所以派不上用場 —— 最低價出現在最後,買了也賣不出去,這正是 Q4 那個錯誤公式的破綻。

應用題// APPLY

App 1

App 2

要回傳買賣是哪兩天:除了最大利潤,還要輸出買日與賣日索引。掃描時要多記什麼?

App 3

最多兩次交易(LC 123):上限兩筆、不重疊。O(1) 變數還夠嗎?要維護哪幾個狀態?(提示:買一、賣一、買二、賣二四個狀態)

// ANSWERS
A1

最佳解一定是某組 (買日 b, 賣日 s) 且 b < s。我們枚舉每一天當賣日 s:固定 s 後,利潤 prices[s]-prices[b] 要最大就得 prices[b] 最小,而 b 只能在 s 左邊,所以最佳買點就是「s 左側的最低價」。掃描到 s 時,minPrice 恰好是這個值。每個賣日都取到它自己的最佳買點,再對所有賣日取 max → 涵蓋了所有可能的最佳組合,不會漏。

A2

必須先算利潤、再更新 minPrice。因為「今天賣」的買點只能是今天以前的最低;若先把今天價併入 minPrice,就等於允許「買在今天、也賣在今天」。以 [2,4] 為例本無差(仍算對),但語意上會把「同日買賣」當成合法買點。交換後利潤不會變負(頂多為 0,買賣同日),所以答案數值常常仍對 —— 但為了語意正確、也為了推廣到「要輸出買賣日」的版本,務必先算再更新。

A3

初值 0 對應題目允許的「不交易」選擇 —— 若所有買賣都虧,最佳決策是什麼都不做,利潤 0。用 INT_MIN 當初值,遇到全程下跌會回傳一個負的最大利潤,違反「無法獲利回 0」。所以 0 既是初值也是「保底」下限。

A4

反例 [7,6,4,3,1]:max=7、min=1,公式給 6,但正解是 0。錯在忽略「買必須在賣之前」的時間約束 —— 這裡最低價 1 出現在最後一天,而最高價 7 在第一天,你不可能「先低買後高賣」。正確做法是掃描時 minPrice 只看當前賣日的左側,天然滿足先後順序。

A5

設差分 diff[i] = prices[i] - prices[i-1]。第 b 天買、第 s 天賣的利潤 prices[s]-prices[b],把中間逐日展開會望遠鏡相消:= (prices[b+1]-prices[b]) + (prices[b+2]-prices[b+1]) + … + (prices[s]-prices[s-1]) = diff[b+1]+…+diff[s]。也就是「連續持有期間的利潤 = 那段差分之和」。所以求最大利潤 = 求差分陣列的最大連續子陣列和,正是 Kadane;初值 0 對應「空區段(不交易)」。

App 1

LC 122:可多次買賣時,最大利潤 = 把所有相鄰上漲的差都收下 —— sum(max(0, prices[i]-prices[i-1]))。用差分視角看很直觀:多次交易等於可以自由挑選差分陣列裡所有正的元素相加(每個上漲區段都能單獨吃下),而本題(單筆)只能挑一段連續。所以 122 是「取所有正差分」,121 是「取最大連續子段」,同一個差分陣列上的兩種取法。

App 2

掃描時除了 minPrice,多記 minPrice 對應的買日索引 buyIdx;每當 prices[i]-minPrice 刷新 maxProfit 時,記下 賣日 i 與當前 buyIdx 為最佳對。更新 minPrice 時同步更新 buyIdx=i。這樣結束時就握有最佳買賣兩天的索引。仍是 O(n)/O(1)。

App 3

LC 123:最多兩筆、不重疊。O(1) 仍夠,維護四個狀態:buy1(第一次買後的最大「-成本」)、sell1(第一次賣後最大利潤)、buy2(在 sell1 基礎上再買)、sell2(第二次賣後總利潤)。轉移:buy1=max(buy1,-p)、sell1=max(sell1,buy1+p)、buy2=max(buy2,sell1-p)、sell2=max(sell2,buy2+p),答案 sell2。這是本題「單狀態」往「多狀態機」的自然推廣,也是 LC 188(最多 k 筆)的雛形。
