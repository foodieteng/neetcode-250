完整實作 · 一次掃描// O(n)/O(1)

p121.cpp C++17 
class Solution {
public:
    int maxProfit(vector<int>& prices) {
        int maxProfit = 0, minPrice = prices[0], n = prices.size();

        for (int i = 1; i < n; i++) {
            maxProfit = max(maxProfit, prices[i] - minPrice);  // 今天賣、買在史上最低
            minPrice = min(minPrice, prices[i]);            // 再更新最低買點
        }

        return maxProfit;
    }
};
兩行的順序是關鍵:先用舊的 minPrice(= 今天之前的最低)算「今天賣」利潤,再把今天價納入 minPrice。 這保證買點嚴格在賣日之前或同日;若同日,利潤是 0,被 max 自然吸收 —— 對應「不交易、獲利 0」的合法解。

同一題 · 差分 + Kadane 視角// AS MAX SUBARRAY

// 把「每日漲跌」當陣列 → 求最大連續子陣列和(最大子陣列 = 最大利潤) int maxProfit(vector<int>& prices) { int cur = 0, best = 0; for (int i = 1; i < (int)prices.size(); i++) { int diff = prices[i] - prices[i - 1]; // 今天相對昨天的漲跌 cur = max(diff, cur + diff); // Kadane:接續前段 or 重啟 best = max(best, cur); } return best; } 一段連續持有的總利潤 = 期間每日漲跌之和 → 最大利潤 = 最大子陣列和

「第 b 天買、第 s 天賣」的利潤 prices[s]-prices[b] 剛好等於差分陣列 diff[b+1..s] 的總和(望遠鏡相消)。 於是「最大利潤」= 差分陣列的「最大連續子陣列和」,正是 Kadane。兩版都是 O(n)/O(1),只是把同一件事講成「跑動最小」或「跑動累加」。

邊界與陷阱// PITFALLS

陷阱症狀正解
先更新 min 再算利潤 同日先壓低 min 再算 → 可能買賣同日,語意錯(雖利潤仍 0) 先算利潤(舊 min),再更新 min
maxProfit 初值設 INT_MIN 一路下跌時回傳負值,違反「不獲利回 0」 初值 0(可選擇不交易)
回傳「最高價−最低價」 沒管先後,最低價可能在最高價之後 → 買不到 必須買在賣之前:掃描時 min 只看左側
minPrice 初值取 0 股價 ≥ 0,min=0 讓利潤虛高 初值 prices[0]

細節走讀// WALKTHROUGH

// prices = [7,1,5,3,6,4] init: minPrice=7, maxProfit=0 i=1 (價1): profit=1-7=-6 → maxProfit=max(0,-6)=0; minPrice=min(7,1)=1 i=2 (價5): profit=5-1=4 → maxProfit=max(0, 4)=4; minPrice=1 i=3 (價3): profit=3-1=2 → maxProfit=max(4, 2)=4; minPrice=1 i=4 (價6): profit=6-1=5 → maxProfit=max(4, 5)=5; minPrice=1 i=5 (價4): profit=4-1=3 → maxProfit=max(5, 3)=5; minPrice=1 回傳 maxProfit = 5(第 1 天買 1、第 4 天賣 6)
看 i=1:利潤是負的,maxProfit 靠初值 0 擋住不變;但這一步把 minPrice 壓到 1,為後面所有賣日鋪好最佳買點。 這就是「跑動最小」的價值 —— 一個變數同時服務所有後續的賣日決策。
