完整實作 · 達標就縮// O(n)

p209.cpp C++17 
class Solution {
public:
    int minSubArrayLen(int target, vector<int>& nums) {
        long long total = 0;
        int minL = INT_MAX, l = 0;

        for (int r = 0; r < nums.size(); r++) {
            total += nums[r];                       // 擴張累加

            while (total >= target) {              // 達標 → 拚命縮左求最短
                minL = min(minL, r - l + 1);      // 仍合格 → 更新最短
                total -= nums[l];
                l++;
            }
        }

        return minL == INT_MAX ? 0 : minL;   // 從沒達標 → 0
    }
};
收縮條件是 while (total >= target) —— 「達標就縮」,和求最長的「違規才縮」相反。縮之前先 minL = min(...),因為當前這個合格視窗可能就是最短,縮掉一格後可能就不合格了。 total 用 long long(target 到 10⁹、和可能更大)。minL 從沒被更新過就代表整段和都不到 target,回 0。

Follow-up · 前綴和 + 二分// O(n log n)

// 正數 → 前綴和 prefix 嚴格遞增 → 對每個右界二分找最短左界 int minSubArrayLen(int target, vector<int>& nums) { int n = nums.size(); vector<long long> pre(n + 1, 0); for (int i = 0; i < n; i++) pre[i + 1] = pre[i] + nums[i]; // pre[i]=前 i 個的和 int minL = INT_MAX; for (int r = 1; r <= n; r++) { // 找最大的 l 使 pre[r] - pre[l] >= target,即 pre[l] <= pre[r]-target long long need = pre[r] - target; int lo = upper_bound(pre.begin(), pre.begin() + r, need) - pre.begin(); if (lo > 0) minL = min(minL, r - (lo - 1)); // 子陣列 [lo-1, r-1] } return minL == INT_MAX ? 0 : minL; } pre 遞增才能二分:對每個右界 r,找最靠右、和仍 ≥ target 的左界 → O(log n)/右界

這是題目 follow-up 要的 O(n log n)。因為全是正數,前綴和 pre 嚴格遞增 → 可二分。對每個右界 r,要找最短的合格子陣列,就是找最靠右的左界 l 使 pre[r] - pre[l] ≥ target,即 pre[l] ≤ pre[r] - target,用 upper_bound 定位。它比滑動視窗慢(多一個 log),但示範「前綴和單調 → 二分」這個常用套路,也是通往含負數版(862 單調隊列)的踏腳石。

邊界與陷阱// PITFALLS

陷阱症狀正解
縮之後才更新 minL 縮掉一格可能已不合格 → 漏掉最短 先 minL=min(...) 再縮
忘了「不存在回 0」 回傳 INT_MAX minL==INT_MAX ? 0 : minL
total 用 int 和可能超過 INT_MAX 用 long long
把滑動視窗用在含負數 縮左不一定變小 → 邏輯崩 負數改前綴和 + 單調隊列(862)
求最短卻「違規才縮」 套錯求最長的模板 求最短要「達標就縮」
