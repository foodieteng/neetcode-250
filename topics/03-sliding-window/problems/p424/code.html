完整實作 · 眾數視窗// O(n)

p424.cpp C++17 
class Solution {
public:
    int characterReplacement(string s, int k) {
        unordered_map<char, int> freq;
        int maxL = 0, l = 0, n = s.size();
        int maxF = 0;                              // 視窗內眾數次數(只增不減)

        for (int r = 0; r < n; r++) {
            maxF = max(maxF, ++freq[s[r]]);          // 擴張 + 更新眾數

            while (r - l + 1 - maxF > k) {            // 要換的字元 > k → 不合法
                freq[s[l++]]--;                     // 縮左
            }

            maxL = max(maxL, r - l + 1);              // 記最長合法視窗
        }

        return maxL;
    }
};
r - l + 1 - maxF = 視窗長 − 眾數次數 = 要替換的字元數。> k 就縮左。maxF 縮視窗時不減小 —— 這不是 bug:偏大的 maxF 只讓視窗「懶得縮」、保持大小平移,而 maxL 只記曾達到的最大合法長度,不會被高估(見 概念頁 §5)。省掉每次重算眾數的 O(26)。

// 因為視窗只增不縮(靠 maxF 單調),縮左其實每步最多一次 → 可用 if int characterReplacement(string s, int k) { vector<int> freq(26, 0); int maxF = 0, l = 0, res = 0; for (int r = 0; r < (int)s.size(); r++) { maxF = max(maxF, ++freq[s[r] - 'A']); if (r - l + 1 - maxF > k) { // 至多縮一格,窗大小不再變小 freq[s[l++] - 'A']--; } res = max(res, r - l + 1); // res 其實 = 最終視窗長度 } return res; } maxF 單調不減 → 一旦視窗合法過某長度,就不會再變短,只會平移 → while 可換成 if

進一步的觀察:因為 maxF 只增不減,視窗長度 r-l+1 一旦到某個大小就再也不會縮小(不合法時 r 進一格、l 也進一格,大小不變)。所以那個 while 每步最多執行一次,可直接寫成 if,語意等價。用 vector<int>(26) 取代雜湊常數更小。兩版都 O(n)。

邊界與陷阱// PITFALLS

陷阱症狀正解
縮窗時重算/回退 maxF 以為要維護「當前」眾數 → 多做 O(26) maxF 只增不減即可,答案不變
合法條件寫反 用 maxF - (len) > k 之類 (r-l+1) - maxF > k 才縮
對 26 字母各跑一遍 可過但慢 26 倍 眾數一次涵蓋所有字母
k 很大 k ≥ n 時整串都能換 條件天然成立,回傳 n
把 maxL 當視窗長 while 版 maxL 才是真答案 每步 maxL=max(maxL, r-l+1)
