題目簡介// THE QUESTION

QUESTION

給字串 s 和整數 k,你可以把最多 k 個字元換成任意大寫字母。求換完後,同一字母連續子字串的最大長度。 例:s="AABABBA", k=1 → 4;s="ABAB", k=2 → 4。

這是 3(最長無重複) 的同骨架變體 —— 一樣的可變長視窗,只是把「合法條件」換掉。 一個視窗要能靠 ≤ k 次替換變成「全同一字母」,只需把非眾數的字元都換掉;所以合法 ⟺ 視窗長度 − 視窗內出現最多的字元次數 ≤ k。右擴張、不合法才縮,答案取最長合法視窗。

2

為什麼「可變長視窗 + 眾數」// WHY

× REJECT · O(26·n²)
枚舉子字串 + 檢查

對每個子字串數字元、算要換幾個。平方級以上,n 到 10⁵ 超時。

△ 固定目標字母
對 26 個字母各跑一次視窗

枚舉「保留哪個字母」,對每個字母做視窗,O(26n)。可行但多做 26 倍;其實一次就能兼顧所有字母。

VARIABLE WINDOW · ADOPT ↓
● ADOPT · O(n) / O(26)
合法 ⟺ 視窗長 − 眾數次數 ≤ k

右指針擴張、記各字元計數與眾數次數 maxF;當 (r-l+1) - maxF > k(要換的字元超過 k)就縮左,直到合法。每步更新 maxL。 「非眾數字元數」就是最少替換數,眾數自然涵蓋所有 26 個字母的最佳選擇 —— 一趟 O(n) 搞定。

3

核心策略 · 三步驟// STRATEGY

  1. 擴張 + 記眾數 —— r 前進,freq[s[r]]++,maxF = max(maxF, freq[s[r]])。
  2. 不合法就縮 —— 當 (r-l+1) - maxF > k(替換數超過 k),freq[s[l++]]--。
  3. 記最長 —— maxL = max(maxL, r-l+1)。掃完回傳 maxL。

為什麼 maxF「只增不減」也對// THE STALE MAXFREQ

縮視窗時不重算 maxF,答案照樣正確

程式碼裡 maxF 只更新變大、縮視窗時從不減小。這看起來像 bug —— 縮完後真正的眾數次數可能比 maxF 小,maxF 是「歷史最大值」而非「當前視窗的眾數」。但答案仍然正確,原因是:

我們要的是最長合法視窗的長度。maxL 只會在「找到更長的合法視窗」時更新。一個偏大的 maxF 只會讓合法條件 len - maxF ≤ k 更寬鬆,使視窗不縮;但只有當「當前 maxF 真的來自一個那麼大的眾數」時,才可能撐出對應長度的答案。視窗長度永遠不會超過「歷史上真的出現過那個眾數次數」時的合法長度,所以 maxL 不會被高估。

直覺版:maxF 偏大只是讓視窗「懶得縮」,窗會保持同樣大小平移;而 maxL 記的是曾經達到的最大長度,一旦真的能靠某個眾數換到更長,maxF 就會被更新到那個值、maxL 跟著增加。所以省下每次縮窗重算眾數的 O(26),答案不變 —— 這是本題最有名的優化陷阱。

複雜度// COMPLEXITY

T = O(n)　　S = O(26) = O(1)

r、l 各走一趟(攤還 O(1)/步)。maxF 不重算 → 每步 O(1);若每次縮窗都重掃 26 個字母找眾數,則是 O(26n)(仍線性)。計數表固定 26 → O(1) 空間。

面向成本說明
右指針掃描O(n)每字元進視窗一次
左指針收縮攤還 O(1)全程最多 n 次
眾數維護O(1)maxF 只增不減
空間O(26)字母計數表

一句洞察// INSIGHT

同 3 的可變長骨架,把合法條件換成 「視窗長 − 眾數次數 ≤ k」(要替換的非眾數字元 ≤ k)。眾數自動涵蓋所有字母,一趟 O(n)。 招牌陷阱:maxF 只增不減也對,因為 maxL 只記曾達到的最大合法長度。
