最長重複字元替換

// REVIEW · 複習與問答

重點回顧// RECAP

關鍵一句話
類型可變長視窗(求最長),換合法條件
合法條件視窗長 − 眾數次數 ≤ k
替換數= 非眾數字元數 = 長 − maxF
眾數一次涵蓋所有 26 字母
maxF只增不減也正確(招牌陷阱)
while→if視窗不縮 → 每步至多縮一格
複雜度O(n) 時間 / O(26) 空間
2

觀念題// CONCEPT Q

Q1

為什麼「一個視窗要能變成全同字母」的最少替換數 = 視窗長 − 眾數次數?為什麼保留眾數最省?

Q2

為什麼只維護一個 maxF 就能同時兼顧 26 個字母,不用對每個字母分別跑?

Q3

招牌陷阱:縮視窗時 maxF 不減小,它可能大於當前視窗的真實眾數。為什麼答案 maxL 仍不會被高估?

Q4

承上,為什麼那個 while 可以改成 if?視窗長度為什麼「只增不減」?

Q5

本題和 3(最長無重複) 骨架相同。唯一的差別在哪一行?一句話概括「可變長求最長」這一族的通用模式。

3

範例 Trace// k=0 退化成「最長同字元」

// s = "AABBBCA", k = 0(不能換 → 找最長連續同字元) r=0 A: maxF=1; 1-1=0≤0 [0,0] maxL=1 r=1 A: maxF=2; 2-2=0≤0 [0,1] maxL=2 ("AA") r=2 B: maxF=2; 3-2=1>0 → 縮到 [2,2] "B" maxL=2 r=3 B: maxF=2; 2-2=0≤0 [2,3] maxL=2 r=4 B: maxF=3; 3-3=0≤0 [2,4] maxL=3 ("BBB") r=5 C: 縮到 [5,5]; r=6 A: 縮到 [6,6] 回傳 3;k=0 時「替換數 ≤ 0」= 視窗全同字元 → 就是最長連續同字母
k=0 讓合法條件變成 「長度 − 眾數 ≤ 0」= 視窗內全是同一字母 —— 本題直接退化成「最長連續相同字元」。 這說明 k 是「容許的雜質量」:k=0 不容雜質、k 越大容許越多非眾數字元,視窗能拉越長。合法條件裡的 k 就是可變長視窗的「鬆緊旋鈕」。

應用題// APPLY

App 1

LC 1004 最長連續 1(可翻 k 個 0):0/1 陣列,最多把 k 個 0 翻成 1,求最長全 1 段。合法條件變成什麼?為什麼比本題更簡單(不用眾數)?

App 2

求那段子字串:除了長度,還要輸出替換後的那段。掃描時多記什麼?

App 3

可變長視窗總表:把 3、424、209、76 放一起。「求最長 vs 求最短」的收縮時機分別是什麼?怎麼從題目一眼判斷?

// ANSWERS
A1

要把視窗變成「全同一字母」,你會選一個字母保留、其餘全換掉。換掉的數量 = 視窗長 − 保留字母的次數。要換得最少,當然保留「出現最多次」的那個字母(眾數)——換掉的就是「非眾數的字元數」= 視窗長 − 眾數次數(maxF)。保留任何非眾數字母都會換更多,所以眾數最省。合法 ⟺ 這個最少替換數 ≤ k。

A2

因為「該保留哪個字母」的最佳選擇,永遠是當前視窗的眾數。我們不需要預先決定保留哪個字母、也不需對 26 個字母各試一遍 —— 只要在滑動時追蹤眾數次數 maxF,合法條件 len - maxF ≤ k 就自動用了「最省的保留選擇」。眾數是誰會隨視窗變,但我們只在乎它的次數,而 maxF 幫我們記住了。一個變數,涵蓋所有字母的最佳情況。

A3

maxL 只在「找到更長的合法視窗」時更新。假設某時刻 maxF 是歷史眾數(偏大),它讓 len - maxF ≤ k 更寬鬆、視窗不縮,於是視窗保持某個大小平移。但這個大小當初是「真的有一個 maxF 這麼大的眾數」時才合法達到的 —— 也就是說 maxL 記的那個長度,對應到某個真實存在過的合法視窗。偏大的 maxF 只會維持、不會憑空放大長度;要 maxL 再變大,必須真的出現更大的眾數把 maxF 推高。所以 maxL 不會被高估。

A4

因為 maxF 只增不減,視窗長度 r-l+1 一旦達到某值就不會再變小。當視窗不合法(len - maxF > k),r 進一格的同時 l 也進一格,視窗大小維持不變(平移);只有當 maxF 變大讓條件放寬時,視窗才會變長。既然每個 r 頂多讓視窗「不變或變長」,那個 while 每步最多執行一次 → 可換成 if。此時最終的 r-l+1 就等於答案(視窗長度只增)。

A5

唯一差別在收縮的合法條件那一行:3 是 while (freq[s[r]] > 1)(有重複就縮);424 是 while ((r-l+1) - maxF > k)(要換的超過 k 就縮)。其餘(右擴張、每步更新 maxL)一模一樣。「可變長求最長」的通用模式:右指針一直擴張;當視窗「不合法」時縮左到重新合法;每步用 r-l+1 更新最大長度。換題只換「什麼叫合法」這個判斷式。

App 1

LC 1004:0/1 陣列,保留的一定是 1,要換的就是視窗內 0 的個數。合法 ⟺ 視窗內 0 的個數 ≤ k。比本題簡單:不用維護眾數 maxF,只要數 0(zeros)—— while (zeros > k) { if (nums[l]==0) zeros--; l++; },每步更新最長。因為只有兩種值、且明確要保留 1,「保留哪個」不必用眾數決定,直接數要換的 0 即可。

App 2

多記最佳視窗的起點:每當 r-l+1 刷新 maxL 時,記 bestStart = l、bestLen = r-l+1。掃完用 s.substr(bestStart, bestLen) 取那段。若還要輸出替換後的字串,把那段裡的非眾數字元都改成眾數即可(眾數可在那段重新數一次,O(bestLen))。仍是線性。

App 3

可變長視窗總表:
‧ 求最長(3、424、1004):右擴張,視窗「不合法」時縮,答案取 max(r-l+1)。
‧ 求最短(209、76):右擴張,視窗「已合格」時就拚命縮(想更短),每次合格更新 min(r-l+1)。
一眼判斷:題目問「最長」→ 違規才縮;問「最短」→ 達標就縮。收縮時機正好相反,這是整個可變長視窗家族的分類主軸。
