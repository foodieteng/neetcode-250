題目簡介// THE QUESTION

QUESTION

給字串 s,求不含重複字元的最長子字串長度。子字串是連續的一段。 例:"abcabcbb" → 3("abc");"pwwkew" → 3("wke");"bbbbb" → 1。

這是可變長滑動視窗的代表題。和 219(定長) 對照:219 的視窗大小固定(≤ k+1),而這題的視窗大小會隨資料伸縮 —— 收縮的依據不是「幾個」,而是「視窗還合不合法(有沒有重複)」。右指針一路擴張,一旦出現重複就從左收縮,答案取歷程中最長的合法視窗。

2

為什麼「可變長滑動視窗」// WHY

× REJECT · O(n²)~O(n³)
枚舉所有子字串

試每個 (i,j) 再檢查有無重複。n 到 5×10⁴ → 平方級以上,超時。

△ 每次重掃
固定左端、右端往後掃到撞重複

對每個左端重新往右掃。有重複時左端只前進一格又從頭掃,重複工作 → 仍是 O(n²)。

SLIDING WINDOW · ADOPT ↓
● ADOPT · O(n) / O(k)
r 擴張、遇重複才縮 l,窗長取最大

右指針 r 每步把 s[r] 記入計數並擴張;若 s[r] 讓視窗出現重複,就 l 右移、移出字元直到重複消失。每步用 r-l+1 更新 maxL。 l、r 各只往右走一趟、從不回頭 → O(n);計數表最多裝字元集大小 k(字母/ASCII)→ O(k) 空間。

3

核心策略 · 三步驟// STRATEGY

  1. 擴張 —— r 從左到右,freq[s[r]]++ 把新字元納入視窗。
  2. 收縮到合法 —— 當 freq[s[r]] > 1(視窗有重複),freq[s[l++]]-- 移出左端,直到重複消失。
  3. 記錄最長 —— maxL = max(maxL, r - l + 1)。掃完回傳 maxL。

定長 vs 可變長 · 收縮的依據不同// THE TWO FLAVORS

收縮條件 = 這兩類滑動視窗的分水嶺

兩類滑動視窗都是「右擴張、左收縮」,差別只在什麼時候收縮:

‧ 定長(219):收縮看大小 —— r - l > k 就縮,窗長上限固定,和內容無關。
‧ 可變長(本題):收縮看內容合不合法 —— 視窗出現重複才縮,縮到合法為止,窗長隨資料浮動。

一句話:定長看「幾個」,可變長看「合不合法」。可變長又常分兩種目標:求最長合法視窗(本題、424)——「違規才縮、答案取 max」;求最短合格視窗(209、76)——「達標就拚命縮、答案取 min」。認出目標是最長還是最短,就知道收縮邏輯往哪擺。

6

複雜度// COMPLEXITY

T = O(n)　　S = O(k)

r 前進 n 步;l 全程最多也前進 n 步(攤還 O(1)/步)—— 兩指針各走一趟,不回頭。計數表最多裝字元集大小 k(小寫字母 26、ASCII 128)→ O(k)(視為常數也可寫 O(1))。

面向成本說明
右指針掃描O(n)每字元進視窗一次
左指針收縮攤還 O(1)全程最多移動 n 次
計數表操作O(1)雜湊 / 陣列
空間O(k)字元集大小
7

一句洞察// INSIGHT

可變長視窗:右擴張、違規(重複)才縮左、答案取最長。收縮依據是「內容合不合法」,不是固定大小 —— 這是它和 219 定長 的分水嶺。 左右指針各走一趟 → O(n)。同一骨架換個「合法條件」就解 424、209、76。
