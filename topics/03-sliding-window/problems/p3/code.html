最長無重複字元子字串

完整實作 · 計數收縮// O(n)

p3.cpp C++17 
class Solution {
public:
    int lengthOfLongestSubstring(string s) {
        int n = s.size();
        int maxL = 0, l = 0;
        unordered_map<char, int> freq;          // 視窗內每個字元的計數

        for (int r = 0; r < n; r++) {
            freq[s[r]]++;                       // 擴張:納入 s[r]

            while (freq[s[r]] > 1) {            // 有重複 → 收縮到合法
                freq[s[l++]]--;                  // 移出左端
            }

            maxL = max(maxL, r - l + 1);       // 記錄最長合法視窗
        }

        return maxL;
    }
};
收縮條件 while (freq[s[r]] > 1) 很巧:只有剛加入的 s[r] 可能造成重複(其餘字元在此之前都無重複),所以只需盯著 freq[s[r]]。 一直移出左端直到那個字元的計數回到 1,視窗就恢復「無重複」。l 只增不減,和 r 各走一趟 → O(n)。

變體 · 記位置一步跳過// LAST-INDEX JUMP

// 存「每個字元上次出現的索引」,l 直接跳過去,不逐格縮 int lengthOfLongestSubstring(string s) { vector<int> last(128, -1); // ASCII → 上次出現的 index int maxL = 0, l = 0; for (int r = 0; r < (int)s.size(); r++) { if (last[s[r]] >= l) // 上次出現還在視窗內 → 跳過它 l = last[s[r]] + 1; // l 一步跳到重複字元的右邊 last[s[r]] = r; maxL = max(maxL, r - l + 1); } return maxL; } l = max(l, last[s[r]]+1);逐格 while 收縮換成「一步跳」,同為 O(n) 但常數更小

原版用 while 一格格移出左端;這版存每個字元上次出現的索引,遇到重複時 l 直接跳到那個位置的右邊,省掉逐格搬移。 注意 last[s[r]] >= l 的判斷 —— 只有「上次出現還在目前視窗內」才需要跳,否則那個重複早已在視窗外、不影響。用 vector<int>(128) 取代雜湊也讓常數更小。

陷阱症狀正解
收縮條件寫錯 用 if 只縮一次,重複沒清乾淨 while (freq[s[r]] > 1) 縮到合法
跳版忘了 >= l l = last[s[r]]+1 可能倒退 l = max(l, last[s[r]]+1)
空字串 s="" 應回 0 迴圈不執行,maxL=0 天然正確
只算子序列不算子字串 誤把不連續也算進來 視窗是連續段,天然是子字串
回傳子字串而非長度 題目只要長度 return maxL

// s = "pwwkew" r=0 'p': freq{p:1} 窗"p" [0,0] maxL=1 r=1 'w': freq{p:1,w:1} 窗"pw" [0,1] maxL=2 r=2 'w': freq[w]=2>1 → 縮:移出 p(l=1)、移出舊 w(l=2),freq[w]=1 窗"w" [2,2] maxL=2 r=3 'k': freq{w:1,k:1} 窗"wk" [2,3] maxL=2 r=4 'e': freq{w:1,k:1,e:1} 窗"wke" [2,4] maxL=3 ← 刷新 r=5 'w': freq[w]=2>1 → 縮:移出舊 w(l=3),freq[w]=1 窗"kew" [3,5] maxL=3 回傳 maxL = 3(最長無重複子字串,如 "wke" / "kew")
看 r=2 的收縮:while 一路移出 p、w,不是只縮一格 —— 直到重複的 w 計數回到 1。 這就是「可變長」的精髓:收縮量由「恢復合法」決定,不是固定值。答案可能出現在中段("wke"),所以每步都要更新 maxL。
