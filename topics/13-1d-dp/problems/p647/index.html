題目簡介// THE QUESTION

QUESTION

給字串 s,回傳它裡面回文子字串的總數。子字串要連續;不同起訖位置即使內容相同也分開計算(如 "aaa" 有 6 個)。

這題是 5 最長回文子字串 的雙胞胎:同一張 dp[i][j] 表、同一條遞迴、同樣依長度填。唯一差別是 —— 5 記「最長」,647 記「有幾個」。把「更新最長」換成「res++」就好。

2

為什麼用區間 DP// WHY INTERVAL DP

A 暴力:枚舉所有子字串逐一判斷 O(n³)
O(n²) 個子字串 × 每個回文檢查 O(n)。n=1000 太慢。
× REJECT · TLE
B 中心擴展:每個中心往兩邊擴,邊擴邊數 O(n²) / O(1)
每成功擴一步就是一個新回文,res++。空間更省;但要分奇偶中心。
○ OK · 可用
ADOPT
C 區間 DP:每個 true 格 = 一個回文子串 O(n²) / O(n²)
dp[i][j]=s[i]==s[j] && dp[i+1][j-1]。填表時只要 dp[i][j]=true 就 res++;掃完整張表,累計值就是答案。與 5 共用骨架。
● ADOPT · O(n²)
3

核心策略 · 三步驟// STRATEGY

  1. base 計數:長度 1 全部 true → res += n;長度 2 頭尾相同者 res++。
  2. 轉移計數:長度由小到大,dp[i][j]=s[i]==s[j] && dp[i+1][j-1],為 true 就 res++。
  3. 回傳 res:整張表為 true 的格子數,就是回文子字串總數。

關鍵推導// COUNT = TRUE CELLS

LEMMA · true 格與回文子串一一對應

每個回文子字串由它的起點 i 與終點 j 唯一決定,對應表上恰好一格 (i,j)。反之,dp[i][j]=true 代表 s[i..j] 是回文,也就是一個回文子字串。

所以「回文子字串的數量」= 「表上 true 格的數量」,不重不漏。因此不需要另外去重 —— 每格只被填一次、只數一次。這也解釋了為什麼 "aaa" 的答案是 6 而不是 3:三個 "a"、兩個 "aa"、一個 "aaa",位置不同就是不同子字串。

複雜度// COMPLEXITY

T = O(n²)　　S = O(n²)

填 O(n²) 格,每格 O(1)。空間為那張表 O(n²)。中心擴展版時間一樣、空間 O(1)。
