範例 Trace// s = "aba"

// 區間 DP 計數 s = a b a (index 0 1 2) 長度1:dp[0][0]=dp[1][1]=dp[2][2]=true → res += 3 (res=3) 長度2:dp[0][1]=(a==b)? no ; dp[1][2]=(b==a)? no → res 不變 長度3:dp[0][2]=(a==a) 且 dp[1][1]=true → true → res++ (res=4) return res = 4 → "a","b","a","aba"
注意 "aba" 的長度 2 一個都沒有(ab、ba 都不是回文),但長度 3 的 "aba" 靠內層 dp[1][1]="b" 成立。 回文可以「跳過」某些長度:中間沒有偶回文,不代表更長的奇回文不存在。

應用題// APPLY

App 1. 若要同時回傳最長回文與回文總數,一次掃描能不能都拿到?怎麼改?
App 2. 131 分割回文串:要把 s 切成若干段、每段都是回文,列出所有切法。這張 dp 表能當什麼用?
App 3. 若字串長到 10⁵、只要總數,O(n²) 太慢。用什麼線性演算法能在 O(n) 內算出回文子字串總數?
// ANSWERS
A1. 因為每個回文子字串由它的 (起點 i, 終點 j) 唯一決定,對應表上恰好一格;反過來每個 dp[i][j]=true 也正好代表一個回文子字串。這是一一對應,所以「數量 = true 格數」,不重不漏。每格只被填一次、只在變 true 當下數一次,不會重複。
A2. 因為子字串以位置區分,不是以內容去重。"aaa" 的 6 個是:三個長度 1 的 "a"(位置 0、1、2)、兩個長度 2 的 "aa"(0–1、1–2)、一個長度 3 的 "aaa"(0–2)。內容雖重複,但起訖不同就算不同子字串,共 3+2+1=6。
A3. 狀態、轉移、填表順序完全一樣,差別只在「遇到 dp[i][j]=true 時做什麼」:5 是「用 L 更新最長答案 ans」,647 是「res++」。可以說是同一份程式碼、換掉最後一步的動作 —— 這就是為什麼掌握其中一題,另一題幾乎免費。
A4. count() 從中心往兩邊擴,每次 while 成功(s[left]==s[right])代表「當前這對邊界之內是一個回文」,c++。因為以同一中心、往外擴 k 步會得到 k 個以此為中心、半徑遞增的回文(每多對稱一層就多一個),所以擴展成功的次數 c 就是這個中心貢獻的回文數。兩種中心(奇/偶)加總即全部。
A5. 不會。當 L=2,j=i+1,轉移要查內層 dp[i+1][j-1]=dp[i+1][i]。但 i+1 > i,這是下三角格子,程式從沒設過它(初始 false)。於是條件 s[i]==s[j] && dp[i+1][i] = … && false = false,不會 res++。所以長度 2 的回文只在 base ② 被數一次,即使外層從 L=2 起也安全 —— 這是「空區間視為 false」自然帶來的保護。
App 1. 可以,一次掃描兩者都拿。同一張 dp 表,填到 dp[i][j]=true 時同時做兩件事:res++(計數)以及「若 L 比目前最長大就更新 ans={i,L}」(記最長)。兩個統計互不干擾,共用同一輪 O(n²) 填表即可,不必掃兩次。
App 2. 這張 dp[i][j](某段是不是回文)可當 131 的預處理表:回溯切割時,要判斷「s[i..j] 能不能當一段」,直接 O(1) 查 dp[i][j],不必每次重新驗回文。先 O(n²) 建表,再 DFS 枚舉切點,把每次回文檢查從 O(n) 降到 O(1)。這是「回文 DP 表當子程序」的典型用法。
App 3. Manacher 演算法,O(n)。它一次求出每個中心的最大回文半徑陣列;而以某中心為心的回文個數,正好等於該中心的半徑值(半徑每 +1 就多一個回文)。所以把所有中心的半徑加總,就是回文子字串總數,線性完成。等於把 A4 的「邊擴邊數」用 Manacher 的半徑陣列一次算好,免去重複擴展。
