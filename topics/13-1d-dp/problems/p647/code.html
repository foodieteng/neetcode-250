寫法 A · 區間 DP(數 true 格)// INTERVAL DP

p647.cpp C++17 
class Solution {
public:
    int countSubstrings(string s) {
        int n = s.size();
        vector<vector<bool>> dp(n, vector<bool>(n, false));
        int res = 0;

        // base ①:長度 1,每個都算一個回文
        for (int i = 0; i < n; i++) {
            dp[i][i] = true;
            res++;
        }

        // base ②:長度 2,頭尾相同再 +1
        for (int i = 0; i < n - 1; i++) {
            if (s[i] == s[i + 1]) {
                dp[i][i + 1] = true;
                res++;
            }
        }

        // 轉移:長度由小到大,為 true 就 res++
        for (int L = 3; L <= n; L++) {
            for (int i = 0; i + L - 1 < n; i++) {
                int j = i + L - 1;
                if (s[i] == s[j] && dp[i + 1][j - 1]) {
                    dp[i][j] = true;
                    res++;
                }
            }
        }

        return res;
    }
};

和 5 逐字一樣的三段填表,只把「更新 ans」換成 res++。提醒:轉移迴圈的長度從 3 起(長度 2 已在 base ② 數過);若寫成從 2 起也不會重複,因為 L=2 時內層 dp[i+1][j-1]=dp[i+1][i] 是下三角、恆為 false,條件不成立。

寫法 B · 中心擴展(邊擴邊數)// EXPAND · O(1)

p647_expand.cpp C++17 
class Solution {
public:
    int countSubstrings(string s) {
        int n = s.size(), res = 0;
        for (int center = 0; center < n; center++) {
            res += count(s, center, center);       // 奇數長度中心
            res += count(s, center, center + 1);   // 偶數長度中心
        }
        return res;
    }

    int count(const string& s, int left, int right) {
        int n = s.size(), c = 0;
        while (left >= 0 && right < n && s[left] == s[right]) {
            left--; right++; c++;       // 每成功擴一步 = 一個新回文
        }
        return c;
    }
};

比 DP 更省:每個中心往外擴,每擴成功一次就多一個回文,直接累加,不必存表 → 空間 O(1)。奇中心 (c,c) 抓奇數長度、偶中心 (c,c+1) 抓偶數長度,合起來不重不漏。

複雜度分析// COMPLEXITY

T = O(n²)　　S = O(n²)(A) / O(1)(B)

A:填 O(n²) 格表。B:2n 個中心 × 每次最多擴 O(n) = O(n²),只用幾個變數。

邊界與陷阱// PITFALLS

陷阱後果正解
以為 "aaa" 只有 3 個漏數 "aa"×2、"aaa"×1位置不同即不同 → 共 6
長度 2 又在 L 迴圈重數偶回文被算兩次L 從 3 起(或倚賴內層恆 false)
中心擴展只做奇中心漏掉所有偶數長度回文奇 + 偶兩種中心都做
DP 按 i,j 順序填內層未算 → 少數外層跑長度由小到大

註:兩種寫法都 O(n²);要更省空間或更快初始化就用中心擴展。n ≤ 1000 下皆綽綽有餘。
