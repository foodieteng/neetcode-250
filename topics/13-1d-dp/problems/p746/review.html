面向要點
狀態dp[i] = 到達位置 i 的最小花費
轉移dp[i] = min(dp[i−1]+cost[i−1], dp[i−2]+cost[i−2])
basedp[0] = dp[1] = 0(免費起步)
付費時機費用在離開某階時付 → 加來源階的 cost
答案dp[n](頂樓 = 位置 n,不是 n−1)
與 70同骨架:70 相加數方法、746 取 min 求最省

範例 Trace// cost=[1,100,1,1,1,100,1,1,100,1]

// 填表(n=10,頂樓=位置 10) dp[0]=0 dp[1]=0 dp[2]=min(0+1, 0+100)=1 dp[3]=min(0+100, 1+1)=2 dp[4]=min(1+1, 2+1)=2 dp[5]=min(2+1, 2+1)=3 dp[6]=min(2+1, 3+100)=3 dp[7]=min(3+100, 3+1)=4 dp[8]=min(3+1, 4+1)=4 dp[9]=min(4+1, 4+100)=5 dp[10]=min(4+100, 5+1)=6 return dp[10] = 6
看 dp[3]=2:與其付第 1 階的 100,不如「從第 0 階(免費)先付 1 跨到第 2 階,再付 1」——DP 自動幫你繞開昂貴的第 1 階。 這正是貪心會失手的地方:當下看第 2 階便宜,但整體最省的路徑要靠「記住每個位置的最小累積花費」才找得到。

應用題// APPLY

App 1. 若改成每次可爬 1、2 或 3 階,轉移式怎麼變?base 要幾個?
App 2. 若還要回傳最省路徑本身(踩了哪些階),而不只是花費,程式怎麼加?
App 3. 若題目改成「費用是踩上該階就付」(而非離開時付),轉移和答案要怎麼調整?
// ANSWERS
A1. 因為費用是「離開某階往上跨時才付」。到位置 i 這件事,是你從 i−1 或 i−2 跨上來造成的 —— 跨的是那個來源階,所以付的是來源階的費用:dp[i−1]+cost[i−1] 或 dp[i−2]+cost[i−2]。cost[i] 要等你從 i 再往上跨時才會被算到(在計算 dp[i+1]/dp[i+2] 時)。所以當前這一步不加 cost[i]。
A2. 樓梯有 n 階,索引 0..n−1;「頂樓」是最後一階的再上一格,也就是位置 n(你要爬離開樓梯)。所以要求的是到位置 n 的最小花費 dp[n]。從最後一階 n−1 或倒數第二階 n−2 都能跨到頂,剛好對應 dp[n]=min(dp[n−1]+cost[n−1], dp[n−2]+cost[n−2])。若回 dp[n−1] 等於「只到最後一階、還沒踏出樓梯」,少算一步。
A3. 對應題目「你可以從第 0 階或第 1 階起步」。起步只是「站上去」,還沒往上跨,所以不花錢 → 到位置 0、到位置 1 的最小花費都是 0。費用只在「從某階往上跨」時產生,由後面的轉移累加進來。
A4. 因為局部最便宜 ≠ 全局最省。貪心只看眼前哪一階 cost 小,但那可能把你逼進後面更貴的組合。以 [1,100,1,1,1,100,1,1,100,1] 為例:貪心可能被便宜的中間階誘導,卻錯過「用兩步一次跨過昂貴的 100」的機會。DP 記住每個位置的最小累積花費,在每格都把兩條來路都比一次取 min,才保證整條路徑最省。
A5. 只差一個運算:70 是 dp[i]=dp[i−1]+dp[i−2](加法,把方案數加起來);746 是 dp[i]=min(dp[i−1]+cost[i−1], dp[i−2]+cost[i−2])(取 min,在幾條路中挑最優)。這正反映 DP 的兩大類:計數型(相加求「有幾種」)與最優化型(取 min/max 求「最好的一個」)。骨架(線性、依賴前兩項、可滾動 O(1))完全共用。
App 1. 每次可跨 1/2/3 階 → 到位置 i 有三種來源 → dp[i]=min(dp[i−1]+cost[i−1], dp[i−2]+cost[i−2], dp[i−3]+cost[i−3])。base 要涵蓋前幾個能免費起步的位置(dp[0]=dp[1]=dp[2]=0,若允許從 0/1/2 起步),或對 i<3 特判,因為轉移用到 i−3 得先有前三項。滾動優化改成留三個變數。
App 2. 多開一個 from[i] 記「dp[i] 是從 i−1 還是 i−2 轉移來的」(取 min 時記下贏家)。算完後從頂樓 n 沿 from[] 一路回溯到起點、反轉,就是最省路徑上踩過的階。額外空間 O(n),時間不變。(這是所有「求最優值 + 要還原方案」DP 的通法:轉移時記前驅,結束回溯。)
App 3. 若「踩上該階就付」,那 dp[i] 定義成「站上第 i 階的最小花費」,轉移變 dp[i]=cost[i]+min(dp[i−1], dp[i−2])(站上 i 一定付 cost[i],而你是從 i−1 或 i−2 上來)。base dp[0]=cost[0]、dp[1]=cost[1]。頂樓不用付費,答案 min(dp[n−1], dp[n−2])(從最後一或兩階跨出去)。付費時機不同,狀態定義與答案位置就得跟著調 —— 這是 DP 最容易搞混、最該想清楚的一步。
