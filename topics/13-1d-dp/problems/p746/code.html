寫法 A · dp 陣列// BOTTOM-UP

p746.cpp C++17 
class Solution {
public:
    int minCostClimbingStairs(vector<int>& cost) {
        int n = cost.size();
        vector<int> dp(n + 1, 0);          // dp[0]=dp[1]=0:從 0 或 1 免費起步

        for (int i = 2; i <= n; i++)
            dp[i] = min(dp[i - 2] + cost[i - 2],   // 從 i-2 上來,付 cost[i-2]
                        dp[i - 1] + cost[i - 1]);  // 從 i-1 上來,付 cost[i-1]

        return dp[n];                        // n = 頂樓(最後一階再上一格)
    }
};

寫法 B · 滾動兩變數// O(1) SPACE

p746_o1.cpp C++17 
class Solution {
public:
    int minCostClimbingStairs(vector<int>& cost) {
        int n = cost.size();
        int prev2 = 0, prev1 = 0;            // dp[i-2]、dp[i-1]
        for (int i = 2; i <= n; i++) {
            int cur = min(prev2 + cost[i - 2], prev1 + cost[i - 1]);
            prev2 = prev1;                      // 往前滾一格
            prev1 = cur;
        }
        return prev1;
    }
};

和 70 一樣,dp[i] 只看前兩項 → 用兩個變數滾動即可,空間 O(1)。差別只在把 + 換成 min(…+cost)。

複雜度分析// COMPLEXITY

T = O(n)　　S = O(n)(A) / O(1)(B)

單一迴圈填 n−1 格,每格一次 min + 兩次加法。A 存整條 dp;B 只留兩個變數。

邊界與陷阱// PITFALLS

陷阱後果正解
回傳 dp[n−1]停在最後一階、沒到頂頂樓是位置 n → dp[n]
轉移加 cost[i]付錯階的費用付來源階:cost[i−1]/cost[i−2]
base 設非 0起步不該有花費dp[0]=dp[1]=0
用貪心挑每步較便宜局部最省 ≠ 全局最省DP 取 min 才對
