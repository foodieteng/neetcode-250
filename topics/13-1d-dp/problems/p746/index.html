最小花費爬樓梯

// CONCEPT · 一維 DP(取 min)

題目簡介// THE QUESTION

QUESTION

cost[i] 是第 i 階的費用。付了該階的費用後可以往上爬 1 或 2 階。 可從第 0 階或第 1 階起步。求爬到頂樓(最後一階再上一格)的最小總花費。

這是 70 爬樓梯 的最優化版:把「數方法數(相加)」換成「求最小花費(取 min)」。 關鍵一問一樣:「到位置 i 的最後一步從哪來?」——從 i−1(付 cost[i−1])或從 i−2(付 cost[i−2]),兩條路取較省的 → dp[i] = min(dp[i−1]+cost[i−1], dp[i−2]+cost[i−2])。

2

為什麼用 DP// WHY DP

A 貪心:每步挑當下較便宜的 會錯
局部最便宜不代表整體最省 —— 一步貪小便宜可能逼你走進後面更貴的路。
× REJECT · 貪心錯
B 純遞迴枚舉所有路徑 O(2ⁿ)
同一位置的最小花費被重算指數多次(重疊子問題)。
× REJECT · TLE
ADOPT
C 自底向上 DP,每格取 min O(n) / O(1)
dp[i] = 到位置 i 的最小花費;由小到大 dp[i]=min(dp[i−1]+cost[i−1], dp[i−2]+cost[i−2]) 填到 dp[n]。 每格只算一次;只依賴前兩項 → 可壓到 O(1) 空間。
● ADOPT · O(n)
3

核心策略 · 三步驟// STRATEGY

  1. 定義狀態:dp[i] = 到達位置 i 的最小花費(i 從 0 到 n,n = 頂樓)。
  2. 轉移:dp[i] = min(dp[i−1]+cost[i−1], dp[i−2]+cost[i−2]) —— 從下一階或下兩階上來,付「離開那階」的費用。
  3. base + 答案:dp[0]=dp[1]=0(從 0 或 1 免費起步);回傳 dp[n]。

兩個關鍵細節// TWO DETAILS

LEMMA · dp[i] 對誰付費 & 頂樓在哪

付費時機:費用是「離開某階時才付」。所以到位置 i,你是從 i−1 或 i−2 跨上來的,要付的是那個來源階的費用 —— 因此是 dp[i−1] + cost[i−1] 或 dp[i−2] + cost[i−2],不是 cost[i]。dp[0]=dp[1]=0 因為「站上起點免費」,費用只在起步往上跨時才產生。

頂樓 = 位置 n:樓梯有 n 階(索引 0..n−1),而「頂」是最後一階再上一格 = 位置 n。所以答案是 dp[n],不是 dp[n−1]。從最後一或倒數第二階都能跨到頂,這正對應 dp[n]=min(dp[n−1]+cost[n−1], dp[n−2]+cost[n−2])。

複雜度// COMPLEXITY

T = O(n)　　S = O(n) → 可優化 O(1)

一個迴圈從 2 填到 n,每格一次 min。只用到前兩項 → 兩變數即可 → O(1) 空間。
