題目簡介// THE QUESTION

QUESTION

給整數陣列 nums,找出乘積最大的連續子陣列,回傳那個乘積。子陣列要連續、非空;單一元素的乘積就是它自己。

像最大子陣列和(Kadane)的乘積版,但乘法有個陷阱:一個負數會把最大翻成最小、最小翻成最大。所以不能只記「目前最大」—— 必須同時記最大與最小,遇到負數時交換,才不會錯過「負×負變大」的機會。

2

C 同時記 maxP 與 minP,遇負數交換 O(n) / O(1)
maxP/minP = 以 i 結尾的最大 / 最小乘積。nums[i]<0 時先 swap;再各自 max/min(prev×num, num)。答案取全程 maxP 的最大。
● ADOPT · O(n)
3

核心策略 · 三步驟// STRATEGY

  1. 初始:ans = maxP = minP = nums[0]。
  2. 遇負先交換:if (nums[i]<0) swap(maxP, minP)(負數把大小關係翻轉)。
  3. 更新 + 記錄:maxP=max(maxP×num, num)、minP=min(minP×num, num)、ans=max(ans, maxP)。

關鍵推導// WHY SWAP ON NEGATIVE

LEMMA · 負數翻轉大小,所以要交換

以 i 結尾的最大乘積,一定是「以 i−1 結尾的某個乘積 × nums[i]」或「nums[i] 自己重新開頭」。乘以正數時,大的還是大;但乘以負數時,最大會變最小、最小會變最大。

所以候選最大值不只來自「上一步的最大」,也可能來自「上一步的最小 × 負數」。維護 minP 就是為了接住這種翻轉。程式上先 swap(maxP, minP) 再用同一條 max/min(prev×num, num) 更新,等價於「對正數不換、對負數換」—— 一次交換把兩種情況統一處理。

6

複雜度// COMPLEXITY

T = O(n)　　S = O(1)

單次線性掃描,每步常數次乘法與比較。只需 maxP、minP、ans 三個變數 → O(1) 空間。

7

一句洞察// INSIGHT

乘積版 Kadane 的關鍵:負數會讓最小變最大,所以同時記 maxP 與 minP、遇負交換。 「最小」不是沒用的垃圾,它是下一個負數的潛在冠軍。
