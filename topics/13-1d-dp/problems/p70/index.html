題目簡介// THE QUESTION

QUESTION

爬一座 n 階的樓梯。每次可以爬 1 階或 2 階。問有幾種不同的方法能爬到頂?

這是一維 DP 的入門題。關鍵一問:「到第 i 階的最後一步是怎麼跨的?」——只可能是從第 i−1 階跨 1 階,或從第 i−2 階跨 2 階。 這兩群走法不重疊、也不遺漏,所以 到 i 的方法數 = 到 i−1 的 + 到 i−2 的 → dp[i] = dp[i−1] + dp[i−2]。正是費氏數列。

2

為什麼用 DP// WHY DP

A 純遞迴 climb(i)=climb(i-1)+climb(i-2) O(2ⁿ)
同一個 climb(k) 被重複算指數多次(重疊子問題),n 大就爆。
× REJECT · TLE
B 遞迴 + 記憶化 O(n)
存每個 climb(k) 避免重算,可行;但有遞迴堆疊、寫法較繞。
○ OK · 可用
ADOPT
C 自底向上填 dp 表 O(n) / O(1)
從 dp[0]=dp[1]=1 開始,由小到大 dp[i]=dp[i−1]+dp[i−2] 一路填到 dp[n]。 每項只算一次,無遞迴。因為只依賴前兩項,還能壓到 O(1) 空間。
● ADOPT · O(n)
3

核心策略 · 三步驟// STRATEGY

  1. 定義狀態:dp[i] = 爬到第 i 階的方法數。
  2. 轉移:dp[i] = dp[i−1] + dp[i−2](最後一步跨 1 或跨 2)。
  3. base + 順序:dp[0]=dp[1]=1;由小到大填到 dp[n],回傳 dp[n]。

為什麼是「相加」而非「相乘」// THE COUNT

LEMMA · 最後一步分類、不重不漏

把「爬到第 i 階的所有走法」依最後一步分成兩類:① 最後一步跨 1 階(前面已站在 i−1);② 最後一步跨 2 階(前面已站在 i−2)。這兩類互斥(最後一步只能是其中一種)且窮盡(沒有第三種跨法),所以總數 = 兩類之和 = dp[i−1] + dp[i−2]。

相加是因為在數「有幾種選擇」(加法原理);若是要走「先做 A 再 做 B」的連續步驟才會相乘。這題最後一步二選一,是分類加總。base dp[0]=1 代表「站在原地不動也算一種」,讓 dp[2]=dp[1]+dp[0]=2 剛好對(兩種:1+1、2)。
