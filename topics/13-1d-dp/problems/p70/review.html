範例 Trace// n = 4

// 填表 dp[0]=1 dp[1]=1 dp[2]=dp[1]+dp[0]=1+1=2 dp[3]=dp[2]+dp[1]=2+1=3 dp[4]=dp[3]+dp[2]=3+2=5 return dp[4] = 5
// 驗證:列出爬 4 階的 5 種走法 1+1+1+1 1+1+2 1+2+1 2+1+1 2+2 共 5 種 ✓(和 dp[4] 一致)
注意 dp[4]=5 剛好等於「到第 3 階(3 種)接著跨 1 階」+「到第 2 階(2 種)接著跨 2 階」。 每種走法都被它的最後一步歸到唯一一類,不重不漏 —— 這就是加法原理在 DP 上的體現。

應用題// APPLY

App 1. 746 Min Cost Climbing Stairs:每階有成本,求爬到頂的最小花費。狀態/轉移怎麼改?(提示:+ 換成 min)
App 2. 198 House Robber:不能偷相鄰兩間,求最大金額。它和爬樓梯的轉移結構像在哪?差在哪?
App 3. 若樓梯上某些階壞掉不能踩(給一個 broken 陣列),轉移要怎麼調整?
// ANSWERS
A1. 把「爬到第 i 階的所有走法」依最後一步分兩類:最後跨 1 階(來自 i−1)、最後跨 2 階(來自 i−2)。這兩類互斥(最後一步只能是其一)又窮盡(沒有別的跨法),所以總數 = 兩類數量相加(加法原理)。相乘是用在「先做完 A 再 做 B」的連續步驟(乘法原理),這裡不是。
A2. dp[0]=1 代表「爬到第 0 階(起點)算一種方法 —— 什麼都不做」。它存在的意義是讓 dp[2]=dp[1]+dp[0]=1+1=2 對得上真實答案(到第 2 階有 1+1、2 兩種)。若把 dp[0] 設成 0,dp[2] 會變成 1,整條表全錯。(另一種等價寫法是 base 設 dp[1]=1, dp[2]=2 從 i=3 起跑,避免用到 dp[0]。)
A3. 純遞迴會把同一個 climb(k) 重複展開:climb(5) 要 climb(4)+climb(3),climb(4) 又要 climb(3)+climb(2)……climb(3) 被算了很多遍,呼叫樹是二元展開 → O(2ⁿ)。DP 靠「每個子問題只算一次、把結果存起來」(記憶化 / 填表)消掉重複,子問題只有 n 個,每個 O(1),總共 O(n)。這就是「重疊子問題」被 DP 解決的典型。
A4. 因為 dp[i] 只依賴最近的兩項 dp[i−1]、dp[i−2],更早的值算完就再也用不到。所以只要留兩個變數 prev1/prev2 一路往前滾,不必存整條陣列,空間 O(n)→O(1)。凡是「當前狀態只依賴前面固定幾項」的 DP 都能這樣滾動壓縮(例如只依賴前一列的二維 DP 可壓成一列)。
A5. 每次能跨 1/2/3 階,則到第 i 階的最後一步有三種來源 → dp[i] = dp[i−1] + dp[i−2] + dp[i−3](這是 Tribonacci,見 1137)。要 3 個 base:dp[0]=1, dp[1]=1, dp[2]=2(或等價地把小 i 特判),因為轉移用到 i−3,得先有前三項才能開始遞推。
App 1. 746 從「數方法數」變成「求最小成本」:狀態 dp[i] = 站上第 i 階的最小花費;轉移把 + 換成 min —— dp[i] = cost[i] + min(dp[i−1], dp[i−2])(站上 i 要付 cost[i],而你是從 i−1 或 i−2 上來的,取較省的)。答案是 min(dp[n−1], dp[n−2])(可從最後一或兩階直接跨出頂)。「相加數方案」→「取 min/max 求最優」是 DP 的兩大分支。
App 2. 198 打家劫舍的轉移是 dp[i] = max(dp[i−1], dp[i−2] + nums[i]) —— 結構和爬樓梯一樣「只看前兩項」,同樣能滾動成 O(1)。差別在:爬樓梯是計數(相加),打家劫舍是最優化(取 max),且「不能偷相鄰」對應到「選了 i 就得從 i−2 轉移(跳過 i−1)」。兩題是同一套「線性 DP、依賴前兩項」的骨架,只是聚合方式不同。
App 3. 對壞掉的階,直接把它的 dp 設成 0(表示「無法站在這階、0 種走法到這」),其餘照常 dp[i]=dp[i−1]+dp[i−2]。這樣壞階的 0 會自然往後傳:任何需要經過壞階的走法都被歸零。若起點或終點本身壞掉,答案就是 0。(這就是把「障礙」用「該格值=0」融進轉移 —— 和 63 Unique Paths II 處理網格障礙同一招。)
