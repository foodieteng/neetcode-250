寫法 A · 硬幣外圈(你的版本)// COIN OUTER

p322.cpp C++17 
class Solution {
public:
    int coinChange(vector<int>& coins, int amount) {
        vector<int> dp(amount + 1, INT_MAX);
        dp[0] = 0;                       // 湊 0 元用 0 枚

        for (int coin : coins) {
            for (int i = coin; i <= amount; i++) {
                if (dp[i - coin] != INT_MAX)   // i-coin 湊得出來才接
                    dp[i] = min(dp[i], dp[i - coin] + 1);
            }
        }

        return dp[amount] == INT_MAX ? -1 : dp[amount];
    }
};

i 從 coin 起由小到大掃(完全背包方向),讓同一種硬幣能被重複使用。dp[i−coin] != INT_MAX 的檢查是必要的:若那格還是 ∞(湊不出),+1 會整數溢位。

寫法 B · 金額外圈(動畫視角)// AMOUNT OUTER

p322_amt.cpp C++17 
class Solution {
public:
    int coinChange(vector<int>& coins, int amount) {
        // amount+1 當作 ∞（湊 amount 最多也只用 amount 枚 1 元）
        vector<int> dp(amount + 1, amount + 1);
        dp[0] = 0;

        for (int i = 1; i <= amount; i++) {
            for (int coin : coins) {
                if (coin <= i)
                    dp[i] = min(dp[i], dp[i - coin] + 1);
            }
        }

        return dp[amount] > amount ? -1 : dp[amount];
    }
};

同一條遞迴、同樣答案,只是迴圈內外對調:對每個金額試每種硬幣。用 amount+1 當 ∞ 哨兵(湊 amount 最多用 amount 枚 1 元,不可能超過),就免去溢位檢查,最後用 > amount 判斷湊不出。這正是動畫演的視角。

複雜度分析// COMPLEXITY

T = O(amount × 種類)　　S = O(amount)

兩種寫法都是雙層迴圈,總計 amount × coins.size() 次更新。amount ≤ 10⁴、種類 ≤ 12。

4

邊界與陷阱// PITFALLS

陷阱後果正解
INT_MAX + 1 溢位變成極小負數 → 亂答先查 dp[i−c]!=INT_MAX,或用 amount+1 當 ∞
用貪心挑大面額[1,3,4] 湊 6 得 3 而非 2DP 對每金額比較所有硬幣
金額由大到小掃變成 0/1 背包(每幣一次)完全背包要小到大掃
忘記 dp[0]=0全部湊不出 → 全 ∞base dp[0]=0 是起點

註:面額可大到 2³¹−1,但因 i 從 coin 起(或 coin ≤ i 才處理),超過 amount 的硬幣自然被跳過,不會越界。
