範例 Trace// coins={1,2,5}, amount=11

// 填表(答案應為 3) dp[0]=0 dp[1]=1 dp[2]=1 dp[3]=2 dp[4]=2 dp[5]=1 (5) dp[6]=2 (5+1) dp[7]=2 (5+2) dp[8]=3 dp[9]=3 dp[10]=2 (5+5) dp[11]=min(dp[10]+1, dp[9]+1, dp[6]+1)=min(3,4,3)=3 (5+5+1) return dp[11] = 3
dp[11] 有兩條都是 3 枚的路:dp[10]+1(5+5 再加 1)與 dp[6]+1(5+1 再加 5)。 min 不在乎是哪一條,只要件數最少 —— 這也是為什麼 DP 只存「最少幾枚」而不存「具體是哪些硬幣」。

應用題// APPLY

App 1. 518 零錢兌換 II:改求「湊出金額的組合方法數」。轉移從 min(...)+1 換成什麼?為什麼這時硬幣必須放外圈?
App 2. 279 完全平方數:求最少幾個平方數湊出 n。它其實就是這題的什麼特例?coins 換成什麼?
App 3. 若每種硬幣只有一枚(0/1 背包),程式要改哪一行?為什麼那個改動就能讓「每種只用一次」?
// ANSWERS
A1. 貪心假設「先拿大的一定不虧」,但面額不一定相容。反例 coins=[1,3,4]、amount=6:貪心先拿 4,剩 2 只能 1+1,共 3 枚;最優是 3+3 = 2 枚。大面額佔走額度後,剩下的湊法反而更差。DP 對每個金額比較所有硬幣的結果取 min,才不會被貪心的短視害到。
A2. 因為硬幣無限量(完全背包)。由小到大掃時,算 dp[i] 用到的 dp[i−coin] 是本輪已更新的值 —— 裡面可能已經放過同一種 coin,於是這枚 coin 疊上去就等於「同種再用一次」。若由大到小掃,dp[i−coin] 還是上一輪的舊值(尚未放過本種硬幣),每種硬幣就只會被算進去一次,那是 0/1 背包的語意。方向決定「能不能重複取」。
A3. 拿掉會整數溢位。若 dp[i−coin] 還是 INT_MAX(那個金額湊不出),dp[i−coin] + 1 = INT_MAX + 1 溢位成極小的負數,min 會誤選它,把 dp[i] 汙染成一個假的「很小」值,答案全錯。所以要嘛先檢查 != INT_MAX,要嘛改用 amount+1 當哨兵(+1 不會溢位),兩種都能避開。
A4. 對最少件數(min)而言,兩種順序答案一樣。因為 dp[i] 最終取的是「所有硬幣、所有路徑」的最小值,而 min 滿足交換律與結合律 —— 不管先枚舉硬幣還是先枚舉金額,只要每個 (i, coin) 組合都被處理過、且 dp[i−coin] 在用到前已為當前最優,最小值就一樣。硬幣外圈時,每種硬幣把整條金額軸掃一遍;金額外圈時,每個金額把所有硬幣試一遍。兩者覆蓋的更新集合相同。
A5. 差在聚合運算:最少件數用 dp[i]=min(dp[i], dp[i−c]+1)(找最省);方法數(518)用 dp[i]+=dp[i−c](把各種湊法累加)。對計數版,順序變得敏感:硬幣必須放外圈、金額內圈,才是算「組合數」(不計順序,如 1+2 與 2+1 算同一種);若金額外圈、硬幣內圈,會把不同順序也算進去,變成「排列數」(那是 377 組合總和 IV)。min 版因為只取極值、與計數方式無關,才對順序免疫。
App 1. 轉移改成累加:dp[i] += dp[i−coin],base dp[0]=1(湊 0 有 1 種空組合)。硬幣必須放外圈、金額內圈:這樣每種硬幣「整批」被考慮一次,枚舉的是無序組合(先決定用多少個 coin₁、再 coin₂…),避免把 1+2 和 2+1 當兩種。若順序反過來就會重複計數成排列。這是計數型完全背包對迴圈順序敏感的經典陷阱。
App 2. 279 就是本題把 coins 換成所有 ≤ n 的完全平方數(1, 4, 9, 16, …)、amount = n 的特例,一樣求「最少幾個湊出 n」。轉移完全相同 dp[i]=min(dp[i], dp[i−sq]+1)。差別只在硬幣集合是動態產生的平方數,而非題目給的陣列 —— 骨架一字不改。
App 3. 把「金額由小到大」改成「金額由大到小」掃(內層 for i = amount down to coin),就變成 0/1 背包、每種硬幣只用一次。原因如 A2:由大到小時 dp[i−coin] 取到的是尚未放入本種硬幣的舊值,所以這枚 coin 至多被算進一次。只改迴圈方向、不動轉移式,就能在「無限量 ↔ 各一枚」之間切換 —— 這是背包家族最該記住的一個開關。
