題目簡介// THE QUESTION

QUESTION

給一組硬幣面額 coins(每種無限量供應)和目標金額 amount,回傳湊出該金額所需的最少硬幣數。湊不出來回傳 -1。

這是完全背包的經典款:每種硬幣可重複用,求「最少件數湊到目標」。用一維 DP:dp[i] = 湊出金額 i 的最少硬幣數,對每種硬幣 c 試 dp[i] = min(dp[i], dp[i−c] + 1)。

2

為什麼不能貪心// WHY DP

A 貪心:每次拿最大面額 O(n)
coins=[1,3,4]、amount=6:貪心 4+1+1=3 枚,最優卻是 3+3=2 枚。面額不「整除相容」時貪心會錯。
× REJECT · 反例
B 純遞迴列舉所有組合 指數
每步選任一硬幣,分支爆炸;大量重疊子問題(湊同一金額被重算)。
△ REJECT · TLE
ADOPT
C 完全背包一維 DP:湊每個金額取 min O(amount × 種類)
dp[i] = min over c ( dp[i−c] + 1 )。從小金額往大填,每個金額嘗試每種硬幣、取最省。硬幣可重複 → 接的是同層的 dp[i−c]。
● ADOPT · O(n·A)
3

核心策略 · 三步驟// STRATEGY

  1. 初始化:dp[0]=0(湊 0 不需硬幣),其餘設 ∞(尚無法湊)。
  2. 轉移:對每個金額 i、每種硬幣 c ≤ i,dp[i] = min(dp[i], dp[i−c] + 1)。
  3. 回傳:dp[amount] 仍是 ∞ → 回傳 -1,否則回傳它。

關鍵推導// WHY dp[i-c], NOT A LAYER

LEMMA · 完全背包 vs 0/1 背包的一線之差

「最少硬幣湊 i」的最優解裡,一定至少用了某一種硬幣 c 當最後一枚。拿掉這枚,剩下就是「湊 i−c 的最優解」,所以 dp[i] = dp[i−c] + 1,對所有可行 c 取 min。

關鍵在 dp[i−c] 是同一輪、已更新過的值 —— 因為硬幣無限量,放完一枚 c 後還能再放 c。這正是完全背包(金額由小到大掃)和 0/1 背包(每件僅一次、金額須由大到小掃)的分水嶺:方向決定能否重複取。

6

複雜度// COMPLEXITY

T = O(amount × 硬幣種類)　　S = O(amount)

雙層迴圈:每個金額 × 每種硬幣。amount ≤ 10⁴、種類 ≤ 12 → 至多約 1.2×10⁵ 次,很快。空間為 dp 陣列 O(amount)。

一句洞察// INSIGHT

完全背包最少件數:dp[i] = min over 硬幣 c 的 (dp[i−c] + 1)。 硬幣無限量 → 金額由小到大掃、接同層 dp[i−c] 就自然允許重複取;湊不到留 ∞ → 回傳 -1。
