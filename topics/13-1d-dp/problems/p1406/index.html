題目簡介// THE QUESTION

QUESTION

一排石頭,各有分數。Alice 先手,兩人輪流從最前面拿 1、2 或 3 顆,分數累加。雙方都最優下,回傳 "Alice" / "Bob" / "Tie"。

這是賽局 DP 的代表題。核心技巧:不分別記兩人分數,只記「相對分差」 —— dp[i] = 輪到我、從第 i 顆起,我能領先對手的最大差值。這招把兩人的 minimax 壓成一個一維陣列。

2

相關練習 · 同一招// PRACTICE

「相對分差」賽局 DP 三連發

下面兩題和本題是同一個心法(dp 存「當前玩家減對手」的最優差值,遞迴時減去對手的最優),很適合連著練:

● LC 486 · Predict the Winner ↗
從兩端任拿一顆。dp[i][j] = 面對區間 [i,j] 的最大分差 = max(a[i]−dp[i+1][j], a[j]−dp[i][j−1])。二維區間版的同一招。

● LC 1690 · Stone Game VII ↗
也是兩端拿,但得分是「拿走後剩餘石頭的總和」。dp[i][j] = max(sum(i+1,j)−dp[i+1][j], sum(i,j−1)−dp[i][j−1]),配前綴和。相對分差 + 區間 DP。

三題共通:把「兩人各自最優」化成「一個人的最大分差」,對手的貢獻用『減』帶入。差別只在「怎麼拿」(前面 1–3 顆 / 兩端一顆)與「得分定義」。

3

核心 · 為什麼「減」dp[next]// THE MINUS

LEMMA · 換人 = 對手的領先要扣回來

dp[i] = 「輪到我、從 i 起」我能領先的最大差值。我拿走 i..i+j(和 take)後,換對手從 i+j+1 開始 —— 這時「我」變成他,他能領先的最大差值就是 dp[i+j+1]。

那個 dp[i+j+1] 是對手相對於我的領先,對我來說是扣分。所以我這步的淨分差 =

dp[i] = maxj∈{0,1,2} ( take(i..i+j) − dp[i+j+1] )

「減」就是 minimax 的精髓:我最大化「我的所得 − 對手之後能拉開的差」。base dp[n]=0(沒石頭,平)。由後往前填,最後 dp[0] 就是 Alice 相對 Bob 的最終分差:>0 Alice、<0 Bob、=0 Tie。

4

核心策略 · 三步驟// STRATEGY

  1. 狀態:dp[i] = 從第 i 顆起、當前玩家能領先的最大分差;dp[n]=0。
  2. 由後往前:i 從 n−1 到 0;累加 take,試拿 1/2/3 顆。
  3. 轉移:dp[i] = max(dp[i], take − dp[i+j+1]);看 dp[0] 判勝負。

複雜度// COMPLEXITY

T = O(n)　　S = O(n) → 可優化 O(1)

每個 i 只試 3 種拿法 → 常數 → 總 O(n)。dp[i] 只依賴 dp[i+1..i+3] → 留 3 個變數即可 O(1) 空間。n ≤ 5×10⁴。

一句洞察// INSIGHT

賽局 DP 的萬用招:只記「相對分差」,對手的最優用『減』帶入。 dp[i] = max(take − dp[next]) —— 一個陣列搞定兩人 minimax。486、1690 同招。
