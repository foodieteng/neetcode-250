觀念題// QUIZ

Q1. 為什麼轉移是 take − dp[next] 而不是 take + dp[next]?那個「減」在賽局裡代表什麼?
Q2. 為什麼只記「相對分差」一個量就夠,不必分別記 Alice、Bob 各自的分數?
Q3. 為什麼要由後往前填?由前往後會出什麼問題?
Q4. 石頭值可以是負的。為什麼「每次盡量拿 3 顆」是錯的貪心?max(take − dp[next]) 怎麼自動處理?
Q5. 這題和 486 Predict the Winner 都用「相對分差」,但一個是一維、一個是二維。差在題目的什麼設定?

範例 Trace// [1,2,3,-9] → Alice

// 填表(答案應為 Alice) dp[4]=0 dp[3]= -9 − dp[4] = -9 dp[2]= max(3−dp[3]=12, (3+(-9))−dp[4]=-6) = 12 dp[1]= max(2−dp[2]=-10, 5−dp[3]=14, (2+3-9)−dp[4]=-4) = 14 dp[0]= max(1−dp[1]=-13, 3−dp[2]=-9, 6−dp[3]=15) = 15 dp[0] = 15 > 0 → "Alice"
dp[0] 選 j=2:Alice 拿 1+2+3=6,把那顆 −9 留給 Bob。6 − dp[3] = 6 − (−9) = 15 —— 「減一個負數」= 加,正好反映「把爛石頭逼給對手,對手反而扣分」。這就是相對分差的威力。

應用題// APPLY

App 1. 486 Predict the Winner:改成從兩端任拿一顆。寫出它的二維轉移 dp[i][j],並說明「減」出現在哪。
App 2. 1690 Stone Game VII:兩端拿,但得分是「拿走後剩餘石頭的總和」。轉移要怎麼配前綴和?為什麼還是相對分差?
App 3. 若把「拿 1/2/3 顆」改成「拿 1..K 顆」,或石頭排成環,DP 要怎麼調整?複雜度變多少?
// ANSWERS
A1. 因為拿完這步就換對手。dp[next] 的定義是「輪到下一位玩家時,他能領先他的對手(也就是現在的我)的最大分差」。那是對手相對於我的領先 —— 對我而言是負貢獻,所以要減。這一減就把「換人」這件事編碼進去了:我最大化「我拿到的 take 減去對手接下來能把差距拉開多少」,正是 minimax(我 max、對手也 max 他自己 = 對我 min)。
A2. 因為勝負只看兩人分數的差,不看絕對值。與其記 (Alice分, Bob分) 兩個量、狀態翻倍,不如直接記「當前玩家 − 對手」這一個差。由於每步都是「當前玩家」在決策,而角色會交替,用同一個 dp[i](永遠站在「輪到誰就是誰」的視角)配上「減」來換視角,就能一路遞推。最後 dp[0] 是 Alice(第一個當前玩家)相對 Bob 的差。相對量把二維(兩人分數)壓成一維。
A3. 因為 dp[i] 依賴更靠後的 dp[i+1], dp[i+2], dp[i+3](拿完後對手從那裡開始)。要先把靠後的算好,才能算靠前的,所以由後往前(i = n−1 … 0)。若由前往後,算 dp[i] 時 dp[i+j+1] 還是初始值(INT_MIN / 未定),整個遞推塌掉。base dp[n]=0 是最靠後的錨點。
A4. 因為石頭可負,多拿不一定好 —— 拿到一堆負石頭反而扣自己分,或把好石頭留給對手。貪心「每次拿最多」會踩雷。max(take − dp[next]) 對每個 i 把「拿 1 / 2 / 3 顆」三種都算一遍,自動比較「我這步拿多少」與「留下的局面讓對手能領先多少 dp[next]」,選淨分差最大的。有時 take 小但 dp[next] 更負(把爛攤子丟給對手),那支反而勝出 —— DP 自然涵蓋,不需要人為判斷。
A5. 差在「拿的位置」決定剩下的是不是一段連續前綴。1406 只從最前面拿 1–3 顆,剩下的永遠是「從某個 i 到尾」的後綴 → 用一維 dp[i] 就夠。486 從兩端任拿,剩下的是「區間 [i,j]」,兩端都會變 → 得用二維 dp[i][j]。核心「相對分差 + 減對手」完全一樣,只是狀態維度隨「剩餘局面的自由度」從一維升成二維。
App 1. dp[i][j] = 面對區間 [i,j],當前玩家能領先的最大分差。轉移:dp[i][j] = max( a[i] − dp[i+1][j], a[j] − dp[i][j−1] ) —— 拿左端 a[i]、剩 [i+1,j] 換對手(減 dp[i+1][j]);或拿右端 a[j]、剩 [i,j−1](減 dp[i][j−1])。「減」一樣出現在「換對手後他的最優」。base dp[i][i]=a[i]。答案 dp[0][n−1] ≥ 0 → 先手不敗。
App 2. 1690 的得分是「拿走一顆後,剩餘石頭的總和」。用前綴和 pre,區間 [i,j] 的總和 S(i,j)=pre[j+1]−pre[i]。拿左端 a[i] 後剩 [i+1,j],得分 = S(i+1,j);拿右端得分 = S(i,j−1)。轉移:dp[i][j] = max( S(i+1,j) − dp[i+1][j], S(i,j−1) − dp[i][j−1] )。仍是相對分差 + 減對手,只是「得分」從「拿走那顆的值」換成「剩餘和」,靠前綴和 O(1) 取得。答案 dp[0][n−1](這裡題目要最大化分差本身)。
App 3. 「拿 1..K 顆」:內層 j 從 0 跑到 K−1,轉移不變,複雜度變 O(nK)(K 大時可用單調隊列/前綴優化降回近 O(n))。環形:因為首尾相接,得像 213 打家劫舍 II 那樣拆成幾個線性子問題(固定「第一段從哪切」)分別算再合併,或在狀態多帶「起點」資訊;通常環形賽局要小心「誰先誰後」與邊界,複雜度乘上枚舉切點的因子。核心的「相對分差」框架不變,變的是狀態要不要多一維來描述環的耦合。
