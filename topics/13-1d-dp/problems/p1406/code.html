寫法 A · 相對分差 DP(你的版本)// SCORE DIFF

p1406.cpp C++17 
class Solution {
public:
    string stoneGameIII(vector<int>& stoneValue) {
        int n = stoneValue.size();
        vector<int> dp(n + 1, INT_MIN);
        dp[n] = 0;                        // base:沒石頭 → 分差 0

        for (int i = n - 1; i >= 0; i--) {      // 由後往前
            int take = 0;
            for (int j = 0; j < 3 && i + j < n; j++) { // 拿 1/2/3 顆
                take += stoneValue[i + j];
                dp[i] = max(dp[i], take - dp[i + j + 1]); // 減對手最優
            }
        }

        if (dp[0] > 0) return "Alice";
        if (dp[0] < 0) return "Bob";
        return "Tie";
    }
};

take − dp[i+j+1] 是全題靈魂:take 是我這步拿到的,dp[i+j+1] 是換對手後他的最大領先 —— 對我是扣分,所以減。dp 存的是「當前玩家減對手」的相對分差,一個陣列就涵蓋兩人的 minimax,不必分開記兩人分數。

寫法 B · 滾動三變數// O(1) SPACE

p1406_o1.cpp C++17 
class Solution {
public:
    string stoneGameIII(vector<int>& s) {
        int n = s.size();
        int dp1 = 0, dp2 = 0, dp3 = 0;       // dp[i+1]、dp[i+2]、dp[i+3](界外=0)

        for (int i = n - 1; i >= 0; i--) {
            int take = 0, best = INT_MIN;
            int nxt[3] = {dp1, dp2, dp3};
            for (int j = 0; j < 3 && i + j < n; j++) {
                take += s[i + j];
                best = max(best, take - nxt[j]);
            }
            dp3 = dp2; dp2 = dp1; dp1 = best;       // 視窗往前滑
        }

        int d = dp1;                            // dp[0]
        return d > 0 ? "Alice" : d < 0 ? "Bob" : "Tie";
    }
};

dp[i] 只用到 dp[i+1..i+3] → 三個變數滾動就夠,空間 O(1)。界外(i≥n)視為 0,對應 base dp[n]=0。算完 best 後把視窗往前滑一格。

複雜度分析// COMPLEXITY

T = O(n)　　S = O(n)(A) / O(1)(B)

每個 i 常數(至多 3)次運算 → O(n)。A 存整條 dp;B 只留三個變數。n ≤ 5×10⁴。

4

邊界與陷阱// PITFALLS

陷阱後果正解
take + dp[next](加)沒換人視角 → 全錯要減對手最優 take − dp[next]
分別記兩人分數狀態爆炸、難寫只記「相對分差」一個量
由前往後填dp[i+j+1] 尚未算由後往前(先算靠後的)
忘記負分石頭以為多拿一定好take 可能為負,靠 max 自動避開

註:石頭值可負(−1000..1000),所以「拿越多越好」不成立 —— 有時只拿 1 顆、把爛石頭留給對手更優。max(take − dp[next]) 已涵蓋這種盤算。
