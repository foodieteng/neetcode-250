寫法 A · 完全背包 DP// int 就夠

p279.cpp C++17 
class Solution {
public:
    int numSquares(int n) {
        vector<int> dp(n + 1, INT_MAX);   // min-DP 不會溢位 → int 即可
        dp[0] = 0;

        for (int i = 1; i <= n; i++) {
            for (int j = 1; j * j <= i; j++) {   // 平方數當「硬幣」,可重複
                int k = j * j;
                if (dp[i - k] != INT_MAX)       // 防 INT_MAX + 1 溢位
                    dp[i] = min(dp[i], dp[i - k] + 1);
            }
        }

        return dp[n];
    }
};

和 322 幾乎一字不差,只是「硬幣」是 j*j。型別:這裡 int 就夠 —— 因為是 min(不像 377 的加法會累積放大),dp[i] 永遠很小(≤ 4)。你原碼用 unsigned int 無害但多餘。留 dp[i−k] != INT_MAX 這個守衛是對的:避免對 ∞ 做 +1。

寫法 B · BFS 最短路// SHORTEST PATH

p279_bfs.cpp C++17 
class Solution {
public:
    int numSquares(int n) {
        if (n == 0) return 0;
        vector<char> visited(n + 1, 0);
        queue<int> q; q.push(0); visited[0] = 1;
        int level = 0;

        while (!q.empty()) {
            level++;                                 // 每一層 = 多用一個平方數
            for (int sz = q.size(); sz > 0; sz--) {
                int cur = q.front(); q.pop();
                for (int j = 1; j * j <= n; j++) {
                    int nx = cur + j * j;
                    if (nx == n) return level;          // 第一次碰到 n = 最短
                    if (nx < n && !visited[nx]) {
                        visited[nx] = 1; q.push(nx);
                    }
                }
            }
        }
        return 0;
    }
};

另一個視角:把 0…n 當節點,「加一個平方數」當邊,求 0 → n 的最短路 = 最少平方數個數。BFS 一層一層擴,第一次碰到 n 就是答案。visited 避免重複入隊。和 DP 同複雜度,但常因 Lagrange(答案 ≤ 4)只擴幾層就結束、實務很快。

複雜度分析// COMPLEXITY

T = O(n√n)　　S = O(n)

DP:每個 i 試 √i 個平方數。BFS:節點 n、每點 √n 條邊,同級但因答案 ≤ 4 通常提早結束。
