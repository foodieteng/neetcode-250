範例 Trace// n = 13

// 填表(答案應為 2) 平方硬幣 = {1,4,9} dp[0]=0 dp[4]=1(4) dp[9]=1(9) dp[13]=min(dp[12]+1, dp[9]+1, dp[4]+1) =min(3+1, 1+1, 1+1)=2 (4+9 或 9+4) return dp[13] = 2
13 = 4 + 9 剛好兩個平方數。dp[13] 靠 dp[9]+1(在 9 上加一個 4)或 dp[4]+1(在 4 上加一個 9)得到 —— 兩條路都是 2。 這也印證 Lagrange:任何數至多 4 個平方數,13 只要 2 個。

應用題// APPLY

App 1. 已知答案恆 ∈ {1,2,3,4},能不能不用 DP、用數學 O(√n) 直接判斷?(提示:先判 1 個、再判 2 個,再用 Legendre 三平方和定理判 4 個。)
App 2. 若把「平方數」換成「立方數」或任意給定集合,DP 要改哪裡?Lagrange 那個「≤ 4」還成立嗎?
App 3. 若 n 到 10⁹,O(n√n) 的 DP 記憶體/時間都不行。這時只剩哪條路可走?為什麼數學解在這裡變成唯一選擇?
// ANSWERS
A1. 「硬幣」= 完全平方數 {1,4,9,16,…}(≤ n);「金額」= n;「最少枚數」= 最少平方數個數。轉移 dp[i]=min(dp[i], dp[i−coin]+1) 完全照搬,只是 coin 由 j*j 動態生成。所以會了 322 就等於會了 279。
A2. 關鍵在聚合運算。377 用加法計數:dp[i]+=dp[i−num],值會不斷累積放大,排列數指數成長,中途就破 2³¹ → 需要 unsigned/long long。279 用 min 求極值:dp[i]=min(…, dp[i−k]+1),值只會取小、不會累積,上界就是 i(全用 1),實際 ≤ 4。永遠遠小於 INT_MAX,int 綽綽有餘。加法會爆、min 不爆,這就是型別選擇的分水嶺。
A3. 防 INT_MAX + 1 的有號溢位。dp 用 INT_MAX 當「還湊不出來(∞)」的哨兵;若 dp[i−k] 還是 INT_MAX 就直接 +1,會溢位成極小負數,再被 min 誤選,把 dp[i] 汙染成假的小值 → 全錯。加上守衛「只有 dp[i−k] 不是 ∞ 才更新」就安全。(或改用 n+1 當 ∞ 哨兵,+1 不會溢位,就不必守衛。)
A4. 因為聚合是 min,而 min 對枚舉順序免疫(交換律、結合律)。不管「金額外層、平方數內層」還是「平方數外層、金額內層」,每個 dp[i] 最終都收斂到同一個最小值。這和 377(計數,順序決定排列 vs 組合)正好相反 —— 求極值不在乎順序,做計數才在乎。所以 279 怎麼寫迴圈都對,377/518 則必須擺對外層。
A5. 在找同一個東西:最少平方數個數。BFS 把 0…n 當節點、「加一個平方數」當權重 1 的邊,求 0→n 的最短路。BFS 一層層擴,層數 = 已用的平方數個數;因為所有邊權相同,BFS 第一次觸及 n 時的層數必是最短(不可能有更少層的路徑先到)。所以第一次碰到就能回傳。DP 是同一最短路問題的「填表版」,兩者答案一致。
App 1. 可以,O(√n) 純數學:① 若 n 本身是完全平方數 → 1。② 由 Legendre 三平方和定理:n 不能寫成 3 個平方數之和 ⟺ n = 4ᵃ(8b+7);先把 n 除盡 4,若餘數 mod 8 == 7 → 答案是 4。③ 否則試 n = a² + b²(掃 a 到 √n 看剩下是不是平方)→ 成立則 2。④ 都不是 → 3。四步都是 O(√n),不需 DP 陣列。
App 2. DP 只要把內層的「硬幣集合」換掉:立方數就是 for j: k=j*j*j,任意集合就照 322 給的陣列。轉移式一字不改。但 Lagrange 的「≤ 4」只對平方數成立 —— 那是數論特例(四平方和定理)。換成立方數,答案上界是別的(華林問題:立方數需至多 9 個);任意集合則可能無解(如硬幣都比 n 大或湊不出)。所以 DP 通用,但「答案有小上界」這種捷徑是平方數專屬。
App 3. 只剩數學解(App 1 的 O(√n))。n=10⁹ 時 O(n√n) 的 DP 需要約 10⁹ 大小的陣列(記憶體爆)且 ~3×10¹³ 次運算(超時),完全不可行;BFS 同樣要 O(n) 空間也不行。而 Legendre/Lagrange 的判斷只需 O(√n) ≈ 3×10⁴ 步、O(1) 空間。當 n 巨大、又剛好有數論結構可用時,數學解不是「更快」而是「唯一可行」 —— 這也是為什麼值得認識這題背後的定理。
