題目簡介// THE QUESTION

QUESTION

給整數 n,回傳「和為 n 的完全平方數」的最少個數。完全平方數 = 1, 4, 9, 16, 25…(可重複用)。

這題根本就是 322 零錢兌換:把硬幣集合換成「≤ n 的平方數」 {1,4,9,16,…},求「湊出 n 的最少個數」。同一套完全背包 min,幾乎免費。

2

就是 322,換平方硬幣// SAME AS COIN CHANGE

對照 · 一模一樣的完全背包 min

狀態:dp[i] = 湊出 i 的最少平方數個數。dp[0]=0,其餘 ∞。
轉移:對每個平方數 k=j²(k≤i),dp[i] = min(dp[i], dp[i−k] + 1)。

和 322 唯一的差別是「硬幣」不是題目給的陣列,而是動態產生的平方數(for j: k=j*j)。因為是完全背包(平方數可重複用),金額由小到大掃;因為是 min,迴圈順序免疫(見 377 的排列/組合對比 —— 那是計數才敏感,求極值不敏感)。

3

這裡 unsigned「不需要」// vs 377

你的碼用了 vector<unsigned int>。但和 377 不同 —— 這題其實不需要 unsigned,普通 int 就夠:

題聚合值會不會爆型別
377加法(計數)會 —— 排列數指數成長,中途 > 2³¹需 unsigned / long long
279(本題)min(求極值)不會 —— 答案 ≤ 4,dp[i] 永遠很小int 就夠

關鍵:加法會累積放大 → 可能溢位;min 只會取小 → 永遠不爆。這題 dp[i] 最多就是 i(全用 1),而由 Lagrange 定理實際 ≤ 4。所以 unsigned 是無害但多餘的習慣;寫 int 更乾淨。用 INT_MAX 當 ∞ 時,記得像你一樣加 dp[i−k] != INT_MAX 的守衛避免 +1 溢位即可。

4

核心策略 · 三步驟// STRATEGY

  1. 初始化:dp[0]=0,其餘 ∞(或 INT_MAX + 守衛)。
  2. 轉移:i 從 1 到 n;內層 j 從 1、j*j≤i,dp[i]=min(dp[i], dp[i−j*j]+1)。
  3. 回傳:dp[n](必有解 —— Lagrange 保證 ≤ 4)。

複雜度 + 數學捷徑// LAGRANGE

DP:T = O(n√n)　　S = O(n)

每個 i 試約 √i 個平方數 → O(n√n)。數學捷徑:Lagrange 四平方和定理保證答案 ∈ {1,2,3,4};用 Legendre 三平方和定理可 O(√n) 直接判斷(見複習頁)。
