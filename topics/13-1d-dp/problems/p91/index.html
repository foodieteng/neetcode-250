題目簡介// THE QUESTION

QUESTION

'A'→"1"、'B'→"2"、…、'Z'→"26"。給一串數字 s,問有幾種合法的解碼方式。每個位置可當單獨一位(1–9)或與下一位合成兩位(10–26);前導 0 不合法。無法解碼回傳 0。

這是 70 爬樓梯 的帶條件版:每一步的方法數 = 「切 1 位」的方法數 + 「切 2 位」的方法數 —— 但兩條路各自要先通過合法碼檢查(1 位不能是 0;2 位要落在 10–26)。

2

為什麼用 DP// WHY DP

A 純遞迴:每步試切 1 位 / 2 位 O(2ⁿ)
和爬樓梯一樣有大量重疊子問題,指數爆炸。
× REJECT · TLE
B 遞迴 + 記憶化(由後往前) O(n)
存每個起點的解碼數,可行;但有遞迴堆疊、常數較大。
○ OK · 可用
ADOPT
C 自底向上一維 DP(帶條件相加) O(n) / O(1)
dp[i] = 前 i 個字元的解碼數。單碼合法 → +dp[i−1];雙碼合法 → +dp[i−2]。只依賴前兩項 → 可 O(1) 空間。
● ADOPT · O(n)
3

核心策略 · 三步驟// STRATEGY

  1. 特判 + base:首字 s[0]=='0' → 回傳 0;否則 dp[0]=dp[1]=1。
  2. 單碼:若 s[i−1] != '0',dp[i] += dp[i−1]。
  3. 雙碼:若 "10" ≤ s[i−2..i−1] ≤ "26",dp[i] += dp[i−2];回傳 dp[n]。

關鍵推導// WHY ADD TWO EDGES

LEMMA · 最後一碼二選一,方法數相加

考慮 s[0..i−1] 的最後一個字母是怎麼來的,只有兩種互斥方式:

① 最後一碼是單一位 s[i−1](需 ≠ 0):那前面 s[0..i−2] 隨便怎麼解都行 → dp[i−1] 種。
② 最後一碼是兩位 s[i−2..i−1](需 10–26):那前面 s[0..i−3] 隨便解 → dp[i−2] 種。

兩種不重疊、合起來窮盡,所以 dp[i] = 合法時的 dp[i−1] + 合法時的 dp[i−2]。這和爬樓梯「最後一步跨 1 或跨 2」同構,差別只在每條邊多了一道合法性閘門:閘門不過,那一項就不加。

6

複雜度// COMPLEXITY

T = O(n)　　S = O(n) → 可優化 O(1)

單一迴圈掃過每個字元,每格常數次比較。只用到前兩項 → 兩個變數 → O(1) 空間。

7

一句洞察// INSIGHT

解碼方法 = 爬樓梯 + 合法性閘門:單碼(≠0)加 dp[i−1]、雙碼(10–26)加 dp[i−2]。 同一套「依賴前兩項的計數 DP」,只是每條邊要先過檢查 —— 大量 0 或超過 26 的兩位,會把方法數砍成 0。

// ONE-LINE TAKEAWAY
dp[i] = (單碼≠0 ? dp[i−1]:0) + (雙碼 10–26 ? dp[i−2]:0)。爬樓梯 + 合法閘門。O(n)、可 O(1)。
