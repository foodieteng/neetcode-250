面向要點
狀態dp[i] = 前 i 個字元的解碼方法數
單碼s[i−1] != '0' → dp[i] += dp[i−1]
雙碼"10" ≤ s[i−2..i−1] ≤ "26" → dp[i] += dp[i−2]
basedp[0]=1、dp[1]=1;首字 0 直接回 0
無解兩邊皆不合法 → dp[i]=0,帶到 dp[n]

Q1. 為什麼 dp[i] 是把 dp[i−1] 和 dp[i−2] 相加(而非相乘或取 max)?兩者各代表什麼?
Q2. 單碼那條邊為什麼要排除 '0'?雙碼那條邊為什麼下界是 10 而不是 00?
Q3. 用字串比較 w >= "10" && w <= "26" 判雙碼合法,為什麼會等價於數值 10 ≤ 值 ≤ 26?
Q4. 當某個位置兩條邊都不合法(如 "100" 最後的 0),程式怎麼自動回傳 0,而不需要特別偵測「無解」?
Q5. 這題和 70 爬樓梯 的轉移長得幾乎一樣,唯一差別是什麼?這說明了 1-D 計數 DP 的什麼共性?

範例 Trace// s = "11106"

// 填表(答案應為 2) s = 1 1 1 0 6 dp[0]=1 dp[1]=1 i=2: '1'≠0 +dp[1]=1 ; "11"∈[10,26] +dp[0]=1 → dp[2]=2 i=3: '1'≠0 +dp[2]=2 ; "11"∈[10,26] +dp[1]=1 → dp[3]=3 i=4: '0' 單碼跳過 ; "10"∈[10,26] +dp[2]=2 → dp[4]=2 i=5: '6'≠0 +dp[4]=2 ; "06" 前導零，不在[10,26] 跳過 → dp[5]=2 return dp[5] = 2 → "1 1 10 6"=AAJF、"11 10 6"=KJF
注意 i=4 的 '0':它只能當雙碼 "10" 的一部分(單碼被擋),把 dp[3]=3 的路徑砍掉、只留 dp[2]=2。 一個 0 就像一道窄門:它強迫前面那位必須是 1 或 2 與它結對,否則整條路徑作廢。

應用題// APPLY

App 1. 639 解碼方法 II:字串可含萬用字元 '*'(代表 1–9)。轉移會怎麼變複雜?'*' 單碼、'*' 雙碼各有幾種?
App 2. 若把映射改成 'A'→"1" … '?'→"63"(碼可到三位,1–63),轉移式要怎麼加一條邊?依賴窗口變多大?
App 3. 若不要方法數,而要列出所有解碼字串,還適合用這個計數 DP 嗎?該換成什麼手法?複雜度差在哪?
// ANSWERS
A1. 因為「s[0..i−1] 的最後一碼」只有兩種互斥取法:最後切 1 位(前面貢獻 dp[i−1] 種),或最後切 2 位(前面貢獻 dp[i−2] 種)。兩群方案不重疊、合起來窮盡所有解法,所以相加。相乘會把「獨立選擇」的語意搞錯(這裡不是兩件事都做),取 max 則是最佳化題才用 —— 這題要「數總數」,故用加法。
A2. 單碼排除 '0':因為沒有字母對應 "0"(映射從 "1" 開始),單獨一個 0 無法解碼。雙碼下界是 10:兩位碼必須是 10–26,"00"…"09" 有前導零,代表第一位是 0,既不是合法兩位碼、第一位也不能單獨成碼,所以整個作廢。把下界設 10 剛好一次擋掉所有前導零情形。
A3. 因為 w 是剛好兩個字元的數字字串,長度固定。定長數字字串的字典序與數值大小完全一致(逐位比較,高位優先,和數值比大小同理)。所以 "10" <= w <= "26" 等價於 10 ≤ 數值 ≤ 26。若長度不定就不成立(如 "9" > "26" 字典序),但這裡永遠兩位,安全。也可寫成 int v = (s[i−2]−'0')*10 + (s[i−1]−'0') 再比,較直觀。
A4. 當兩條邊都不合法,迴圈那一輪不加任何東西,dp[i] 維持初始值 0。之後的每一格都依賴前面,而 0 會透過相加傳播下去:任何用到這個 0 的路徑都貢獻 0。最終 dp[n] 若為 0,就代表沒有任何合法解碼 —— 這是加法 DP 的自然性質,不必額外偵測「卡住了」。
A5. 唯一差別是每條邊多了一道合法性閘門:爬樓梯 dp[i]=dp[i−1]+dp[i−2] 無條件相加;解碼則是「單碼合法才加 dp[i−1]、雙碼合法才加 dp[i−2]」。這說明 1-D 計數 DP 的共性:骨架都是「最後一步的幾種合法選擇,方法數相加」,不同題目只是換掉「有哪些選擇、各自的合法條件」。爬樓梯的選擇永遠合法,解碼則要過濾。
App 1. '*' 讓每條邊變成「多種可能的加權」。單碼:'*' 代表 1–9,共 9 種,故 dp[i] += 9·dp[i−1];數字非 0 則 +1 種。雙碼要按組合分情況:如 "1*" = 11–19 共 9 種、"2*" = 21–26 共 6 種、"**" = 11–26 去掉 20 共 15 種、"*x" 視 x ≤6 或 >6 給 2 或 1 種。把「+dp[i−k]」換成「+(種數)·dp[i−k]」,並記得對大質數取模。骨架不變,係數變多。
App 2. 多加第三條邊:若末 3 位 s[i−3..i−1] 落在合法三位碼區間(如 1–63 中的三位,即 100…?—— 依映射而定),則 dp[i] += dp[i−3]。依賴窗口從 2 變 3,所以要 3 個 base、滾動優化留 3 個變數 —— 正是 泰波那契 那種「往回看 k 項」的推廣,只是每條邊仍各帶合法條件。
App 3. 不適合用計數 DP —— 它只保留「數量」,丟掉了具體字串。要列舉就改用 DFS / 回溯:從左到右,每步嘗試切 1 位(非 0)或切 2 位(10–26),遞迴到底就收集一個解。複雜度差很多:方法數本身可能指數級(如全 1 字串),所以列舉是 O(答案數量 × n),無法像計數那樣壓到 O(n) —— 因為光是輸出就指數多。計數用 DP、列舉用回溯,是這類題的分水嶺。
