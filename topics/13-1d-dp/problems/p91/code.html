寫法 A · dp 陣列// BOTTOM-UP

p91.cpp C++17 
class Solution {
public:
    int numDecodings(string s) {
        int n = s.size();
        if (s[0] == '0') return 0;      // 首字是 0 → 無法解碼

        vector<int> dp(n + 1, 0);
        dp[0] = 1; dp[1] = 1;          // 空字串 1 種;首字 1 種

        for (int i = 2; i <= n; i++) {
            if (s[i - 1] != '0')            // 單碼:末 1 位非 0
                dp[i] += dp[i - 1];

            string w = s.substr(i - 2, 2);       // 末 2 位
            if (w >= "10" && w <= "26")    // 雙碼:10–26
                dp[i] += dp[i - 2];
        }

        return dp[n];
    }
};

這裡用字串比較 w >= "10" && w <= "26" 判雙碼合法很巧:兩位數字字串字典序與數值大小一致,所以 "10"…"26" 恰好框住合法區間;"01"、"27"、"99" 都自然落在外面。

寫法 B · 滾動兩變數// O(1) SPACE

p91_o1.cpp C++17 
class Solution {
public:
    int numDecodings(string s) {
        if (s[0] == '0') return 0;
        int prev2 = 1, prev1 = 1;          // dp[i-2]、dp[i-1]

        for (int i = 2; i <= (int)s.size(); i++) {
            int cur = 0;
            if (s[i - 1] != '0') cur += prev1;       // 單碼
            string w = s.substr(i - 2, 2);
            if (w >= "10" && w <= "26") cur += prev2; // 雙碼
            prev2 = prev1;                      // 往前滾一格
            prev1 = cur;
        }
        return prev1;
    }
};

因為 dp[i] 只看前兩項,兩個變數 prev1/prev2 一路往前滾就夠,空間降到 O(1)。注意 cur 每輪要先歸零再依條件累加 —— 兩條邊都不合法時 cur 保持 0,代表此處無解(整串就會回傳 0)。

邊界與陷阱// PITFALLS

陷阱後果正解
忘記首字 '0' 特判"06" 誤判有解開頭先 if(s[0]=='0') return 0
單碼沒排除 '0'"10" 多算一種單碼需 s[i−1] != '0'
雙碼漏下界 10"01"…"09" 前導零算進去下界 "10",擋掉前導零
雙碼漏上界 26"27"…"99" 誤判合法上界 "26"

註:兩條邊都不合法時 dp[i] 維持 0,並會把 0 一路帶到 dp[n] —— 這正是「整串無法解碼回傳 0」的自然結果,不需另外判斷。
