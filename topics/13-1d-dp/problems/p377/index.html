題目簡介// THE QUESTION

QUESTION

給相異整數 nums 和目標 target,問有幾種組合加起來等於 target(元素可重複用)。注意:順序不同算不同 —— (1,2) 與 (2,1) 是兩種。

題目雖叫「組合」,但因為順序算數,它其實在數排列數。這正是它和 322 零錢兌換 / 518 的分水嶺 —— 差別只在迴圈順序。下面兩件事講清楚:①排列 vs 組合的迴圈順序、②為何用 unsigned int。

2

排列 vs 組合 · 迴圈順序// ORDER MATTERS?

核心 · 外層放誰,決定數排列還是組合

兩題轉移一模一樣都是 dp[i] += dp[i−num],差別只在迴圈誰在外:

● 本題 377(排列):for i(金額) → for num。每個金額都能挑任何 num 當「最後一個數」,於是 (1,2) 和 (2,1) 在不同金額被分開數到 → 排列。
● 518 零錢兌換 II(組合):for num(硬幣) → for i。一種硬幣「整批」處理完才換下一種,枚舉的是無序組合(先決定用幾個 1、再幾個 2…)→ (1,2) 和 (2,1) 只算一次 → 組合。

口訣:要排列(順序算) → 金額外層、num 內層;要組合(順序不算) → 物品外層、金額內層。 這和 139 序列切割也要位置外層 是同一個「外層決定計數語意 / 依賴順序」的大主題。

題外層 / 內層數什麼(1,2)vs(2,1)
377(本題)金額外 / num 內排列(順序算)算 2 次
518 零錢 II硬幣外 / 金額內組合(順序不算)算 1 次
322 零錢(min)任一皆可最少枚數—(min 免疫順序)

unsigned int vs long long · 溢位與 mod// OVERFLOW

這題的 DP 只做加法,而排列數長很快 —— 中途 dp[i] 可能超過 int 上限 2³¹−1。型別選擇因此變成正確性問題:

型別溢位時的行為這題
int未定義行為(UB) —— C++ 對有號溢位不保證任何結果,常見變成負值/亂數× 危險
unsigned int有定義:溢位環繞 mod 2³²(超過就從 0 重來)○ 可用(見下)
long long64 位,這題的值不會溢位(< 2⁶³)● 最直觀安全
unsigned int 的行為 = mod 2³²(實測)

unsigned int 的算術是模 2³²(= 4294967296):任何運算結果都自動取 mod 2³²,超過就環繞。實測 nums=[1,2,3] 把 target 加大到真值跨過 2³¹:

// target=37,真值 = 3,831,006,429(介於 2³¹ 與 2³²) long long(真值) : 3831006429 unsigned int : 3831006429 ✓ 正確(< 2³²,裝得下) int(有號) : -463960867 ✗ 溢位 UB(3831006429 − 2³² 的位元被當成負數)

為什麼 unsigned 就算中途環繞、最後仍正確?因為整個 DP 只有加法,而模運算對加法封閉:(a+b) mod m = ((a mod m)+(b mod m)) mod m。所以 dp[target](unsigned)永遠等於「真值 mod 2³²」 —— 即使中途某些 dp[i] 爆過 2³² 環繞了,最後這一格仍是對的 mod 值。

LeetCode 保證答案 fits 32-bit(真值 < 2³¹),於是「真值 mod 2³²」就等於真值本身,static_cast<int> 轉回去剛好正確。unsigned int 是「我只要 mod 2³² 正確、且答案裝得下」的競賽慣用寫法;面試想穩、想好懂,直接用 long long 最省事。(注意:若真值落在 [2³¹, 2³²),unsigned 值對、但 cast 回 int 會變負 —— 這種 case 不在題目保證內。)

核心策略 · 三步驟// STRATEGY

  1. 狀態 + base:dp[i] = 湊出 i 的排列數;dp[0]=1(空排列一種)。
  2. 金額外層(排列):i 從 1 到 target;num 內層:每個 num≤i。
  3. 轉移:dp[i] += dp[i−num];用 unsigned / long long 防溢位;回傳 dp[target]。

一句洞察// INSIGHT

同一條 dp[i]+=dp[i−num]:金額外層 = 排列(順序算)、物品外層 = 組合。 加法會爆 int → 用 unsigned(mod 2³² 環繞、加法保持一致)或直接 long long。
