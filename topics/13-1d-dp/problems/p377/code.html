寫法 A · unsigned int(你的版本)// MOD 2³²

p377.cpp C++17 
class Solution {
public:
    int combinationSum4(vector<int>& nums, int target) {
        vector<unsigned int> dp(target + 1, 0);  // 溢位 → mod 2³² 環繞(有定義)
        dp[0] = 1;                            // 空排列一種

        for (int i = 1; i <= target; i++) {       // ← 金額外層 → 排列
            for (int& num : nums) {
                if (i - num >= 0) {
                    dp[i] += dp[i - num];       // 累加「最後一個數 = num」的排列數
                }
            }
        }

        return static_cast<int>(dp[target]);       // 答案 < 2³¹ → 轉回 int 正確
    }
};

用 unsigned int 而非 int:中途 dp[i] 可能超過 2³¹−1,有號溢位是 UB;unsigned 溢位環繞 mod 2³²,是有定義的。因為 DP 只做加法、模對加法封閉,dp[target] 永遠 = 真值 mod 2³²;題目保證答案 < 2³¹,static_cast<int> 剛好對。詳見概念頁第 3 節。

寫法 B · long long(面試建議)// SAFE & CLEAR

p377_ll.cpp C++17 
class Solution {
public:
    int combinationSum4(vector<int>& nums, int target) {
        vector<long long> dp(target + 1, 0);       // 64 位,這題不會溢位
        dp[0] = 1;

        for (int i = 1; i <= target; i++)
            for (int num : nums)
                if (num <= i)
                    dp[i] += dp[i - num];

        return (int)dp[target];                     // 答案保證 fits int
    }
};

long long 有 64 位,對本題(答案 fits 32-bit、中途值遠小於 2⁶³)不會溢位、也不需 mod 推理,可讀性最好 —— 面試時直接這樣寫、口頭說一句「用 long long 防中途溢位」即可。unsigned int 省一半記憶體、是競賽慣用,但要懂它「mod 2³²」的語意。

複雜度分析// COMPLEXITY

T = O(target × 種類)　　S = O(target)

兩層迴圈 target × nums.size(),每格一次加法。target ≤ 1000、種類 ≤ 200 → 至多 2×10⁵,很快。

邊界與陷阱// PITFALLS

陷阱後果正解
用 int 累加中途 >2³¹ → 有號溢位 UB、變負unsigned int 或 long long
硬幣外層、金額內層變成數組合(518)、少算金額外層、num 內層(排列)
忘記 dp[0]=1全 0,答案永遠 0base:空排列一種
把 unsigned 當「只是防負」誤解它的 mod 語意懂它是 mod 2³² 環繞

註:因為 i − num ≥ 0 的守衛,超過 i 的 num 自然被跳過;nums 元素相異但這裡不影響正確性(重複也只是多枚舉一次)。
