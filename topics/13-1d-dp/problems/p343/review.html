面向要點

不切j × (i−j):另一塊 i−j 留整塊

觀念題// QUIZ

Q1. 用一句話說出 j×(i−j) 和 j×dp[i−j] 各代表什麼?為什麼兩項都要?
Q2. 在這個定義下 dp[2] 是 1 還是 2?dp[3] 呢?為什麼不是 2 和 3?
Q3. 舉一個「不切贏」和一個「切贏」的具體 (i,j),並算出兩項的值。
Q4. 為什麼「小塊(2、3)不要切、大塊(≥4)才切」?用 dp[2]、dp[3]、dp[5] 的值說明。
Q5. 數學捷徑為什麼是「盡量拆 3」而不是拆 2?餘數為 1 時為什麼要把一個 3 換成 4?

範例 Trace// n = 10 → 36

// DP:dp[0..10] dp = [1,1,1,2,4,6,9,12,18,27,36] dp[10]: j=3 不切 3×7=21 / 切 3×dp[7]=3×12=36 → 切贏 (也可 j=4:不切4×6=24 / 切 4×dp[6]=4×9=36) dp[10] = 36(= 3×3×2×2 或 3×3×4)
// 數學:盡量拆 3 10 → 拿一個 3(product=3, n=7)→ 再拿一個 3(product=9, n=4) n=4 ≤ 4 收尾:9 × 4 = 36 = 3 × 3 × 4 = 36
36 = 3×3×4:兩個 3 加一個 4(=2×2)。這正是「盡量拆 3、餘 1 換 4」的結果 —— 10 = 3+3+3+1 會有孤單的 1(×1 浪費),所以把一個 3+1 併成 4,得 3+3+4。

應用題// APPLY

App 1. 若改用「dp[i] = 把 i 拆成 ≥1 塊(允許整塊)的最大乘積」這種定義,轉移會變成什麼?為什麼那時只需一項 j×dp[i−j]?
App 2. 若 n 很大(如 10⁹),O(n²) DP 不行。數學版 O(n) 也慢。怎麼把「盡量拆 3」做到 O(1) / O(log n)?(提示:用 n%3 分類 + 快速冪。)
App 3. 若答案要對大質數取模(拆分後乘積很大),數學版怎麼調整?為什麼不能先算完再取模?
// ANSWERS
A1. j×(i−j) = 把 i 切成 j 和 i−j 兩塊、i−j 不再切(留整塊)的乘積;j×dp[i−j] = 把 i−j 繼續拆到最優的乘積。兩項都要,是因為 dp[i−j] 的定義是「一定要拆 ≥2 塊」,它不包含「i−j 留整塊」這個選項 —— 那個選項只能靠 j×(i−j) 補上。少了它,像 dp[4] 就會漏掉「2×2=4」而誤算成 2。
A2. dp[2]=1、dp[3]=2。因為這個定義要求「一定拆成 ≥2 塊」:2 只能拆成 1+1 → 乘積 1;3 拆成 1+2 → 乘積 2。「2 整塊」「3 整塊」不算拆,不在 dp 裡。你原本註解寫 dp[2]=2 是把「允許整塊」的定義混進來了 —— 兩種定義各自自洽,但不能混用。
A3. 不切贏:i=4, j=2 → 不切 2×2=4、切 2×dp[2]=2×1=2 → 不切贏,dp[4]=4。切贏:i=8, j=3 → 不切 3×5=15、切 3×dp[5]=3×6=18 → 切贏,dp[8]=18。前者「小塊 2 留整塊」較好,後者「大塊 5 繼續拆」較好。
A4. 看拆了會不會變大:dp[2]=1 < 2、dp[3]=2 < 3 —— 2、3 拆了反而變小,所以留整塊(不切)。dp[4]=4 = 4(拆成 2×2 打平),dp[5]=6 > 5、dp[6]=9 > 6 —— ≥5 拆了會變大,所以繼續拆(切)。臨界在 4:2、3 別碰,≥4 開始有拆的價值。這也是為什麼最優解裡只會出現 2 和 3(不會留下 ≥4 的整塊)。
A5. 因為固定總和下,想讓乘積大就要讓每塊的「單位乘積效率 x^(1/x)」高。整數中 3^(1/3) ≈ 1.442 最高,勝過 2^(1/2) ≈ 1.414,所以盡量湊 3。餘數為 1 時,若硬留一個孤單的 1,乘上去等於白費(×1 不增大);把「一個 3 + 這個 1」= 4 拆成 2×2=4 比 3×1=3 大,所以要換。餘數為 2 就直接留一個 2(2 本身效率也不差,且無法再湊 3)。
App 1. 若 dp[i] 允許「整塊」(≥1 塊),轉移變成 dp[i] = max over j ( j × dp[i−j] ),且 base 讓 dp[k] 至少等於 k 本身(整塊)。此時只需一項,因為 dp[i−j] 已經包含「i−j 留整塊」(它 ≥ i−j)。最後題目要求「至少拆一刀」,再在頂層對 n 強制拆一次(max over j ( j × dp[n−j] ))即可。這是另一種等價寫法 —— 定義變了,轉移也跟著少一項,但別和「一定拆」的定義混用。
App 2. 直接用 n 除以 3 的商與餘數分類:設 q = n/3, r = n%3。① r==0 → 答案 3^q;② r==1 → 拿一個 3 出來配那個 1 成 4 → 3^(q−1) × 4;③ r==2 → 3^q × 2。其中 3^q 用快速冪 O(log n) 算(需取模時尤其必要)。特判小 n(2、3)。這樣不用迴圈逐個減 3,整體 O(log n)。
App 3. 若要對質數 M 取模,乘積在計算過程中就要邊乘邊 % M(用快速冪的模版本算 3^q mod M),不能先算完再取模 —— 因為真值可能是天文數字(3^(10⁹/3)),任何整數型別都裝不下、會溢位成垃圾。模運算對乘法封閉((a·b) mod M = ((a mod M)(b mod M)) mod M),所以全程保持在模空間內即可。注意:取模後就失去比較大小的能力,所以「拆 3 最優」這個決策必須在取模之前用數學證明定好,不能在模空間裡比大小。
