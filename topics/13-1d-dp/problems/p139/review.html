Q5. 這題(序列切割)和 322 完全背包 都是「一維、可重複使用元素」,為什麼一個位置外層、一個物品外層?差在問題結構的哪一點?

範例 Trace// s="applepenapple"

// 位置外層填表 dict={apple(5),pen(3)} n=13, dp[0]=T i=5 : "apple" → dp[0]=T 且 s[0..5]="apple" ✓ → dp[5]=T i=8 : "pen" → dp[5]=T 且 s[5..8]="pen" ✓ → dp[8]=T i=13: "apple" → dp[8]=T 且 s[8..13]="apple"✓ → dp[13]=T return dp[13] = true(apple | pen | apple)
關鍵在 i=13:它靠 dp[8],而 dp[8] 是 i=8 時用 "pen" 設真的。 位置外層保證掃到 i=13 前,i=8 早已處理完 —— 所以 dp[8] 是可靠的真。詞外層則會在 dp[8] 變真之前就掃過 i=13,錯過這條路。

應用題// APPLY

App 1. 140 單字拆分 II:要列出所有切法(不只 true/false)。還適合用這個布林 DP 嗎?該搭配什麼?複雜度為何可能爆炸?
App 2. 91 解碼方法 也是序列切割(每步切 1 或 2 位)。它的外層是什麼?和本題的「位置外層」是不是同一個道理?
App 3. 若把「能否切開」改成「最少切成幾段」或「有幾種切法」,轉移的聚合運算要各自換成什麼?外層還是位置嗎?
// ANSWERS
A1. 看答案的形狀:若是「把序列切成有順序、落在固定位置的片段」(詞會交錯)→ 位置當外層,每個位置試所有選擇;若是「挑一堆物品湊數、順序無關」(背包)→ 物品當外層,容量掃描方向決定重複。一句話:有序切割用位置外層,無序組合用物品外層。
A2. 詞外層先跑 "apple":單趟由小到大掃位置,掃到 i=13 時要看 dp[8],但此刻 dp[8] 還是 false(它要靠 "pen" 才會真,而 "pen" 還沒輪到)→ dp[13] 被判 false。接著跑 "pen",在 i=8 把 dp[8] 設真 —— 但迴圈不會回頭重掃 i=13,所以 dp[13] 永遠卡在 false。漏掉了 apple|pen|apple 這條合法切法。
A3. 因為詞外層下,dp[i] 是否被設真,取決於「它依賴的 dp[j] 有沒有在更早處理的詞裡先變真」。把字典順序一換,哪些 dp[j] 先就緒就變了,結果跟著變:{"ca","rs"} 對、{"rs","ca"} 錯。正確的 DP 不該對輸入順序敏感 —— 會敏感,就代表它違反了「用到子問題時子問題已定案」的前提,是結構性錯誤,不是調一調就好。
A4. dp[0]=true 代表「空字串可以被切開」(切成零個詞),是遞推的錨點:每個 dp[i] 最終都要回溯到某個 dp[0] —— 第一個詞的前綴就是 dp[0]。若忘了設,dp[i−L] 永遠找不到一個為真的起點,整張表全 false,任何字串都被判「不能切」。它就是 base case,少了它整個 DP 塌掉。
A5. 差在「元素有沒有固定位置」。完全背包挑的是無序的硬幣多重集合 —— 用了 3 個 1 元、2 個 5 元,先放哪個都一樣,所以可以「一種硬幣處理完再換下一種」(物品外層),用容量升冪允許重複。Word Break 的詞卻各自佔據字串上一段固定位置、會交錯(apple 在 0–5、pen 在 5–8、apple 又在 8–13),不能重排成「所有 apple 再所有 pen」,所以無法用物品外層,必須位置外層逐格把所有更短前綴定案。核心分野:組合(無序)可物品外層,切割(有序、定位)必須位置外層。
App 1. 布林 DP 只回答「能不能」,不記路徑,不適合直接列舉。要列出所有切法,常用 DFS / 回溯 + 記憶化:先用本題的 dp(或反向可達性)剪枝「這個位置往後還切得完嗎」,再 DFS 枚舉每個切點收集答案。複雜度可能指數級 —— 如 s="aaaa...", dict={a,aa,aaa,...} 切法數本身就指數多,光輸出就爆。所以 140 的重點是「用 DP 判可行性剪枝 + 回溯列舉」,而非單純 DP。
App 2. 是同一個道理。91 的 dp[i] = 前 i 位的解碼方法數,外層就是位置 i(由小到大),內層是「這一步切 1 位還是 2 位」的選擇。它也是序列切割:片段(1 位碼 / 2 位碼)落在固定位置、會交錯,所以位置外層。139 的「詞」對應 91 的「切 1/2 位」,骨架完全一致 —— 只是 91 的「字典」是固定的兩種切法。
App 3. 只換聚合運算,外層仍是位置。① 最少切幾段:dp[i] 存最小段數,dp[i] = min over w ( dp[i−L] + 1 ),base dp[0]=0、其餘 ∞。② 有幾種切法:dp[i] 存方法數,dp[i] += dp[i−L](對每個吻合的詞累加),base dp[0]=1。可行性用 OR、最優用 min、計數用 + —— 這正是 1-D DP 家族「同骨架、換聚合」的共性,而序列切割的位置外層三者通用。
