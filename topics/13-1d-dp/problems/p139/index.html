題目簡介// THE QUESTION

QUESTION

給字串 s 和字典 wordDict,問 s 能否被切成一段段字典裡的詞(詞可重複使用)。能則 true。

dp[i] = 前 i 個字元 s[0..i−1] 能否被完整切開。答案 dp[n]。這是「序列切割 DP」的代表題 —— 也是最多人在迴圈順序上寫錯的一題。下面先把心法講清楚。

2

迴圈順序心法// THE LOOP-ORDER PATTERN

核心一問 · 「dp[i] 依賴誰?那些子問題必須先算好」

寫這類 DP 前,先回答三件事:
① 狀態是什麼(dp 的 index 代表什麼)?
② dp[i] 依賴哪些更小的 dp[j]?
③ 這些依賴要怎麼掃才能保證「用到時已算好」?

答案就決定了外層迴圈:外層 = 掃「狀態維度」、把依賴先鋪好;內層 = 枚舉「這一步的選擇」。搞反了,就會拿到「還沒算好」的子問題,答案時對時錯。

怎麼分序列切割 vs 背包?問自己:答案是「把序列切成有順序的片段」還是「挑一堆物品湊數(順序無關)」?

  • 序列切割(139/91):每個詞落在固定位置、會交錯,不能「一種詞處理完再換下一種」→ 必須 位置當外層,每個位置試所有詞。
  • 背包(322/416):挑的是無序組合,可以「一種物品一種物品地加」→ 物品當外層,容量掃描方向決定重複與否。

為什麼「詞當外層」會錯// THE COMMON BUG

✗ 詞外層、位置內層(單趟) 錯
for word: for i…。處理某個詞、掃到位置 i 時,它依賴的 dp[i−L] 可能要靠還沒處理的別的詞才會變真 → 漏解。且答案會隨字典順序改變。
× REJECT · 順序相依
● 位置外層、詞內層 O(n·k·L)
for i: for word…。算 dp[i] 時,所有 dp[j] (j<i) 都已被全部詞考慮過、定案 → 穩定正確。
● ADOPT · 位置外層
實測 · 詞外層真的會錯

s="applepenapple", dict={"apple","pen"}:

詞外層先處理 "apple":掃到 i=13 時要 dp[8],但 dp[8] 要靠 "pen"(還沒處理)才會真 → dp[13] 被判 false。等處理 "pen" 讓 dp[8] 變真時,早就掃過 i=13 了、不會回頭。

結果:位置外層 = true(正確)、詞外層 = false(錯)。把字典從 {"rs","ca"} 改成 {"ca","rs"},詞外層的答案還會跟著變 —— 這就是「順序相依」的鐵證。你當初那版除了語法(i 沒宣告、迴圈條件不完整)之外,骨架也選錯了外層。

4

核心策略 · 三步驟// STRATEGY

  1. 狀態 + base:dp[i] = 前 i 字元可切;dp[0]=true(空字串)。
  2. 位置外層:i 從 0 到 n;詞內層:每個詞 w(長 L)。
  3. 轉移:若 i−L≥0 && dp[i−L] && s[i−L..i]==w → dp[i]=true。回傳 dp[n]。

複雜度// COMPLEXITY

T = O(n × k × L)　　S = O(n)

n 個位置 × k 個詞 × 每次比對 L。n≤300、k≤1000、L≤20。把字典放進 unordered_set、內層改枚舉「切點 j」查子串,可寫成 O(n²·L)。

7

一句洞察// INSIGHT

序列切割題的鐵律:位置當外層(升冪)、選擇當內層。 因為片段落在固定位置、會交錯,算 dp[i] 前必須先把所有更短前綴定案 —— 這正是背包(物品外層)分不清時最容易踩的雷。
