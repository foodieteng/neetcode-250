寫法 A · 位置外層(你修對的版本)// POSITION OUTER

p139.cpp C++17 
class Solution {
public:
    bool wordBreak(string s, vector<string>& wordDict) {
        int n = s.size();
        vector<bool> dp(n + 1, false);
        dp[0] = true;                       // 空字串可切

        for (int i = 0; i <= n; i++) {          // ← 位置外層,由小到大
            for (string w : wordDict) {          // ← 詞內層,枚舉選擇
                int L = w.size();
                if (i - L >= 0 && dp[i - L] &&
                    s.substr(i - L, L) == w) {
                    dp[i] = true;
                }
            }
        }

        return dp[n];
    }
};

外層 i 由小到大掃,保證算 dp[i] 時所有 dp[j] (j<i) 都已被全部詞考慮過、定案。i−L≥0 是防越界的守衛;dp[i−L] 確認「前綴切得開」,substr==w 確認「這段正好是這個詞」。

寫法 B · set + 切點// HASH SET

p139_set.cpp C++17 
class Solution {
public:
    bool wordBreak(string s, vector<string>& wordDict) {
        unordered_set<string> dict(wordDict.begin(), wordDict.end());
        int n = s.size();
        vector<bool> dp(n + 1, false);
        dp[0] = true;

        for (int i = 1; i <= n; i++) {          // 位置外層
            for (int j = 0; j < i; j++) {       // 內層枚舉切點 j
                if (dp[j] && dict.count(s.substr(j, i - j))) {
                    dp[i] = true;
                    break;                          // 找到一種就夠
                }
            }
        }

        return dp[n];
    }
};

仍是位置外層,只是內層改成枚舉「上一個切點 j」,用 unordered_set 以 O(1) 均攤查子串在不在字典。當字典很大(k 大於 n)時,枚舉切點比枚舉每個詞更省。break 是因為只要存在一種切法即可。

複雜度分析// COMPLEXITY

A:T = O(n·k·L)　　B:T = O(n²·L)　　S = O(n)

A 每個位置試 k 個詞、比對 L;B 每個位置試 n 個切點、每次 substr + 雜湊 約 L。字典大用 A,字串長用 B。
