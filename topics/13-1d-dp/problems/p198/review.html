範例 Trace// nums=[2,1,1,2]

// 滾動版走一遍(答案應為 4) prev2=0 prev1=0 money=2: cur=max(0, 0+2)=2 → prev2=0 prev1=2 money=1: cur=max(2, 0+1)=2 → prev2=2 prev1=2 money=1: cur=max(2, 2+1)=3 → prev2=2 prev1=3 money=2: cur=max(3, 2+2)=4 → prev2=3 prev1=4 return prev1 = 4 (搶頭尾:2 + 2)
這正是貪心會踩雷的例子:貪心先搶中間的房只能拿 2,DP 卻算出「跳過中間、搶頭尾」= 2+2=4。 DP 每一步都同時保留「不搶」與「搶」兩條路的最優,才不會被局部大值誤導。

應用題// APPLY

App 1. 213 打家劫舍 II:房子排成環(首尾也相鄰)。怎麼用「跑兩次直線版」解決?
App 2. 若還要回報實際搶了哪幾間(不只金額),程式要怎麼調整?
App 3. 把限制改成「不能搶連續三間裡的某種組合」或「相鄰距離至少 k」,轉移式會怎麼變?
// ANSWERS
A1. 因為相鄰不能同時搶。一旦決定搶第 i 間,第 i−1 間就被禁掉,合法的接續起點必須退到 i−2。所以搶 i 的最優 = 「前 i−2 間的最優 dp[i−2]」+ nums[i]。若接 dp[i−1],就可能同時把 i−1 和 i 都搶了,違反規則。
A2. 局部最大 ≠ 全域最大。反例 [2,1,1,2]:貪心先搶最大的房(頭一個 2),之後受相鄰限制只能再拿一個 1,共 3;但最優是搶頭尾 2+2=4。貪心無法「為了尾巴放棄眼前」,DP 則比較兩條路。
A3. 因為「不搶 i」的定義就是第 i 間的錢不拿,前 i 間的最優等於前 i−1 間的最優,原封不動 = dp[i−1]。加 nums[i] 是「搶」那一支才做的事,兩支互斥,不能混。
A4. 不能亂調。cur 必須先用舊的 prev1(=dp[i−1])、prev2(=dp[i−2])算完,才把窗口往前滾:prev2=prev1(舊 dp[i−1] 變成下一輪的 dp[i−2])、prev1=cur。若先寫 prev1=cur,下一行的 prev2=prev1 就抓到被覆蓋的新值,全錯。順序:先算 cur → prev2 收舊 prev1 → prev1 收 cur。
A5. 差在把 + 換成 max(且多了 +nums[i])。爬樓梯 dp[i]=dp[i−1]+dp[i−2] 是計數(兩類走法相加);打家劫舍 dp[i]=max(dp[i−1], dp[i−2]+nums[i]) 是最佳化(兩個選擇取較好)。同一套「線性 DP、依賴前兩項」骨架,聚合運算不同 —— 計數用 +/求極值用 max/min,這是 1-D DP 家族的共同分水嶺。
App 1. 環形的難點是「第一間和最後一間也相鄰,不能同時搶」。把它拆成兩個直線子問題:(a) 允許搶第一間、但排除最後一間 → 對 nums[0 .. n−2] 跑本題;(b) 排除第一間、允許最後一間 → 對 nums[1 .. n−1] 跑本題。兩者取 max。因為「首尾同搶」被兩種切法都排除掉,結果一定合法。(單一房子要特判直接回傳 nums[0]。)
App 2. 除了值,還要能回溯決策。作法一:保留整條 dp,從尾往前走 —— 若 dp[i]==dp[i−1] 代表沒搶第 i 間(往 i−1 退),否則搶了(記錄第 i 間、跳到 i−2)。作法二:多存一個 take[i] 布林陣列記每格的選擇。滾動 O(1) 版丟掉了歷史,要回報路徑就得放棄 O(1) 空間、留下 dp 或決策陣列。
App 3. 依賴窗口變寬。若「相鄰距離至少 k」(搶了 i 就要跳過 k−1 間),轉移變成 dp[i]=max(dp[i−1], dp[i−k]+nums[i]) —— 搶 i 時接的是 dp[i−k] 而非 dp[i−2]。本題是 k=2 的特例。這和「泰波那契往回看三項」是同一種「調整依賴窗口」的思路:改的是「往回接哪一格」,骨架不動。
