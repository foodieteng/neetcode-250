寫法 A · dp 陣列(sentinel dp[0]=0)// BOTTOM-UP

p198.cpp C++17 
class Solution {
public:
    int rob(vector<int>& nums) {
        int n = nums.size();
        vector<int> dp(n + 1, 0);
        dp[0] = 0;              // sentinel:搶 0 間 = 0 元
        dp[1] = nums[0];       // 只有第 1 間 → 直接搶它

        for (int i = 2; i <= n; i++) {
            // dp[i] 覆蓋前 i 間房(第 i 間 = nums[i-1])
            dp[i] = max(dp[i - 2] + nums[i - 1],  // 搶第 i 間
                        dp[i - 1]);            // 不搶第 i 間
        }

        return dp[n];
    }
};

這裡用 sentinel 技巧:dp 大小開 n+1、dp[0]=0 代表「零間房」,於是第 i 間房對應 nums[i−1],迴圈能從 i=2 起統一寫轉移,不必特判第二間。答案在 dp[n]。

寫法 B · 滾動兩變數// O(1) SPACE

p198_o1.cpp C++17 
class Solution {
public:
    int rob(vector<int>& nums) {
        int prev2 = 0, prev1 = 0;      // dp[i-2]、dp[i-1]

        for (int money : nums) {
            int cur = max(prev1, prev2 + money); // 不搶 vs 搶
            prev2 = prev1;                     // 往前滾一格
            prev1 = cur;
        }

        return prev1;
    }
};

因為 dp[i] 只看 前兩項,整條陣列用不到 —— 兩個變數 prev1/prev2 一路往前滾就夠,空間降到 O(1)。這版本連 base 都不用特判:從 prev1=prev2=0 起掃,第一間自然算成 max(0, 0+nums[0])=nums[0]。

複雜度分析// COMPLEXITY

T = O(n)　　S = O(n)(A) / O(1)(B)

單一迴圈掃 n 間房,每格一次 max。A 存整條 dp;B 只留兩個變數。

邊界與陷阱// PITFALLS

陷阱後果正解
搶 i 時接 dp[i−1]搶了相鄰兩間 → 違規搶 i 只能接 dp[i−2]
用貪心挑最大[2,1,1,2] 得 2 而非 4DP 比較搶/不搶兩條路
寫法 A 少 dp[1] basedp[2] 少一條來源dp[1]=nums[0] 必設
index 對錯(A 版偏移 1)第 i 格用成 nums[i]A 版第 i 間 = nums[i−1]

註:題目保證 nums.length ≥ 1,所以寫法 A 直接寫 dp[1]=nums[0] 安全。若要更保險,滾動版(B)天生對空陣列也回傳 0。
