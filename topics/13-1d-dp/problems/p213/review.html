重點回顧// RECAP

面向要點
新約束房子成圈:第 0 與 n−1 間也相鄰,不能同搶
核心手法剪成兩條直線,各跑一次 198 線性版
Run A去尾 nums[0..n−2](保留頭)
Run B去頭 nums[1..n−1](保留尾)
答案max(Run A, Run B)
為何對頭尾不同搶 = 至少一端不搶 → 必落在 A 或 B
特判n==1 回傳 nums[0]
2

觀念題// QUIZ

Q1. 為什麼不能直接把 198 的線性解套上去?用 [2,7,9,3,1] 說明。
Q2. 為什麼「去頭一次、去尾一次」這兩條直線就足以涵蓋所有合法解?會不會漏掉什麼?
Q3. 「頭尾都不搶」的方案有沒有被算到?它落在 Run A 還是 Run B?
Q4. 為什麼要特判 n==1?若不特判,take(nums, 0, n−2) 會發生什麼?
Q5. 這題整體是 O(n) 還是 O(2n)?兩次線性掃描為什麼不算「更慢」?
3

範例 Trace// nums=[1,2,3,1]

// Run A · 去尾 [1,2,3](house 0..2) prev2=0 prev1=0 money=1: cur=max(0,0+1)=1 → 0,1 money=2: cur=max(1,0+2)=2 → 1,2 money=3: cur=max(2,1+3)=4 → 2,4 Run A = 4 (搶 house 0、2 = 1+3)
// Run B · 去頭 [2,3,1](house 1..3) prev2=0 prev1=0 money=2: cur=max(0,0+2)=2 → 0,2 money=3: cur=max(2,0+3)=3 → 2,3 money=1: cur=max(3,2+1)=3 → 3,3 Run B = 3 (搶 house 1 或 house 3 那類)
// 取 max 答案 = max(4, 3) return 4
Run A 保留了頭(house 0),讓「搶頭尾兩端」中的頭端方案得以成立(1+3); Run B 保留了尾。兩輪聯手,任何「至少犧牲一端」的合法搶法都被涵蓋,取 max 就是全域最優。

應用題// APPLY

App 1. 337 打家劫舍 III:房子變成一棵二元樹(父子不能同搶)。線性/環形的手法還能用嗎?該換成什麼?
App 2. 若題目改成「環形排列,但可搶相鄰,只是不能搶超過 k 間」,DP 狀態要怎麼加維度?
App 3. 一般化:凡是「線性 DP + 一條環形/首尾耦合約束」的題目(如環形塗色、圓桌問題),都能用「固定/排除某個接縫點,跑幾次線性版」的套路嗎?舉一個你想到的例子。
// ANSWERS
A1. 因為線性版不知道「頭尾相鄰」。[2,7,9,3,1] 線性最優是搶 house 0、2、4 = 2+9+1=12;但在圈上 house 0 和 house 4 是鄰居,不能同搶,這個 12 是非法的。必須強制至少放棄一端,答案才會降到合法的 11。
A2. 不會漏。環形唯一的新限制就是「頭尾不同搶」,等價於說頭、尾至少有一間沒被搶。① 沒搶頭 → 問題落在 nums[1..n−1](Run B);② 沒搶尾 → 落在 nums[0..n−2](Run A)。任何合法解至少滿足其一,所以被 A 或 B 涵蓋;而 A、B 內部是普通直線問題,不會產生「頭尾同搶」的非法解。兩輪取 max 剛好不多不少。
A3. 有被算到,而且同時落在 Run A 和 Run B 裡(它既沒搶頭、也沒搶尾,兩個區間都容得下它)。被重複考慮不影響正確性 —— 取 max 只在意最大值,重複的可行解不會讓答案變錯。這也是為什麼兩種切法就夠,不必再補「頭尾都不搶」的第三種。
A4. 當 n==1,take(nums, 0, n−2) = take(nums, 0, −1) 是空區間,去尾把唯一的房也切掉了,會回傳 0 而漏算。所以先特判:圈裡只有一間房時根本沒有「相鄰」問題,直接回傳 nums[0]。(滾動版 B 因為用 for i=s..e,空區間自然回 0,但仍要靠 n==1 特判保證答案是 nums[0]。)
A5. 是 O(n)。兩次線性掃描是 2n 次基本運算,大 O 記法丟掉常數係數 2,仍是 O(n)。「跑兩次」只把常數因子變大一點,不改變成長級別 —— 相較於為環形硬寫一套帶狀態的 DP,重用兩次線性版又快又不易錯,是划算的取捨。
App 1. 直線/環形的「陣列 DP」不能直接用,因為結構變成樹。改用樹形 DP(樹上 DFS + 後序回傳兩個狀態):每個節點回傳 {搶這個節點的最優, 不搶這個節點的最優}。搶這個 → 兩個小孩都必須「不搶」;不搶這個 → 每個小孩取 max(搶, 不搶)。本質仍是「選/不選 + 相鄰約束」,只是約束從「陣列相鄰」變成「父子相鄰」,聚合從一維滾動變成樹的後序合併。
App 2. 加一維「已搶間數」:dp[i][j] = 考慮到第 i 間、已搶 j 間時的最優;轉移在 j 上 +1(搶)或不變(不搶)。環形照樣用「排除接縫點」拆兩次直線,只是每次直線 DP 都多帶 j 這一維,複雜度變 O(nk)。核心的「環形 → 幾次直線」框架不變,變的是每次直線問題的狀態維度。
App 3. 可以,這是通用套路:環形問題 = 在某個「接縫」上做有限枚舉,把每種枚舉固定後化成直線問題。本題枚舉「頭搶/不搶」(其實只需枚舉「至少一端不搶」→ 兩次)。例子:環形塗色(相鄰不同色)可固定第一格的顏色再對其餘做直線 DP;圓桌安排、環形加油站、環形子陣列最大和(把「跨越接縫」拆成 總和 − 中間最小子陣列)都是同一精神 —— 把環的耦合點孤立出來單獨處理,其餘退化成熟悉的線性 DP。
