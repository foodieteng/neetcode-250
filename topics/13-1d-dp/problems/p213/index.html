題目簡介// THE QUESTION

QUESTION

和 198 打家劫舍 一樣不能搶相鄰兩間,但這次房子排成一個圈 —— 第一間和最後一間也相鄰。求不觸發警報能搶到的最多金額。

只多了一條約束:頭尾不能同時搶。與其重寫一套環形 DP,不如把圈剪成兩條直線,各自套用 198 的線性解,再取較大者 —— 用已解過的子問題拼出答案。

2

為什麼「跑兩次」// WHY TWO RUNS

A 直接套 198 線性解 錯
線性版可能同時搶頭尾;在圈上頭尾相鄰 → 違規。[2,7,9,3,1] 線性得 12(搶 0,2,4),但 0、4 相鄰不合法。
× REJECT · 違規
B 為環形重寫一套 DP 複雜
要在狀態裡多記「有沒有搶第一間」,轉移變麻煩、易錯。能重用就別重寫。
△ REJECT · 過度
ADOPT
C 剪開成兩條直線,各跑一次 198,取 max O(n)
Run A:搶 nums[0..n−2](去尾)。Run B:搶 nums[1..n−1](去頭)。兩種切法都保證頭尾不同搶,答案 = max(A, B)。
● ADOPT · O(n)
3

核心策略 · 三步驟// STRATEGY

  1. 特判:只有 1 間 → 回傳 nums[0];2 間 → max(nums[0], nums[1])。
  2. 剪兩刀:Run A 對 nums[0..n−2]、Run B 對 nums[1..n−1],各跑一次線性打家劫舍。
  3. 取 max:答案 = max(Run A, Run B)。

關鍵推導// WHY IT'S CORRECT

LEMMA · 兩條直線覆蓋所有合法解

環形唯一的新麻煩,是第一間與最後一間不能同時搶。任何合法方案,一定滿足下面兩者之一(至少一個):

① 沒搶第一間 → 它等價於在 nums[1..n−1] 上的直線問題(Run B)。
② 沒搶最後一間 → 它等價於在 nums[0..n−2] 上的直線問題(Run A)。

因為「頭尾不同搶」代表至少有一端沒被搶,所以每個合法解都落在 Run A 或 Run B 的範圍內。兩者各自求出直線最優、再取 max,就涵蓋了全部合法情況,且絕不會誤放「頭尾同搶」的非法解。

6

複雜度// COMPLEXITY

T = O(n)　　S = O(n) → 可優化 O(1)

跑兩次線性掃描 = 2 × O(n) = O(n)。每次線性版都可用兩變數滾動 → O(1) 空間。

7

一句洞察// INSIGHT

環形不用重寫 DP —— 「頭尾不同搶」= 至少一端不搶,所以去頭一次、去尾一次,各跑線性版取 max。 把一個難的環形問題,化成兩個已解的直線子問題,是很常見的降維技巧。
