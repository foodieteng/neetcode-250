寫法 A · 兩次線性(dp 陣列)// TWO RUNS

p213.cpp C++17 
class Solution {
public:
    int rob(vector<int>& nums) {
        int n = nums.size();
        if (n == 1) return nums[0];            // 圈裡只有一間 → 沒有相鄰約束
        if (n == 2) return max(nums[0], nums[1]); // 兩間互為鄰居 → 取大

        // 去尾 nums[0..n-2]  vs  去頭 nums[1..n-1],取 max
        return max(take(nums, 0, n - 2), take(nums, 1, n - 1));
    }

    // 198 線性打家劫舍,套用在區間 nums[s..e]
    int take(vector<int>& nums, int s, int e) {
        int n = e - s + 1;
        vector<int> dp(n + 1, 0);
        dp[1] = nums[s];                     // 區間第一間直接搶

        for (int i = 2; i <= n; i++) {
            dp[i] = max(dp[i - 2] + nums[s + i - 1],  // 搶
                        dp[i - 1]);                  // 不搶
        }
        return dp[n];
    }
};

take(nums, s, e) 就是 198 的 sentinel 版,只是把「第 i 間」對應到區間內的 nums[s+i−1]。rob() 只負責剪兩刀、取 max。

寫法 B · take 滾動兩變數// O(1) SPACE

p213_o1.cpp C++17 
class Solution {
public:
    int rob(vector<int>& nums) {
        int n = nums.size();
        if (n == 1) return nums[0];
        return max(take(nums, 0, n - 2), take(nums, 1, n - 1));
    }

    int take(vector<int>& nums, int s, int e) {
        int prev2 = 0, prev1 = 0;        // dp[i-2]、dp[i-1]
        for (int i = s; i <= e; i++) {
            int cur = max(prev1, prev2 + nums[i]); // 不搶 vs 搶
            prev2 = prev1;
            prev1 = cur;
        }
        return prev1;
    }
};

滾動版連 n==2 都不用特判:區間 [0..0]、[1..1] 各自只掃一間,take 天然回傳那間的錢,max 後就是兩間取大。空間降到 O(1)。

複雜度分析// COMPLEXITY

T = O(n)　　S = O(n)(A) / O(1)(B)

兩次線性掃描 = 2n 次比較 = O(n)。A 每輪開一條 dp;B 每輪只用兩個變數。

邊界與陷阱// PITFALLS

陷阱後果正解
直接套 198 線性版可能同搶頭尾 → 違規去頭 / 去尾各跑一次取 max
忘記 n==1 特判take(0,−1) 空區間、漏算n==1 直接回傳 nums[0]
兩區間切錯漏房或重疊到頭尾去尾 [0..n−2]、去頭 [1..n−1]
以為要三種切法多算「頭尾都不搶」其實被涵蓋兩種就夠(至少一端不搶)

註:「頭尾都不搶」的方案,同時屬於 Run A 與 Run B 的可行域,兩輪都會考慮到,不需要第三種切法。
