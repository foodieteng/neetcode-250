最長遞增子序列

題目簡介// THE QUESTION

QUESTION

給整數陣列 nums,回傳最長嚴格遞增子序列的長度。子序列可不連續(挑出部分元素、保持原順序),但必須嚴格遞增。

兩條經典路線:O(n²) DP(dp[i] = 以 i 結尾的 LIS 長度,往回找前驅)與 O(n log n) 貪心 + 二分(維護 tails[],用 lower_bound 替換)。兩者都要掌握 —— 面試常先寫 DP,再被追問優化。

2

兩種解法// TWO ROUTES

A 枚舉所有子序列 O(2ⁿ)
每個元素選或不選,再檢查是否遞增。指數爆炸。
× REJECT · TLE
B DP:以 i 結尾的 LIS O(n²)
dp[i]=1+max(dp[j])(j<i, nums[j]<nums[i])。直觀好寫,n≤2500 可過。
○ OK · 可用
ADOPT
C tails + 二分(lower_bound) O(n log n)
tails[k] = 長度 k+1 遞增子序列的最小結尾。每個 num 用二分找位置替換或接長,tails.size() 就是 LIS 長度。
● ADOPT · O(n log n)
3

核心策略// STRATEGY

  1. DP 版:dp[i] 初始 1;j 從 i−1 往回,nums[j]<nums[i] 就 dp[i]=max(dp[i], dp[j]+1)。答案 max(dp)。
  2. tails 版:對每個 num,lower_bound(tails, num):落在尾端就 push_back,否則覆蓋那個位置。
  3. 關鍵:tails 恆遞增,所以能二分;它的長度是 LIS,但內容不一定是某條真正的 LIS。

lower_bound vs upper_bound// STRICT vs NON-DECREASING

這對函式是 tails 解法的靈魂開關。兩者都在已排序區間二分,回傳一個迭代器:

函式回傳「第一個…」tails 用它 →子序列類型
lower_bound(v, x)第一個 ≥ x 的位置相等的也被替換嚴格遞增(本題)
upper_bound(v, x)第一個 > x 的位置相等的往後、可接長非嚴格遞增(允許相等)
LEMMA · 一個字之差,決定「能不能相等」

遇到與現有結尾相等的 num 時:lower_bound 會指到那個相等值(第一個 ≥ num)並覆蓋它 —— 長度不增,等於「不允許相等」→ 嚴格遞增。upper_bound 會跳過相等值、指到第一個更大的(或尾端),於是相等的 num 可能接長 → 允許相等(非遞減)。

最鮮明的例子 nums=[7,7,7,7]:嚴格 LIS 用 lower_bound = 1(每個 7 都覆蓋同一格);最長非遞減子序列用 upper_bound = 4(每個 7 都接在後面)。想允許相等,就把 lower_bound 換成 upper_bound,其餘一字不改。

複雜度// COMPLEXITY

DP:T = O(n²)　　tails:T = O(n log n)　　S = O(n)

DP 雙層迴圈 n²;tails 每個元素一次二分 log n。n≤2500 兩者都可過,但 n 大時只有 O(n log n) 撐得住。

7

一句洞察// INSIGHT

DP 記「以 i 結尾的最長」、取全體 max;tails 記「各長度的最小結尾」、用二分維護長度。 要嚴格遞增用 lower_bound、要允許相等改 upper_bound —— 一字之差。
