重點回顧// RECAP

面向要點
DP 狀態dp[i] = 以 nums[i] 結尾的 LIS 長度
DP 轉移dp[i]=1+max(dp[j] : j<i, nums[j]<nums[i]),答案 max(dp)
tails 定義tails[k] = 長度 k+1 遞增子序列的最小結尾
tails 操作lower_bound:接長(push)或覆蓋;長度 = LIS
嚴格 vs 非遞減lower_bound(嚴格)/ upper_bound(允許相等)
複雜度DP O(n²) / tails O(n log n)

觀念題// QUIZ

Q1. DP 版為什麼回傳 max(dp) 而不是 dp[n−1]?
Q2. tails 解法中,「覆蓋某個位置」為什麼不會讓最終長度變短?它到底改善了什麼?
Q3. tails 陣列的內容是否就是一條真正的 LIS?若不是,為什麼它的長度還是對的?
Q4. 把 lower_bound 換成 upper_bound 會得到什麼?用 [7,7,7,7] 說明兩者答案差在哪、為什麼。
Q5. tails 為什麼永遠保持遞增?這個性質對 O(n log n) 為什麼是必要的?

範例 Trace// nums=[10,9,2,5,3,7,101,18]

// tails 版逐步(答案應為 4) 10 : push tails=[10] 9 : 覆蓋 pos0 tails=[9] 2 : 覆蓋 pos0 tails=[2] 5 : push tails=[2,5] 3 : 覆蓋 pos1(5→3) tails=[2,3] 7 : push tails=[2,3,7] 101: push tails=[2,3,7,101] 18 : 覆蓋 pos3(101→18) tails=[2,3,7,18] return tails.size() = 4
注意最後 tails=[2,3,7,18] 剛好是一條真的 LIS,但這是巧合 —— 中途 tails 曾是 [2,3,7,101],把 101 覆蓋成 18 讓「長度 4 的最小結尾」更小。 長度一路都對(4),但 tails 的內容只保證「各長度的最小可能結尾」,不保證是某條實際子序列。

應用題// APPLY

App 1. 354 俄羅斯套娃信封:二維 (w,h) 要嚴格巢狀。為什麼先按 w 升冪、w 相同時 h 降冪排序,再對 h 跑 LIS?那個「降冪」是為了什麼?
App 2. 若還要還原出一條實際的 LIS(不只長度),tails 解法要額外記什麼?或改用哪種解法比較好回溯?
App 3. 673 最長遞增子序列的個數:要數有幾條最長的。O(n²) DP 要多維護什麼陣列?轉移怎麼分「更長」與「等長」兩種情況?
// ANSWERS
A1. 因為 dp[i] 只算「以 i 結尾的 LIS」,而真正的最長遞增子序列可能結尾在中間某個位置,不一定是最後一個元素。例如遞增段在前、後面接一堆小數,LIS 結尾在中段。所以要掃過整條 dp 取最大值。dp[n−1] 只反映「必須用到最後一個元素」的情況,會漏掉更長的解。
A2. 覆蓋只把「長度 k+1 的結尾值」換成更小的 num,長度 k+1 這件事不變。改善的是「潛力」:結尾越小,之後越容易有元素比它大、能接上去繼續變長。所以覆蓋不減長度、只讓未來更可能接長 —— 是純粹的優化,不會弄短答案。真正增加長度的是 push_back(遇到比所有結尾都大的 num)。
A3. 不是,tails 的內容通常不是一條實際存在的 LIS(見範例:[2,3,7,18] 雖巧合成立,但中途覆蓋會混入不同子序列的結尾)。但它的長度永遠正確,因為可以證明:tails[k] 始終等於「所有長度 k+1 的遞增子序列中最小的結尾」這個量,是良好定義且單調的;每次操作都正確維護它,所以 tails.size()(最大的有效 k+1)就是 LIS 長度。長度對、內容不一定對。
A4. 換成 upper_bound 會得到最長非遞減(允許相等)子序列。差別在遇到與現有結尾相等的 num:lower_bound 指向那個相等值(第一個 ≥ num)並覆蓋 → 長度不增(不允許相等,嚴格);upper_bound 跳過相等、指向第一個更大的或尾端 → 相等的 num 可接長(允許相等)。[7,7,7,7]:lower_bound 每個 7 都覆蓋第 0 格 → 長度 1;upper_bound 每個 7 都 push → 長度 4。
A5. 因為每次操作要嘛 push_back 一個「比全部大」的值(尾端仍最大,保持遞增),要嘛把某位置覆蓋成 num,而 num 恰好 ≥ 前一格(它是第一個 ≥ num 的位置,代表前一格 < num)且 < 原值(更小),覆蓋後仍夾在中間 → 遞增不破。這個單調性正是能用二分的前提:唯有已排序的陣列,lower_bound 才是 O(log n),整體才 O(n log n)。若 tails 不遞增,就只能線性掃、退回 O(n²)。
App 1. 排序後對 h 跑 LIS,是把二維巢狀降成一維遞增。w 升冪保證後面的信封 w 不小於前面;但相同 w 不能巢狀(要嚴格大)。若相同 w 的 h 也升冪,LIS 可能把兩個同 w、h 遞增的信封當成可套 —— 但它們 w 相等,不合法。把相同 w 的 h 排成降冪,同 w 群內 h 遞減,LIS 就最多只取一個(因為要遞增),自動排除「同寬」的假巢狀。這是用排序技巧把一維的約束編碼進去。
App 2. tails 解法不利於回溯(內容非真序列)。要還原一條 LIS,常用 O(n²) DP + parent 指標:算 dp[i] 時記錄「是從哪個 j 接來的」prev[i]=j,最後從 max(dp) 的位置沿 prev 往回走、反轉即得序列。或在 tails 版額外記每個 num 被放進 tails 時的「位置索引」與前驅,能在 O(n log n) 內重建,但實作較繁 —— 面試要序列通常給 O(n²) + parent 最清楚。
App 3. 多維護 count[i] = 以 i 結尾的最長遞增子序列條數。轉移時對每個 j<i 且 nums[j]<nums[i]:若 dp[j]+1 > dp[i](發現更長)→ dp[i]=dp[j]+1; count[i]=count[j](長度刷新,條數繼承 j 的);若 dp[j]+1 == dp[i](等長)→ count[i] += count[j](多一批同長的接法)。最後把所有 dp[i]==maxLen 的 count[i] 加總。核心是把「最佳值」與「達到最佳值的方法數」分開兩個陣列同步維護 —— 這是「最優值 + 計數」型 DP 的通用套路。
