重點回顧// RECAP

面向要點
定義T₀=0, T₁=1, T₂=1,之後 = 前三項之和
轉移t[i] = t[i−1] + t[i−2] + t[i−3]
base三個:t[0]=0, t[1]=1, t[2]=1
特判n==0→0、n≤2→1(避免越界)
優化只依賴前三項 → 三變數滾動 O(1)
與費氏依賴窗口 2→3,骨架相同

觀念題// QUIZ

Q1. 為什麼 base 要三個、迴圈從 i=3 起?和費氏(兩個 base、從 2 起)差在哪個數字、為什麼?
Q2. 為什麼要先特判 n==0 和 n≤2?不特判會怎樣?
Q3. 滾動三變數版本 a=b; b=c; c=cur; 的順序為什麼不能亂?寫反會怎樣?
Q4. 純遞迴 trib(n)=trib(n-1)+trib(n-2)+trib(n-3) 為什麼比費氏的純遞迴還慢?
Q5. 若定義成「加前 k 項的線性遞迴」,base 數、滾動變數數、每格加法次數各是多少?

範例 Trace// n = 5

// 填表 t[0]=0 t[1]=1 t[2]=1 t[3]=t[0]+t[1]+t[2]=0+1+1=2 t[4]=t[1]+t[2]+t[3]=1+1+2=4 t[5]=t[2]+t[3]+t[4]=1+2+4=7 return t[5] = 7
// 滾動三變數(等價) a=0 b=1 c=1 i=3: cur=0+1+1=2 → a=1 b=1 c=2 i=4: cur=1+1+2=4 → a=1 b=2 c=4 i=5: cur=1+2+4=7 → a=2 b=4 c=7 return c = 7
泰波那契數列 0,1,1,2,4,7,13,24,44,… 每項都是前三項之和 —— 和費氏 0,1,1,2,3,5,8,…(前兩項之和)長得很像,差別只在「往回看三項還是兩項」。同一個填表骨架,窗口大小不同而已。

應用題// APPLY

App 1. 把 1137 看成「爬樓梯:每次可跨 1、2 或 3 階,求方法數」——為什麼兩者答案結構一樣?轉移式對得起來嗎?
App 2. 若 n 很大(例如 10⁹),線性 O(n) 太慢。有什麼方法能做到 O(log n)?(提示:矩陣快速冪)
App 3. 若答案可能超出 int(改成大 n 且要精確值),程式要怎麼調整?
// ANSWERS
A1. 因為轉移 t[i]=t[i−1]+t[i−2]+t[i−3] 用到 i−3,要先備好三個初始值 t[0],t[1],t[2] 才能從 i=3 開始遞推。費氏轉移只用到 i−2,所以兩個 base、從 i=2 起。差別的那個數字就是「依賴窗口大小」:泰波那契往回看 3 項,費氏 2 項 —— base 數與起始 i 都跟著它走。
A2. 當 n≤2 時,陣列 t(n+1) 的大小 < 3,但我們要寫 t[0]=0; t[1]=t[2]=1; —— t[2](甚至 t[1])可能越界,是未定義行為 / 崩潰。所以先特判把小情況直接回傳(T₀=0、T₁=T₂=1),保證進到建表分支時 n≥3、陣列夠大。
A3. a,b,c 代表 t[i−3],t[i−2],t[i−1]。算完 cur=a+b+c(=t[i])後,要把窗口整體往前滑一格:新的 t[i−2] 是舊 b、新 t[i−1] 是舊 c、新的最前是 cur。所以 a=b; b=c; c=cur; 必須由左而右:先把 b 存進 a,再把 c 存進 b,最後 cur 進 c。若順序寫反(例如先 c=cur),就會用被覆蓋過的新值去更新別人,結果全錯。
A4. 純遞迴每次展開成三個子呼叫(費氏是兩個),呼叫樹的分支度更大,trib(k) 被重複計算的次數以約 3ⁿ 級別成長(費氏約 2ⁿ)。同樣的「重疊子問題」問題更嚴重,更需要 DP / 記憶化把每個子問題壓到只算一次,降到 O(n)。
A5. 「加前 k 項」的線性遞迴:需要 k 個 base(前 k 個初始值)、滾動優化留 k 個變數、每格做 k−1 次加法、迴圈從 i=k 起。費氏 k=2、泰波那契 k=3。這是同一套模板,k 一改全部跟著改 —— 掌握這個「參數化」就一次通吃整個家族。
App 1. 對得起來。「每次可跨 1/2/3 階,到第 i 階的方法數」= 最後一步從 i−1(跨1)、i−2(跨2)或 i−3(跨3)來,三類不重不漏 → f[i]=f[i−1]+f[i−2]+f[i−3],正是泰波那契的遞迴。差別只在 base 的語意(方法數 vs 數列定義值)與初值設定,但轉移結構完全相同 —— 這也是為什麼 70(跨 1/2)= 費氏、跨 1/2/3 = 泰波那契。
App 2. 用矩陣快速冪:把 [t[i], t[i−1], t[i−2]]ᵀ 的遞推寫成一個 3×3 轉移矩陣 M 乘上上一個向量。那麼 Tₙ 可由 Mⁿ 得到,而 Mⁿ 用快速冪(平方倍增)在 O(log n) 次矩陣乘法內算完,每次矩乘是常數 3³。總複雜度 O(log n),適合 n 極大的情形。(費氏也有同款 2×2 矩陣快速冪。)
App 3. 換更大的整數型別:long long(到約 9.2×10¹⁸)或 unsigned long long;若還要更大就用大數(BigInteger)或每步取模(若題目要 mod)。本題 n≤37、答案保證 < 2³¹,用 int 剛好;但一旦放寬 n,先想「答案會不會溢位」是必要的一步 —— DP 的值域和 n 一樣要先估。
