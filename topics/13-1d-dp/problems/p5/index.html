題目簡介// THE QUESTION

QUESTION

給一個字串 s,回傳它最長的回文子字串(連續、且正著讀反著讀一樣)。1 ≤ s.length ≤ 1000。

關鍵觀察:一個回文剝掉頭尾兩字後,剩下的仍是回文。反過來,只要「頭尾字元相同」且「中間那段是回文」,整段就是回文 —— 這個遞迴結構讓我們用區間 DP,由短的子字串推出長的。

2

為什麼用區間 DP// WHY INTERVAL DP

A 暴力:枚舉所有子字串逐一判斷 O(n³)
O(n²) 個子字串,每個回文檢查 O(n)。n=1000 → 約 10⁹,超時。
× REJECT · TLE
B 中心擴展:每個中心往兩邊擴 O(n²) / O(1)
很好、空間更省;但要分「奇中心 / 偶中心」兩種展開,細節易錯。
○ OK · 可用
ADOPT
C 區間 DP:dp[i][j] 由內層推外層 O(n²) / O(n²)
dp[i][j] = s[i..j] 是否回文 = s[i]==s[j] && dp[i+1][j-1]。依長度由小到大填表,系統化、好推廣(數回文子串 647 同套路)。
● ADOPT · O(n²)
3

核心策略 · 三步驟// STRATEGY

  1. base:所有長度 1 的 dp[i][i]=true;長度 2 的 dp[i][i+1]= (s[i]==s[i+1])。
  2. 轉移:長度 L 從 3 到 n,dp[i][j]= s[i]==s[j] && dp[i+1][j-1]。
  3. 記錄最長:每次 dp[i][j]=true 時,用 L 更新答案的起點與長度。

關鍵推導// WHY FILL BY LENGTH

LEMMA · 填表順序決定正確性

轉移 dp[i][j] = s[i]==s[j] && dp[i+1][j-1] 用到的內層 dp[i+1][j-1],長度剛好比 dp[i][j] 少 2。所以只要先把所有短的算完,再算長的,內層一定已就緒。

這就是為什麼外層迴圈跑長度 L(由小到大),而不是按 i 或 j 順序 —— 在二維表上,這相當於沿對角線一條一條往右上推。長度 1、2 當 base 先鋪好,長度 3 以上才能安全遞推。

6

複雜度// COMPLEXITY

T = O(n²)　　S = O(n²)

表格有 O(n²) 格,每格 O(1) 轉移。空間為那張表 O(n²)。若改用中心擴展,時間一樣 O(n²) 但空間可降到 O(1)。

7

一句洞察// INSIGHT

回文的遞迴結構是「頭尾相同 + 內層也是回文」。 區間 DP 把它變成 dp[i][j]=s[i]==s[j] && dp[i+1][j-1],依長度由短到長填表,內層永遠先算好。
