重點回顧// RECAP

面向要點
狀態dp[i][j] = s[i..j] 是不是回文
轉移dp[i][j] = s[i]==s[j] && dp[i+1][j-1]
base長度 1 全 true;長度 2 看頭尾是否相同
填表順序長度 L 由小到大(內層短 2,先算好)
取答案每次 true 用 L 更新最長起點/長度
替代解中心擴展:同 O(n²) 時間、O(1) 空間

觀念題// QUIZ

Q1. 為什麼轉移是 s[i]==s[j] && dp[i+1][j-1]?這兩個條件各代表什麼?
Q2. 為什麼外層迴圈要跑「長度 L」而不是直接雙層跑 i、j?按 i,j 順序會出什麼問題?
Q3. 為什麼長度 1 和長度 2 要當 base 單獨處理,不能丟進長度 ≥3 的通式?
Q4. 中心擴展法為什麼要對每個位置做兩種中心((c,c) 與 (c,c+1))?只做一種會漏什麼?
Q5. 這題(最長回文子字串)和「最長回文子序列(516)」的 DP 差在哪?為什麼一個要求連續、一個不用?

範例 Trace// s = "cbbd"

// 區間 DP 填表 s = c b b d (index 0 1 2 3) 長度1:dp[0][0]=dp[1][1]=dp[2][2]=dp[3][3]=true ans={0,1} 長度2:dp[0][1]=(c==b)? no dp[1][2]=(b==b)? yes → true ans={1,2} dp[2][3]=(b==d)? no 長度3:dp[0][2]=(c==b)? no ; dp[1][3]=(b==d)? no 長度4:dp[0][3]=(c==d)? no ans={1,2} → return s.substr(1,2) = "bb"
注意長度 4 的 dp[0][3] 根本沒機會靠內層 —— 頭尾 c≠d 直接 false。 區間 DP 的美在於:只要頭尾一不合,整格立刻 false,不必再看內層;頭尾相合時才花那一次 O(1) 查表看內層。

應用題// APPLY

App 1. 647 回文子串計數:要數出總共有幾個回文子字串。同一張 dp 表怎麼改?
App 2. 516 最長回文子序列:改成可不連續挑字元。轉移式會變成什麼?為什麼變成求長度而非布林?
App 3. 若字串長度到 10⁵,O(n²) 太慢。你聽過哪個 O(n) 專門解最長回文的演算法?(提示:Manacher)
// ANSWERS
A1. s[i]==s[j] 是「頭尾兩字相同」;dp[i+1][j-1] 是「剝掉頭尾後的內層也是回文」。兩者同時成立,s[i..j] 才是回文 —— 這正是回文的遞迴定義:一層對稱的殼 + 一個對稱的核。少任一個都不行:頭尾相同但內層不對稱(如 "abca")不是回文;內層是回文但頭尾不同(如 "xbbz")也不是。
A2. 因為轉移用到內層 dp[i+1][j-1],它的長度比 dp[i][j] 少 2。若按 i 遞增、j 遞增的自然順序填,算到 dp[i][j] 時 dp[i+1][…](下一列)還沒算 —— 內層是「未來」的格子,拿到的是初始 false,答案全錯。以長度由小到大當外層,保證每次要用的內層(短 2)都已完成。在二維表上這相當於沿對角線往右上推。
A3. 因為通式 dp[i][j]=s[i]==s[j] && dp[i+1][j-1] 需要「內層」存在。長度 1(i==j)剝掉頭尾後是空,長度 2(j=i+1)剝掉後 i+1 > j-1 也是空 —— 內層索引會交叉/越界,dp[i+1][j-1] 沒有意義。所以直接規定:長度 1 必回文、長度 2 看頭尾,當作遞迴的錨點。長度 ≥3 內層才保證是合法區間。
A4. 因為回文有奇偶兩種對稱中心。奇數長度回文(如 "aba")的中心是單一字元 → 從 (c, c) 往外擴;偶數長度回文(如 "bb")的中心是兩字之間的縫 → 從 (c, c+1) 往外擴。只做奇中心會完全漏掉所有偶數長度回文。兩種各做一遍、共 2n 個中心,才覆蓋全部。
A5. 子字串(本題)要連續,所以 dp[i][j] 是布林(這段是不是回文),頭尾不同就整段作廢。子序列(516)可跳字元,所以 dp[i][j] 存「s[i..j] 內最長回文子序列的長度」:s[i]==s[j] 時 = dp[i+1][j-1]+2,不同時 = max(dp[i+1][j], dp[i][j-1])(可各丟一端繼續找)。連續版頭尾不合直接 false;子序列版頭尾不合還能退一端,所以是求 max 的最佳化,而非布林判定。
App 1. 把 dp 從「找最長」改成「計數」:每當 dp[i][j]==true 就 count++。因為每個為真的 (i,j) 恰對應一個回文子字串,掃完整張表累加即得總數。轉移與填表順序完全不變,只是把「更新最長」換成「累加計數」—— 這就是 647,和本題共用同一套區間 DP 骨架。
App 2. 狀態改成長度:dp[i][j] = s[i..j] 內最長回文子序列長度。轉移:s[i]==s[j] → dp[i+1][j-1] + 2(頭尾配對,長度 +2);s[i]!=s[j] → max(dp[i+1][j], dp[i][j-1])(丟掉頭或丟掉尾,取較好)。base dp[i][i]=1。變成求 max 是因為子序列允許「放棄一端繼續找」,不像子字串必須連續、頭尾不合就死。
App 3. Manacher 演算法,O(n)。它在字元間插入分隔符把奇偶中心統一,並利用「已知回文的對稱性」重用先前算過的半徑,避免每個中心都從頭擴展。實務上最長回文若要 O(n) 就用它;但 Manacher 細節多、面試較少要求手寫,通常先給 O(n²) 的 DP 或中心擴展,再口頭提一句「可用 Manacher 優化到線性」即可。
