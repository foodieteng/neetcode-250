寫法 A · 布林 DP(你的版本)// 0/1 KNAPSACK

p416.cpp C++17 
class Solution {
public:
    bool canPartition(vector<int>& nums) {
        int target = accumulate(nums.begin(), nums.end(), 0);
        if (target & 1) return false;       // 總和為奇數 → 不可能平分

        target /= 2;                       // 目標:湊出總和的一半

        vector<bool> dp(target + 1, false);
        dp[0] = true;                     // 空集合湊出 0

        for (int num : nums) {
            for (int i = target; i >= num; i--) {  // 由大到小 → 每個數用一次
                if (dp[i - num]) {
                    dp[i] = true;
                }
            }
        }

        return dp[target];
    }
};

核心在內層 i = target … num 遞減:更新 dp[i] 時,dp[i−num] 讀到的是還沒放本個 num 的值,所以每個數對每個和至多貢獻一次(0/1)。把它改成遞增就會變成完全背包。

寫法 B · bitset 位元平移// ~64× FASTER

p416_bitset.cpp C++17 
class Solution {
public:
    bool canPartition(vector<int>& nums) {
        int sum = accumulate(nums.begin(), nums.end(), 0);
        if (sum & 1) return false;
        int target = sum / 2;

        bitset<10001> dp;              // 位 s = 和 s 是否可達;target ≤ 10000
        dp[0] = 1;

        for (int num : nums)
            dp |= dp << num;            // 每個可達和 s → s+num 也可達

        return dp[target];
    }
};

dp << num 把「所有可達的和」整體往上平移 num,|= 併回去 —— 一行就完成「每個可達和 s → s+num 也可達」,正是動畫 BAND 2 的操作。用 64-bit 字並行,常數快約 64 倍。(這裡不需在意 0/1 vs 完全:每個 num 只 shift 一次,天然是選/不選。)

複雜度分析// COMPLEXITY

T = O(n × target)　　S = O(target)

A:雙層迴圈 n × target。B:同樣 n × target 但除以字長 64 → O(n × target / 64)。空間都是 O(target)。
