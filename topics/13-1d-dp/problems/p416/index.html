題目簡介// THE QUESTION

QUESTION

給整數陣列 nums,問能否把它分成兩個子集,且兩邊總和相等。能則回傳 true,否則 false。

關鍵轉化:兩邊相等 ⟺ 每邊都是總和的一半。所以總和必須是偶數,問題變成:能不能挑出一個子集,其和恰為 target = 總和 / 2? 這正是經典的 0/1 背包(子集和) —— 每個數只能選或不選一次。

2

為什麼用 0/1 背包// WHY DP

A 枚舉所有子集 O(2ⁿ)
n 到 200,2²⁰⁰ 完全不可能。大量子集和重複。
× REJECT · TLE
B 貪心:湊近 target 就好 錯
子集和沒有貪心結構,[1,5,11,5] 這類要精準命中 target,不是「接近」。
× REJECT · 反例
ADOPT
C 0/1 背包:dp[s] = 和 s 是否可達 O(n × target)
dp[s] = 用已看過的數能否湊出和 s。每放一個 num,凡 dp[s−num] 可達 → dp[s] 也可達。內層由大到小掃保證每個數只用一次。
● ADOPT · O(n·T)
3

核心策略 · 三步驟// STRATEGY

  1. 轉化:總和為奇數 → 直接 false;否則 target = 總和 / 2。
  2. 0/1 背包:dp[0]=true;對每個 num,s 從 target 到 num(由大到小),dp[s] |= dp[s−num]。
  3. 回傳:dp[target]。

關鍵推導// WHY SWEEP DOWN

LEMMA · 由大到小掃 = 每個數只用一次

放入 num 時要更新 dp[s] |= dp[s−num]。若內層 s 由大到小掃,算 dp[s] 時 dp[s−num] 還是「尚未放入本個 num」的舊值 —— 所以這個 num 對每個 s 至多貢獻一次,正是 0/1 背包(每個數選或不選)。

反之若由小到大掃,dp[s−num] 會是「已放過本 num」的新值,同一個數就能被重複累加 —— 那是完全背包(322)。同一份轉移式,只靠內層掃描方向就切換「各一次 ↔ 無限量」。這是背包家族最該記牢的開關。

6

複雜度// COMPLEXITY

T = O(n × target)　　S = O(target)

target ≤ 總和/2 ≤ 200×100/2 = 10⁴,n ≤ 200 → 至多約 2×10⁶,很快。空間為布林陣列 O(target)。用 bitset 可再快約 64 倍。
