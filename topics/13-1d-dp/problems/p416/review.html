Q2. 為什麼內層 i 要由大到小掃?若由小到大會發生什麼錯誤?用「同一個數被用兩次」說明。

Q4. 這題和 322 零錢兌換 的程式骨架幾乎一樣,唯一的關鍵差別是哪裡?各自對應背包的什麼變體?
Q5. bitset 版 dp |= dp << num 為什麼等價於「每個可達和 s → s+num 也可達」?為什麼它不需要擔心由大到小/由小到大?

範例 Trace// nums=[1,2,3,4]

// 可達和逐步擴張(target=5) sum=10 (偶) → target=5 dp[0]=T 可達 {0} 放 1: dp[1]=T 可達 {0,1} 放 2: dp[3]=dp[1], dp[2]=dp[0] 可達 {0,1,2,3} 放 3: dp[5]=dp[2], dp[4]=dp[1] 可達 {0,1,2,3,4,5} dp[5]=true → return true(子集 {2,3}=5)
dp[5] 是靠 dp[5−3]=dp[2](= 用 {2} 湊出 2)再加這個 3 得到的 —— 對應子集 {2,3}。 DP 只記「和 5 可達」這個布林,不記是哪些數;但每個 true 背後都對應至少一個合法子集。

應用題// APPLY

App 1. 494 目標和:每個數前面放 + 或 −,問有幾種方式讓總和 = target。怎麼把它化簡成一個子集和問題?(提示:設正號子集和為 P)
App 2. 1049 最後一塊石頭 II:把石頭分兩堆相撞,求最小剩餘。為什麼它等於「讓兩堆盡量接近總和一半」?和本題什麼關係?
App 3. 若不只要 true/false,還要回報具體是哪個子集,程式要怎麼調整?為什麼一維滾動陣列在這時會不夠用?
// ANSWERS
A1. 若能分成兩個等和子集,兩邊各是總和的一半 S/2;反之只要能挑出一個子集和為 S/2,剩下的自動也是 S/2,兩邊相等。所以問題等價於「存在子集和 = S/2」。而 S/2 必須是整數,若 S 是奇數,S/2 不是整數,不可能有子集和等於它,直接 false。
A2. 由大到小掃,更新 dp[i] 時 dp[i−num] 讀到的是本輪還沒放入這個 num 的舊值,所以 num 至多被用一次(0/1)。若由小到大:算到 dp[i] 前,dp[i−num] 可能已在同一輪被本 num 設成 true,於是 dp[i] 相當於「用了兩個 num」。例:num=3,先把 dp[3] 設 true,接著 dp[6]=dp[3] 又用一次 3 → 誤判 {3,3} 可達,但陣列裡只有一個 3。由大到小就杜絕這種重複。
A3. dp[s] = 「用目前為止看過的數,能不能湊出恰好和 s 的子集」。dp[0]=true 表示空集合湊出和 0 —— 這是一切遞推的起點:每個 dp[s]=dp[s−num] 最終都要回溯到 dp[0] 這個「什麼都不選」的基底。少了它,所有格子都無從變 true,整張表恆 false。
A4. 骨架都是「對每個物品、掃金額、dp[i] 由 dp[i−w] 推」。唯一關鍵差別是內層掃描方向:416 由大到小(每個數只用一次,0/1 背包);322 由小到大(硬幣無限量,完全背包)。另外聚合也不同:416 是布林可達性(|=),322 是最小件數(min(...)+1)。記住「方向 = 用一次還是無限次」這個開關,兩題就統一了。
A5. bitset 的第 s 位代表「和 s 可達」。dp << num 把每個為 1 的位整體左移 num,也就是把每個可達和 s 變成 s+num;|= 再把新舊併起來,等於「原本可達的,加上『各自 +num』後可達的」。它不需擔心方向,是因為右側 dp << num 用的是這次迴圈開始前的整份舊 dp(先算好整個位移結果,再一次 OR 回去),本輪的更新不會回頭餵給自己,天然就是「每個 num 只作用一次」。
App 1. 設正號那組的和為 P、負號組為 N,則 P + N = S(總和)、P − N = target。兩式相加得 P = (S + target) / 2。於是「有幾種 ± 方式湊出 target」= 「有幾個子集其和 = (S+target)/2」—— 變成子集和計數(注意這裡要數方法數,轉移用 dp[s] += dp[s−num],不是布林)。前提是 (S+target) 為偶且非負。這就是把本題的布林可達性換成計數版。
App 2. 兩堆石頭和分別為 A、B(A+B=S),相撞後剩 |A−B|。要最小化 |A−B| = |S − 2A|,就是讓某一堆 A 盡量接近 S/2 但不超過。於是對 target=S/2 跑 0/1 背包,找出 ≤ S/2 的最大可達子集和 best,答案 = S − 2·best。和本題完全同一張 dp 表,只是本題問「能否剛好 = S/2」,1049 問「最接近 S/2 是多少」—— 掃 dp 找最大的 true 即可。
App 3. 要回報子集就得能回溯。作法一:保留二維 dp[i][s](前 i 個數能否湊 s),從 dp[n][target] 往回走:若 dp[i−1][s] 已 true 代表沒選第 i 個,否則選了(s -= nums[i−1])。作法二:記錄每個和「由哪個數達成」。一維滾動陣列就地覆蓋、丟掉了「第幾個物品」的歷史,無法區分某個 true 是哪一輪、用哪些數得到的,所以要回報具體子集就得放棄 O(target) 空間、改存二維表或決策記錄。
