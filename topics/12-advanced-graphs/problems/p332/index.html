題目簡介// THE QUESTION

QUESTION

給一堆機票 tickets[i] = [from, to],每張剛好用一次,從 "JFK" 出發, 重建整條行程。若有多條合法行程,回傳字典序最小的那條。題目保證至少有一條合法行程。

「每條邊用剛好一次」= 找一條歐拉路徑(Eulerian path)。把機場當節點、機票當有向邊, 要走過每條邊各一次、且從 JFK 起。多條解時取字典序最小 → 每步優先走字母最小的目的地。

2

為什麼用 Hierholzer// WHY THIS

A 回溯枚舉所有行程 指數級
嘗試每種走法、遇死路就回退,最壞是階乘 / 指數級。300 張票會 TLE。
× REJECT · TLE
B 貪心走最小、直接順序輸出 會卡死
只顧走字母最小,可能提早走進死路、把還沒用的票困住 → 走不完。順序輸出是錯的。
△ REJECT · 提早卡死
BOTH FAIL · ADOPT
C Hierholzer:貪心走最小 + 死路先入答案 + 最後反轉 O(E log E)
用 multiset 讓每個機場的出邊自動字典序排好。DFS 一路走最小的邊並邊走邊刪; 走到沒有出邊的節點(死路)就把它 push 進 routes(後序),回退繼續。 全部走完後 reverse —— 死路節點自然被排到正確的尾端,detour(繞路)也會被正確接回。
● ADOPT · STABLE
3

核心策略 · 三步驟// STRATEGY

  1. 建圖:graph[from] 用 multiset<string> 存目的地 —— 自動字典序、又能 O(log) 刪除。
  2. DFS 消邊:在 u 時,不斷取出最小的出邊 v、刪掉它、遞迴進 v;直到 u 無出邊。
  3. 後序 push + 反轉:u 無出邊時 routes.push_back(u)。整個 DFS 完成後 reverse(routes),即為答案。

為什麼「後序 + 反轉」是對的// CORRECTNESS

CLAIM · 死路節點必是尾端
  1. 當 DFS 卡在某節點 u(無出邊),u 一定是目前這段路的終點 —— 沒有票能再飛出去。它就是行程(這一段)的最後一個機場,所以最早被 push。
  2. 回退時若某個更早的節點還有未用的邊,DFS 會從那裡再走一個環(detour)回到自己;這個環會被 push 在「已 push 的尾段」之前,反轉後剛好插進正確位置。
  3. 因此 routes 是行程的逆序,最後 reverse 還原。這正是動畫裡 KUL 字母最小卻排到最後的原因 —— 它是死路,最早入 routes。
  4. 因為每一步都走「當前最小」,得到的歐拉路徑就是字典序最小的那條。

複雜度// COMPLEXITY

設 E = 機票數。每條邊被走一次、從 multiset 刪一次(O(log E));reverse 為 O(E)。

T = O(E log E)　　S = O(E)

建圖 + 每條邊一次 O(log E) 刪除;遞迴堆疊與 routes 皆 O(E)。

項目量級說明
TimeO(E log E)multiset 排序 / 刪除的 log 因子
SpaceO(E)圖 + 遞迴堆疊 + routes
遞迴深度O(E)最壞一條長鏈(E ≤ 300,無虞)

一句洞察// INSIGHT

別在走的時候急著輸出。走到死路才 push、最後反轉 —— 死路自動落到尾端,繞路自動接回中間。 每步走最小的邊,就順便拿到字典序最小的歐拉路徑。
