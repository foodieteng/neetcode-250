重點回顧// RECAP

面向要點
問題本質gcd>1 建圖問連通 ⇒ 邊數 O(n²) 不可建
關鍵轉換節點改成「質數」；數字把自己的質因數併成一組
等價命題全連通 ⇔ 出現過的質數落在同一分量
工具最小質因數篩 spf[] + 併查集（路徑壓縮 + 按秩）
兩個 bug篩守門 spf[j]==j；檢查用 Find(spf[v])
特例n==1→true；含 1 且 n>1→false

觀念題// QUIZ

Q1. 為什麼「gcd(a,b) > 1」可以等價換成「a、b 共用至少一個質因數」？
Q2. 為什麼要把「質數」當作併查集的節點，而不是把「數字」當節點？這帶來什麼規模上的差別？
Q3. 篩最小質因數時，內層守門條件為什麼是 spf[j] == j？若誤寫成 spf[j] == i 會怎樣？
Q4. 最後的連通性檢查為什麼是 Find(spf[v]) 而不是 Find(v)？
Q5. 陣列出現 1 時應如何處理？n==1 與 n>1 為何結論不同？

範例 Trace// nums = [4, 3, 12, 8]

// SETUP mx = 12 → spf[] 篩後：spf[4]=2 spf[3]=3 spf[12]=2 spf[8]=2 parent[0..12] 全部 = -1，anchor = -1
// v = 4 (= 2·2) p=spf[4]=2, prev=-1 → anchor 尚未設 ⇒ anchor = 2 prev=2, v=4/2=2, p=spf[2]=2 prev=2 ⇒ Union(2, 2) 無效（同根） prev=2, v=2/2=1 → 停 分量：{2} 3、5… 尚未出現
// v = 3 (= 3) p=spf[3]=3, prev=-1，但 anchor 已是 2 ⇒ 什麼都不做 prev=3, v=3/3=1 → 停 分量：{2}、{3} ← 3 目前是孤島！
// v = 12 (= 2·2·3) p=2, prev=-1（anchor 已設）→ 略；prev=2, v=6 p=2, Union(2,2) 無效；prev=2, v=3 p=3, prev=2 ⇒ Union(2, 3) 合併！；v=1 停 分量：{2, 3} ← 12 同時含 2、3，把兩座島接起來
// v = 8 (= 2·2·2) 全是質因數 2，Union(2,2) 皆無效 分量不變：{2, 3}
// CHECK (anchor = 2) Find(spf[4]) = Find(2) = 2 == Find(2) ✓ Find(spf[3]) = Find(3) = 2 == Find(2) ✓ Find(spf[12]) = Find(2) = 2 == Find(2) ✓ Find(spf[8]) = Find(2) = 2 == Find(2) ✓ // OUTPUT: true
注意 3 一度是孤島，直到 12 這個同時含 2 和 3 的數字把它接回主分量。 這正是「數字把自己的質因數併成一組」在做的事。

應用題// APPLY

App 1. 若值域放大到 10⁹，開不下 spf[] 陣列，你會怎麼對每個數字做質因數分解？整體複雜度變成多少？
App 2. 若不維護 anchor，改成「數 distinct 根的個數」來判斷是否全連通，程式要怎麼寫？兩種寫法各有什麼取捨？
App 3. 進一步問「把整張圖變連通，最少要再加幾條邊（合併幾個分量）」，在這個質因數 DSU 上該怎麼回答？
// ANSWERS
A1. 把 a、b 做質因數分解，gcd(a,b) 就是兩者質因數（取次方較小者）的乘積。gcd > 1 代表這個公因數至少含一個質數 ⇒ 兩者至少共用一個質因數；反之若無共用質因數則 gcd=1。所以「可走訪」＝「共用質因數」。
A2. 用數字當節點的話，要決定「哪兩個數字該連」仍需兩兩比較質因數，逃不掉 O(n²)。改用質數當節點：每個數字只把自己 ≤ log₂A 個質因數連起來，節點數 = 出現過的質數個數（很少），邊也線性。連通性檢查因此從 O(n²) 降到近線性。
A3. spf[j] == j 代表 j 還沒被任何更小的質數標記過，此刻用當前質數 i（由小到大）寫入，即得最小質因數（先到先得）。若寫成 spf[j] == i：由於 spf[j] 初值是 j，幾乎永遠不等於 i，條件幾乎不成立 ⇒ 篩形同虛設，spf 全留初值 j，之後分解合成數會出錯。
A4. 我們 Union 的節點是質數，只有質數的 parent 被寫過。合成數 v（如 12）從來沒被 union，Find(v) 會回到它自己、與任何質數不同根，且 v 可能超出有意義範圍。取 spf[v]（v 的一個質因數，必在 v 的分量裡）才能正確代表 v 的所屬分量。
A5. 1 沒有質因數，與任何數 gcd = 1，無法與別人走訪。n == 1 時陣列只有它自己，沒有任何「一對」需要連通 ⇒ true；n > 1 只要出現 1，它就是永遠到不了的孤點 ⇒ false。程式用開頭 if(n==1) return true 與迴圈內 if(v==1) return false 分別處理。
App 1. 改用試除法：對每個數字用 2..√v 試除取質因數，單數 O(√v)，整體 O(n√A)。若要更快可先線性篩出 √(10⁹) ≈ 31623 以內的質數，只拿這些質數試除，剩下的殘值（>1）本身就是一個大質數。分解出的質因數同樣做 union，其餘框架不變。
App 2. 收集所有出現過的質數，對每個取 Find 丟進 set 數 distinct 根；size == 1 ⇒ 全連通。取捨：anchor 版本 O(n·α)、常數更小、無額外容器；數根版本語意更直觀、且能順便回報「幾個分量」，但要多一個 set 掃過所有質數（O(質數種類·α)）。兩者結論一致。
App 3. 答案是（質數分量數 − 1）：把 k 個分量接成一張連通圖，最少加 k−1 條邊（等同「把 k 棵樹接成一棵」）。在 DSU 上就是統計 distinct 根的個數 k，回傳 k−1。若 k=1 代表已連通，加 0 條。
