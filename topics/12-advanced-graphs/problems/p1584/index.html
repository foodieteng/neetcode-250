題目簡介// THE QUESTION

QUESTION

平面上 n 個點 points[i] = (x, y)。任兩點之間可連一條邊,成本 = 兩點的曼哈頓距離 |x₁−x₂| + |y₁−y₂|。求把所有點連通(任兩點間有路徑)所需的最小總成本。

「用最小總成本把所有點連成一片、且不需要環」——這正是最小生成樹(Minimum Spanning Tree)的定義。 把每個點看成頂點、任兩點的曼哈頓距離看成邊權,答案 = 這張完全圖的 MST 權重和。n−1 條邊剛好連通 n 個點。

BOTH FAIL · ADOPT
C 最小生成樹 → Prim 或 Kruskal O(n²·log n)
① Prim(lazy):從一個點開始長樹,min-heap 每次取「樹→樹外」最便宜的跨邊,把新點併入。 ② Kruskal(Union-Find):所有邊照權重由小到大取出,不成環就加(union),加滿 n−1 條為止。 兩者都是貪心且都保證最優,皆 O(n²·log n)。

核心策略// STRATEGY

  1. 建模:頂點 = 各點;邊權 = 任兩點的曼哈頓距離。要在這張完全圖上求 MST。
  2. 兩種算法(見下一頁完整碼):
    • A · Prim:inMST 標記已在樹中的點。從 (0, 起點) 出發,min-heap 反覆彈出最小跨邊;若目標點已在樹中(過期條目)就跳過,否則併入、累加權重、把它到其他點的邊入堆。連滿 n 點結束。
    • B · Kruskal:把所有點對的邊丟進 min-heap(或排序)。逐條取最小,用 Union-Find 檢查兩端是否已同集合:不同就 union、累加;相同(成環)就丟棄。加到 n−1 條即完成。
  3. 收尾:Prim 連滿 n 點、Kruskal 加滿 n−1 條邊時,累計的邊權和就是答案。

複雜度// COMPLEXITY

T = O(n² · log n)　　S = O(n²)

完全圖有 ≈ n²/2 條邊。Kruskal 要處理全部邊(排序/堆),Prim lazy 也會把 O(n²) 條邊入堆,故都是 O(n²·log n)。 稠密圖的最優是 Prim 用陣列(不用 heap)的 O(n²) 版本。

一句洞察// INSIGHT

看到「把所有點用最小總成本連通」就是 MST。邊權 = 曼哈頓距離,套 Prim 或 Kruskal 直接解 —— 關鍵是別把它誤當成「最短路徑」。
