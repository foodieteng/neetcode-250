重點回顧// RECAP

面向要點
問題本質完全圖(邊權 = 曼哈頓距離)的最小生成樹
A · Prim從一點長樹,min-heap 取最小跨邊,inMST lazy 去環,連滿 n 點
B · Kruskal所有邊由小到大,Union-Find 不成環才收,加滿 n−1 條
兩者關係稠密完全圖,兩招得同一棵 MST,皆 O(n²·log n)
易錯誤當最短路;Prim 漏 lazy 去環;Kruskal 比較子方向反
稠密最優Prim 用陣列(不用 heap)可到 O(n²) 時間 / O(n) 空間

範例 Trace · B Kruskal// [[0,0],[2,2],[3,10],[5,2],[7,0]] → 20

// POINTS · 邊權 = 曼哈頓距離 P0(0,0) P1(2,2) P2(3,10) P3(5,2) P4(7,0) 所有點對的邊照權重排序: 3:P1-P3 · 4:P0-P1 · 4:P3-P4 · 7:P0-P3 · 7:P0-P4 · 7:P1-P4 · 9:P1-P2 · …
// KRUSKAL 加邊(Union-Find) w=3 P1-P3 不同集合 → 收。累計 3 {P1,P3} w=4 P0-P1 不同集合 → 收。累計 7 {P0,P1,P3} w=4 P3-P4 不同集合 → 收。累計 11 {P0,P1,P3,P4} w=7 P0-P3 / P0-P4 / P1-P4 皆已同集合 → 成環,全部略過 w=9 P1-P2 不同集合 → 收。累計 20 {P0,P1,P2,P3,P4}
// RESULT 已加 n−1 = 4 條邊 → 5 點全連通,結束 MST 邊:P1-P3(3) · P0-P1(4) · P3-P4(4) · P1-P2(9) // OUTPUT: 3 + 4 + 4 + 9 = 20
遠在天邊的 P2(3,10)不是靠「離某點最近」接入,而是靠整張圖裡對它最省的那條邊(P1–P2=9)。 這正是 MST 與最短路的分別:看的是連通整體的代價,不是單點的距離。

應用題// APPLY

App 1. 若 n 高達 10⁴,把所有 n²/2 條邊入堆會爆記憶體,該怎麼優化?(提示:稠密圖 Prim)
App 2. 若題目改成「已有部分點事先連好(免費)」,再求把其餘連通的最小成本,兩種寫法各怎麼改?
App 3. 若不要總成本,而要輸出MST 用了哪些邊,Prim / Kruskal 各怎麼記錄?
// ANSWERS
A1. 最短路徑求的是「某個起點到各點的個別最短距離」,結果是一棵以起點為根、讓每個點到根最近的樹;而這題要的是「用最小總成本讓所有點彼此連通」,也就是最小生成樹。兩者常常給出不同的樹:最短路樹在意「到根的距離」,MST 在意「所有邊權的總和」。把這題丟給 Dijkstra 會答錯。
A2. 因為同一個點 u 可能被多條不同權重的邊推進 heap(它從樹裡不同點都可達)。當 u 第一次被彈出(以最小跨邊)就已入樹;之後堆裡剩下的都是過期條目。若不 continue,會把 u 重複計入、cnt 也多加,答案偏大甚至死循環。這行就是 lazy Prim 的「延遲刪除」。
A3. Union(u,v) 回傳 false 代表 u 與 v 早已在同一個集合(同一連通塊)。此時再加 u–v 這條邊會形成環——對「樹」是多餘的,而且既然它們已連通,這條邊對連通性毫無貢獻,自然不收(也不加權)。只有回傳 true(真的合併了兩塊)才計入答案。
A4. n 個點的生成樹恰好有 n−1 條邊。Prim 是「點導向」:每併入一個新點 cnt++,連滿 n 個點就成樹。Kruskal 是「邊導向」:每收一條有效邊 add++,收滿 n−1 條就成樹。兩個計數是同一件事的兩面(n 點 ⇔ n−1 邊),到達即可提前結束、不必看完剩下的大邊。
A5. 切分定理(cut property):對任一種「把點分成兩群」的切分,橫跨兩群的邊中權重最小的那條,必屬於某棵 MST。Prim 的切分是「樹內 vs 樹外」,每步取的最小跨邊正好命中;Kruskal 由小到大加邊,每條被收的邊都是它連接的那兩塊之間的最小邊。兩者都在反覆套用切分定理,所以貪心安全、結果最優。
App 1. 用稠密圖 Prim(陣列版):維護 minDist[v] = 各樹外點到樹的最小距離,每輪 O(n) 掃一遍挑最小、把它併入、再 O(n) 更新鄰居距離。完全不入堆,時間 O(n²)、空間 O(n),對稠密完全圖是最優,記憶體也不會爆。
App 2. Kruskal 最自然:先把「已免費連好」的每組點預先 union(成本 0),再照常跑;已同集合的邊會自動被跳過。Prim 則把那些點全部標成 inMST=true 當作初始樹、並把它們到樹外的邊都先入堆,再開始長。兩者本質都是「起始就有若干已連通塊」。
App 3. Kruskal:每次 Union 回傳 true 時,把當前邊 (u,v) 記進結果清單即可。Prim:改存 (權重, 目標點, 來源點),當某點以最小跨邊入樹時,記下「來源 → 目標」這條邊。兩者最後都得到 n−1 條 MST 邊。
