範例 Trace// k=3, row=[[1,2],[3,2]], col=[[2,1],[3,2]]

// ROW TOPO (edges 1→2, 3→2) indegree: 1:0 2:2 3:0 queue 初始 = [1, 3] pop 1 → order=[1] (2 的入度 2→1) pop 3 → order=[1,3] (2 的入度 1→0 → 入隊) pop 2 → order=[1,3,2] rowOrder = [1,3,2] ⇒ 1@列0, 3@列1, 2@列2
// COL TOPO (edges 2→1, 3→2) indegree: 1:1 2:1 3:0 queue 初始 = [3] pop 3 → order=[3] (2 的入度 1→0 → 入隊) pop 2 → order=[3,2] (1 的入度 1→0 → 入隊) pop 1 → order=[3,2,1] colOrder = [3,2,1] ⇒ 3@行0, 2@行1, 1@行2
// PLACE pos[v] = (row, col) 1 → (0, 2) 3 → (1, 0) 2 → (2, 1) matrix[0][2]=1 matrix[1][0]=3 matrix[2][1]=2
// RESULT c0 c1 c2 r0 [ 0 0 1 ] r1 [ 3 0 0 ] r2 [ 0 2 0 ] // OUTPUT: [[0,0,1],[3,0,0],[0,2,0]] ✓ 通過驗證
驗算 rowCondition [1,2]:1 在列 0、2 在列 2,0 < 2 ✓。 colCondition [2,1]:2 在行 1、1 在行 2,1 < 2 ✓。題目要「任一」解,故與官方範例不同排版也對。

應用題// APPLY

App 1. 若題目只問「是否存在合法矩陣」(回傳 bool),程式可以怎麼精簡?複雜度有何改變?
App 2. 若 rowConditions 給的是「a 在 b 下面 k 列以上」這種帶距離的限制,單純拓樸還夠嗎?會往哪個方向想?
App 3. 當條件自我矛盾時,如何進一步找出「環上有哪些數字」以便回報使用者?
// ANSWERS
A1. 一個數字的列只由 rowConditions 決定、行只由 colConditions 決定,兩者互不影響。合成一張圖會把「列限制」和「行限制」混為同一維度的先後關係,製造出不存在的環、或把本來各自可解的兩個問題錯誤綁死。兩維度獨立 ⇒ 分開各排一次才正確。
A2. [a, b] 表「a 必須排在 b 前面(較小索引)」,方向是 a → b。入度 indegree[v] = 「還有幾個數字必須排在 v 之前」。入度為 0 代表 v 沒有未滿足的前置,可以安全放到當前最前位置 —— 這正是 Kahn 演算法一次取一個入度 0 節點的依據。
A3. 無環的 DAG,每個節點都會恰好入隊、出隊一次,最終 order 長度 = k。若存在環,環上每個節點的入度永遠 > 0(彼此互相卡著),永遠進不了佇列,於是 order.size() < k。所以 order.size() != k 就是「排不完 = 有環 = 條件矛盾」,回傳空矩陣。
A4. rowOrder 是 1..k 的一個排列,掃一遍得到的 rowPos 是「數字 → 列索引」的雙射:不同數字必有不同列。colPos 同理是不同行的雙射。因此任兩個數字至少在「列」上就不同(其實行也不同),k 個數字落在 k 個相異格,像一個排列矩陣,絕不重疊。
A5. 數字的值域是 1..k,程式直接用數值當索引(如 indegree[v]、pos[v]、graph[u])。若只開到 k,存取 [k] 就會越界。開 k + 1 讓索引 0..k 都合法(索引 0 空著不用),寫起來最直接。
App 1. 不必真的建矩陣,只要兩次拓樸都成功(長度都等於 k)即存在合法解 ⇒ 回傳 true,否則 false。省掉 O(k²) 的矩陣建置,時間降到 O(k + E)、空間 O(k + E)(不再需要 k×k 陣列)。
App 2. 純拓樸只保證「相對先後」,無法保證「恰好差 k 列」這種帶權距離約束。這時要轉成差分約束系統(difference constraints):把每個限制寫成 pos[b] − pos[a] ≥ w,用最短 / 最長路(Bellman-Ford / SPFA)求解,負環即代表無解。
App 3. 跑完 Kahn 後,入度仍 > 0 的節點就一定在環上或被環「擋在後面」。要精準抓環,可改用 DFS 三色標記(白 / 灰 / 黑):在遞迴堆疊中(灰色)又遇到灰色節點,即偵測到一條回邊,沿堆疊回溯即可列出環上的數字。
