最小體力路徑

題目簡介// THE QUESTION

QUESTION

rows×cols 高度網格。從左上走到右下(上下左右移動),一條路徑的體力 effort = 路徑上相鄰兩格高度差 |Δ| 的最大值。 求所有路徑中最小的 effort。

把相鄰格之間連一條邊、權重 = |高度差|。一條路徑的 effort = 該路徑上最大的邊權。 答案 = 所有路徑中「最大邊權」的最小值 —— 又是最小化瓶頸(minimax)路徑,與 778 同型,只是成本改成邊權 |Δ|。

A DFS 枚舉所有路徑 指數級

× REJECT · TLE
B 當成「邊權和最小」的最短路 目標錯
要最小化的是路徑上邊權的最大值,不是總和。一般 Dijkstra 累加會算錯。
△ REJECT · 目標函數錯
BOTH FAIL · ADOPT
C 最小化瓶頸 → 三種標準解法 O(mn·log)
① Union-Find / Kruskal:邊照 |Δ| 由小到大加入,src 與 dst 連通那刻的權重即答案。 ② Dijkstra 變形:effort 用 max(當前, 邊權) 鬆弛,彈出終點即答案。 ③ 二分答案 + BFS:可達性對門檻單調,二分最小可行 effort。三者同為 O(mn·log(mn))。
● ADOPT · 3 WAYS
3

核心策略// STRATEGY

  1. 重述:答案 = 起點到終點所有路徑中,max(相鄰 |Δ|) 的最小值。
  2. 三種算法(見下一頁完整碼):
    • A · Union-Find:枚舉所有相鄰邊、依 |Δ| 排序、逐一 union,一旦 Find(src)==Find(dst) 回傳當前邊權。
    • B · Dijkstra 變形:dist[cell] = 到此格的最小 effort,鬆弛用 max(d, |Δ|);彈出終點即答案。
    • C · 二分答案 + BFS:二分 effort 門檻 mid,BFS 只跨 |Δ| ≤ mid 的邊,檢查能否到終點。

一句洞察// INSIGHT

effort 只看路徑上最陡的一步。把「相鄰高度差」當邊權,答案就是最小化最大邊權的瓶頸路徑 —— Union-Find、max-Dijkstra、二分答案三招通吃。
