K 站中轉最便宜航班

1

題目簡介// THE QUESTION

QUESTION

有 n 座城市與一批單向航班 flights[i] = [from, to, price]。求從 src 飛到 dst、 最多中轉 k 站（at most k stops）的最低總票價；到不了就回傳 -1。

「中轉 k 站」= 路徑最多用 k+1 條邊（k 個中間城市，加頭尾）。這個「邊數上限」正是本題與一般最短路的差別 —— 純粹的 Dijkstra 只顧「最便宜」，會挑出中轉太多的違規路徑。1 ≤ n ≤ 100 · 0 ≤ k < n · price ≤ 10⁴。

2

為什麼用 Bellman-Ford// WHY BELLMAN-FORD

A DFS 枚舉所有路徑 O(nᵏ⁺¹)
窮舉每條 ≤ k+1 條邊的路徑。分支數隨 k 指數爆炸，稠密圖直接 TLE。
× REJECT · TLE
B 純 Dijkstra（只比價格） O(E log V)
貪心只鎖「最便宜」，會回傳中轉超過 k 的更便宜路 —— 忽略了邊數上限。
△ REJECT · WRONG
BOTH FAIL · ADOPT
C Bellman-Ford · 跑 k+1 輪（或分層 BFS） O(k·E)
每輪用 temp 凍結快照鬆弛所有邊 ⇒ 一輪剛好多用一條邊。跑 k+1 輪 = 最多 k 次中轉，邊數自動封頂。
● ADOPT · CORRECT

關鍵洞察：Bellman-Ford 一輪鬆弛，若每輪都以本輪開頭的凍結值當來源，則第 r 輪之後 dist[v] = 「從 src 出發、最多用 r 條邊到 v」的最低票價。題目「最多 k 站」= 最多 k+1 條邊 ⇒ 跑 k+1 輪剛好。

3

核心策略 · 三步驟// STRATEGY

維護 dist[v] = 目前找到、抵達 v 的最低票價。初始 dist[src]=0，其餘 ∞。

  1. 凍結快照 · 每輪一開始把 dist 複製成 temp。本輪所有鬆弛只讀 temp，不讀本輪剛改過的 dist。
  2. 掃全部邊鬆弛 · 對每條 (u,v,w)，若 temp[u]+w < dist[v] 就更新 dist[v]。因讀 temp，一輪內不會連鎖多跳。
  3. 跑滿 k+1 輪 · 重複步驟 1–2 共 k+1 次。最後 dist[dst] 即答案（仍 ∞ ⇒ -1）。

為何步驟 1 不能省 · 若直接讀 dist，同一輪內 src→a 剛更新的值會立刻被 a→b 接著用， 一輪就偷跑了兩條邊、破壞「一輪 = 一條邊」的計數 ⇒ 可能回傳中轉超過 k 的更便宜路（答案偏小、違規）。

4

動畫演示 · 兩種寫法各一段// n=4, src=0→dst=3, k=1 → 700

官方樣本：航班 0→1:100 · 1→2:100 · 2→0:100 · 1→3:600 · 2→3:200、src=0 · dst=3 · k=1。 兩種寫法都跑 k+1=2 輪/層，看它們如何讓答案是 700 而非 300：

5

為何「一輪 = 一條邊」// WHY ONE EDGE PER ROUND

INVARIANT · 第 r 輪之後 dist[v] 的意義
  1. dist[v] = 從 src 出發、最多用 r 條邊抵達 v 的最低票價。
  2. 歸納：第 r 輪的每個鬆弛 dist[v] ← temp[u] + w，temp[u] 是「≤ r−1 條邊到 u」的最佳值，接上邊 (u,v) ⇒「≤ r 條邊到 v」。因讀 temp、不讀本輪剛改的 dist，同一輪不會串兩條邊。
  3. 「最多 k 站」= 最多 k 個中間城市 = 路徑最多 k+1 條邊。所以跑 k+1 輪後的 dist[dst] 正是答案。
  4. 邊權非負，多繞不會更便宜；故「≤ k+1 條邊」的最佳解不會漏掉更短邊數的更優解（它們在更早的輪已算進 dist）。

實測佐證：對 20 萬組隨機圖，本解與「列舉 exactly-e 條邊 DP」的暴力參考解答案完全一致、零反例； 官方三個樣本亦全數吻合（700 / 200 / 500）。兩種寫法（Bellman-Ford / 分層 BFS）互相對拍也一致。

複雜度// COMPLEXITY

T = (k+1) × O(E) = O(k·E)　　S = O(V)

n ≤ 100、E ≤ n² ⇒ 上限約 100 × 10⁴ = 10⁶，瞬殺。

項目數量備註
TimeO(k·E)k+1 輪 × 每輪掃 E 條邊
SpaceO(V)dist + 每輪 temp（BFS 版是 queue，最壞 O(E)）
輪數k+1「最多 k 站」= 最多 k+1 條邊
數值型別int總價 ≤ (n−1)·10⁴ ≈ 10⁶；相加前判 ≠ INF 防溢位

一句洞察// INSIGHT

「最多 k 站」= 「最多 k+1 條邊」= 「跑 k+1 輪 Bellman-Ford（或 k+1 層 BFS）」。每輪用 temp 凍結快照鬆弛， 讓一輪只前進一條邊、邊數自動封頂。省掉凍結 ⇒ 一輪偷跑多條邊 ⇒ 中轉超標、答案偏小。
