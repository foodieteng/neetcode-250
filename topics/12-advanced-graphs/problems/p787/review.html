範例 Trace · 跑一次 code 看 dist 怎麼變// WORKED TRACE

用官方樣本 1（就是動畫那張）。n=4、src=0、dst=3、k=1 ⇒ 跑 2 輪。

// INPUTn = 4, src = 0, dst = 3, k = 1
flights:
  0 → 1  (100)
  1 → 2  (100)
  2 → 0  (100)
  1 → 3  (600)
  2 → 3  (200)
// INIT
dist = [ 0,  ∞,  ∞,  ∞ ]      ← 只有 src=0 是 0
// 第 0 輪  (temp = [0, ∞, ∞, ∞])
只有邊 0→1 的來源 temp[0]=0 有限：
  dist[1] = min(∞, 0 + 100) = 100
其餘邊來源都是 ∞ ⇒ 無效
dist = [ 0, 100,  ∞,  ∞ ]     ← 「≤ 1 條邊」能到的最低價
// 第 1 輪  (temp = [0, 100, ∞, ∞])
1→2 : dist[2] = min(∞, temp[1]+100) = 200
1→3 : dist[3] = min(∞, temp[1]+600) = 700
2→3 : 想用 temp[2]，但 temp[2]=∞（本輪凍結）⇒ 無效！
      （所以 dist[3] 保持 700，不會被壓成 0→1→2→3 = 300）
dist = [ 0, 100, 200, 700 ]
// OUTPUTdist[dst=3] = 700
（路徑 0 → 1 → 3，恰好中轉 1 站）
觀察：0→1→2→3 只要 300，卻用了兩次中轉（超過 k=1）。正是 temp[2]=∞ 這一格把它擋在門外 —— temp 快照就是「中轉數不准偷跑」的守門員。

// ANSWERS

先自己想過再對答案 — Q 對應 Q、App 對應 App。

A1 · 核心對應

「中轉 k 站」數的是中間經過的城市。一條 src → a₁ → … → dst 若中間有 k 個城市，它就有 k+1 條邊（頭尾各接一段）。

Bellman-Ford「一輪 = 多允許一條邊」，所以要涵蓋「最多 k+1 條邊」的路徑，就得跑 k+1 輪。只跑 k 輪會漏掉剛好用滿 k 站的合法路。

A2 · temp 快照

會讓一輪偷跑多條邊。直接讀 dist 時，本輪 src→a 剛更新的 dist[a]，會馬上被同輪稍後的 a→b 讀去接力，於是一輪就走了 2、3、… 條邊。

// 不用 temp 的後果（本範例 k=1）
第 1 輪若讀 dist：
  1→2 先把 dist[2]=200
  2→3 立刻讀新的 dist[2] ⇒ dist[3]=min(700, 200+200)=400
但 0→1→2→3 用了 2 次中轉，超過 k=1 ⇒ 答案錯（違規變便宜）

所以一定要 temp = dist 凍結，讓鬆弛都讀「上一輪結束時」的值。

A3 · 不變量

第 r 輪跑完後，dist[v] = 「從 src 出發、最多用 r 條邊抵達 v」的最低票價。

歸納：每次鬆弛 dist[v] ← temp[u] + w，temp[u] 是「≤ r−1 條邊到 u」的最佳值，接一條邊 (u,v) 就變「≤ r 條邊到 v」。 跑滿 k+1 輪即涵蓋「≤ k+1 條邊」＝「≤ k 站」的所有路徑，dist[dst] 就是答案。

A4 · 為何非 Dijkstra

不能直接套。Dijkstra 的貪心一旦「定型」某點的最短距離就不再更新，但本題有邊數上限，最優解可能要走「貴一點、但中轉更少」的路。

// 反例（本題樣本，k=1）
Dijkstra 會先鎖定 0→1→2→3 = 300（最便宜）
但那是 2 次中轉，違反 k=1
正解是 0→1→3 = 700
⇒ 純 Dijkstra 給 300，WA

（進階：可以把狀態擴成 (node, stops) 再跑 Dijkstra，但那本質上就回到「按邊數分層」的想法。）

A5 · 兩種寫法

兩者都在「凍結本輪」：

  • temp = dist：鬆弛都讀上一輪結束的值，不讀本輪剛改的。
  • sz = q.size()：每層只處理凍結當下的 sz 個節點，新 push 的算下一層。

效果一致 —— 強制一輪/層只前進一條邊，讓「最多 k 站」的邊數上限自動成立。所以兩份 code 對任意輸入答案都相同。

A · App 1

k=2 ⇒ 跑 3 輪。接著第 2 輪（temp = [0,100,200,700]）：

// 第 2 輪
2→3 : dist[3] = min(700, temp[2]+200) = min(700, 200+200) = 400
（此時 temp[2]=200 已是上一輪算好的，允許使用）
答案 = 400   ← 路徑 0→1→2→3，用了 2 次中轉（≤ k=2 合法）

放寬中轉上限，就能解鎖更便宜的 400。可見答案隨 k 遞減（或持平）。

A · App 2

「最多 k 站」跑 k+1 輪、最後讀 dist[dst] 即可。若要「恰好 k 次中轉」，就把狀態拆成邊數維度： dp[e][v] = 恰好用 e 條邊到 v 的最低價，只用 dp[e][*] 推 dp[e+1][*]，最後取 dp[k+1][dst]（不再對更少邊數取 min）。

本題 code 其實就是把這個二維 dp 用「dist + temp 滾動」壓成一維、且對所有 ≤ k+1 邊數取 min 的版本。

A · App 3

差別在有沒有「邊數上限」這個額外限制。

  • P743：純最短路、無邊數限制、邊權非負 ⇒ Dijkstra 貪心最快 O(E log V)。
  • P787：多了「最多 k 站」的限制。Dijkstra 的「定案即不改」會和邊數上限打架，所以改用按輪數（邊數）分層的 Bellman-Ford，跑 k+1 輪剛好封頂。

一句話：無限制 → Dijkstra；限制邊數 → 分層 Bellman-Ford。
