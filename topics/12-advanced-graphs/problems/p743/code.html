完整實作 · Dijkstra// FULL C++

p743_dijkstra.cpp C++ 
class Solution {
public:
    int networkDelayTime(vector<vector<int>>& times, int n, int k) {
        vector<vector<pair<int, int>>> graph(n + 1);
        vector<int> dist(n + 1, INT_MAX);

        for (auto t : times) {
            int u = t[0], v = t[1], w = t[2];
            graph[u].push_back({v, w});
        }

        using P = pair<int, int>;          // {距離, 節點}
        priority_queue<P, vector<P>, greater<P>> pq;  // min-heap

        dist[k] = 0;
        pq.push({0, k});

        while (!pq.empty()) {
            auto [d, u] = pq.top(); pq.pop();

            if (d > dist[u]) continue;          // 過期項 ⇒ 惰性刪除

            for (auto [v, w] : graph[u]) {
                int nd = d + w;
                if (nd < dist[v]) {              // 鬆弛成功 ⇒ 推新項
                    dist[v] = nd;
                    pq.push({nd, v});
                }
            }
        }

        int ans = 0;
        for (int i = 1; i <= n; i++) {
            if (dist[i] == INT_MAX) return -1;  // 有點收不到
            ans = max(dist[i], ans);          // 取最晚收到的時間
        }
        return ans;
    }
};

節點編號 1..n，所以用 n+1 大小的 graph、dist（索引 0 空著）。整份就是標準 Dijkstra 模板 + 最後「取 max、有 ∞ 回 −1」。

複雜度分析// COMPLEXITY

步驟內容成本
建圖掃過所有邊入鄰接表O(E)
heap 彈出最多 O(E) 個項（含過期）O(E log V)
鬆弛推入每條邊最多推一次O(E log V)
取 max掃 dist[1..n]O(V)
T = O((V + E) log V) = O(E log V)

每條邊推一次、每次 heap 操作 log；連通圖 E ≥ V−1。

項目數量備註
TimeO(E log V)n ≤ 100 ⇒ 遠快於時限
SpaceO(V + E)鄰接表 + dist + heap
數值型別int最長路 ≤ (n−1)·100 ≈ 10⁴，int 夠
3

邊界與陷阱// PITFALLS

陷阱說明 / 解法
1-indexed 節點節點編號 1..n，容器開 n+1、迴圈跑 1..n；別誤用 0-based 漏掉節點 n。
忘了惰性刪除判斷少了 if (d > dist[u]) continue; 答案仍對（鬆弛條件會擋），但會重複展開過期項、變慢。標準寫法要留著。
min-heap 方向priority_queue 預設是大根堆；要 greater<P> 才是 min-heap。忘了會變每次取最遠點、邏輯全錯。
pair 順序存 {距離, 節點} 而非 {節點, 距離}，才能讓 heap 按距離排序。
不可達判斷最後掃 dist，只要有一個 == INT_MAX 就 return -1，別漏。
負權不適用Dijkstra 只在邊權非負時正確。本題 w ≥ 1 OK；有負權要改 Bellman-Ford。

        dist[v] = nd

for i in [1, n]:
