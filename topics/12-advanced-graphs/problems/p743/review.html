重點回顧// RECAP

關鍵點內容
問題本質「訊號傳遍全網」= 單源最短路，答案 = max over v 的 dist(k, v)。
選 Dijkstra邊權非負 ⇒ 貪心成立，O(E log V) 比 Bellman-Ford 快。
彈出即定案min-heap 彈出的最小 (d,u)，dist[u] 就是最終最短路。
惰性刪除d > dist[u] ⇒ 過期項，直接 continue；免寫可減鍵 heap。
不可達掃 dist 有 INT_MAX ⇒ 回 -1。
複雜度每邊推一次 heap，總 O(E log V)。

範例 Trace · 跑一次 Dijkstra// WORKED TRACE

用動畫那張圖。n=5、src=1，有向邊如下：

// INPUTn = 5, src = 1
edges:  1→2:2 · 1→3:5 · 2→3:1 · 2→4:7 · 3→4:2 · 4→5:1 · 3→5:6
// INITdist = [_, 0, ∞, ∞, ∞, ∞]      heap = { (0,1) }
// pop (0,1) — 定案 1
1→2: dist[2]=2   1→3: dist[3]=5
dist = [_, 0, 2, 5, ∞, ∞]      heap = { (2,2), (5,3) }
// pop (2,2) — 定案 2
2→3: 2+1=3 < 5 ⇒ dist[3]=3   2→4: dist[4]=9
dist = [_, 0, 2, 3, 9, ∞]      heap = { (3,3), (5,3)過期, (9,4) }
// pop (3,3) — 定案 3
3→4: 3+2=5 < 9 ⇒ dist[4]=5   3→5: dist[5]=9
dist = [_, 0, 2, 3, 5, 9]      heap = { (5,3)過期, (5,4), (9,4)過期, (9,5) }
// pop (5,3) — 過期！
d=5 > dist[3]=3 ⇒ continue（惰性刪除，不處理）
// pop (5,4) — 定案 4
4→5: 5+1=6 < 9 ⇒ dist[5]=6
dist = [_, 0, 2, 3, 5, 6]      heap = { (6,5), (9,4)過期, (9,5)過期 }
// pop (6,5) — 定案 5；之後彈出全過期，略過
dist = [_, 0, 2, 3, 5, 6]
// OUTPUTmax(0, 2, 3, 5, 6) = 6
（節點 5 最晚收到，時間 6）
觀察：(5,3) 被彈出時 dist[3] 早已是 3，這個舊項是節點 2 鬆弛前留下的殘影。 靠 d > dist[u] 一句就安全略過 —— 這就是不用可減鍵 heap 也能正確的關鍵。

// ANSWERS

先自己想過再對答案 — Q 對應 Q、App 對應 App。

A1 · 問題轉化

訊號從 k 同時往外擴散，每個節點「第一次收到」的時間就是它到 k 的最短路徑長。 「全部都收到」必須等最慢的那一個，所以取 max。

不是總和（會重複計算共用路段），也不是平均（題目要的是「全部完成」的時刻，由最大值決定）。

A2 · 貪心正確性

Dijkstra 假設「一旦彈出某點、它的 dist 就定案」。這靠非負邊保證：之後再走的路徑只會更長，不可能繞遠路反而變短。

若有負邊，一條「先走遠、再靠一條負邊補回來」的路可能更短，但那個點可能已被提早定案 ⇒ 錯過更短路。所以負權要改用 Bellman-Ford / SPFA。

A3 · 惰性刪除

它跳過 heap 裡的過期項：某節點 u 的 dist 後來被更短路徑更新過，heap 裡還躺著舊的、較大的 (舊距離, u)， 彈出時 舊距離 > dist[u] 成立 ⇒ 略過。

刪掉這行答案仍會對（進到迴圈後 nd < dist[v] 的鬆弛條件本身就擋得住無效更新）， 但會多花時間去展開已定案節點的出邊、重複做白工，效率變差。標準寫法保留它。

A4 · 實作細節

priority_queue 預設是大根堆（每次給最大）。我們要每次拿「最近的點」，所以加 greater<P> 變 min-heap。

pair 的比較先比 first。存 {距離, 節點} 才能讓 heap 按距離排序；若存 {節點, 距離} 就會按節點編號排，完全錯亂。

A5 · 複雜度

不會爆。heap 裡雖然可能有同一節點的多個項，但「推入」只發生在鬆弛成功時，而每條邊最多觸發一次有效鬆弛 ⇒ 總推入次數 ≤ E。

所以 heap 大小 O(E)、總彈出 O(E)，每次操作 O(log E) = O(log V)，合計 O(E log V)。過期項只是被 continue 快速丟棄，不增加漸進複雜度。

A · App 1

從 k=3 出發，只能走 3→4→5 和 3→5：

// src = 3
dist[3]=0, dist[4]=2, dist[5]=min(6, 2+1)=3
dist[1]=∞, dist[2]=∞   ← 沒有指向 1、2 的路徑
⇒ 有點收不到，回傳 -1

節點 1、2 沒有入邊來自 {3,4,5} 這側，永遠收不到 ⇒ 答案 −1。

A · App 2

建圖時兩個方向都加：graph[u].push_back({v,w}) 之外再 graph[v].push_back({u,w})。演算法本體完全不動。

答案通常會變小（多了反向邊、更多捷徑），且無向連通圖不會出現「收不到」（只要整張圖連通）。

A · App 3

差別在有沒有「邊數上限」這個額外限制。

  • P743：純最短路、無邊數限制、邊權非負 ⇒ Dijkstra 貪心最快 O(E log V)。
  • P787：多了「最多 k 站」的限制。Dijkstra 的「定案即不改」會和邊數上限打架（貴一點但中轉少的路會被錯過）， 所以改用按輪數（邊數）分層的 Bellman-Ford，跑 k+1 輪剛好封頂。

一句話：無限制 → Dijkstra；限制邊數 → 分層 Bellman-Ford。
