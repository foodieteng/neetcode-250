題目簡介// THE QUESTION

QUESTION

一份外星語字典把單字 words 依它們的字母順序由小到大排好了。請反推出這個外星語的 字母順序(回傳任一合法順序)。若順序矛盾(無解),回傳 ""。

關鍵觀察:字典序只在相鄰兩個單字之間洩漏一點資訊 —— 它們第一個不同的字元, 前者一定排在後者前面。把每條這種「誰在前」關係當成一條有向邊,問題就變成經典的拓樸排序。

3

核心策略 · 三步驟// STRATEGY

  1. 收集字母:掃過所有單字,把出現過的字母都放進圖(即使它沒有任何邊,也要出現在答案裡)。
  2. 建邊:對每對相鄰單字,找第一個不同的位置 j,加邊 w1[j] → w2[j] 後即可 break。順手擋掉「前綴矛盾」:若前面完全相同卻 w1 比 w2 長(如 ["abc","ab"])⇒ 無解回 ""。
  3. 拓樸排序:對這張圖做拓樸排序(DFS 後序 或 Kahn BFS 皆可)。排不出完整順序(有環)⇒ 回 ""。

兩種拓樸寫法// DFS vs KAHN

建圖完全一樣,差別只在怎麼拓樸排序。兩種都對,完整程式碼見下一頁:

面向寫法 A · DFS 三色後序寫法 B · Kahn BFS 入度
核心遞迴走訪,後序加入,最後反轉入度 0 進佇列,逐一取出、鄰居入度 −1
判環撞到灰色(訪問中)節點 = 有環最後輸出長度 < 字母數 = 有環
額外狀態color:白 / 灰 / 黑indegree 計數 + 佇列
順序來源逆後序(reverse post-order)層層剝除入度 0
結果多組合法解時兩者可能給出不同但都合法的順序
拓樸排序有兩種標準骨架:DFS 逆後序、與 Kahn 消入度。這題是把它們套進「外星字母圖」的最佳練習題。
6

關鍵細節// CORRECTNESS

CLAIM · 三個一定要顧到的點
  1. 只取第一個不同字元:相鄰字 w1, w2 的字典序只保證「第一個不同處 w1[j] < w2[j]」,之後的字元沒有任何順序資訊,找到就 break。
  2. 前綴矛盾:若掃到 minL 都相同,而 w1 比 w2 長(["abc","ab"]),字典序不可能讓長的排前面 ⇒ 直接 ""。
  3. 孤立字母也要出現:沒有任何邊的字母仍屬於字母表,所以要先把所有出現過的字母放進圖 / 入度表。
  4. 判環:有環代表順序自我矛盾,兩種寫法各有偵測法(灰色 / 長度不足)。
7

複雜度// COMPLEXITY

設 C = 所有單字的字元總數、V = 相異字母數(≤ 26)、E = 邊數(≤ 相鄰對數)。

T = O(C + V + E)　　S = O(V + E)

建圖掃過所有字元 O(C);拓樸排序 O(V + E)。字母上限 26,故常數極小。

項目量級說明
TimeO(C + V + E)建圖 + 拓樸
SpaceO(V + E)鄰接表 + 狀態 / 入度
字母數 V≤ 26邊數 E ≤ 相鄰對數
8

一句洞察// INSIGHT

字典序的線索只藏在相鄰字的第一個不同字元裡。把它抽成有向邊,剩下就是一題拓樸排序 —— 別忘了「前綴矛盾」與「孤立字母」兩個特例。
