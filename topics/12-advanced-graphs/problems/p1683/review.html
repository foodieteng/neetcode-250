重點回顧// RECAP

面向要點
問題本質有向圖強連通分量:一團彼此互達的點
第一趟原圖 DFS,後序壓 stack(完成時間序)
第二趟反圖 DFS,依 stack 頂端起,每樹一 SCC
為何對完成最晚在源頭 SCC;反圖上只框得住自己
縮圖SCC 縮點後是 DAG
複雜度時間 O(V+E),空間 O(V+E)

範例 Trace// cycle + tail

// 輸入 n = 5 邊(原圖 g): 1→2, 2→3, 3→1, 3→4, 4→5 反圖(rg): 2→1, 3→2, 1→3, 4→3, 5→4 預期:{1,2,3} 互達成一組,4、5 各自一組
// 第一趟(g)完成序 dfs1(1): 1→2→3→(1 訪過,退)→ 3 再走 4→5 完成順序(先完成先壓底):5, 4, 3, 2, 1 order: 底 5,4,3,2,1 頂 → top = 1
// 第二趟(rg) 彈 1 → dfs2 在反圖:1→3→2(繞回 1 訪過)→ {1,2,3} = SCC 1 彈 2,3:已分派 → skip 彈 4 → dfs2:4→3(已分派)→ {4} = SCC 2 彈 5 → dfs2:5→4(已分派)→ {5} = SCC 3 輸出 k = 3 comp = [1,1,1,2,3] (節點 1..5) // {1,2,3} · {4} · {5}
關鍵一眼:4 和 5 雖然在原圖連得到後面(3→4→5),但回不去(沒有 5→…→1 的路),所以它們不和環同組。 SCC 要的是雙向互達 —— 反圖那一趟正是在驗證「回得去嗎」。

應用題// APPLY

App 1. 求出所有 SCC 後,怎麼建縮圖(condensation DAG)?接著可以在上面做什麼經典題?
App 2. 2-SAT 怎麼用 SCC 解?(提示:變數 x 與 ¬x 若在同一個 SCC 代表什麼?)
App 3. 若只想知道「整張圖是不是一個大 SCC(任兩點互達)」,需要完整跑 Kosaraju 嗎?有沒有更省的判法?
// ANSWERS
A1. 因為要靠反圖來「隔離」出單一 SCC。第一趟後,完成最晚的點在源頭 SCC。在原圖上從源頭出發能一路走到下游好幾個 SCC,一次 DFS 會把它們全染成同一組,錯。反圖把所有邊反向後,源頭 SCC 變成「匯點」——從它出發只到得了自己這一組,跨不出去,一次 DFS 剛好框住一個 SCC。所以第二趟非反圖不可。
A2. 考慮縮圖(每個 SCC 縮成一點)的兩個 SCC A→B(A 有邊指向 B)。可以證明 A 裡的最大完成時間 > B 裡的最大完成時間:DFS 若先進 A,會在離開 A 前把 B 也走完,A 的根最後才完成;若先進 B,B 走完回不到 A(單向),A 之後才被訪、完成更晚。無論如何 A 完成更晚。推廣:全圖完成時間最大的點,必在沒有任何 SCC 指進來的源頭 SCC。
A3. 不行,順序是正確性的關鍵。若隨便挑起點,可能挑到縮圖的匯點 SCC,在反圖上它反而變成源頭、能走到別組,又染錯。按 stack(完成序反過來)取,保證每次都從當前剩餘圖的源頭開始;把它那組移除後,下一個未分派的 top 又是新的源頭。這個「由源頭往下剝」的順序,正是 Kosaraju 成立的核心。
A4. 反證:若縮圖有環,例如 SCC A→B→…→A,那 A 裡任一點能到 B 裡任一點、B 又能回到 A,代表 A、B 其實彼此互達,依定義它們該是同一個 SCC,矛盾(SCC 是「極大」的互達集合)。所以縮圖不可能有環 → 一定是 DAG。這也是為什麼可以在縮圖上做拓撲排序、DP 等。
A5. 兩者都 O(V+E)。Kosaraju:兩趟 DFS + 一張反圖,邏輯分兩段(排序、染色)非常直觀好記、好講,代價是要存反圖、掃兩遍。Tarjan SCC:一趟 DFS,用 dfn/low + 顯式堆疊 + onStack 標記,當 low[u]==dfn[u] 時彈出一整個 SCC,不需反圖、只掃一遍常數更小,但實作較精巧、容易寫錯。面試想穩、想解釋清楚 → Kosaraju;想炫技或效能吃緊 → Tarjan。
App 1. 先用 Kosaraju 得到每點的 comp[]。掃每條原邊 u→v,若 comp[u] != comp[v] 就在縮圖裡加一條 comp[u] → comp[v](去重)。得到的縮圖是 DAG,接著可做:拓撲排序、DAG 上最長路 / DP、統計「能被多少點到達」等。很多「有環的有向圖」題,第一步就是縮 SCC 變 DAG。
App 2. 2-SAT:每個布林變數 x 拆成兩個點 x、¬x,把每個子句 (a ∨ b) 轉成兩條蘊含邊 ¬a→b、¬b→a,建成「蘊含圖」。跑 SCC 後:若某個 x 與 ¬x 落在同一個 SCC,代表 x 同時要真又要假 → 無解。否則有解,並可依 SCC 的拓撲序(誰的 comp 較晚/較早)決定每個變數取值。SCC 把 2-SAT 從 NP 印象變成線性可解。
App 3. 不用跑完整分組。「整張圖是單一 SCC」⟺ 從任一點 s 出發,在原圖能到所有點,且在反圖也能到所有點。所以只要:① 在 g 上從 s BFS/DFS,看是否覆蓋全部;② 在 rg 上從 s 再一次,看是否覆蓋全部。兩次都 yes 就是一個大 SCC。兩趟遍歷、O(V+E),比完整 Kosaraju 更省心(不必記完成序與分組)。
