題目簡介// THE QUESTION

QUESTION

n×n 網格,grid[i][j] 是該格高度(0..n²-1 的排列)。時間 t 時,所有高度 ≤ t 的格子都被水淹、可游過。 求從左上 (0,0) 游到右下 (n-1,n-1) 的最短時間。

在時間 t 能走的格子 = 高度 ≤ t。要能走完一條路,t 必須 ≥ 這條路上的最高格。 所以答案 = 所有路徑中「路徑最大高度」的最小值(minimax / bottleneck path)。

核心策略// STRATEGY

  1. 重述:答案 = 從起點到終點所有路徑中,max(路徑上的高度) 的最小值。
  2. 三種算法(見下一頁完整碼):
    • A · 上升水位 + Union-Find:把格子當「邊」,依高度由小到大加入、與已加入的鄰居 union;一旦起點與終點同分量,當前高度就是答案。
    • B · Dijkstra 變形:dist[cell] = 到此格所需的最小瓶頸,鬆弛用 max(當前, 鄰居高度);彈出終點即答案。
    • C · 二分答案 + BFS:二分時間 t,用 BFS 檢查「只走 ≤ t 的格」能否到終點;可達性隨 t 單調。

一句洞察// INSIGHT

這題不是求「路徑和最小」,而是求「路徑上最高的一格最小」。想成水位慢慢上升, 第一次讓起終點連通的高度就是答案 —— Union-Find、max-Dijkstra、二分答案三招都能算它。
