重點回顧// RECAP

面向要點
問題本質最小化路徑上的最大高度(minimax / 瓶頸路徑)
A · Union-Find由低到高淹格 + union,連通那刻的高度 = 答案
B · Dijkstra成本 = max(沿途),彈出終點即答案
C · 二分+BFS可達性對 t 單調,二分最小可行 t
複雜度三者皆 O(n² log n)
易錯把它當「路徑和最小」;Dijkstra 鬆弛誤用 +

觀念題// QUIZ

Q1. 為什麼答案是「路徑最大高度的最小值」,而不是路徑高度總和的最小值?
Q2. 寫法 A 為什麼「起點與終點第一次連通那一刻」的高度就是答案?
Q3. 寫法 B 的 Dijkstra 為什麼鬆弛用 max(h, 鄰居) 而不是 h + 鄰居?
Q4. 寫法 C 為什麼能對答案二分?單調性從哪裡來?
Q5. 三種寫法都是 O(n² log n),各自的 log 分別來自什麼?

範例 Trace · A 上升水位// [[0,2,1],[5,8,3],[6,7,4]]

// GRID (值 = 高度) c0 c1 c2 r0 [ 0 2 1 ] START=(0,0) r1 [ 5 8 3 ] r2 [ 6 7 4 ] END=(2,2) 依高度由小到大淹:0,1,2,3,4,5,6,7,8 的位置依序加入
// 上升水位 + UNION h=0 淹 (0,0) 區:{(0,0)=START} h=1 淹 (0,2) 另一島 {(0,2)} h=2 淹 (0,1) → 併 (0,0)、(0,2) 上排連成一片 h=3 淹 (1,2) → 併 (0,2) 接上 (1,2) h=4 淹 (2,2)=END → 併 (1,2) START ↔ END 連通!
// RESULT 連通路徑:0 → 2 → 1 → 3 → 4 (max = 4) 高度 5,6,7,8 從頭到尾沒被淹 // OUTPUT: 4
答案 4 = 這條低窪路徑上的最高格。高牆 5~8 完全用不到 —— 這就是「最小化瓶頸」的畫面: 水只要漲到剛好淹過那條最低的通道即可。

應用題// APPLY

App 1. 若改成「最大化路徑上的最小值」(例如找承載量最大的通道),三種寫法各要怎麼調整?
App 2. 若允許八方向(含斜向)移動,三種寫法分別要改哪裡?複雜度會變嗎?
App 3. 若同一個 grid 要回答很多次「時間 t 能否到達」的查詢,哪種前處理最划算?
// ANSWERS
A1. 因為在時間 t,你能踏上的格子是「高度 ≤ t」。要走完一條路,t 必須 ≥ 這條路上最高的那格(任何一格太高就淹不到、過不去)。所以「能走完某路的最早時間」= 那條路的最大高度;要最早到達 = 在所有路裡挑最大高度最小的那條。與總和無關 —— 只有最高的瓶頸決定能不能過。
A2. 格子依高度由小到大加入,等於「水位逐步上升」。當加入某格(高度 h)後 START 與 END 首次落在同一分量,代表存在一條全程 ≤ h 的路;而在加入它之前不連通,代表沒有全程 < h 的路。所以 h 正是能連通的最小水位 = 答案。這是 Kruskal「加邊到連通」的翻版。
A3. 因為路徑成本被定義成「沿途最大高度」,不是總和。到達鄰居 (nr,nc) 的瓶頸 = max(到我這格的瓶頸 h, 鄰居自己的高度) —— 走過去只會被「更高的那個」卡住。Dijkstra 的貪心(每次取當前瓶頸最小的點)在 max 這個單調運算下依然成立,所以彈出終點時它的瓶頸就是最小值。
A4. 可達性對門檻 t 具單調性:t 越大,可通行的格子只增不減,所以「t 時能到終點」若成立,則所有 > t 也成立。於是存在一個臨界值 t*,t < t* 不可達、t ≥ t* 可達。對這種「false…false true…true」結構就能二分,找第一個 true 即答案。
A5. A:把 n² 個格丟進 min-heap,每次取最小是 O(log n²)=O(log n)(union-find 的 α 視為常數)。B:Dijkstra 的 heap 操作 O(log n),最多推入 O(n²) 次。C:二分值域 0..n²-1 共 O(log n²)=O(log n) 輪,每輪 BFS O(n²)。三者相乘後都是 O(n² log n)。
App 1. 對稱地改成「最大化最小值」:A 改成由高到低加入格子(像最大生成樹),連通那刻的高度就是答案;B 用最大堆,鬆弛改成 min(h, 鄰居),取「當前最小值最大」的點擴展;C 二分不變,BFS 條件改成只走「≥ t」的格,找最大可行 t。
App 2. 只需把 dirs 從 4 個方向改成 8 個(加四個斜向)。三種寫法的主體邏輯都不變。複雜度量級不變(每格鄰居從 4 個變 8 個,只是常數),仍是 O(n² log n)。
App 3. 用寫法 A 的離線做法最划算:把格子依高度排序,逐步 union,並記錄「START 與各點連通時的水位」;或直接記錄每對查詢在何時連通。一次 O(n² log n) 前處理後,每個「t 能否到達」查詢可用當時的連通資訊 O(1)~O(α) 回答,不必每次重跑 BFS。
