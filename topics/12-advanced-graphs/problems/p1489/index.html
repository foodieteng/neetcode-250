MST 關鍵邊與偽關鍵邊

題目簡介// THE QUESTION

QUESTION

給一張帶權無向圖 n 個點、edges[i] = [u, v, w]。找出所有邊分成兩類: 關鍵邊(critical)—— 刪掉它,最小生成樹(MST)權重會變大或圖不再連通; 偽關鍵邊(pseudo-critical)—— 它出現在某些但非全部的 MST 裡。回傳這兩組邊的原始索引。

先把 MST 求出來得到基準權重 mst。一條邊到底多重要,就用兩個小測試量: 拿掉它會不會變貴 / 斷開,以及強迫它先進來還能不能維持一樣便宜。

2

為什麼用「Kruskal + skip / force」// WHY THIS

A 枚舉所有生成樹再比較 指數級
生成樹數量可達指數級(Cayley:完全圖有 nⁿ⁻² 棵),不可能全列。
× REJECT · 爆炸
B 只跑一次 MST 看哪些邊被選 不完整
一次 MST 只給「一棵」樹;等重的邊誰進誰出是任意的,無法區分關鍵 / 偽關鍵。
△ REJECT · 分不出來
BOTH FAIL · ADOPT
C 基準 MST + 對每條邊做 skip / force 兩測試 O(m²·α)
先算基準 mst。對每條邊 e:skip —— 不准用 e 建 MST,若權重 > mst(或無法連通), e 是關鍵邊;否則 force —— 強迫先放 e 再建,若權重仍 = mst,e 是偽關鍵。 每個測試就是一次 Kruskal,乾淨且好證明。
● ADOPT · PROVABLE
3

核心策略 · 三步驟// STRATEGY

  1. 排序 + 基準 MST:把邊照權重排序,跑一次 Kruskal 得基準權重 mst。
  2. 關鍵邊測試(skip):對每條邊 i,建 MST 時跳過 i。若結果 > mst(變貴或斷開)⇒ i 是關鍵邊。
  3. 偽關鍵測試(force):若不是關鍵邊,則強制先放 i 再建 MST。若結果 == mst ⇒ i 是偽關鍵邊。
4

語法補充 · 間接排序慣用法// C++ IDIOM

這三行是很常見的 C++「間接排序 indirect sort」寫法 —— 排序的是「索引陣列」而不是資料本身。逐行拆解:

indirect_sort.cpp C++17 
vector<int> order(m);                       // ① 建一個長度 m 的陣列(值全為 0)

iota(order.begin(), order.end(), 0);       // ② 填入 0,1,2,...,m-1(每條邊的原始索引)

sort(order.begin(), order.end(),            // ③ 排序「索引」,不是排 edges 本身
     [&](int a, int b) {                  //    [&] = lambda 以「參考」抓外部變數(edges)
         return edges[a][2] < edges[b][2];    //    比較兩索引對應邊的權重(第 3 欄)
     });                                    //    回傳 a<b ⇒ a 排前面 ⇒ 權重由小到大
行語法作用 / 目的
① vector<int> order(m); 用「大小建構子」開一個長度 m 的 int 陣列,元素預設全為 0。(注意 vector<int> v(m) 是 m 個 0;vector<int> v{m} 才是一個值 m。)
② iota(begin, end, 0) 來自 <numeric>。從起始值 0 開始遞增填滿:order = {0,1,2,…,m-1}。這樣 order 就是「每條邊的原始編號」。
③ sort(…, cmp) 第三個參數是比較器,決定排序規則。這裡排的是 order(索引),比較時去查 edges[a][2](權重),等於「照權重排索引」。
lambda [&](int a,int b){…} 匿名函式當比較器。[&] = 以參考擷取外層所有變數(這裡需要用到 edges);參數 a,b 是要比較的兩個索引;return a<b 代表「a 應排在 b 前面」⇒ 遞增。
WHY · 為什麼不直接 sort(edges)?
因為答案要回傳邊的原始索引,而且 skip / force 測試都針對「第 i 條原始邊」。若直接對 edges 排序, 原始編號就被打亂、對不回去了。間接排序只重排一份索引清單 order,既能「照權重由小到大掃描」, 又完整保留每條邊的原始編號 —— 一魚兩吃。

為什麼這樣就對// CORRECTNESS

CLAIM · skip 抓關鍵、force 抓偽關鍵
  1. 關鍵邊:若刪掉 e 後 MST 變貴(或不連通),代表沒有一棵 MST 能不用 e ⇒ e 出現在每一棵 MST 裡。這正是 skip 測試 > mst 的意義。
  2. 偽關鍵邊:e 不是關鍵(拿掉不變貴),但強制先放 e 仍能建出權重 = mst 的樹 ⇒ e 出現在某些 MST 裡。這是 force 測試 == mst。
  3. 兩測試互斥且互補:一條在 MST 上的邊,不是關鍵就是偽關鍵;完全用不到的邊兩個測試都不會通過。
  4. force 的實作:先 Union 這條邊、權重先加、已選數 +1,再照權重跑剩下的 Kruskal。
7

複雜度// COMPLEXITY

設 m = 邊數。排序 O(m log m);之後對每條邊最多做 2 次 Kruskal,每次 O(m·α)。

T = O(m log m + m² · α)　　S = O(n + m)

主導項是 m 條邊 × 每條 2 次 Kruskal;α 為反 Ackermann,視為常數。

項目量級說明
TimeO(m² · α)每條邊各 skip / force 一次 Kruskal
SpaceO(n + m)併查集 O(n)、order O(m)
基準 MSTO(m · α)一次 Kruskal
排序O(m log m)間接排序索引

一句洞察// INSIGHT

量一條邊有多重要,就拿掉它看會不會變貴(關鍵)、硬塞它看能不能一樣便宜(偽關鍵)。 兩個 Kruskal 小測試,把每條邊分類完畢。
