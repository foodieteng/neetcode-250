MST 關鍵邊與偽關鍵邊

重點回顧// RECAP

面向要點
關鍵邊刪掉它 MST 變貴或斷開 ⇒ 出現在每一棵 MST
偽關鍵邊非關鍵,但硬塞仍能建出等重 MST ⇒ 出現在某些 MST
工具Kruskal(排序 + 併查集)+ 一個 kruskal(skip, force)
間接排序排 order(索引)而非 edges,保留原始編號
判斷skip 版 > mst → critical;否則 force 版 == mst → pseudo
不連通沒選滿 n−1 條回 INT_MAX,自然 > mst

範例 Trace// e0(0-1,1) e1(1-2,2) e2(1-3,2) e3(2-3,2)

// INDIRECT SORT order 初值 = {0,1,2,3} 依權重(e0=1, e1=2, e2=2, e3=2)排序後 order = [0, 1, 2, 3]
// BASE MST (skip=-1, force=-1) 掃 e0(0-1,1): Union ✓ total=1 add=1 掃 e1(1-2,2): Union ✓ total=3 add=2 掃 e2(1-3,2): Union ✓ total=5 add=3 = n-1 → 停 掃 e3 不需要(已滿) base mst = 5
// e0 → 關鍵測試 (skip=0) 跳過 e0:只剩 e1,e2,e3 連 {1,2,3},節點 0 連不上 add 只到 2 < n-1 → 回 INT_MAX INT_MAX > 5 → e0 是 CRITICAL
// e3 → 偽關鍵測試 (先 skip=3,再 force=3) skip=3: 用 e0+e1+e2 → total=5 = mst → 不是關鍵 force=3: 先放 e3(total=2,add=1),再掃 e0(total=3),e1(total=5,add=3) → total=5 = mst → e3 是 PSEUDO
// RESULT e1、e2 與 e3 同理(三條等重邊,任兩條可組 MST) // OUTPUT: critical = [0] pseudo = [1, 2, 3]
關鍵邊 e0 是唯一連到節點 0 的橋,每棵 MST 都得用它;而 e1/e2/e3 三條等重,MST 只需其中兩條,誰進誰出都行 ⇒ 三條都是偽關鍵。

應用題// APPLY

App 1. 若把比較器改成 edges[a][2] > edges[b][2](由大到小),Kruskal 還對嗎?會變成求什麼?
App 2. 這題的間接排序慣用法,還能用在哪些場景?舉一個「需要保留原始索引又要照某個 key 排序」的例子。
App 3. 當 m 很大、O(m²) 太慢時,往哪個方向優化?(提示:等重邊分組 + 縮點 / bridge)
// ANSWERS
A1. 關鍵邊:刪掉它,MST 的最小權重會變大,或圖直接不連通 —— 等價於它出現在每一棵 MST。偽關鍵邊:它不是關鍵(拿掉不變貴),但存在某一棵權重仍最小的 MST 用到它 —— 即出現在某些、但非全部 MST。差別就是「全部 MST」vs「某些 MST」。
A2. 因為答案要回傳邊的原始索引,且 skip / force 都針對特定原始邊。直接 sort(edges) 會把邊的順序打亂,原始編號就對不回去了。間接排序只重排一份索引清單 order,同時得到「照權重掃描」與「保留原始編號」兩個好處。
A3. skip = 建 MST 時要略過的邊索引(-1 表不略過);force = 要在最前面強制放入的邊索引(-1 表不強制)。三種呼叫:(-1,-1) 求基準 mst、(i,-1) 測關鍵、(-1,i) 測偽關鍵。因為「略過某邊」與「先塞某邊」都只是 Kruskal 起手的小變化,參數化後一個函式全包。
A4. skip 後跑 Kruskal,用 add 記已選邊數。若因為少了這條邊而連不滿 n−1 條,函式回 INT_MAX;若連得滿但被迫改用更重的邊,回傳的 total 會大於 mst。兩種情況的回傳值都 > mst,所以同一個判斷式 skip 版 > mst 就一次涵蓋。
A5. 互斥。一條邊若是關鍵邊,它必然出現在每棵 MST,force 測試也會 == mst,若不先判關鍵就會被誤標成偽關鍵。所以先用 skip 濾掉關鍵邊,else 分支才用 force 判偽關鍵,保證每條邊最多歸一類。
App 1. 不對。Kruskal 的正確性依賴「由小到大貪心」(cut / cycle property)。改成由大到小會變成求最大生成樹(Maximum Spanning Tree),而不是 MST。要保持本題語意就必須遞增排序。
App 2. 任何「要照某個 key 排序、但事後仍需原始位置」的場景都適用:例如排序後要回傳原始下標(如 argsort)、多個平行陣列只想動其中一個的順序、或排完還要對照另一份資料。Python 的 sorted(range(n), key=…)、NumPy 的 argsort 都是同一招。
App 3. 進階做法:把邊按權重分組,逐組處理 —— 同一權重組內,先用「前面已確定的 MST 邊」把圖縮點,再在縮點後的多重圖上找橋(bridge, Tarjan):是橋的即關鍵邊,在環上且被 MST 用到的即偽關鍵。這樣可把每組做到近線性,整體約 O(m log m + m·α),遠優於樸素 O(m²)。
