題目簡介// THE QUESTION

QUESTION

給整數陣列 nums 和整數 k,回傳和剛好等於 k 的連續子陣列個數。 子陣列是連續、非空的一段。例:nums=[1,1,1], k=2 → 2;nums=[1,2,3], k=3 → 2。 注意 nums[i] 可能是負數。

有負數這件事很關鍵 —— 不能用滑動視窗(和不是單調遞增,右指針擴張時和可能變小,無法判斷何時該收縮左指針)。 正解是把它轉成一道「前綴和之差 = k」的計數題,用雜湊表一趟解決。

2

為什麼是「前綴和 + 雜湊」// WHY

× REJECT · TLE
雙層迴圈枚舉所有子陣列

對每個起點累加到每個終點,判斷是否 = k。O(n²),n=2×10⁴ 時約 4 億次 —— 會超時。

× REJECT · 負數失效
滑動視窗

視窗要靠「和單調」才能決定何時收縮左界。有負數 → 和非單調,右移不一定讓和變大,視窗無從判斷。

BOTH FAIL · ADOPT ↓
● ADOPT · O(n)
前綴和 + 雜湊計數

設 prefix[j] = 前 j 項的和。子陣列 (i, j] 的和 = prefix[j] − prefix[i]。 要它 = k,等價於存在一個更早的前綴 prefix[i] = prefix[j] − k。 於是「以 j 結尾、和為 k 的子陣列數」= 之前 prefix = prefix[j]−k 出現過幾次。 邊掃邊用雜湊表記錄「每個前綴和出現的次數」,O(n) 時間、O(n) 空間。

3

核心策略 · 三步驟// STRATEGY

  1. 種子 —— freq[0] = 1,代表「空前綴和 = 0」出現一次,讓「從頭開始」的子陣列也算得到。
  2. 邊掃邊查 —— 累加 prefix += num 後,先 cnt += freq[prefix − k](把所有能配對的更早前綴一次算進)。
  3. 再登記 —— freq[prefix]++。順序不能反:先查再記,否則會把「長度 0 的自己」誤算進去(當 k=0 時尤其致命)。

動畫演示// PREFIX + MAP

[1,-1,1,-1], k=0:前綴和走 0→1→0→1→0,0 重複出現。到最後一步 freq[0]=2, 所以 cnt += 2 —— 這就是為什麼要「加次數」而不是「加 1」。答案 4。

核心恆等式// THE IDENTITY

子陣列和 = 兩個前綴和之差

　sum(i..j) = prefix[j] − prefix[i−1]
　要 = k　⇔　prefix[i−1] = prefix[j] − k

固定右端 j,問題變成「有幾個更早的前綴等於 prefix[j] − k」。這正是雜湊表最擅長的「數某個值出現幾次」。 把減法搜尋轉成 O(1) 查表,是這題從 O(n²) 降到 O(n) 的全部魔法。 種子 freq[0]=1 對應 prefix[i−1] 可以是「第 0 項之前」的空前綴 —— 少了它,凡「從 index 0 開始」的合法子陣列都會被漏數。

複雜度// COMPLEXITY

T = O(n)　　S = O(n)

單趟掃描,每步一次雜湊查詢 + 一次更新,均攤 O(1)。空間是雜湊表最多存 n+1 個不同前綴和。相比 O(n²) 暴力,這是拿 O(n) 空間換一個數量級的時間。

面向成本說明
掃描O(n)每個元素處理一次
每步查表 + 更新O(1)unordered_map 均攤常數
雜湊表空間O(n)最多 n+1 種前綴和
負數不影響前綴和差不需要單調性

一句洞察// INSIGHT

「連續子陣列和 = k」+ 有負數 → 想到前綴和之差:prefix[j] − prefix[i] = k。 把它翻成「查 prefix[j] − k 出現過幾次」,用雜湊表把 O(n²) 打成 O(n)。 種子 freq[0]=1、先查後記,兩個細節缺一不可。
