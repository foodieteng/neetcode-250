完整實作 · 前綴和 + 雜湊// O(n) TIME

p560.cpp C++17 
class Solution {
public:
    int subarraySum(vector<int>& nums, int k) {
        unordered_map<int, int> freq;       // 前綴和 → 出現次數
        int prefix = 0, cnt = 0;
        freq[0] = 1;                        // 種子:空前綴

        for (int num : nums) {
            prefix += num;                     // 走到此處的前綴和
            if (freq.count(prefix - k))        // 先查:有幾個更早前綴 = prefix−k
                cnt += freq[prefix - k];     // 加「次數」,不是加 1
            freq[prefix] += 1;              // 後記:登記自己
        }
        return cnt;
    }
};
兩行順序是靈魂:if…cnt += freq[prefix−k](先查)必須在 freq[prefix]++(後記)之前。 若先登記自己再查,當 k=0 時會把「長度 0 的空子陣列」算進去 → 每個位置多算一次。 這裡的 freq.count(...) 只是避免 operator[] 對不存在的 key 插入 0(其實插了也不影響答案,只是多佔空間),寫成 cnt += freq[prefix−k] 一行也對。

複雜度分析// COST

片段成本為什麼
單趟 forO(n)每元素一次
freq.count + freq[...]均攤 O(1)雜湊查詢與更新
總時間O(n)n 步各 O(1)
空間O(n)最多 n+1 種不同前綴和
T = O(n)　　S = O(n)

溢位注意:prefix 最大約 2×10⁴ × 10³ = 2×10⁷,k 到 10⁷,prefix−k 落在 int 範圍內,安全。

邊界與陷阱// PITFALLS

陷阱症狀正解
忘了種子 freq[0]=1 凡「從 index 0 開始」的子陣列全漏數,如 [1,2],k=3 少算 初始化 freq[0]=1
先記後查(順序反了) k=0 時每格多算一次自己(空子陣列) 一定「先 cnt+= 再 freq[prefix]++」
加 1 而非加次數 前綴和重複時(有負數/零)少算,如 [1,-1,1,-1],k=0 得 3 而非 4 cnt += freq[prefix−k]
誤用滑動視窗 有負數時視窗收縮邏輯失效 前綴和 + 雜湊,不依賴單調性
想省空間用 set 記存在 丟了「出現次數」→ 重複前綴少算 必須用 map 記次數

細節走讀// WALKTHROUGH

// nums=[1,-1,1,-1], k=0(前綴和會重複,考驗「加次數」) freq={0:1} prefix=0 cnt=0 num=1 : prefix=1 need=1-0=1 freq[1]?無 cnt+=0=0 freq={0:1,1:1} num=-1: prefix=0 need=0 freq[0]=1 cnt+=1=1 freq={0:2,1:1} → [1,-1] num=1 : prefix=1 need=1 freq[1]=1 cnt+=1=2 freq={0:2,1:2} → [-1,1] num=-1: prefix=0 need=0 freq[0]=2 cnt+=2=4 freq={0:3,1:2} → [1,-1,1,-1] 和 [1,-1] 4
最後一步是精華:prefix 回到 0,而 0 這個前綴和之前已經出現過 2 次(種子的空前綴 + index 1 那次)。 每一個都能和現在配成一條和為 0 的子陣列,所以 cnt += 2。這就是為何要記「次數」而非「存在」。
