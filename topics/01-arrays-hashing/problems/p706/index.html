題目簡介// THE QUESTION

QUESTION

不用內建雜湊表,自己實作 MyHashMap:put(key, value) 插入/更新、get(key) 取值(沒有回 -1)、remove(key) 移除。 約束:0 ≤ key, value ≤ 10⁶。

這題是 705 設計雜湊集合的雙胞胎 —— 差別只有一個:705 存「在不在」(bool),706 存「對到什麼 value」(int)。 一樣靠 key ≤ 10⁶ 這個界,用直接定址,只是每格從 true/false 換成 value。

2

核心 · int 陣列 + -1 當「不存在」// SENTINEL

key 當索引,格子存 value,-1 = 沒映射

開一個 vector<int> mp,大小 10⁶+1,全部初始化為 -1。mp[key] 存這個 key 對到的 value:

　put(k, v)　→　mp[k] = v
　get(k)　　→　return mp[k]（可能是 -1）
　remove(k)　→　mp[k] = -1（改回哨兵）

-1 當「不存在」的哨兵(sentinel):因為題目保證 value ≥ 0,-1 絕不會是一個真實的 value,所以 get 讀到 -1 就能斷定「這個 key 沒 put 過」。哨兵值必須落在合法 value 範圍之外 —— 這是它能安全代表「空」的前提。

5

哨兵值選錯會出事// SENTINEL PITFALL

若 value 可能等於哨兵,就分不清「有值」和「空」

用 -1 當哨兵只在 value 保證 ≥ 0 時安全。若題目的 value 可以是 -1,那 mp[k] = -1 就分不清是「put(k, -1) 存了真實的 -1」還是「k 根本沒映射」——get 兩種都回 -1,語意崩壞。
通用解法:另開一個 vector<bool> present 專門記「這個 key 有沒有映射」,value 陣列則存任意值。get 先查 present[k] 再回 value[k]。這樣把「存在性」和「值」分開,不管 value 是什麼都不會混淆。哨兵是省一個陣列的偷懶技巧,前提是它落在 value 範圍外。

6

複雜度// COMPLEXITY

直接定址:T = O(1)／op　　S = O(值域)

每操作一次陣列存取。vector<int> 存 10⁶ 個 → 約 4 MB(比 705 的 bool 陣列 125 KB 大,因為 int 4 bytes/格)。key 無界時同樣改雜湊 + 桶,桶裡存 pair<key, value>(見程式頁)。
