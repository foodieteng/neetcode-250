完整實作 · 直接定址 + 哨兵// O(1) PER OP

p706.cpp C++17 
class MyHashMap {
public:
    vector<int> mp;

    MyHashMap() {
        mp.assign(1000001, -1);      // 全 -1 = 都沒映射
    }

    void put(int key, int value) {
        mp[key] = value;            // 存 / 覆蓋
    }

    int get(int key) {
        return mp[key];            // -1 代表不存在
    }

    void remove(int key) {
        mp[key] = -1;              // 改回哨兵
    }
};

和 705 唯一的差:陣列型別 bool → int、「空」用 -1 哨兵而非 false。put 天然支援「更新」——直接覆蓋。

若 value 可能是 -1 · 用 present 陣列// ROBUST

// 把「存在性」和「值」分成兩個陣列,不靠哨兵 class MyHashMap { vector<int> val; // 任意 value(含負數) vector<bool> present; // 這個 key 有沒有映射 public: MyHashMap() : val(1000001), present(1000001, false) {} void put(int key, int value) { val[key] = value; present[key] = true; } int get(int key) { return present[key] ? val[key] : -1; } void remove(int key) { present[key] = false; } }; present 記存在性 → value 可以是任何數(包含 -1),不再和哨兵混淆

本題 value ≥ 0,單一 -1 哨兵就夠、更省。但面試若追問「value 可以是任意整數呢?」,這個 present 雙陣列版就是答案 —— 把「有沒有」和「是多少」拆開,是處理「哨兵可能撞到合法值」的通用手法。

通用版 · 雜湊 + 桶(存 pair)// UNBOUNDED

// key 無界:每桶掛 list<pair<key,value>> class MyHashMap { static const int B = 769; vector<list<pair<int,int>>> buckets; int h(int key) { return key % B; } public: MyHashMap() : buckets(B) {} void put(int key, int value) { for (auto& p : buckets[h(key)]) if (p.first == key) { p.second = value; return; } // 更新 buckets[h(key)].push_back({key, value}); // 新增 } int get(int key) { for (auto& p : buckets[h(key)]) if (p.first == key) return p.second; return -1; } void remove(int key) { buckets[h(key)].remove_if([&](auto& p){ return p.first == key; }); } }; 和 705 桶版一樣,只是桶裡存 (key,value) 對;put 要先找 key 決定更新或新增

桶版和 705 幾乎相同,只把 list<int> 換成 list<pair<int,int>>。put 要先掃桶:找到同 key 就更新 value、沒有才新增(維持「一個 key 一份」)。這正是 unordered_map 的骨架。

邊界與陷阱// PITFALLS

陷阱症狀正解
初始化為 0 而非 -1 get 未 put 的 key 回 0,但 0 是合法 value → 誤判「有映射」 assign(…, -1),-1 當哨兵
value 可能是 -1 仍用哨兵 put(k,-1) 和「沒映射」分不清 改用 present 雙陣列版
桶版 put 不先找 key 同 key put 兩次 → 桶裡兩份,get 拿到舊值 put 先掃桶,找到就更新
陣列開 10⁶ key==10⁶ 越界 大小 1000001

細節走讀// WALKTHROUGH

// 直接定址,-1 哨兵 put(1,10) → mp[1]=10 put(2,20) → mp[2]=20 get(1) → mp[1] = 10 get(3) → mp[3] = -1 (從沒 put → 不存在) put(2,30) → mp[2]=30 (更新:直接覆蓋) get(2) → mp[2] = 30 remove(2) → mp[2]=-1 get(2) → mp[2] = -1 (已移除 → 不存在) 全部 O(1),行為和 std::map 一致
注意 put(2,30) 的「更新」是免費的 —— 直接覆蓋 mp[2],不用先檢查存在。這是直接定址的又一好處: 插入和更新是同一個動作(而桶版要先掃桶區分新增/更新)。
