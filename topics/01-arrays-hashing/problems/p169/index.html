題目簡介// THE QUESTION

QUESTION

給大小為 n 的陣列 nums,找出多數元素 —— 出現次數超過 ⌊n/2⌋ 的那個(保證存在)。 例 [2,2,1,1,1,2,2] → 2(出現 4 次 > 7/2)。

follow-up 問「能不能 線性時間、O(1) 空間?」——雜湊計數是 O(n) 時間但 O(n) 空間、排序取中位是 O(n log n)。 真正漂亮的答案是 Boyer-Moore 投票演算法:一個計數器、一個候選人,一遍掃描,O(1) 空間。

2

核心 · 不同的兩票互相抵銷// PAIR & CANCEL

關鍵直覺 · 多數元素抵銷不完

想像一場「兩兩對消」的遊戲:把兩個不同的元素配成一對、一起丟掉(它們互相抵銷)。不斷這樣配對消除,直到剩下的都相同。

多數元素出現 > n/2 次,比「其他所有元素加起來」還多。所以就算讓它和每個非多數元素一對一抵銷,對面的「彈藥」也不夠消光它 —— 最後一定有多數元素剩下來。Boyer-Moore 就是把這個「配對抵銷」用一個計數器模擬出來,不用真的兩兩配。

3

核心策略 · 候選人 + 票數// CANDIDATE + COUNT

  1. 維護一個候選人 candidate 和票數 cnt(初始 0)。
  2. 掃每個 num:
    　● cnt == 0 → 沒有候選人了,讓 num 當新候選人,cnt = 1。
    　● num == candidate → 同一陣營,cnt++(加持)。
    　● num != candidate → 敵對票,cnt--(一對一抵銷)。
  3. 掃完回傳 candidate。因為多數元素抵銷不完,最後留下的一定是它。

cnt 不是「candidate 的真實出現次數」,而是「candidate 目前領先對手多少票」。cnt 歸零代表「打平了」,此時換人重新開始。

候選人會中途換人:2 →(被 1 抵銷歸零)→ 1 →(又歸零)→ 2。但因為 2 是多數,最後一定回到 2。

為什麼保證正確// WHY IT WORKS

每次 cnt-- 都「消掉一個多數 + 一個非多數」

把每次 cnt-- 看成「candidate 一票 和 當前這個異票 同歸於盡」。能抵銷掉一個多數元素,就得付出一個非多數元素當代價。而非多數元素總共 < n/2 個,多數元素> n/2 個 —— 對面的彈藥不足以消光多數元素。所以無論候選人怎麼換,最後 cnt 一定停在某段「多數元素」上、以正值收尾,candidate 就是它。
但注意:這只在「多數元素確實存在(> n/2)」時保證正確。若不保證存在,演算法會回一個「候選人」,但它不一定真的過半 —— 需要第二遍驗證它的次數(見下一頁)。本題題目保證存在,所以一遍就夠。

6

複雜度// COMPLEXITY

T = O(n)　　S = O(1)

一遍掃描、每步 O(1)。只用兩個變數(候選人 + 票數)→ O(1) 空間,完美回答 follow-up。 對比:雜湊計數 O(n)/O(n)、排序取中位 O(n log n)/O(1) —— Boyer-Moore 兩者兼得。

7

一句洞察// INSIGHT

不同的兩票互相抵銷,多數元素抵銷不完。用一個候選人 + 票數模擬:cnt==0 換人、同票 +1、異票 -1。 cnt 是「領先幅度」不是真次數。O(n) 時間、O(1) 空間。
