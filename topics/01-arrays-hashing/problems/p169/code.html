完整實作 · Boyer-Moore// O(n) TIME · O(1) SPACE

p169.cpp C++17 
class Solution {
public:
    int majorityElement(vector<int>& nums) {
        int cnt = 0, candidate = 0;

        for (int num : nums) {
            if (cnt == 0) {                 // 沒候選人 → 換人
                cnt++; candidate = num;
            } else if (candidate == num) {  // 同票 → 加持
                cnt++;
            } else {                        // 異票 → 抵銷
                cnt--;
            }
        }
        return candidate;                   // 多數元素抵銷不完,必勝出
    }
};

三個分支對應三種情況:歸零換人 / 同票 +1 / 異票 −1。candidate=0 的初值無所謂 —— 第一個元素必然觸發 cnt==0 把它換掉。

關鍵 · cnt 不是「出現次數」// WHAT cnt MEANS

cnt = 目前候選人「領先其他票的淨差」

新手常誤以為 cnt 是 candidate 的真實出現次數 —— 不是。它是「從上次歸零到現在,candidate 這一陣營比對手多出的票數」。
同票 +1(領先擴大)、異票 -1(領先縮小)、歸零就是「這段打成平手」→ 之前的票全數抵銷完畢,從下一個元素重新開一局。

正因為 cnt 只是「淨差」,它可以在中途歸零、candidate 也可以換人好幾次 —— 但每一段被抵銷掉的,都是「等量的多數 + 非多數」。非多數彈藥不足,所以撐到最後那一段一定屬於多數元素。理解「cnt=淨差、不是次數」,是看懂這演算法的鑰匙。

若「不保證存在」· 要第二遍驗證// VERIFY PASS

// Boyer-Moore 只找出「候選人」;不保證存在時要數一次確認 int cnt = 0, candidate = 0; for (int num : nums) { // 第一遍:找候選人 if (cnt == 0) candidate = num; cnt += (num == candidate) ? 1 : -1; } // 第二遍:驗證 candidate 是否真的過半 int occur = 0; for (int num : nums) if (num == candidate) occur++; return occur > nums.size() / 2 ? candidate : -1; // 沒有多數元素回 -1 本題保證存在 → 可省第二遍;但知道「何時該補驗」很重要

Boyer-Moore 的產物是「唯一可能的候選人」——如果有多數元素,它一定是這個 candidate;但如果根本沒有多數元素,candidate 只是「碰巧撐到最後的那個」,不保證過半。 所以在沒保證存在的變體裡,必須第二遍數一次 candidate 的真實次數來確認。本題明說保證存在,才能省掉這遍。

4

三種解法對照// TRADE-OFFS

解法時間空間備註
雜湊計數O(n)O(n)直觀,但空間不達 follow-up
排序取中位O(n log n)O(1)*多數元素必佔中間位置 nums[n/2]
Boyer-MooreO(n)O(1)唯一同時線性 + 常數空間
排序法有個好記的性質:排序後多數元素(> n/2 個)一定橫跨正中間,所以 nums[n/2] 直接就是答案 —— 一行 sort + 回傳中間。 但要達成 follow-up 的「線性 + O(1)」,只有 Boyer-Moore。面試講 Boyer-Moore,能補一句排序中位當備選。

細節走讀// [2,2,1,1,1,2,2]

// candidate 中途換過人,但最後回到多數元素 num=2 cnt==0 → candidate=2, cnt=1 num=2 同票 → cnt=2 num=1 異票 → cnt=1 num=1 異票 → cnt=0 ← 2 的領先被打平 num=1 cnt==0 → candidate=1, cnt=1 ← 換 1 當候選人! num=2 異票 → cnt=0 ← 1 又被打平 num=2 cnt==0 → candidate=2, cnt=1 ← 2 重新當選 return 2 (2 出現 4 次 > 7/2,無論怎麼換都撐到最後)

這個例子刻意讓 candidate 換了兩次(2→1→2),展示「cnt 歸零就換人」的機制。 即使中途 candidate 變成非多數的 1,它也撐不住(下一票就被抵銷)——只有真多數元素能笑到最後。
