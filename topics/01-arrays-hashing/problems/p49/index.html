1

題目簡介// THE QUESTION

QUESTION

給字串陣列 strs,把互為字母異位詞(anagram)的字分在同一組,回傳所有組(順序不拘)。 例:["eat","tea","tan","ate","nat","bat"] → [["eat","tea","ate"],["tan","nat"],["bat"]]。

核心只有一句:兩字互為 anagram ⇔ 把字母排序後完全相同。 於是「排序後的字串」就是每個 anagram 群共有的指紋(canonical key) —— 拿它當雜湊表的 key,同群的字自然落進同一個桶。這是 242 驗證 anagram 的「分組版」。

2

為什麼「排序字串當鍵」// WHY

× REJECT · O(N²·L)
兩兩比較誰是 anagram

對每對字檢查是否 anagram,再併組。O(N²) 對比 × 每次 O(L) —— N 到 10⁴ 時退化嚴重。

△ OK · 更快的鍵
字母計數當鍵

用 26 長度的計數陣列(轉成字串)當 key,免排序,每字 O(L)。更快,但寫起來稍繁瑣(見第 5 節)。

SIMPLE & ENOUGH · ADOPT ↓
● ADOPT · sorted key
sorted(w) 當鍵,一趟分桶

對每個字 w,把它的字母排序得到 key = sorted(w),丟進 map[key]。 同 anagram 群的字排序後 key 相同 → 落進同一桶;掃完把所有桶收集起來就是答案。 單趟 O(N · L log L),程式極短、最好記。

3

核心策略 · 三步驟// STRATEGY

  1. 算鍵 —— 對每個字 w,key = sorted(w)(把字母排序)。
  2. 分桶 —— map[key].push_back(w),把原字放進對應的桶。
  3. 收集 —— 掃完後,把 map 裡每個桶(value)收進答案陣列。

兩種標準鍵的取捨// SORT vs COUNT KEY

排序鍵 sorted(w)計數鍵 count[26]
鍵怎麼算把字母排序成字串數 26 字母次數,拼成字串/tuple
每字成本O(L log L)O(L)
總時間O(N·L log L)O(N·L)
程式3 行,最好寫要組計數字串,稍長
適用字串短、要簡潔字串長、要極速
這正是 242 的「排序 vs 計數」之爭搬到分組場景:兩者都用「anagram 的標準指紋」, 差別只在指紋怎麼算。本題 L ≤ 100 很短,排序鍵完全夠用且最好記;字串很長時計數鍵省下 log L 因子。
6

複雜度// COMPLEXITY

排序鍵:T = O(N · L log L)　　S = O(N · L)

N = 字數、L = 最長字串長。每字排序 O(L log L)、雜湊插入均攤 O(L)(要雜湊整個 key)。空間存下所有字 + 所有 key ≈ O(N·L)。計數鍵版時間降為 O(N·L)。

7

一句洞察// INSIGHT

anagram ⇔ 排序後相同 → 用 sorted(w) 當雜湊 key 分桶,同群自動聚在一起。 「幫每個物件算一個標準指紋、再用雜湊表分組」是通用套路:換成計數鍵可省 log L。
