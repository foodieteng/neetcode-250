完整實作 · 排序鍵分桶// SORTED KEY

p49.cpp C++17 
class Solution {
public:
    vector<vector<string>> groupAnagrams(vector<string>& strs) {
        unordered_map<string, vector<string>> group;

        for (auto w : strs) {
            string key = w;
            sort(key.begin(), key.end());   // 標準指紋 = 排序後字串
            group[key].push_back(w);        // 丟進對應桶
        }

        vector<vector<string>> ans;
        for (auto [k, v] : group)          // 收集所有桶
            ans.push_back(v);
        return ans;
    }
};
一個小效能點:for (auto w : strs) 和 for (auto [k, v] : group) 都是複製。 字串短時無所謂,但更省的寫法是 for (const auto& w : strs)(免拷貝原字)與 for (auto& [k, v] : group), 收集時再 ans.push_back(std::move(v))。功能相同,少幾次深拷貝。

更快的鍵 · 字母計數// O(N·L)

// 免排序:26 計數轉成字串當 key,每字 O(L) vector<vector<string>> groupAnagrams(vector<string>& strs) { unordered_map<string, vector<string>> group; for (auto& w : strs) { int cnt[26] = {0}; for (char c : w) cnt[c - 'a']++; string key; for (int i = 0; i < 26; i++) key += '#' + to_string(cnt[i]); // 例 "#1#0#2..." 避免 11 與 1,1 混淆 group[key].push_back(w); } vector<vector<string>> ans; for (auto& [k, v] : group) ans.push_back(v); return ans; } O(N·L) 時間;分隔符 '#' 是關鍵,否則 count=[1,11] 和 [11,1] 會拼成同一串

分隔符 '#' 不能省:count=[1,11,0…] 若直接拼成 "111…",會和 [11,1,0…] 撞鍵。 加分隔符讓每個計數獨立可辨。本題字母只有小寫、L ≤ 100,兩版都秒過;計數版在字串很長時勝出。

邊界與陷阱// PITFALLS

陷阱症狀正解
用原字當 key eat、tea 各自成組 → 沒分到一起 用 sorted(w) 或計數當 key
排序改到原字 sort(w...) 後把 w 也弄亂 → 桶裡存的是排序後的字 複製一份 key=w 再排序,push 原 w
計數鍵沒加分隔符 [1,11] 與 [11,1] 撞鍵 → 錯併組 每個計數前加 '#'
空字串 "" 排序仍是 "" 天然正確:所有空字串歸同一桶
忘了輸出順序不拘 糾結桶的順序 / 組內順序 題目允許任意順序,不需排

細節走讀// WALKTHROUGH

// strs=["eat","tea","tan","ate","nat","bat"] eat → sort → "aet" group["aet"]=[eat] tea → sort → "aet" group["aet"]=[eat,tea] tan → sort → "ant" group["ant"]=[tan] ate → sort → "aet" group["aet"]=[eat,tea,ate] nat → sort → "ant" group["ant"]=[tan,nat] bat → sort → "abt" group["abt"]=[bat] 收集: [[eat,tea,ate], [tan,nat], [bat]] 3 群(順序不拘)
每個字只做一次「排序 → 插桶」,和其他字零比較 —— 雜湊表把「找同群」這件事變成 O(1) 查表。 這就是雜湊分組的威力:把 O(N²) 的兩兩比對,換成 O(N) 次「算指紋 + 插桶」。
