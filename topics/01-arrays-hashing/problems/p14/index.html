1

題目簡介// THE QUESTION

QUESTION

給一組字串 strs,找出它們全部共有的最長開頭字串(最長共同前綴)。沒有共同前綴就回空字串 ""。 例 ["flower","flow","flight"] → "fl";["dog","racecar","car"] → ""。

「共同前綴」= 每個字串開頭都一樣的那一段。核心觀察:答案不可能比任何一個字串長,也不可能比它們共有的部分長。 所以策略是「先假設一個候選,再一路砍到大家都同意」。

2

核心 · 拿 strs[0] 當候選,越砍越短// SHRINK

水平掃描 · prefix 只會變短、不會變長

共同前綴一定是 strs[0] 的前綴(它得是每個字串的前綴,當然也是第一個的)。所以直接拿 strs[0] 當初始候選 prefix。

接著逐一比對每個字串 str:只要 prefix 不是 str 的開頭,就從右邊砍掉一個字元,砍到它變成 str 的前綴為止。全部比完,剩下的 prefix 就是答案。

為什麼只砍不加:每比一個字串,共同的部分只會更嚴格(交集越取越小),不可能變長。所以 prefix 單調變短 —— 這保證了演算法會收斂、且最多砍到空字串。

3

用 str.find(prefix) == 0 判前綴// THE TRICK

find 回傳「首次出現位置」,== 0 就代表在開頭

str.find(prefix) 回傳 prefix 在 str 裡第一次出現的位置(找不到回 npos)。 若它回傳 0,代表 prefix 正好從 str 的開頭開始 —— 也就是 prefix 是 str 的前綴。

所以 while (str.find(prefix) != 0) prefix.pop_back掉一個 的意思是:「只要 prefix 還沒對齊到 str 開頭,就砍尾」。 邊界很漂亮:當 prefix 砍到空字串時,str.find("") 回傳 0(空字串在任何位置都算「找到」),迴圈自然停下 → 再判 prefix.empty() 直接回 ""。

5

另一種視角 · 垂直掃描// VERTICAL

一欄一欄比,遇到第一個不一致就停

除了「砍前綴」(水平),還有更直觀的「逐欄比對」(垂直):看所有字串的第 0 個字元是否全相同、再看第 1 個…直到某一欄出現不一致、或某個字串到底了就停,前面對齊的那幾欄就是答案。
例 ["flower","flow","flight"]:第 0 欄全 f、第 1 欄全 l、第 2 欄 o/o/i 不一致 → 停,答案 "fl"。
垂直法能「一發現分歧就停」,常常更快(不用把某個字串整個比完);水平法程式更短。兩者都 O(總字元數),選你順手的。

6

複雜度// COMPLEXITY

T = O(S)(S = 所有字元總數)　　Sspace = O(1)*

最壞情況每個字元都要比一次 → O(S)，S = Σ strs[i].length。額外空間只有 prefix(*不計輸出,且它是 strs[0] 的子字串,O(m),m = 最短/首字串長)。strs.length ≤ 200、每字 ≤ 200,非常寬鬆。

一句洞察// INSIGHT
