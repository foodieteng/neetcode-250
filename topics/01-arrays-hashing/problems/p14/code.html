完整實作 · 水平掃描// O(S)

p14.cpp C++17 
class Solution {
public:
    string longestCommonPrefix(vector<string>& strs) {
        string prefix = strs[0];            // 拿第一個字串當初始候選

        for (string str : strs) {
            while (str.find(prefix) != 0) {       // prefix 不在 str 開頭?
                prefix = prefix.substr(0, prefix.size() - 1);  // 砍尾一個
            }
            if (prefix.empty()) return "";          // 砍光了,沒共同前綴
        }
        return prefix;
    }
};

每個字串都把 prefix 砍到「自己的前綴」為止。因為 prefix 只減不增,一旦砍成空字串就能提前 return ""。 str.find(prefix)==0 是「prefix 是 str 前綴」的簡潔判斷。

關鍵 · find 的三種回傳與空字串邊界// find SEMANTICS

str.find(prefix)意思迴圈行為
== 0prefix 從 str 開頭出現 = 是前綴停,換下一個字串
> 0prefix 在 str 中間才出現,不在開頭!= 0,繼續砍
== nposprefix 根本不在 str 裡!= 0,繼續砍
為什麼砍到空字串迴圈會自己停

當 prefix 被砍成 "",str.find("") 依 C++ 規範回傳 0(空字串在任何字串的位置 0「找得到」)。於是 while 條件 != 0 為假,迴圈停止,不會 substr(0, -1) 造成越界。接著 if (prefix.empty()) 抓到空 → 回 ""。空字串這個邊界被 find 的語意天然守住,不用額外 size()>0 判斷。

另解 · 垂直掃描(遇分歧即停)// VERTICAL

// 逐欄比,任一字串到底或不一致就停 string longestCommonPrefix(vector<string>& strs) { for (int i = 0; i < strs[0].size(); i++) { // 第 i 欄 char c = strs[0][i]; for (string& s : strs) if (i == s.size() || s[i] != c) // 某字串到底,或字元不一致 return strs[0].substr(0, i); // 前 i 欄就是答案 } return strs[0]; // 第一個字串本身就是共同前綴 } 一發現分歧立刻回傳,常常不用把字串整個掃完

垂直法的優勢是「早停」:分歧出現在第 k 欄,就只比了 k 欄 × 字串數,不必像水平法可能把某個長字串整個 find。 兩者最壞都 O(S);實務上垂直法在「前綴很短」時更快,水平法勝在程式極短。面試講水平法(短)、能補一句垂直法（早停更快）就完整。

邊界與陷阱// PITFALLS

陷阱症狀正解
陣列含空字串 ["","abc"]:空字串沒有任何前綴 → 共同前綴必為 "" 水平法會把 prefix 砍到空 → 回 "" ✓(自動處理)
用 find 但沒判 ==0 只判「有沒有出現」→ prefix 在中間出現時誤判成前綴 必須 find(prefix) == 0(在開頭)
垂直法忘了判 i==s.size() 某字串比 strs[0] 短時 s[i] 越界 先判「這個字串是不是到底了」
strs 只有一個字串 以為要特判 不用,水平法回 strs[0] 本身 ✓
假設 strs 非空 strs[0] 在空陣列上越界 題目保證 strs.length ≥ 1,可放心
