完整實作 · 荷蘭國旗// ONE PASS · O(1)

p75.cpp C++17 
class Solution {
public:
    void sortColors(vector<int>& nums) {
        int i = 0, l = 0, r = nums.size() - 1;   // l=0界 i=游標 r=2界

        while (i <= r) {
            if (nums[i] == 0) {
                swap(nums[i], nums[l]);          // 0 換到左界
                l++; i++;                        // 換來的必是 1 → i 前進
            } else if (nums[i] == 2) {
                swap(nums[i], nums[r]);          // 2 換到右界
                r--;                             // i 不動!換來的還沒檢查
            } else {
                i++;                             // 1 留中,只前進
            }
        }
    }
};
三個分支的不對稱是全題精髓:遇 0 換左後 i++(l 換來的來自已掃的 1 區、必是 1);遇 2 換右後 i 不動(r 換來的來自未看區,得重判)。 l ≤ i 恆成立,所以 swap(nums[i], nums[l]) 就算 i==l 也無害(換自己)。

對照寫法 · 兩趟計數排序// TWO PASS

// 若不要求 one-pass:數 0/1/2 個數,再覆寫。更好想、但掃兩趟 void sortColors(vector<int>& nums) { int cnt[3] = {0, 0, 0}; for (int x : nums) cnt[x]++; // 第一趟:計數 int idx = 0; for (int v = 0; v < 3; v++) while (cnt[v]--) nums[idx++] = v; // 第二趟:依數量覆寫 } O(n) 時間、O(1) 空間,但掃兩趟;荷蘭國旗一趟就地完成,是 one-pass 追問的答案

計數排序更直觀,值域小時很實用;但它覆寫而非交換,且要兩趟。荷蘭國旗把「計數 + 覆寫」壓成「邊掃邊交換」的一趟原地版。 當值不只三種、但仍是小範圍整數時,計數排序更通用;剛好三種且要 one-pass,選荷蘭國旗。

邊界與陷阱// PITFALLS

陷阱症狀正解
遇 2 後也 i++ 跳過從 r 換來的未檢查元素 → 排序錯 遇 2 只 r--,i 不動
迴圈條件用 i < r i==r 那格漏處理 while (i <= r)
遇 0 忘了 i++ l 換來的 1 又被當新元素重看 → 空轉/錯 遇 0 要 l++ 也 i++
三指針初值錯 r = n 越界 / l 從 1 起 l=0, i=0, r=n−1
空陣列 r=−1,i=0 > r 迴圈不進 天然安全,回傳空

// nums=[2,0,2,1,1,0] (l,i,r 初值 0,0,5) i=0 nums[0]=2 → swap(0,5) [0,0,2,1,1,2] r=4 i=0(不動) i=0 nums[0]=0 → swap(0,0) l=1 i=1 i=1 nums[1]=0 → swap(1,1) l=2 i=2 i=2 nums[2]=2 → swap(2,4) [0,0,1,1,2,2] r=3 i=2(不動) i=2 nums[2]=1 → i=3 i=3 nums[3]=1 → i=4 i=4 > r=3 → 結束 [0, 0, 1, 1, 2, 2]
看第一步:nums[0]=2 換到 r=5,換來 0,i 留在 0。下一輪立刻重判這個 0 → 換到左界。 正是因為 i 沒前進,才不會漏掉這個換來的 0。若當時 i++,這個 0 就永遠留在中間,結果變成 [錯]。
