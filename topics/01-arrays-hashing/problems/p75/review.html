重點回顧// RECAP

關鍵一句話
三指針l=0 區右界、i=游標、r=2 區左界
四段不變量[0,l)=0 · [l,i)=1 · [i,r]=未看 · (r,n)=2
遇 0swap(i,l); l++; i++(換來必是 1)
遇 2swap(i,r); r--,i 不動(換來未檢查)
遇 1i++
本質三路快排 partition 的雛形
複雜度一趟 O(n)、原地 O(1)

觀念題// CONCEPT Q

Q1

為什麼遇 0 交換後 i++,但遇 2 交換後 i 不動?用不變量解釋這個不對稱。

Q2

迴圈條件為什麼是 i <= r 而非 i < r?用 i==r 那一步說明。

Q3

兩趟計數排序也是 O(n)/O(1),為什麼追問還要荷蘭國旗?兩者差在哪?

Q4

荷蘭國旗和 快速排序的 partition 是什麼關係?為什麼叫「三路分割」?

Q5

若把「遇 0」分支寫成 swap(i,l); i++;(忘了 l++),會發生什麼?

範例 Trace// i 不動如何救回換來的元素

// nums=[2,0,1](l,i,r=0,0,2)—— 看遇 2 後不 i++ 的必要性 i=0 nums[0]=2 → swap(0,2) [1,0,2] r=1 i=0(不動) ↑ 若這裡 i++,就跳過換來的 nums[0]=1?不,更糟: 看 [2,0,2] 這種,換來的可能是 0,i++ 會漏掉它 i=0 nums[0]=1 → i=1 i=1 nums[1]=0 → swap(1,0)?l=0 → [0,1,2] l=1 i=2 i=2 > r=1 → 結束 [0, 1, 2]
正因為遇 2 時 i 留在原地,下一輪才會重新檢查從 r 換來的那個元素。 對 [2,0,2,1,1,0] 第一步換來的是 0 —— 若當時 i++,這個 0 會被永遠卡在中間、排不到左邊,結果就錯。「不前進」= 給換來的未知元素一次被判斷的機會。

應用題// APPLY

App 1

四種顏色 / k 種值:若不是 3 種而是 4 種、甚至 k 種小整數,荷蘭國旗還能一趟嗎?該用什麼?

App 2

三路快排(LC 912):把荷蘭國旗的「0/1/2」換成「<pivot / =pivot / >pivot」,就是三路快排。它在大量重複元素時為何遠勝普通兩路快排?

App 3

穩定性:荷蘭國旗是不穩定的(交換會打亂相等元素順序)。若題目要求穩定,且值只有三種,該怎麼做?

// ANSWERS
A1

關鍵在「換來的元素來自哪一段」。遇 0 換到 l:l 位置屬於 [l, i) 這個已掃過的 1 區,換來的必是 1(或 i==l 時是自己),已知已排好,i 可安全前進。遇 2 換到 r:r 位置屬於 [i, r] 這個還沒檢查的未看區,換來的可能是 0/1/2,必須留 i 原地重判。不對稱來自「左邊已知、右邊未知」。

A2

因為 i==r 時,nums[i](= nums[r])這一格還沒被歸類,必須處理。用 i < r 會在 i==r 時就退出,漏掉最後這格 —— 例如 [0](i=r=0)或收尾時剩一個未看元素會被漏排。i <= r 確保「未看區 [i,r] 非空就繼續」,直到 i 越過 r(未看區空)才停。

A3

兩者都 O(n)/O(1),但趟數與寫入次數不同。計數排序掃兩趟(數 + 覆寫),荷蘭國旗一趟(邊掃邊換)。追問明確要 one-pass,考的就是荷蘭國旗這個技巧。實務上兩者都好;但當「讀取代價高」或「只能單次遍歷(如串流的近似)」時,one-pass 更有價值。且荷蘭國旗是「原地交換」,計數排序是「覆寫」—— 若元素是複雜物件(不只是值),交換 vs 重建的成本也不同。

A4

荷蘭國旗就是三路快排的 partition:把「等於 pivot」的元素集中在中間,「小於」丟左、「大於」丟右,一趟切成三段。普通快排是兩路(≤pivot / >pivot),等於 pivot 的元素散在兩邊;三路把「=pivot」獨立成一段,遞迴時跳過中間相等段,只對左右遞迴。本題的 0/1/2 相當於「pivot=1」的一次三路 partition。

A5

會壞。l 是「0 區的右界」,每放一個 0 就要 l++ 讓界前移。忘了 l++:下一個 0 又會換到同一個 l 位置,把剛放好的 0 換走 → 0 區無法增長、元素互相覆蓋,排序錯亂。l 與 0 的個數必須同步:放一個 0,界就前進一格。

App 1

荷蘭國旗專為 3 段設計(l/i/r 剛好切三段);4 種以上就無法用單一個 i 一趟切好(段界之間會互相干擾)。此時改用計數排序:數每種值幾個,再依序覆寫,O(n + k),對任意 k 種小整數都一趟數 + 一趟寫。想 one-pass 且 k 種,通常做不到嚴格一趟就地(除非用額外空間);計數排序的兩趟是最乾淨的通解。

App 2

大量重複時,普通兩路快排會把「等於 pivot」的元素反覆分到子問題裡再排一次 —— 全部相同的陣列會退化成 O(n²)。三路快排把「=pivot」的一整段一次定位、遞迴時完全跳過,重複越多、跳過越多,對「少數不同值 + 大量重複」能到接近 O(n)。這是處理重複鍵的標準優化(如 [5,5,5,…,5] 三路直接一趟結束)。

App 3

荷蘭國旗靠交換,會打亂相等元素的相對順序 → 不穩定(本題值就是全部資訊,看不出差別;但若元素帶附加資料就有差)。要穩定又只有三種值:用計數排序 —— 數完 0/1/2 個數,依序覆寫,同值元素保持原順序(需搭配輸出陣列存原物件才真穩定),O(n) 兩趟。穩定 + 三值 → 計數排序;one-pass 原地但不穩 → 荷蘭國旗。取捨看需求。
