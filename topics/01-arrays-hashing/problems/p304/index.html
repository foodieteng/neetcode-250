題目簡介// THE QUESTION

QUESTION

設計 NumMatrix:建構時吃一個 2D 矩陣,之後多次查詢 sumRegion(r1,c1,r2,c2) = 以 (r1,c1) 為左上角、(r2,c2) 為右下角的矩形內元素總和。要求 sumRegion O(1)。最多 10⁴ 次查詢。

「多次查詢 + 每次 O(1)」是經典的預處理換查詢訊號:建構時花 O(mn) 算好二維前綴和, 之後每次查詢用容斥(加加減減)四個角落值即可。這是 560 一維前綴和 的二維版。

2

為什麼「二維前綴和 + 容斥」// WHY

× REJECT · 每查 O(面積)
每次查詢直接雙層迴圈加

每次 sumRegion 掃整個矩形相加。單查 O(mn),10⁴ 次 × 4×10⁴ 面積 → 太慢。

△ OK · 每查 O(列數)
每列各存一維前綴和

每列一個前綴和,查詢時把 r1..r2 每列的區段和加起來。O(r2−r1)/查 —— 好一些,但不是 O(1)。

O(1) QUERY · ADOPT ↓
● ADOPT · 2D prefix + 容斥
prefix[i][j] = 左上矩形總和,查詢四角容斥

預處理 prefix[i][j] = 從 (0,0) 到 (i-1,j-1) 的矩形總和。查詢時: 大矩形 − 上長條 − 左長條 + 左上角(角落被減兩次要加回)。四次查表 + 加減 = O(1),建表 O(mn)。

3

核心策略 · 建表 + 查詢// STRATEGY

  1. 多開一圈 0 邊界 —— prefix 尺寸 (m+1)×(n+1),第 0 列/行全 0,省掉邊界判斷。
  2. 建表(容斥累積) —— prefix[i][j] = 上 + 左 − 左上 + matrix[i-1][j-1]。
  3. 查詢(容斥抽取) —— prefix[r2+1][c2+1] − prefix[r1][c2+1] − prefix[r2+1][c1] + prefix[r1][c1]。

兩個容斥公式// BUILD & QUERY

① 建表:上 + 左 − 左上 + 自己

prefix[i][j] = prefix[i-1][j] + prefix[i][j-1] − prefix[i-1][j-1] + matrix[i-1][j-1]

「上方矩形」和「左方矩形」相加時,左上角那塊被算了兩次,要減掉一次;最後補上當前格 matrix[i-1][j-1]。這就是容斥。

② 查詢:大矩形 − 兩長條 + 左上角

sum = prefix[r2+1][c2+1] − prefix[r1][c2+1] − prefix[r2+1][c1] + prefix[r1][c1]

要留下的是「右下大矩形扣掉上方與左方多出來的部分」。上長條和左長條重疊在左上角,那塊被減了兩次,所以要加回一次。 兩條公式其實是同一個容斥原理:一個用來「一格一格累積」,一個用來「一次抽出任意矩形」。邊界那圈 0 讓 r1=0 / c1=0 時的 prefix[0][…]、prefix[…][0] 自然是 0,不必特判。

複雜度// COMPLEXITY

建表 = O(mn)　　每次查詢 = O(1)　　空間 = O(mn)

建構時掃一遍矩陣填前綴和;之後每次 sumRegion 固定四次查表 + 三次加減。Q 次查詢總成本 O(mn + Q),遠勝「每查 O(mn)」的 O(Q·mn)。空間多存一個 (m+1)×(n+1) 前綴表。

階段成本說明
建構(預處理)O(mn)填滿前綴表
單次 sumRegionO(1)四查表 + 加減
Q 次查詢O(mn + Q)預處理攤到多次查詢
空間O(mn)(m+1)×(n+1) 前綴表

一句洞察// INSIGHT

多次矩形和查詢 → 預處理二維前綴和(左上矩形總和),查詢用容斥四角:大矩形 − 上長條 − 左長條 + 左上角。 建表與查詢是同一容斥的兩面。多開一圈 0 邊界免特判。建 O(mn)、查 O(1)。
