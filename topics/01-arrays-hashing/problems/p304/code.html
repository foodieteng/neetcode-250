完整實作 · 二維前綴和// BUILD O(mn) · QUERY O(1)

p304.cpp C++17 
class NumMatrix {
public:
    int m, n;
    vector<vector<int>> prefix;

    NumMatrix(vector<vector<int>>& matrix) {
        m = matrix.size(), n = matrix[0].size();
        prefix.assign(m + 1, vector<int>(n + 1, 0));  // 多開一圈 0

        for (int i = 1; i <= m; i++)
            for (int j = 1; j <= n; j++)            // 建表:上 + 左 − 左上 + 自己
                prefix[i][j] = prefix[i-1][j] + prefix[i][j-1]
                             - prefix[i-1][j-1] + matrix[i-1][j-1];
    }

    int sumRegion(int r1, int c1, int r2, int c2) {      // 查詢:容斥四角
        return prefix[r2+1][c2+1] - prefix[r1][c2+1]
             - prefix[r2+1][c1] + prefix[r1][c1];
    }
};
兩個公式,一個容斥:建表把「上矩形 + 左矩形」相加、扣掉重複的「左上矩形」、補當前格;查詢把「右下大矩形」扣掉上、左兩長條、加回被扣兩次的左上角。 多開一圈 0(m+1, n+1) 讓 r1=0 或 c1=0 時的邊界項自然是 0 —— 索引 +1 的位移就是為了對齊這圈邊界。

索引位移為什麼是 +1// THE +1 OFFSET

prefix[i][j] 存的是 matrix 前 i 列、前 j 行(即到 matrix[i-1][j-1])的總和。所以 matrix 座標和 prefix 座標差 1:

// matrix 座標 → prefix 座標的對應 prefix[i][j] = sum of matrix[0..i-1][0..j-1] prefix[0][*] = 0 (空的上邊界) prefix[*][0] = 0 (空的左邊界) 查矩形 (r1,c1)-(r2,c2)(matrix 座標,inclusive): 右下大矩形含到 (r2,c2) → prefix[r2+1][c2+1] 上長條含到 row r1-1 → prefix[r1][c2+1] 左長條含到 col c1-1 → prefix[r2+1][c1] 左上角含到 (r1-1,c1-1) → prefix[r1][c1] r1=0 → prefix[0][…]=0(上長條空);c1=0 → prefix[…][0]=0(左長條空),自動正確

這就是邊界那圈 0 的用途:當查詢矩形貼著上邊(r1=0)或左邊(c1=0),要減掉的長條是空的,對應 prefix[0][…] 或 prefix[…][0] 剛好 = 0。 少了這圈 0,就得寫一堆 if (r1>0) 的特判。

3

邊界與陷阱// PITFALLS

陷阱症狀正解
建表忘了減左上 上 + 左 把左上矩形算兩次 → 前綴全錯偏大 − prefix[i-1][j-1]
查詢忘了加回左上角 左上角被兩長條各減一次 → 結果偏小 + prefix[r1][c1]
沒開 0 邊界 r1=0/c1=0 時索引 -1 越界 prefix 開 (m+1)×(n+1)
索引位移搞混 查詢忘了 +1 或建表用了 matrix 同索引 prefix 比 matrix 座標大 1
每次查詢重建前綴表 把建表放進 sumRegion → 每查 O(mn) 建表只在建構子做一次
