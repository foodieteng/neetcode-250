題目簡介// THE QUESTION

QUESTION

給一個未排序整數陣列 nums,回傳不在陣列裡的最小正整數。 必須做到 O(n) 時間 且 O(1) 額外空間。 例:[3,4,-1,1] → 2、[7,8,9,11,12] → 1、[1,2,0] → 3。

兩個約束綁死了做法:O(1) 空間不准開雜湊表,O(n) 時間不准排序。 唯一的出路是把輸入陣列自己當雜湊表 —— 讓每個值 v 去坐它「命定的位子」index v−1, 再掃一遍看誰的位子空著。這招叫 循環排序(cyclic sort)。

2

為什麼是「原地循環排序」// WHY

× REJECT · SORT
先排序再掃

排完後線性找第一個斷點。對,但 O(n log n) —— 違反 O(n) 時間要求。

× REJECT · O(n) SPACE
丟進 hash set

把值全放進 set,再從 1 往上找第一個不在的。O(n) 時間沒錯,但 O(n) 額外空間 —— 違反 O(1) 要求。

BOTH FAIL · ADOPT ↓
● ADOPT · IN-PLACE
陣列即雜湊 · 讓值 v 歸位到 index v−1

把 hash set 的功能「內建」進原陣列:值 v 該待在 index v−1。 用 swap 把每個 1..n 的值送回家,越界值(≤0 或 >n)丟著不管。 排完掃一遍,第一個「位子和值對不上」的 index i → 答案 i+1。 O(n) 時間、O(1) 空間,兩個約束同時滿足。

3

核心策略 · 三步驟// STRATEGY

  1. 界定答案範圍 —— 只有 1..n 的值可能是答案,答案必在 [1, n+1](見第 5 節)。
  2. 歸位 —— 對每個值,只要它在 [1,n] 且沒待在家(nums[i] != nums[nums[i]-1]),就 swap 送它回 index v−1,反覆直到手上的值越界或家裡已是它。
  3. 掃描找空位 —— 從左到右,第一個 nums[i] != i+1 的位置 → 答案 i+1;若全部對上 → 答案 n+1。

[3,4,-1,1]:把 3 送回 index 2、4 送回 index 3、換出來的 1 送回 index 0; 越界的 -1 留原地。排成 [1,-1,3,4] 後掃描,index 1 該是 2 卻是 -1 → 答案 2。

5

為什麼答案一定在 [1, n+1]// LEMMA · PIGEONHOLE

n 個位子最多蓋住 1..n,缺的第一個 ≤ n+1

陣列有 n 個元素。考慮正整數 1, 2, …, n:

‧ 若 1..n 全部到齊 → 缺的第一個正數是 n+1。
‧ 若 1..n 裡有任何一個缺席 → 那個缺席者 ≤ n,答案就是它。

所以答案永遠落在 [1, n+1]。這條界推出兩個關鍵推論: (1) 只有 1..n 的值有資格當答案,所以只需替它們安排位子; (2) 越界值(≤0 或 >n)永遠不是答案,可以無視 —— 它們只是佔著「不屬於任何人」的格子,掃描時自然露餡。

6

複雜度// COMPLEXITY

T = O(n)　　S = O(1)

看似有「迴圈裡又套 while」像 O(n²),其實是攤還 O(n):每次 swap 都把一個值放進它最終的家,而到家的格子被 nums[i] != nums[nums[i]-1] 這個守衛擋住、不會再被換出。全程 swap 次數 ≤ n。空間只用了幾個變數 —— 排序就地在原陣列上完成。

面向成本說明
歸位(外圈 + 內 while)O(n)總 swap 次數 ≤ n(每次固定一個家)
掃描找空位O(n)單趟線性
額外空間O(1)只有 swap 的暫存,原地改陣列
常數因子小純整數比較與 swap,無雜湊、無配置

一句洞察// INSIGHT

當題目要 O(1) 空間 + O(n) 時間 又和「值域 = 位置」有關,想到把陣列自己當雜湊表: 讓值 v 坐 index v−1,缺的位子就是答案。答案界在 [1, n+1],越界值一律無視。
