重點回顧// RECAP

關鍵一句話
答案範圍永遠在 [1, n+1] —— 鴿籠原理:n 個位子最多蓋住 1..n
核心技巧陣列即雜湊,值 v 歸位到 index v−1(循環排序)
越界值≤0 或 >n 一律無視,不佔有效格子
防無窮迴圈守衛 x != nums[x-1]:家裡已坐對人就停手
複雜度攤還 O(n) 時間、O(1) 空間 —— 每次 swap 固定一個家
找答案第一個 nums[i] != i+1 → i+1;全對 → n+1

範例 Trace// [7,8,9,11,12]

// 全部越界的極端例:答案該是 1 nums=[7,8,9,11,12], n=5 i=0: x=7 → 7>5 越界 → 立刻停(不歸位) i=1: x=8 → 越界 → 停 i=2: x=9 → 越界 → 停 i=3: x=11 → 越界 → 停 i=4: x=12 → 越界 → 停 歸位後 nums 原封不動 = [7,8,9,11,12] 掃描: i=0 nums[0]=7 != 1 ✗ → return 1 1
沒有任何 1..5 的值 → 所有格子都「空著」(坐著越界的過客),第一格就對不上 → 答案 1。 這正印證:越界值再多也不影響答案,它們只是佔位、掃描時全露餡。

應用題// APPLY

App 1

LC 268 Missing Number:[0,n] 內 n+1 個數缺一個,找出來。循環排序怎麼改?還有沒有更省的 XOR / 求和法?

App 2

LC 448 Find All Numbers Disappeared:1..n 範圍、長度 n,找出全部缺席的數。除了循環排序,能不能用「把 nums[|v|-1] 標成負數」的原地標記法?

App 3

LC 442 Find All Duplicates:每個數出現一或兩次,找出所有出現兩次的。同樣值域 1..n,如何用原地技巧一趟找出重複?

// ANSWERS
A1

陣列只有 n 個元素,最多裝下 1..n 這 n 個不同正數。若它們全在,最小缺席者就是 n+1;若缺了其中一個,缺席者 ≤ n。兩種情況答案都 ≤ n+1,所以只需替 1..n 安排位子。

A2

能無視。由 A1,答案必在 [1, n+1],≤0(不是正數)和 >n(超過可能答案)都不可能是答案。它們唯一的作用是「佔著某個格子」讓真正該在那的值進不來 —— 但那個該在的值本來就缺席,所以那格本來就會對不上,掃描照樣抓到。無視越界值不會漏答案。

A3

會無窮迴圈。輸入 [1,1]:i=0 時 x=1,家是 index 0,但 nums[0] 已經是 1。沒有 x != nums[x-1] 守衛的話,swap(x, nums[0]) 換了等於沒換(1↔1),條件永遠成立 → 卡死。守衛的意思是「家裡已經坐著對的人(重複值)就別再換」。

A4

關鍵在攤還:每一次 swap 都把某個值放進它最終的家,而到家的格子被守衛 x != nums[x-1] 鎖住、不會再被換出。因此整支程式的 swap 總次數 ≤ n(至多 n 個家各被敲定一次),加上兩趟線性掃描,合計 O(n)。「單一 while 最壞可能跑很多次」不代表總量是平方 —— 要看全程加總。

A5

換成 if 只換一次就往下走,換回來的值沒繼續歸位。例:[3,4,-1,1],i=0 用 if 把 3 換到 index 2 後停,此時 index 0 變成 -1,但原本該被繼續處理的鏈就斷了,後面某些值永遠回不了家 → 掃描答錯。必須用 while 把「一路換出來的值」全部安置到越界為止。

App 1

LC 268:值域是 0..n,把值 v 放到 index v(注意這裡不是 v−1),掃描第一個 nums[i] != i → 答案 i;全對則答案 n。更省的做法:求和 n(n+1)/2 − Σnums,或 XOR 把 0..n 和所有元素全 XOR 起來,剩下的就是缺的數 —— 都 O(n) 時間、O(1) 空間且不改陣列。

App 2

LC 448:兩招都行。循環排序後,凡 nums[i] != i+1 的 i+1 就是缺席者,收集全部。更輕巧的是負號標記法:遍歷,對每個值 v 把 nums[|v|-1] 標成負(表示「|v| 出現過」);再掃一遍,仍為正的 index i 代表 i+1 沒出現過。用正負號當「存在位」,同樣 O(1) 空間。

App 3

LC 442:同樣負號標記。遍歷每個值 v,看 nums[|v|-1]:若已經是負,代表 |v| 之前來過 → 是重複,收進答案;否則把它標負。一趟 O(n)、O(1) 空間。這系列(41 / 448 / 442 / 268）的共同 DNA:值域 = 1..n(或 0..n),就把索引當雜湊,用歸位或正負號原地記錄存在性。
