完整實作 · 手持值(carry)版// O(n) TIME · O(1) SPACE

p41.cpp C++17 
class Solution {
public:
    int firstMissingPositive(vector<int>& nums) {
        int n = nums.size();

        for (int x : nums) {                       // x = 手上要歸位的值
            while (1 <= x && x <= n && x != nums[x - 1]) {
                swap(x, nums[x - 1]);    // 把 x 放進家,x 換回原本住那的值
            }
        }

        for (int i = 0; i < n; i++) {
            if (nums[i] != i + 1) return i + 1;  // 第一個對不上 → 答案
        }
        return n + 1;                          // 1..n 全到齊
    }
};
關鍵是那個 for (int x : nums) 的 x 是「複本」:它像一隻手,先抓起 nums[i], 在 while 裡把手上的值放進它的家 nums[x−1]、順手撿起原本住那的值,再繼續放 —— 直到手上的值越界(≤0 或 >n)或家裡已是它。 守衛 x != nums[x-1] 讓「家裡已坐對人」(重複值)時立刻停手,避免無窮迴圈。

等價寫法 · 直接換 nums[i]// CLASSIC

// 更常見的教科書寫法:不用手持變數,直接盯著 nums[i] 換 for (int i = 0; i < n; i++) { while (1 <= nums[i] && nums[i] <= n && nums[i] != nums[nums[i] - 1]) { swap(nums[i], nums[nums[i] - 1]); // 把 nums[i] 送回它的家 } } // 之後掃描完全一樣:第一個 nums[i] != i+1 → i+1 兩種寫法最終排出的陣列一致(每個 1..n 的值都在自己的家),掃描結果相同

差別只在「誰當手」:carry 版用區域變數 x 當手(nums[i] 本身不被 while 直接改); 經典版直接盯著 nums[i] 換到它歸位。兩者的攤還複雜度都是 O(n) —— 每次 swap 都固定一個家,到家後被守衛擋住不再動,總 swap 次數 ≤ n。 經典版更好推理不變量,carry 版少寫一次 nums[i] 索引、讀起來清爽。

複雜度分析// AMORTIZED

片段成本為什麼
外圈 forO(n)n 個起點各進 while 一次
內圈 while(全程加總)O(n)每次 swap 固定一個家;家被守衛鎖住不再換 → 總 swap ≤ n
掃描O(n)單趟
空間O(1)就地改原陣列,只用暫存變數
T = O(n) + O(n) + O(n) = O(n)　　S = O(1)

別被「for 裡有 while」嚇到當成 O(n²):攤還分析看的是總工作量,不是最壞單步。整個歸位過程,每個位子最多被「敲定」一次。

邊界與陷阱// PITFALLS

陷阱症狀正解
while 條件漏掉 x != nums[x-1] 遇到重複值(如 [1,1])一直互換 → 無窮迴圈 家裡已是對的值就停手
用 if 而非 while 換一次就走,換來的值沒繼續歸位 → 漏擺 必須 while 反覆放到手上的值越界
沒過濾越界值 nums[x-1] 對 x≤0 或 x>n → 越界存取 條件先卡 1 <= x && x <= n
索引差一 值 v 放到 index v 而非 v−1 家 = v−1;掃描比 i+1
全到齊時忘了回 n+1 [1,2,3] 掃完沒斷點卻不回 4 迴圈後 return n+1

細節走讀// WALKTHROUGH

// carry 版跑 [3,4,-1,1](n=4) 起手 nums=[3,4,-1,1] i=0: x=3 → 1<=3<=4 且 3!=nums[2](-1) → swap → nums=[3,4,3,1], x=-1 x=-1 越界 → 停手(3 已坐進 index 2 的家) i=1: x=4 → 4!=nums[3](1) → swap → nums=[3,4,3,4], x=1 x=1 → 1!=nums[0](3) → swap → nums=[1,4,3,4], x=3 x=3 → 3==nums[2](3) 家裡已是它 → 停手 i=2: x=3 → 3==nums[2](3) → 立刻停 i=3: x=4 → 4==nums[3](4) → 立刻停 歸位完成 nums=[1,4,3,4] 掃描: i=0 nums[0]=1==1 ✓ i=1 nums[1]=4!=2 ✗ → return 2 2
注意 carry 版的中途狀態會出現「暫時的重複」(如 [3,4,3,1])—— 那是手上還握著待丟的值造成的假象, 等外圈把值 i+1 帶到 index i 時就修正。最終每個到家的格子都滿足 nums[i]=i+1,掃描才是真相。
