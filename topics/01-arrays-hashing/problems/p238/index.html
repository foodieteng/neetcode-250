除自身以外的乘積

// CONCEPT · 前綴積 × 後綴積

1

題目簡介// THE QUESTION

QUESTION

給整數陣列 nums,回傳 answer,其中 answer[i] = 除了 nums[i] 以外所有元素的乘積。 必須 O(n) 時間、不可用除法。追問:O(1) 額外空間(輸出陣列不算)。例:[1,2,3,4] → [24,12,8,6]。

關鍵洞察:answer[i] = (i 左邊全部的積) × (i 右邊全部的積)。 把「左積」和「右積」分別用兩趟掃描算出來相乘,就繞開了除法、也達成 O(n)。 更漂亮的是:用輸出陣列自己當暫存,額外空間只剩兩個純量 → O(1)。

2

為什麼「前綴積 × 後綴積」// WHY

× REJECT · 除法
全部乘起來再除以 nums[i]

算總積 P,answer[i]=P/nums[i]。但題目禁止除法,而且有 0 時除法直接崩(除以 0、或 0 讓總積歸零)。

× REJECT · O(n²)
對每個 i 重乘一遍其他元素

雙層迴圈,每個 answer[i] 都掃一次其他 n−1 個相乘。O(n²) —— 違反 O(n)。

TWO SWEEPS · ADOPT ↓
● ADOPT · O(n) / O(1)
前綴積 × 後綴積,輸出陣列當暫存

Pass 1(左→右):ans[i] = 目前的 prefix(i 左邊乘積),再 prefix *= nums[i]。 Pass 2(右→左):ans[i] *= suffix(i 右邊乘積),再 suffix *= nums[i]。 兩趟後 ans[i] = 左積 × 右積。不用除法、O(n) 時間、O(1) 額外空間(只用 prefix/suffix 兩個純量)。

3

核心策略 · 兩趟掃// STRATEGY

  1. 初始化 —— ans 全填 1。prefix = 1。
  2. Pass 1(左→右) —— ans[i] *= prefix; prefix *= nums[i];。此後 ans[i] = i 左邊所有元素的積。
  3. Pass 2(右→左) —— suffix=1;ans[i] *= suffix; suffix *= nums[i];。乘上右積後 ans[i] 完成。

兩個關鍵細節// KEY DETAILS

① 為什麼「先寫入 ans,再更新累積」→ 不含自己

Pass 1 的順序是先 ans[i] = prefix、後 prefix *= nums[i]。所以寫進 ans[i] 的 prefix 是「還沒乘上 nums[i] 自己」的值 —— 恰好是 i 左邊(不含 i)的積。若順序反過來,就會把自己乘進去。Pass 2 的 suffix 同理。「先用後更新」正是排除自身的機關。

② 為什麼這招天生免疫「零」

除法版遇到 0 會崩:一個 0 讓總積歸零,P/nums[i] 對非零位得 0(錯)、對 0 位除以 0(爆)。 前綴×後綴從不做除法,只是把「左邊的積」和「右邊的積」相乘 —— 0 自然只影響「跨過它」的那些位置。 有兩個以上的 0 → 每格左右積必含一個 0 → 全 0;恰一個 0 → 只有那一格的左右積都不含 0、其餘全 0。邏輯天然正確,不必特判。

複雜度// COMPLEXITY

T = O(n)　　S = O(1) 額外

兩趟線性掃描。追問的 O(1) 空間靠「輸出陣列自己扛前綴積、再原地乘後綴積」達成 —— 額外只用 prefix、suffix 兩個純量(輸出陣列依題意不計)。若允許額外陣列,可用 left[]、right[] 兩個陣列,概念一樣但空間 O(n)。

一句洞察// INSIGHT

answer[i] = 左積 × 右積。兩趟掃:輸出陣列先扛前綴積,再原地乘後綴積。 「先用後更新」排除自身、免疫零、免除法,額外空間只剩兩個純量 → O(1)。這是 前綴和 思路的乘法雙向版。
