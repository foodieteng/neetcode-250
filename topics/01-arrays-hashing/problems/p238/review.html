除自身以外的乘積

重點回顧// RECAP

關鍵一句話
核心式answer[i] = 左積 × 右積(不含自己)
兩趟Pass 1 左→右填左積;Pass 2 右→左乘右積
排除自身先寫入 ans、後更新累積 → 累積不含自己
O(1) 空間輸出陣列當暫存,只留 prefix/suffix 兩純量
不用除法結構上免疫零,無需特判 0 的個數
複雜度O(n) 時間 / O(1) 額外空間

觀念題// CONCEPT Q

Q1

為什麼 Pass 1 要「先 ans[i] *= prefix、後 prefix *= nums[i]」?兩行對調會怎樣?

Q2

除法版 總積 / nums[i] 為什麼不行?列出遇到 0 的兩種崩法。

Q3

前綴×後綴為什麼「天生免疫零」?恰有一個 0、有兩個以上 0 時,答案各長什麼樣?

Q4

追問要 O(1) 額外空間。輸出陣列為什麼「不算」額外空間?這裡真正的額外空間是多少?

Q5

這題和 560 前綴和 的共通思路是什麼?為什麼這裡要「雙向」(前綴+後綴)而 560 只要單向?

範例 Trace// 含一個 0:[1,2,0,4]

// 免疫零:只有「不跨過 0」的那格非零 初始 ans=[1,1,1,1] Pass 1(左積): ans[0]=1 prefix: 1→1 ans[1]=1 prefix: 1→2 ans[2]=2 prefix: 2→0 (乘進 0) ans[3]=0 prefix: 0→0 → ans=[1,1,2,0] Pass 2(右積): ans[3]*=1 →0 suffix: 1→4 ans[2]*=4 →8 suffix: 4→0 (乘進 0) ans[1]*=0 →0 suffix: 0→0 ans[0]*=0 →0 suffix: 0→0 → ans=[0,0,8,0] [0,0,8,0]
唯一非零的是 ans[2]=8 —— 因為 index 2 剛好是那個 0 的位置,「除了它以外」的乘積 1×2×4=8 不含 0。 其餘每格的左右積都跨過了那個 0,自然歸零。全程沒有任何特判,邏輯自動對 —— 這就是免疫零的美。
4

應用題// APPLY

App 1

允許除法 + 保證無 0:若題目放寬(可除法、且無 0),一行 總積/nums[i] 是否更好?比前綴×後綴省在哪、風險在哪?

App 2

LC 152 乘積最大子陣列:同樣是「乘積 + 前綴」味道,但要找最大乘積連續子陣列。為什麼要同時維護最大和最小兩個前綴積?(提示:負負得正)

App 3

區間乘積查詢:多次詢問「[l,r] 區間乘積」。前綴積 + 除法 O(1) 查,但遇 0 失效。無 0 保證下怎麼做?有 0 時要換什麼結構?

// ANSWERS
A1

因為寫進 ans[i] 的 prefix 必須是「不含 nums[i] 自己」的左積。先寫入時 prefix 還只累積了 nums[0..i-1],正好是純左積;之後才 prefix *= nums[i] 把自己吞進去,留給下一格。對調成先更新,ans[i] 就會多乘一個自己 → 答案錯。

A2

兩種崩法(以 [1,2,0,4],總積 P=0 為例):(1) 非 0 位 ans[i]=P/nums[i]=0/nums[i]=0,但正解該是「含 0 的乘積」→ 錯成 0;(2) 0 位 ans=P/0=0/0 → 除以零。就算特判 0 的個數也囉嗦。加上題目明文禁止除法,此路不通。

A3

因為 ans[i]=左積×右積,一個 0 只會落在某些格子的左段或右段裡,讓那些「跨過 0」的格子乘積為 0。恰一個 0(在 index k):只有 ans[k] 非零(它的左右積都不含那個 0),其餘全 0。兩個以上 0:任何一格的左右積至少含一個 0 → 全部為 0。全靠乘法結構自動成立,不必數 0。

A4

因為輸出陣列是題目要求你回傳的結果,本來就必須佔 O(n),不是「為了演算法額外開的」工作空間 —— 空間複雜度分析裡通常不計輸出。這裡真正的額外空間只有 prefix、suffix(和 i、n)幾個純量 = O(1)。技巧就是「讓輸出陣列兼職暫存」:先扛前綴積,再原地被後綴積乘上去。

A5

共通思路:用「前綴聚合」把區間查詢降成 O(1) —— 560 是前綴和,238 是前綴積。差別在「除自身」需要的是「左邊 和 右邊」兩段的積相乘,而每一段都要「不含自己」→ 必須雙向掃(前綴給左、後綴給右)。560 找「和為 k」只需單向的前綴和之差(prefix[j]−prefix[i]),一個方向就夠。「除自身」的對稱性逼出雙向。

App 1

無 0 且可除法時,總積/nums[i] 確實更短、只掃兩趟變一趟半(一趟算總積、一趟除),常數更小。但風險:(1) 只要有一個 0 就崩(需保證無 0);(2) 整數除法要求乘積整除(這裡成立);(3) 溢位風險略高(總積比分段積大)。前綴×後綴犧牲一點常數,換來無 0 假設、無除法的穩健 —— 面試預設用它更安全。

App 2

LC 152:乘積的麻煩是負號 —— 一個負數會把「目前最大」翻成最小、最小翻成最大。所以要同時維護以 i 結尾的最大積 mx 和最小積 mn:遇到負數時交換兩者角色(max(nums[i], mn*nums[i]) 之類)。這樣才能捕捉「兩個負數相乘變大正數」的機會。核心仍是前綴積思路,但因負號要帶兩個極值。

App 3

無 0:預處理前綴積 pre[i],查 [l,r] = pre[r]/pre[l-1],O(1)(需可除)。有 0:除法失效。改記「前綴中 0 的個數」zeroCnt[i] + 「非零元素的前綴積」nzPre[i]:查詢時若區間內 zeroCnt 差 > 0 → 答案 0;否則用非零前綴積相除。把「0 的存在性」和「非零積」拆開處理,是有 0 情境的通用解法。
