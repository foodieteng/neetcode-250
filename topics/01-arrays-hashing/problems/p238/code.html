除自身以外的乘積

完整實作 · 前綴 × 後綴// O(1) EXTRA

p238.cpp C++17 
class Solution {
public:
    vector<int> productExceptSelf(vector<int>& nums) {
        int n = nums.size();
        vector<int> ans(n, 1);

        int prefix = 1;                    // Pass 1:左→右,累積左積
        for (int i = 0; i < n; i++) {
            ans[i] *= prefix;          // 先寫入「不含自己」的左積
            prefix *= nums[i];         // 再把自己乘進累積
        }

        int suffix = 1;                    // Pass 2:右→左,累積右積
        for (int i = n - 1; i >= 0; i--) {
            ans[i] *= suffix;          // 原地乘上「不含自己」的右積
            suffix *= nums[i];         // 再把自己乘進累積
        }
        return ans;
    }
};
兩趟的順序都是「先用累積、後更新累積」 —— 這是排除自身的機關。 Pass 1 寫進 ans[i] 的 prefix 尚未乘 nums[i],所以是純左積;更新才把自己吞進去,給下一格用。 Pass 2 對稱。兩趟後 ans[i] = 左積 × 右積,額外只用 prefix、suffix 兩個 int。

除法版為什麼行不通// NO DIVISION

// 「總積除以自己」看似 O(n),但兩個致命問題 long P = 1; for (int x : nums) P *= x; // 總積 for (int i = 0; i < n; i++) ans[i] = P / nums[i]; // ← 有 0 就爆 情境 A nums=[1,2,0,4]:P=0 非 0 位:ans = 0/nums[i] = 0 ✗（正解該是含 0 的乘積,非 0） 0 位 :ans = 0/0 ✗ 除以零 情境 B 題目明文「不可用除法」→ 直接不合法 前綴×後綴不做除法,0 只自然影響「跨過它」的位置,無需特判

就算硬要用除法處理 0(數有幾個 0、特判),邏輯會變得囉嗦又易錯。前綴×後綴結構上就免疫零 —— ans[i] 是左右兩段積相乘,某個 0 落在誰的左或右,就只影響那些「跨過 0」的格子,完全不必數 0、不必特判。

陷阱症狀正解
先更新累積再寫 ans 把 nums[i] 自己也乘進去 → 每格多乘自己 先 ans[i] *= prefix,後 prefix *= nums[i]
ans 沒初始化為 1 *= 從 0 開始 → 全 0 vector<int> ans(n, 1)
Pass 2 用新陣列 不是原地乘 → 多一個 O(n) 陣列,失 O(1) 在同一個 ans 上 *= suffix
用除法 遇 0 崩、且違反題目限制 前綴×後綴,不除
擔心溢位加 long long 題目已保證前綴/後綴積在 int 內 直接用 int 即可
