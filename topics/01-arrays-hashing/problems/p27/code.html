完整實作 · 覆寫法(保序)// O(n) TIME · O(1) SPACE

p27.cpp C++17 
class Solution {
public:
    int removeElement(vector<int>& nums, int val) {
        int i = 0;                          // 寫指標:下一個要放的位置

        for (int num : nums) {              // 讀指標:逐一掃過
            if (num == val) continue;         // 是 val → 跳過,i 不動

            nums[i++] = num;              // 非 val → 寫入並前進
        }
        return i;                             // [0, i) 全是非 val,i = k
    }
};

nums[i++] = num 一行做兩件事:先寫入 nums[i]=num,再後綴 ++ 讓 i 前進。 continue 跳過 val 時 i 完全不動 —— 寫指標只在保留時前進,是整段的靈魂。

另解 · 和尾巴交換(少寫、不保序)// SWAP

// 遇 val 就換到尾端並縮短範圍,val 稀少時更省寫入 int removeElement(vector<int>& nums, int val) { int n = nums.size(); int i = 0; while (i < n) { if (nums[i] == val) { nums[i] = nums[n - 1]; // 把尾端搬過來蓋掉 val n--; // 縮短有效範圍(不 i++,要重驗換來的值) } else { i++; // 保留,前進 } } return n; } 只在遇到 val 時寫一次;保留元素完全不搬 → val 越少越划算,但順序被打亂

注意換值後不能 i++ —— 從尾端換過來的新值還沒檢驗,可能也是 val,要留在原地重驗。 覆寫法最多寫 n 次(每個保留元素一次);交換法只寫「val 的個數」次。val 稀少 → 交換法;要保序 → 覆寫法。

同一個「寫指標」模式// THE PATTERN

題目寫指標保留的條件
27 移除元素num != val
26 移除排序陣列重複num != nums[i-1](和已寫入的最後一個不同)
283 移動零num != 0,最後把 [i, n) 補 0
這三題骨架一模一樣:一個寫指標 i,掃過陣列,只在「符合保留條件」時 nums[i++]=num。 差別只在那個保留條件。學會 27,26 / 283 只是換一行判斷 —— 「就地篩選 + 寫指標」是陣列題的基本功。

邊界與陷阱// PITFALLS

陷阱症狀正解
跳過時也 i++ 寫指標和讀指標同步前進 → val 沒被擠掉,前 k 個混入 val val 時 continue,i 不動
交換法換值後 i++ 從尾端換來的值沒重驗,若它也是 val 就漏掉 換值後不 i++,原地重驗
想「刪除」元素 用 erase 每次 O(n) 搬移 → 整體 O(n²) 覆寫而非刪除,O(n)
糾結 i 之後的垃圾值 想把 [i, n) 清乾淨 不用,題目明說「其餘不重要」
空陣列 以為要特判 不用,迴圈不跑,回 i=0 ✓

// 寫指標 i 只在保留時前進 初始 nums=[0,1,3,0,2] i=0 讀 0(idx0) == val → 跳過 i=0 讀 1(idx1) != val → nums[0]=1, i=1 [1,1,3,0,2] 讀 3(idx2) != val → nums[1]=3, i=2 [1,3,3,0,2] 讀 0(idx3) == val → 跳過 i=2 讀 2(idx4) != val → nums[2]=2, i=3 [1,3,2,0,2] return k = 3 前 3 個 [1,3,2] = 所有非 0 元素(順序保留)
// 寫指標 ≤ 讀指標,覆寫從不傷及未讀資料 每次寫 nums[i] 時,i 都 <= 當前讀到的索引 寫 nums[0] 時讀到 idx1 → 0 < 1 ✓(蓋掉已讀的 idx0) 寫 nums[2] 時讀到 idx4 → 2 < 4 ✓(蓋掉已讀的 idx2) 被覆寫的都是「已讀過」的位置 → 安全就地
