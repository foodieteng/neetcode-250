// 覆寫法 初始 [3,2,2,3] i=0 讀 3 == val → 跳過 i=0 讀 2 != val → nums[0]=2, i=1 [2,2,2,3] 讀 2 != val → nums[1]=2, i=2 [2,2,2,3] 讀 3 == val → 跳過 i=2 return k=2 前 2 個 [2,2]

應用題// APPLY

App 1. LC 26 移除排序陣列的重複:保留每個值一份。寫指標的「保留條件」要改成什麼?為什麼需要陣列已排序?
App 2. LC 283 移動零:把所有 0 移到尾端、非 0 保序在前。和本題差在哪?多做一步是什麼?
App 3. 若要移除的不是「等於某值」,而是「滿足某條件」的元素(如「移除所有偶數」「移除長度 < 3 的字串」),程式怎麼改?這體現什麼通用結構?
// ANSWERS
A1. 因為 i 代表「已保留元素的下一個空位」——只有真的保留(寫入)一個元素,那個空位才被填掉、需要往後移。跳過 val 時什麼都沒寫進去,空位還在原地,i 當然不動。
若跳過時也 i++:i 就和讀指標同步前進,退化成「原地不動」——每個 nums[i]=num 都寫回自己的位置,val 從沒被擠掉。結果前 k 個裡混著 val,而且回傳的 i 會等於原長度 n(不是真正的 k)。寫指標「只在保留時前進」正是「把 val 擠出前段」的機制 —— 缺口(讀與寫的差)就是被擠掉的 val 數。
A2. 因為寫指標 i 永遠 ≤ 讀指標。每讀一個元素讀指標必前進;而只有「保留」才寫、才 i++,所以 i 的前進不會比讀指標快。
於是 nums[i] = num 覆寫的位置 i,要嘛是已經讀過的(i < 讀指標,那格的原值早就處理完了)、要嘛正好是當前讀的位置(i == 讀,寫回自己,無害)。絕不會 i > 讀指標 —— 也就是永遠不會覆蓋到「後面還沒讀到」的資料。這個不變式(i ≤ fast)是所有「就地寫指標」演算法能安全在原陣列上操作、不需額外空間的根本原因。
A3. ● 覆寫法:每個保留元素都寫一次 → 最多 n 次寫入。保序(保留元素相對順序不變)。
● 交換法:只在遇到 val 時把尾端元素換過來 → 只寫「val 的個數」次。不保序(尾巴的元素被搬到前面)。
何時選交換法:當 val 很稀少(要刪的沒幾個)、且不要求保序時 —— 交換法幾乎不動資料,寫入次數遠少於覆寫法。反之,要保序就只能覆寫法;val 很多時兩者寫入量差不多。本題「順序可變」給了選交換法的自由,但覆寫法更直觀、更常見,面試寫覆寫法即可,能提一句交換法的取捨加分。
A4. 因為從尾端換過來的那個新值還沒被檢驗過。交換法在 nums[i]==val 時執行 nums[i] = nums[n-1]; n--; —— 現在 nums[i] 是原本尾端的值,它可能也等於 val!若這時 i++ 跳過它,就會把一個沒檢查的 val 留在前段,漏刪。
所以要留在原地(不 i++)重新檢驗換過來的值:下一輪迴圈再判一次 nums[i]==val,若還是 val 就繼續換、若不是才 i++。
對比覆寫法:覆寫法的讀指標永遠只往前(每個原始元素只讀一次),不會有「換來的值要重驗」的問題 —— 這是交換法特有的坑。「就地把資料搬過來」時,務必想清楚搬過來的東西還需不需要再處理一次。
A5. 因為 vector::erase(it) 刪除一個中間元素時,要把它後面的所有元素往前搬一格 —— 單次就是 O(n)。若對每個 val 都 erase,最壞(半數是 val)就是 O(n) 次刪除 × 每次 O(n) 搬移 = O(n²)。
寫指標覆寫法只掃一遍、每步 O(1) → O(n)。關鍵差異:erase 是「真的刪除 + 每次搬移剩餘全部」;覆寫法是「不刪除,只把要留的往前蓋」——把 n 次各自 O(n) 的搬移,合併成一次 O(n) 的掃描。
這也是題目強調「就地、且 k 之後不重要」的用意:它刻意允許你不真的刪除,好讓 O(n) 解成立。凡是看到「回傳有效長度、其餘不管」,就該想到寫指標覆寫而非 erase。
App 1. 26 保留每個值一份,寫指標的保留條件改成「和上一個保留的不同」:if (i == 0 || num != nums[i-1]) nums[i++] = num;(nums[i-1] 是最後一個已寫入的值)。
為什麼需要已排序:排序後相同的值一定相鄰,所以「和前一個保留的不同」就足以判斷「這是新的值」。若沒排序,相同值散落各處,只比「上一個」會漏 —— 那就得改用 hash set 記錄看過的值(退化成 O(n) 空間)。「排序讓相同者相鄰」是很多雙指標題能用 O(1) 空間的前提,26 正是靠這個把「去重」變成「比相鄰」。
App 2. 283 的第一步和 27 幾乎一樣:寫指標把所有非 0 元素(保留條件 num != 0)依序往前寫,寫完 i 就是非 0 元素的個數。
多做一步:27 到這裡就結束(其餘不管);但 283 要求尾端補 0 —— 所以再跑一個迴圈把 [i, n) 全設成 0。也就是 283 = 27(移非 0)+ 尾端填 0。
差別的本質:27 只在乎「前 k 個對、其餘垃圾」;283 在乎「整個陣列都要對(前段非 0 保序、後段全 0)」。所以 283 不能留垃圾,要多一步清尾。同一個寫指標骨架,依「其餘要不要管」決定要不要補收尾。
App 3. 把「num == val」這個判斷,換成任意的「該不該保留」條件即可,骨架完全不動:
for (auto& x : arr) if (shouldKeep(x)) arr[i++] = x; return i;
● 移除偶數 → 保留條件 x % 2 != 0。
● 移除長度 < 3 的字串 → 保留條件 s.size() >= 3。
這體現的是「就地 partition / filter」的通用結構:一個寫指標,把「通過謂詞(predicate)的元素」壓到前段,回傳通過的個數。它其實就是 std::stable_partition / std::remove_if 的手寫版 —— C++ 標準庫的 nums.erase(remove_if(nums.begin(), nums.end(), pred), nums.end()) 就是這個模式的封裝。學會「寫指標 + 一個保留謂詞」,等於掌握了所有「就地篩選陣列」的題型。
