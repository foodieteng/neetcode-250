題目簡介// THE QUESTION

QUESTION

給陣列 nums 和值 val,就地移除所有等於 val 的元素。設非 val 的元素有 k 個,要讓 nums 的前 k 個是那些非 val 元素(順序可變),回傳 k。k 之後的值不重要。

「就地移除、回傳有效長度」是慢/快雙指標的招牌。關鍵領悟:陣列元素刪不掉(大小固定),但可以用要保留的值把前面覆寫掉 —— 用一個寫指標指向「下一個該放的位置」,只把非 val 的值依序寫進去。

2

核心 · 讀指標掃、寫指標只在保留時前進// SLOW / FAST

兩個角色 · 讀(快)與寫(慢)

● 讀指標(快):逐一走過每個元素(就是 for (int num : nums) 的 num),不管值是什麼都往前。
● 寫指標 i(慢):指向「下一個要放非 val 元素的位置」,只在保留一個元素時才 +1。

規則:讀到 num == val → 跳過(寫指標不動);num != val → nums[i] = num; i++(寫進去、寫指標前進)。

每遇到一個 val,寫指標就比讀指標多落後一格 —— 這個「缺口」正好就是已跳過的 val 個數。掃完後,[0, i) 塞滿了所有非 val 元素,i 就是答案 k。

3

為什麼「邊讀邊覆寫」不會出錯// WHY SAFE

寫指標永遠 ≤ 讀指標,不會蓋到還沒讀的資料

寫的位置 i 永遠不會超過正在讀的位置 —— 因為每保留一個才寫一格,而讀指標每步都前進,寫指標 i ≤ 讀指標。所以 nums[i] = num 覆寫的,要嘛是已經讀過的位置、要嘛是正在讀的位置(i == 讀 時寫回自己,無害)。絕不會覆蓋到「後面還沒讀到」的資料。這就是為什麼可以安心就地覆寫、不用額外陣列。

4

動畫演示// [0,1,3,0,2], val=0 → k=3

讀指標掃過每個元素;寫指標 i 只在保留時前進。遇到 0 就跳過,缺口拉大。最後前 3 個 = [1,3,2]。

另一種策略 · 和尾巴交換// SWAP WITH END

當 val 很少時,「換到尾巴」寫入次數更少

本題的覆寫法會把每個保留元素都寫一次(即使它本來就在對的位置)。另一種:遇到 val 時,把它和陣列尾端的元素交換、並縮短有效長度 —— 這樣只動到 val,保留元素完全不搬。當 val 很稀少時,交換法的寫入次數遠少於覆寫法。
代價:交換法會打亂順序(把尾巴的元素換到前面),而覆寫法保留原順序。本題「順序可變」所以兩者都行;若題目要保序,只能用覆寫法。覆寫=保序、穩;交換=少寫、快(但亂序)。

6

複雜度// COMPLEXITY

T = O(n)　　S = O(1)

讀指標掃一遍 O(n),每步 O(1)。就地覆寫,除了兩個指標沒有額外空間 → O(1)。覆寫法保序;交換法同樣 O(n)/O(1) 但寫入更少、不保序。

7

一句洞察// INSIGHT

寫指標 i 指「下一個該放的位置」,只在保留時前進。讀到 val 跳過、否則 nums[i++]=num。 因 i ≤ 讀指標,覆寫安全。掃完 i = k。
