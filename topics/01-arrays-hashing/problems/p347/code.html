完整實作 · 頻率桶排序// O(n)

p347.cpp C++17 
class Solution {
public:
    vector<int> topKFrequent(vector<int>& nums, int k) {
        int n = nums.size();
        unordered_map<int, int> counts;
        for (int x : nums) counts[x]++;              // ① 計數

        // ② 桶:buckets[f] = 出現 f 次的所有值(大小 n+1,最高頻可能是 n)
        vector<vector<int>> buckets(n + 1);
        for (auto const& [val, freq] : counts)
            buckets[freq].push_back(val);

        vector<int> result;
        // ③ 從最高頻的桶往回收集
        for (int i = n; i >= 0 && result.size() < k; --i) {
            for (int num : buckets[i]) {
                result.push_back(num);
                if (result.size() == k) break;
            }
        }
        return result;
    }
};
三段一氣呵成:計數 → 分桶 → 由高往低收集。外層 for i = n..0 讓「先掃到的桶」就是「頻率最高的桶」,索引本身完成了排序。 迴圈條件 result.size() < k + 內層 break 雙重保險,湊滿 k 立刻停,不多掃低頻桶。

對照寫法 · 大小 k 的最小堆// O(n log k)

// 維持 k 個「最高頻」的最小堆:堆頂是門檻,較大者擠掉它 vector<int> topKFrequent(vector<int>& nums, int k) { unordered_map<int,int> counts; for (int x : nums) counts[x]++; // 最小堆存 (freq, val);超過 k 個就丟掉頻率最小的 priority_queue<pair<int,int>, vector<pair<int,int>>, greater<>> pq; for (auto& [val, freq] : counts) { pq.push({freq, val}); if ((int)pq.size() > k) pq.pop(); // 擠掉最低頻 } vector<int> res; while (!pq.empty()) { res.push_back(pq.top().second); pq.pop(); } return res; } O(n log k) 時間、O(n+k) 空間;k 遠小於 n 或串流情境時常用

最小堆為什麼是「最小」而非最大堆:我們只想留 k 個最高頻,用大小 k 的最小堆,堆頂就是「目前第 k 高的門檻」;新來的頻率若比堆頂大就擠掉堆頂。這樣堆永遠 ≤ k 個,每次操作 O(log k)。 桶排序不帶 log,但堆在「k 很小」或「資料是串流、頻率上限未知」時更合適。

邊界與陷阱// PITFALLS

陷阱症狀正解
桶陣列開 n(非 n+1) 頻率 = n 時 buckets[n] 越界 大小 n + 1(頻率 0..n)
從低頻往高掃 收到的是最低頻 → 答案相反 for i = n; i>=0; --i
忘了湊滿就停 把整桶都塞進去 → 超過 k 個 size<k 條件 + 內層 break
用最大堆裝全部 堆大小 n → O(n log n),失去 log k 優勢 用大小 k 的最小堆
結構化綁定型別 [val,freq] 順序記反 → 桶放錯 map 是 {值 → 頻率},別搞反
