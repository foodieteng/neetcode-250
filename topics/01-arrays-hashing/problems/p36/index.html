題目簡介// THE QUESTION

QUESTION

判斷一個 9×9 數獨盤是否有效(只看已填格,不必可解)。三條規則: 每列、每行、每個 3×3 宮內的數字 1–9 都不能重複。空格是 '.'。

三條規則是三個獨立的「不重複」條件。最直接的做法:給每列、每行、每宮各一個集合,掃過每個已填格, 把數字丟進它所屬的三個集合;只要任一集合已經有這個數字,就違規、回 false。 唯一的技術點是:一個格子 (r,c) 屬於哪個宮?

2

為什麼「三組集合、一趟掃」// WHY

△ WORKS · 掃三趟
分別檢查列、行、宮

寫三段迴圈各查一種規則。對,但程式重複、要小心宮的座標換算,容易寫錯邊界。

× REJECT · 慢
對每格重新掃它的列/行/宮

每個格子都重掃整列、整行、整宮找重複。O(81 × 27) 級的重複工作,沒必要。

ONE PASS · ADOPT ↓
● ADOPT · 一趟三組集合
row[9] / col[9] / box[9],邊掃邊記

開三組各 9 個集合。掃到已填格 (r,c)=d:算 boxId = r/3*3 + c/3, 查 row[r]、col[c]、box[boxId] 是否已有 d —— 有就 false,沒有就三組都插入。 一趟掃完 81 格,每格常數次雜湊操作。乾淨、對稱、不易錯。

3

核心策略 · 三步驟// STRATEGY

  1. 三組集合 —— row[9]、col[9]、box[9],各存「這一列/行/宮出現過的數字」。
  2. 算宮號 —— 對每個已填格 (r,c):boxId = r/3*3 + c/3(整數除法)。
  3. 查+插 —— 若 d 已在三者任一 → return false;否則三組都插入 d。掃完回 true。

boxId = r/3*3 + c/3 為什麼對// THE FORMULA

把 (r,c) 壓成 0..8 的宮編號

9×9 盤被切成 3×3 = 9 個宮,排成 3 列 3 行的宮陣。用整數除法定位:

　r/3 ∈ {0,1,2}　→ 這格在第幾「帶列」(上/中/下)
　c/3 ∈ {0,1,2}　→ 這格在第幾「帶行」(左/中/右)
　boxId = (r/3) * 3 + (c/3)　→ 把 (帶列, 帶行) 攤平成 0..8

這就是二維座標壓成一維索引的老把戲:行號 × 每列格數 + 列號。這裡「每列宮數」= 3。 驗算:(0,0)→0、(2,2)→0(同宮)、(4,4)→1*3+1=4(正中宮)、(8,8)→2*3+2=8(右下宮)。 每個宮拿到唯一的 0..8,三組集合就能各認各的宮。

6

複雜度// COMPLEXITY

T = O(81) = O(1)　　S = O(81) = O(1)

盤固定 9×9,掃 81 格、每格常數次操作 → 常數時間。以 N×N 一般化就是 O(N²) 時間、O(N²) 空間(集合最多存 N² 個數)。用 bitmask 取代集合可把每個列/行/宮壓成一個 int(第 d 位 = 有沒有數字 d),更省更快(見程式頁)。

面向成本說明
掃描O(81)每格一次
每格查+插O(1)三次雜湊 count + 插入
空間O(81)27 個集合,共存 ≤ 81 數
bitmask 版更快27 個 int,位元運算

一句洞察// INSIGHT

三條獨立的「不重複」規則 → 三組集合(row/col/box),一趟掃邊查邊插。 宮號用 r/3*3 + c/3 把 (r,c) 壓成 0..8 —— 二維攤平成一維的通用公式。任一集合撞到就 false。
