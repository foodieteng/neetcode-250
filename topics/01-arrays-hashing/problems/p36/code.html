完整實作 · 三組集合// ONE PASS

p36.cpp C++17 
class Solution {
public:
    bool isValidSudoku(vector<vector<char>>& board) {
        vector<unordered_set<char>> row(9), col(9), box(9);

        for (int r = 0; r < 9; r++) {
            for (int c = 0; c < 9; c++) {
                if (board[r][c] == '.') continue;       // 空格跳過

                int boxId = r / 3 * 3 + c / 3;       // (r,c) 屬哪個宮
                char digit = board[r][c];

                if (row[r].count(digit) || col[c].count(digit)
                    || box[boxId].count(digit)) return false;

                row[r].insert(digit);
                col[c].insert(digit);
                box[boxId].insert(digit);
            }
        }
        return true;
    }
};
三組集合完全對稱:每個已填格對 row[r]、col[c]、box[boxId] 各做一次「查+插」。 || 短路:任一集合已有 digit 就立刻 return false,不必查完三個。 整數除法 r/3 自動捨去小數,把 0,1,2→0、3,4,5→1、6,7,8→2。

更省的寫法 · 位元遮罩// BITMASK

// 每個列/行/宮用一個 int:第 d 位 = 數字 d 出現過 bool isValidSudoku(vector<vector<char>>& board) { int row[9] = {0}, col[9] = {0}, box[9] = {0}; for (int r = 0; r < 9; r++) for (int c = 0; c < 9; c++) { if (board[r][c] == '.') continue; int d = board[r][c] - '1'; // 0..8 int bit = 1 << d; int b = r / 3 * 3 + c / 3; if ((row[r] & bit) || (col[c] & bit) || (box[b] & bit)) return false; // 該位已亮 → 重複 row[r] |= bit; col[c] |= bit; box[b] |= bit; } return true; } 27 個 int 取代 27 個集合;查=&、插=|=,無雜湊開銷,更快更省記憶體

數字只有 1–9,用一個 int 的低 9 位就能表示「哪些數字出現過」。row[r] & bit 查存在、row[r] |= bit 插入。 這把雜湊集合的查/插換成單一位元運算,常數更小、零配置 —— 值域小又固定時的首選。

邊界與陷阱// PITFALLS

陷阱症狀正解
boxId 寫成 r/3 + c/3*3 帶列帶行對調 → 宮號錯,誤判 r/3*3 + c/3(帶列 ×3 + 帶行)
忘記跳過 '.' 把 '.' 當數字插入 → 多個 '.' 誤判重複 if (board[r][c]=='.') continue;
先插後查 插入後才查 → 永遠查到剛插的自己 → 全部誤判重複 先查三組,無重複才插
只查一種規則 漏查宮 → 宮內重複沒抓到 row、col、box 三者都查
bitmask 用 -'0' 數字 1–9 → 位 1..9,浪費第 0 位(無害但不齊) 用 -'1' 對到位 0..8

// 一個宮內重複的例子(其餘格 '.') (0,0)=5 boxId=0/3*3+0/3=0 查 row0/col0/box0 皆空 → 插 box0={5} (0,1)=3 boxId=0 查 皆無 3 → 插 box0={5,3} (1,0)=6 boxId=0 查 皆無 6 → 插 box0={5,3,6} (2,2)=5 boxId=2/3*3+2/3=0 查 box0 已有 5 → return false ★ false
關鍵在 (2,2):它和 (0,0) 座標差很遠,但 boxId 都算出 0 —— 因為同屬左上宮。 box[0] 這個集合把整個宮的數字聚在一起,才能抓到「同宮不同格」的重複 5。 這正是為什麼需要 boxId 公式,而不能只用 (r,c) 判斷。
