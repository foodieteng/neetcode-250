題目簡介// THE QUESTION

QUESTION

給未排序整數陣列 nums,回傳最長連續元素序列的長度。序列的數不必在陣列中相鄰,但數值要連續(如 1,2,3,4)。 必須 O(n) 時間。例:[100,4,200,1,3,2] → 4(序列 1,2,3,4)。

「O(n) 時間」這個硬性要求是全題的樞紐 —— 它把最直覺的排序(O(n log n))擋在門外。 正解:丟進雜湊集合,再用一個「只從每段連續數的最左端起算」的技巧,把看似 O(n²) 的巢狀迴圈壓成攤還 O(n)。

2

為什麼「集合 + 起點法」// WHY

× REJECT · O(n log n)
排序後數連段

排序後掃一遍數最長連續段。對、好寫,但 O(n log n) —— 違反題目的 O(n) 要求。

× REJECT · O(n²)
集合但「每個數都往右數」

丟進集合後,對每個數都 while(x+1 in set) 往右數。[1,2,…,n] 時每個數都重數整段 → O(n²)。

FIX WITH START-CHECK · ADOPT ↓
● ADOPT · O(n)
集合 + 只從最左端起算

丟進 unordered_set(去重 + O(1) 查)。對每個數 x,只有當 x−1 不在集合(x 是某段的最左端)才往右數; x−1 在集合就跳過。這樣每段只被它的起點數一次,內層 while 的總步數 = 所有段長相加 = n。攤還 O(n)。

3

核心策略 · 三步驟// STRATEGY

  1. 建集合 —— 把所有數丟進 unordered_set,自動去重、O(1) 查詢。
  2. 找起點 —— 遍歷集合,只處理「x−1 不在集合」的 x(它是一段的最左端);否則跳過。
  3. 往右數 —— 從起點 while(x+1 in set) 累加長度,更新最長。回傳最長。

為什麼巢狀迴圈仍是 O(n)// LEMMA · AMORTIZED

每個數最多被「碰」兩次

乍看「for 裡有 while」像 O(n²),但關鍵在 if (seen.count(s-1)) continue; 這道守衛:

‧ 外層 for:每個數被檢視一次(判斷是不是起點)。
‧ 內層 while:只有起點會進入往右數;而每個數屬於恰好一段,只會被那段的起點在 while 裡走過一次。

所以所有內層 while 的步數加總 = 每段長度相加 = n。加上外層 n 次檢視,總共 2n 次操作 → O(n)。 守衛擋掉了「非起點也往右數」的重複工作 —— 沒有它,[1..n] 這種輸入就會退化成 O(n²)。這一行 continue 就是 O(n) 的全部祕密。

複雜度// COMPLEXITY

T = O(n)　　S = O(n)

建集合 O(n)、遍歷 + 攤還往右數 O(n)。空間是集合存 n 個數。注意 unordered_set 是均攤 O(1) 查詢(最壞碰撞退化,但隨機資料下穩定)。用排序則是 O(n log n) 時間、O(1)~O(n) 空間,換取不同取捨。

空間O(n)集合存所有數
vs 排序O(n log n)更省空間但不達 O(n) 時間

一句洞察// INSIGHT

要 O(n) 又和「連續值」有關 → 丟進雜湊集合,再只從每段最左端(x−1 不在集合)往右數。 守衛 continue 保證每個數只被走一次,巢狀迴圈攤還成 O(n)。這是「攤還分析」最漂亮的入門例子。
