完整實作 · 集合 + 起點法// O(n)

p128.cpp C++17 
class Solution {
public:
    int longestConsecutive(vector<int>& nums) {
        unordered_set<int> seen(nums.begin(), nums.end());  // 去重 + O(1) 查
        int maxL = 0;

        for (int s : seen) {
            if (seen.count(s - 1)) continue;   // s-1 在 → s 非起點,跳過

            int curL = 0;
            while (seen.count(s)) {       // 從起點往右數
                s++; curL++;
            }
            maxL = max(maxL, curL);
        }
        return maxL;
    }
};
if (seen.count(s - 1)) continue; 是整題的靈魂 —— 它保證只有「一段的最左端」才進入 while。 少了這行,程式仍正確,但 [1,2,…,n] 這種輸入會讓每個數都重數整段,退化成 O(n²)。 這一行把「重複往右數」剪掉,是 O(n²) → O(n) 的唯一關鍵。

為什麼遍歷 set,不是 nums// DEDUP MATTERS

迴圈跑的是 for (int s : seen) 而非 for (int s : nums)。差別在重複值: 若 nums=[1,2,2,2,3],遍歷 nums 會對三個 2 各做一次起點判斷 —— 雖然都不是起點(1 在集合)會被 continue 擋掉, 但這破壞了「攤還 O(n)」的乾淨論證(重複值可能是 O(重複數) 次多餘檢視)。遍歷去重後的 set,每個「值」只檢視一次,論證才嚴格成立。

// 遍歷 set:每個 distinct 值恰好一次起點判斷 nums=[1,2,2,2,3] → seen={1,2,3} for s in {1,2,3}: s=1: count(0)? 否 → 起點,往右數 1,2,3 → curL=3 s=2: count(1)? 是 → 跳過 s=3: count(2)? 是 → 跳過 3(重複的 2 不會造成多餘工作)

邊界與陷阱// PITFALLS

陷阱症狀正解
省略 continue 守衛 每個數都往右數,[1..n] 退化 O(n²) 只從 x−1 不在集合的起點數
遍歷 nums 而非 set 重複值造成多餘檢視,弄髒 O(n) 論證 for (int s : seen)
空陣列 nums=[] → 集合空,迴圈不進 maxL=0 初值即正確回 0
往右改壞了 s while 裡 s++ 破壞外層迭代變數 此處 s 是複本(range-for by value),安全
用 map 存次數 只需存在性,map 多花空間 用 unordered_set 即可

三個起點(100、200、1)各自往右數一次;4、3、2 因左鄰在集合被 continue 秒跳。 注意 起點 1 的 while 走過 1,2,3,4 這四個數 —— 它們之後被檢視時都是「非起點」,不再重數。 每個數:被檢視一次 + 至多被某起點走一次 = 攤還 O(n)。
