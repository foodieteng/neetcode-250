完整實作// O(n) TIME · O(n) SPACE

p217.cpp C++17 
class Solution {
public:
    bool containsDuplicate(vector<int>& nums) {
        int n = nums.size();
        unordered_set<int> seen;

        for (int i = 0; i < n; i++) {
            if (seen.count(nums[i])) return true;   // 先查:已看過 → 重複
            seen.insert(nums[i]);              // 再放:留給後面比對
        }
        return false;                            // 掃完沒撞到 → 全相異
    }
};

seen.count(x) 回傳 0 或 1(set 不會有重複鍵),當布林用剛好。 先 count 再 insert 的順序是靈魂:查的是「之前」看過的,不能查到自己剛放的。

更短的寫法 · 用 insert 的回傳值// ONE-LINER TRICK

// insert 回傳 pair<iterator, bool>,second = 有沒有成功插入 unordered_set<int> seen; for (int x : nums) if (!seen.insert(x).second) return true; // 插入失敗 = 已存在 = 重複 return false; // 或用 set 一次比大小: unordered_set<int> s(nums.begin(), nums.end()); return s.size() != nums.size(); // 集合去重後變短 = 有重複

insert 的 .second 是「這次有沒有真的插進去」——重複值會插入失敗回 false,一次搞定「查 + 放」。 第二種「建集合比大小」最短,但會掃完全部(沒有提前結束),重複很早出現時反而較慢。要提前結束就用逐一 insert 的寫法。

三種解法對照// TRADE-OFFS

解法時間空間備註
雙層迴圈O(n²)O(1)n 大就 TLE,不用
排序 + 看相鄰O(n log n)O(1)*省空間但破壞順序;*不算排序遞迴堆疊
雜湊集合O(n)O(n)首選,可提前結束
面試標準答案是雜湊集合。若面試官追問「不能用額外空間呢?」→ 答排序法(O(n log n) / O(1)), 展示你懂時間 vs 空間的取捨。這是 Arrays & Hashing 章節反覆出現的主軸。
4

邊界與陷阱// PITFALLS

陷阱症狀正解
先 insert 再 count 每個值都查到自己剛放的 → 永遠 true 先 count 再 insert
用 map 數次數 可行但多餘 —— 只問「有沒有」不需要次數 set 就夠(要次數才用 map)
忘了提前結束 掃完全部才判斷,浪費(重複可能在很前面) 撞到就 return true
ordered set 用 std::set(紅黑樹)→ 每次 O(log n),慢 unordered_set(雜湊)→ 均攤 O(1)
空 / 單元素陣列 以為要特判 不用,迴圈自然回 false
