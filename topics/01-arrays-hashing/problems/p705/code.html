完整實作 · 直接定址// O(1) PER OP

p705.cpp C++17 
class MyHashSet {
public:
    vector<bool> mp;

    MyHashSet() {
        mp.assign(1000001, false);   // 0..10^6 每個 key 一格
    }

    void add(int key) {
        mp[key] = true;
    }

    void remove(int key) {
        mp[key] = false;
    }

    bool contains(int key) {
        return mp[key];
    }
};

vector<bool> 是位元壓縮的特化,10⁶ 個布林只佔約 125 KB(1 bit/元素),不是 1 MB。remove 不用檢查存在與否 —— 設 false 對「本來就不在」也無害。

通用版 · 雜湊函式 + chaining// UNBOUNDED KEYS

// key 無界時:固定桶數 + 每桶一條 list 處理碰撞 class MyHashSet { static const int B = 769; // 質數桶數(減少碰撞) vector<list<int>> buckets; int h(int key) { return key % B; } // 雜湊函式 public: MyHashSet() : buckets(B) {} void add(int key) { auto& b = buckets[h(key)]; for (int x : b) if (x == key) return; // 已存在就不重複加 b.push_back(key); } void remove(int key) { buckets[h(key)].remove(key); // list::remove 刪所有等值 } bool contains(int key) { for (int x : buckets[h(key)]) if (x == key) return true; return false; } }; 桶數固定、空間 O(B + 元素數);同桶元素少 → 平均 O(1)、最壞 O(n)

質數桶數(如 769)讓 key % B 分佈更均勻、減少碰撞。同桶的 key 用 list 串起來(separate chaining),add 要先掃桶避免重複。這就是 std::unordered_set 的簡化骨架 —— 直接定址表是它「key 能直接當索引、桶數 = 值域、每桶最多一個」的極端特例。

直接定址 vs 雜湊+桶// TRADE-OFFS

直接定址(本題)雜湊 + 桶(通用)
需要 key 有界?要(≤ 10⁶)不用
雜湊函式不需要(key 即索引)key % B
碰撞永不發生會,用 chaining 解
時間/opO(1)平均 O(1),最壞 O(n)
空間O(值域)(固定 10⁶)O(桶+元素)(隨用量)
面試策略:先寫直接定址(利用題目給的 key ≤ 10⁶),再補一句「若 key 無界,改用固定桶數 + chaining,這也是 unordered_set 的原理」。展現你既會用捷徑、也懂通用設計。

邊界與陷阱// PITFALLS

陷阱症狀正解
陣列開 10⁶ 而非 10⁶+1 key == 10⁶ 時 mp[10⁶] 越界 大小 1000001(含上界)
桶版 add 不查重複 同一 key 加兩次 → 桶裡出現兩份 add 先掃桶,存在就 return
key 無界卻用直接定址 開 2³² 陣列 → 記憶體爆 值域無界 → 雜湊 + 桶
桶數用合數(如 1000) 某些分佈碰撞集中,退化嚴重 用質數桶數(769 等)
