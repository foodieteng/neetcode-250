範例 Trace// 基數排序含負數(本題解法)

// radixSort([-5, 53, -8, 24, 6]) —— 負數如何處理 拆分: pos = [53, 24, 6] neg = [5, 8] (存絕對值 |-5|=5, |-8|=8) pos 基數(個位→十位): 個位: 53→3, 24→4, 6→6 → [53, 24, 6] 十位: 53→5, 24→2, 6→0 → [6, 24, 53] neg 基數(絕對值,個位): 個位: 5→5, 8→8 → [5, 8] 組裝: reverse(neg)=[8,5] → 取負 [-8,-5] 放前面;pos 接後面 [-8, -5] + [6, 24, 53] [-8, -5, 6, 24, 53]
負數的巧思:基數只認非負數字位,所以把負數變成絕對值來排。排好後 [5,8] 是「絕對值小→大」,但作為負數應該是「−5 > −8」,所以要反轉成 [8,5]、取負成 [-8,-5],才是「最負在最前」。這是把「無法直接處理的輸入」轉成「能處理的形式、再轉回來」的通用套路。

應用題// APPLY

App 1

該選哪個排序?(a) 需要穩定且要外部排序(資料放不進記憶體);(b) 記憶體極省、要最壞保證;(c) 值都是 0–10⁶ 的整數。各選一個並說明。

App 2

快排的 partition 變形:本題用 Lomuto(單指針)。Hoare partition(雙指針對撞)有什麼優點?為什麼實務上更常用?

App 3

std::sort 是什麼?C++ 的 std::sort 用 introsort。為什麼要混合快排 + 堆積 + 插入三種?各自負責什麼?

// ANSWERS
A1

穩定= 值相等的元素排序後維持原本的相對順序。例 [(2,a),(1,x),(2,b)] 依數字排,穩定排序保證 (2,a) 仍在 (2,b) 前。插入穩定:只在 a[j] > key(嚴格大於)才右移,相等時停手插在後面 → 不越過相等元素。選擇不穩定:每趟把最小值 swap 到前面,這個遠距交換可能把某個元素跳過另一個相等元素,打亂相對順序(如 [2a,2b,1] 選 1 換到前 → [1,2b,2a],兩個 2 順序反了)。穩定性在「多關鍵字排序」時很重要。

A2

只靠比較的排序等價於走一棵決策樹:每次比較二選一,把可能排列砍半。要區分 n 個元素的 n! 種排列,樹至少 log₂(n!) = Θ(n log n) 層深 → 最壞至少 O(n log n)。基數 / 桶不做比較:它們用「值本身當陣列索引」直接定位元素(計數排序、桶號),繞過決策樹。代價是只適用小範圍整數 / 定長鍵(才能當索引),不是萬用。

A3

最壞發生在每次 partition 都極度不平衡 —— 用「固定取最右為 pivot」時,對已排序(或逆序)陣列,pivot 每次都是最大/最小,一邊 0 個一邊 n−1 個 → 遞迴深度 n、總 O(n²),還可能爆遞迴棧。避免:隨機化 pivot(隨機挑一個和最右交換)或三數取中(取 l/mid/r 的中位數),讓最壞情況機率極低;工業級再加「遞迴太深就轉堆積排序」(introsort)。

A4

因為基數是「從低位到高位,一位一位排」,靠後面的高位排序時不破壞前面低位已排好的相對順序來累積正確性。這只有在每一位都穩定時才成立:高位相同的元素,要保留它們在低位排序後的先後。若某位用不穩定排序,高位相同的兩個數可能被亂序,低位資訊就丟了 → 整體錯誤。程式裡「由後往前 + 前綴和計數」正是為了穩定(相同位的數維持原順序)。

A5

快排常最快是因為:原地、快取友善(連續存取、少額外配置),常數因子小,平均 O(n log n) 但常數比合併小。合併:穩定、最壞保證,但需 O(n) 額外空間 + 資料搬移多,常數較大;不過它是外部排序(資料放不進記憶體)與連結串列排序的首選。堆積:原地、最壞保證,但存取跳躍(不快取友善),實務常數大。取捨:要速度用快排、要穩定用合併、要原地+最壞保證用堆積。

App 1

(a) 穩定 + 外部排序 → 合併排序:天然穩定,且能分塊排序後合併(外部歸併),適合資料放不進記憶體。 (b) 記憶體極省 + 最壞保證 → 堆積排序:O(1) 原地、最壞仍 O(n log n),不像快排會退化。 (c) 0–10⁶ 整數 → 基數 / 計數排序:值域是小整數,可用值當索引達 O(n),快過任何比較排序。

App 2

Hoare partition:左右兩指針對撞,各自找到「該換到另一邊」的元素就交換。相較 Lomuto,它平均交換次數少約 3 倍、對已排序輸入更友善,常數更小,所以工業實作(含經典 introsort)多用 Hoare 或其變形。缺點:Hoare 的 pivot 最終不保證落在正確位置(返回的是分界,不是 pivot 就位處),遞迴邊界要小心寫。Lomuto 較直觀、好教學,但效能略遜。

App 3

introsort(introspective sort)混合三種,各補彼此的短板:快排當主力(平均最快、原地、快取友善);當遞迴深度超過 ~2 log n(偵測到快排要退化成 O(n²))就切換到堆積排序保證最壞 O(n log n);當子區間很小(如 ≤ 16)就改用插入排序(小陣列常數最小、快取最佳)。三者合體 → 平均快、最壞有保證、小段高效,這就是 std::sort 又快又穩健的秘密。
