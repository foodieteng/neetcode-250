題目簡介// THE QUESTION

QUESTION

給整數陣列 nums,升序排序後回傳,不可用內建排序,並儘量達到 O(n log n)。 這題是把排序演算法本身攤開來練 —— 這張卡一次收錄 九種經典排序的機制、複雜度與取捨。

九種排序分三大類:簡單排序 O(n²)(氣泡 / 選擇 / 插入)、進階比較排序 O(n log n)(希爾 / 快排 / 合併 / 堆積)、 非比較排序 O(n)(桶 / 基數)。比較排序的下界是 O(n log n);非比較排序靠「值域是小整數」跳過比較,才能到線性。

九大排序全景比較// THE BIG TABLE

排序平均最壞空間穩定一句話記憶
氣泡 BubbleO(n²)O(n²)O(1)穩定相鄰換,大的冒到尾
選擇 SelectionO(n²)O(n²)O(1)不穩選最小換到前
插入 InsertionO(n²)O(n²)O(1)穩定整理手牌,近序 O(n)
希爾 ShellO(n^1.3)~O(n²)O(1)不穩間隔插入,先粗後細
快排 QuickO(n log n)O(n²)O(log n)不穩pivot 分左右,遞迴
合併 MergeO(n log n)O(n log n)O(n)穩定分治 + 雙指針合併
堆積 HeapO(n log n)O(n log n)O(1)不穩max-heap,取根到尾
桶 BucketO(n)~O(n²)O(n)看桶內依值域散桶再串
基數 RadixO(d·n)O(d·n)O(n)穩定逐位穩定計數排序
面試預設選擇:通用又穩 → 合併;原地又保證 O(n log n) → 堆積;平均最快、可原地 → 快排(記得隨機 pivot 防最壞)。 值是小整數 → 基數 / 桶。小陣列 → 插入。工業級 std::sort 是 introsort(快排 + 堆積 + 插入的混合)。
6

為什麼比較排序無法快過 O(n log n)// LOWER BOUND

n! 種排列 · 每次比較只能二選一

只靠「兩兩比較」的排序,本質是走一棵決策樹:每次比較有兩個結果(< 或 ≥),把可能性砍半。要區分 n 個元素的 n! 種排列,決策樹至少要 log₂(n!) 層深 —— 由 Stirling 公式,log₂(n!) = Θ(n log n)。 所以任何比較排序最壞至少 O(n log n),合併 / 堆積已達到這個下界。基數 / 桶之所以能到 O(n),正是因為它們不做比較 —— 用「值當索引」直接定位,繞過了這棵決策樹。

7

一句洞察// INSIGHT

排序的分水嶺是「比不比較」:比較排序下界 O(n log n)(合併 / 堆積達標);想更快就別比較 —— 用值當索引(基數 / 桶)。 再往上是穩定性、原地、最壞保證這三個維度的取捨。記住這張全景表,就掌握了排序的全貌。
