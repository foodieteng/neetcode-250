完整實作 · hash map(通用)// O(n) · UNICODE-READY

p242.cpp C++17 
class Solution {
public:
    bool isAnagram(string s, string t) {
        if (s.size() != t.size()) return false;   // 長度不同,免比

        unordered_map<char, int> freq;

        for (char c : s) {
            freq[c]++;                        // s 建表:各字元 +1
        }

        for (char c : t) {
            if (freq.count(c) == 0 || --freq[c] < 0)
                return false;                  // t 消耗:沒這鍵、或減到負 → 偽
        }
        return true;                          // 長度已相等 + 沒人變負 → 真
    }
};

freq.count(c) == 0 || --freq[c] < 0 靠短路求值:先確認鍵存在(不存在直接偽,不會執行右邊的 --freq[c] 而誤建新鍵);存在才 -- 並看是否變負。

2

關鍵 · 那個 count(c)==0 || 守門// THE GUARD

少了它,--freq[c] 會偷偷幫不存在的鍵建一個

C++ 的 freq[c] 對 unordered_map 有個陷阱:若鍵不存在,operator[] 會自動插入一個值為 0 的項,再回傳它的參考。所以 --freq[c] 在鍵不存在時,會先建一個 0、再減成 −1 —— 看起來像「減到負數」,但其實是 t 有個 s 根本沒有的字元。

兩種情況(s 沒這字元、s 這字元用完了)都該回 false,結果湊巧都被 --freq[c] < 0 抓到。所以就算拿掉 count(c)==0 也能正確 —— 但會污染 map(悄悄新增一堆值為 −1 的垃圾鍵)。加上 count 守門語意更清楚、也不亂建鍵,是好習慣。

3

若只有小寫英文 · int[26]// FASTER

// 值域已知(a-z),用固定陣列比 hash map 快 bool isAnagram(string s, string t) { if (s.size() != t.size()) return false; int cnt[26] = {0}; for (char c : s) cnt[c - 'a']++; // s ++ for (char c : t) if (--cnt[c - 'a'] < 0) return false; // t -- return true; } 下標 c-'a' 直接定址,無雜湊開銷。O(1) 空間(固定 26 格)

陣列版不用 count 守門 —— 因為 cnt[26] 全部預設 0、每個字元都有格子,不存在「鍵不存在」的問題。 --cnt[c-'a'] < 0 就直接判偽。值域小且已知 → 陣列;值域大或未知(Unicode)→ hash map,兩者都是計數,只差容器。

邊界與陷阱// PITFALLS

陷阱症狀正解
不先擋長度 "a" vs "ab":s 建 {a:1},t 消完 a 剩 b 但迴圈已結束 → 誤判 true 開頭 s.size()!=t.size() → false
先 -- 再 count 短路失效,不存在的鍵被 [] 建出來 count(c)==0 || --freq[c]<0(count 在前)
結尾又檢查全零 多餘 —— 長度已相等 + 沒人變負就必然全零 沒人變負 → 直接 return true
陣列版忘記 = {0} cnt[26] 未初始化 → 垃圾值 → 全錯 int cnt[26] = {0};
Unicode 用 int[26] 非 a-z 字元下標越界 / 錯亂 用 unordered_map<char,int>(或依編碼調整)

細節走讀// TWO CASES

// 真:s="anagram", t="nagaram" 建表 s = "anagram": {a:3, n:1, g:1, r:1, m:1} 消耗 t = "nagaram": n → freq[n]:1→0 a → freq[a]:3→2 g → freq[g]:1→0 a → freq[a]:2→1 r → freq[r]:1→0 a → freq[a]:1→0 m → freq[m]:1→0 全程沒有任何字元變負(a 剛好被消 3 次到 0) return true
// 偽:s="rat", t="car" 建表 s: {r:1, a:1, t:1} 消耗 t: c → freq.count('c')==0 ← s 根本沒有 c 立刻 return false(不用消完)

兩種偽的成因:"car" 的 c 是「s 沒有的字元」(count==0 抓到); 若是 "aab" vs "abb",則是 b 用超過(freq[b] 從 1 減到 −1,<0 抓到)。兩種偽,兩個守門條件各抓一種。
