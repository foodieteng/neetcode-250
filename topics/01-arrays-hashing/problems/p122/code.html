完整實作 · 貪心一趟// O(n) / O(1)

p122.cpp C++17 
class Solution {
public:
    int maxProfit(vector<int>& prices) {
        int n = prices.size();
        int mx = 0, start = prices[0];       // start = 前一天的價

        for (int i = 1; i < n; i++) {
            if (start < prices[i])            // 今天比昨天貴
                mx += prices[i] - start;     // 賺下這段漲幅
            start = prices[i];               // 前一天前移
        }
        return mx;
    }
};
start 就是「前一天的價格」,每輪比較後無條件前移 start = prices[i]。 整段程式沒有「買點/賣點」的概念 —— 貪心把它化簡成只看相鄰兩天的差,漲就收、跌就跳。 等價的極簡寫法:for i in 1..n: mx += max(0, prices[i]−prices[i−1])。

對照寫法 · hold / free 狀態機// DP

// 通用的股票 DP:兩個狀態,和貪心殊途同歸(此題答案相同) int maxProfit(vector<int>& prices) { int hold = INT_MIN, free = 0; // hold=持股中最好, free=空手最好 for (int p : prices) { int nh = max(hold, free - p); // 今天持股:昨持 or 昨空手買入 int nf = max(free, hold + p); // 今天空手:昨空 or 昨持賣出 hold = nh; free = nf; } return free; // 最後空手獲利最大 } O(n)/O(1),對「無限次交易」等價於貪心;但 122 用貪心一行即可,DP 是留給有手續費/冷凍期的變體

這個狀態機是整個股票系列的母模板 —— 加手續費(714)只需在賣出項扣 fee;加冷凍期(309)則拆成三狀態。122 因為沒有這些限制,才退化成純貪心。

陷阱症狀正解
想找「單一最低買、最高賣」 那是 121(只能一次交易)的解;用在 122 會少賺 122 無限次 → 加總每段漲幅
把負日差也加進去 下跌日算成負利潤 → 總額被拉低 只在 prices[i] > prices[i−1] 時加
忘了持股上限而糾結 擔心「同時買多股」—— 其實貪心天然滿足最多 1 股 每個正日差 = 一次昨買今賣,不重疊持股
n==1 未處理 只有一天 → 迴圈不進,回 0(本題已正確) start=prices[0],迴圈從 i=1 起
