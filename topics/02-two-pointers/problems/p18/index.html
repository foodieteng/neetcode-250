題目簡介// THE QUESTION

QUESTION

給陣列 nums 和 target,回傳所有不重複的四元組 [nums[a],nums[b],nums[c],nums[d]],四個索引互異且四數和 = target。 例:nums=[1,0,-1,0,-2,2], target=0 → [[-2,-1,1,2],[-2,0,0,2],[-1,0,0,1]]。

這是 3Sum(15) 的再加一層:同樣「排序 → 固定 → 對撞 → 去重」,只是外層從固定一個數變成固定兩個(i 和 j),內層仍是 167 的對撞找兩數。 兩個新重點:多一層外循環(去重也多一處) 與 四數相加可能溢位 int,要用 long long。

2

為什麼「固定兩數 + 對撞」// WHY

× REJECT · O(n⁴)
四層迴圈枚舉

試所有 (a,b,c,d)。就算 n 只有 200,n⁴=1.6×10⁹ 已偏慢,還要另外 set 去重。

△ 想太多
固定兩數 + 雜湊找兩數

內層改雜湊找 target-nums[i]-nums[j]-nums[l]。時間一樣 O(n³) 但要處理雜湊去重、額外空間,遠不如排序後對撞乾淨。

SORT + TWO POINTERS · ADOPT ↓
● ADOPT · O(n³) / O(1)*
排序 → 固定 i,j → 對撞找兩數 → 去重

排序後,外兩層固定 nums[i]、nums[j](各自跳過相鄰重複),內層在 [j+1, n-1] 用對撞找 nums[l]+nums[r] = target-nums[i]-nums[j]。 和 3Sum 完全同一套骨架,只是多一個固定的數 → O(n³),除輸出外常數空間。四數和用 long long 累加避免溢位。

* 不計答案輸出;原地排序 O(1)(遞迴堆疊 O(log n))。

3

核心策略 · 四步驟// STRATEGY

  1. 排序 —— sort(nums),給單調性 + 讓相同值相鄰。
  2. 外層固定 i(去重) —— nums[i]==nums[i-1] 就跳過。
  3. 次外層固定 j(去重) —— j>i+1 && nums[j]==nums[j-1] 就跳過。
  4. 內層對撞 + 去重 —— l=j+1,r=n-1,依 long long sum 與 target 比較移動;命中收下後跳相鄰相同的 l、r。

比 3Sum 多的兩件事// WHAT'S NEW

多一處去重 · 用 long long 防溢位

① 去重從三處變四處:外層 i(比 nums[i-1])、次外層 j(比 nums[j-1])、命中後的 l、r。 注意 j 的去重條件是 j > i+1 && nums[j]==nums[j-1] —— j > i+1 確保每個 i 底下 j 的第一個值仍會處理,只跳同一 i 內重複的 j(和外層 i>0 同理)。

② 溢位:nums[i] 可達 ±10⁹,四個相加最大約 4×10⁹,超過 int 上限(約 2.1×10⁹)。若用 int 累加會溢位、比較出錯。解法:long long sum = (long long)nums[i] + nums[j] + nums[l] + nums[r]; —— 第一個轉 long long,後面自動提升。target 也可能是負的,但因為兩邊都在 long long 下比較,沒問題。

複雜度// COMPLEXITY

T = O(n³)　　S = O(1)(不計輸出)

排序 O(n log n) 被吸收;主體是「外兩層 i,j 共 O(n²) × 內層對撞 O(n)」= O(n³)。除答案外只用兩個索引 → O(1)。相較 O(n⁴) 暴力快一個量級,n≤200 綽綽有餘。

面向成本說明
排序O(n log n)一次,被 O(n³) 蓋過
外兩層 i,jO(n²)固定每對起點
內層對撞O(n)l+r 合計走一趟
總時間 / 空間O(n³) / O(1)不計輸出

一句洞察// INSIGHT

kSum 階梯:每多一個數,就多固定一層,內層永遠是 167 的對撞。3Sum 固定 1 層是 O(n²),4Sum 固定 2 層是 O(n³)。 去重就是「每個固定層比前一個、命中後跳 l/r」。別忘了四數和用 long long。
