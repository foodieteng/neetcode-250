重點回顧// RECAP

關鍵一句話
結構3Sum 外再套一層固定 j
內層對撞找 nums[l]+nums[r] = target-nums[i]-nums[j]
去重四處i、j(比前一個)· 命中後 l、r
j 條件j > i+1 && nums[j]==nums[j-1]
溢位四數和用 long long(第一個轉型)
複雜度O(n³) 時間 / O(1) 空間(不計輸出)
階梯2Sum→3Sum→4Sum→kSum,每層 +1 固定

Q5

範例 Trace// 全等值 + 溢位邊界

// nums=[2,2,2,2,2], target=8 → 唯一解 [2,2,2,2] 排序後 [2,2,2,2,2] i=0(2), j=1(2): l=2,r=4 → 2+2+2+2=8 → 收下 [2,2,2,2]; l++,r-- 跳重複左 nums[3]==nums[2] → l++ → l=r → 內層結束 i=0, j=2..: nums[j]==nums[j-1] → continue(去重 j) i=1..: nums[i]==nums[i-1] → continue(去重 i) ans = [[2,2,2,2]](四層去重合力,只回一組)
// 溢位邊界:nums=[1000000000]×4, target=-1(無解但別溢位) sum = (long long)1e9 + 1e9 + 1e9 + 1e9 = 4e9 // long long 正確保存 若用 int:4e9 溢位成負數 → 誤判 sum < target → 邏輯錯 long long 讓 4e9 正確 > target=-1 → r--,不會亂跳
全等值 [2,2,2,2,2] 是去重的壓力測試:四層去重把 i、j 的重複 2、以及命中後 l 的重複 2 全擋掉,只回一組。 而溢位邊界提醒:資料範圍到 10⁹、又要相加多個,幾乎都得先想 long long。

App 2

App 3

4Sum II(LC 454):給四個陣列 A,B,C,D,問各取一個使和為 0 的組合數。為什麼這題不能用本題的對撞、而要用雜湊?時間差在哪?

// ANSWERS
A1

nums[i] 可達 ±10⁹,四個相加最大約 4×10⁹,超過 int 上限 2.147×10⁹ → 溢位變負數,sum < target 之類的比較就全亂了。C++ 加法從左往右:((a+b)+c)+d,第一步 a+b 就是 int+int,若不先轉型這裡已溢位。把最左 operand 轉 long long,之後每步都是 long long+int→long long,全程 64 位元安全。轉型放後面(如 (long long)(a+b+c+d))無效,因為括號內已用 int 算完。

A2

寫 j > 0 會誤跳每個 i 底下 j 的第一個值:當 nums[j] 剛好等於外層 nums[i](即 nums[j]==nums[j-1]==nums[i])時,j=i+1 這第一個合法 j 會被錯跳,漏解。正確比較點是 i+1:j > i+1 表示「只在同一個 i 內、j 不是第一個」才去重,讓每個 i 的第一個 j 一定進內層 —— 和外層 i > 0 是完全對稱的道理(每個層級跳自己這層的重複,保留第一個)。

A3

四處:① 外層 i i>0 && nums[i]==nums[i-1] —— 防同一個 nums[i] 起點重複整批。② 次外層 j j>i+1 && nums[j]==nums[j-1] —— 防同一 i 下 nums[j] 重複。③ 命中後 l nums[l]==nums[l-1]、④ 命中後 r nums[r]==nums[r+1] —— 防同一 (i,j) 下內層產生相同左/右值。四個各守一層,任缺一個都可能吐出重複四元組。

A4

一句話:「在 3Sum 外面再包一層固定的 j 迴圈(含它自己那處去重),並把和改成 long long」。內層對撞三分支完全不變。時間從 O(n²) 變 O(n³) 是因為固定的層數多了一層:3Sum 是「固定 1 個 × 內層 O(n)」= O(n²),4Sum 是「固定 2 個(O(n²))× 內層 O(n)」= O(n³)。每加一個數,複雜度就乘上一個 n。

A5

l、r 是索引,範圍 [0, n) 且 n≤200(甚至 ≤10⁵ 的題也遠在 int 內),不會溢位,用 int 就好。真正需要 64 位元的是「數值的加總」 —— 也就是 sum,因為它把四個可達 10⁹ 的值相加。分清「索引」和「數值累加」:前者小、後者可能大,是判斷要不要 long long 的準則。

App 1

kSum 模板:排序一次。kSum(start, target, k):若 k==2 → 對撞雙指針(基底,就是 167 那段,含 l/r 去重);否則從 start 掃 i(跳 nums[i]==nums[i-1]),對每個 i 遞迴 kSum(i+1, target-nums[i], k-1),把回傳的每組前面補上 nums[i]。去重放在每一層的固定迴圈與基底的 l/r。整體時間 O(n^{k-1})。so 2Sum 是基底、3Sum/4Sum 只是遞迴深度不同。

App 2

固定 i,j 後,內層 [l,r] 已排序。最小可能四數和 = nums[i]+nums[j]+nums[j+1]+nums[j+2](取最小兩個);若它 > target,再大只會更大 → 這個 j(甚至 i)之後都無望,break。最大可能和 = nums[i]+nums[j]+nums[n-2]+nums[n-1];若它 < target,這個 j 太小 → continue 換下一個 j。都要用 long long 算界。這些剪枝把常數大幅壓低,最壞仍 O(n³)。

App 3

LC 454 4Sum II:四個獨立陣列各取一個、只求組合數(不需列出、不需去重)。因為元素來自不同陣列、又不用去重,「排序 + 對撞」的前提(同一陣列、要列元素)不適用。正解是折半雜湊(meet in the middle):先把 A[i]+B[j] 的所有和存進 unordered_map 計數(O(n²)),再枚舉 C[k]+D[l] 查 -(C+D) 出現幾次累加(O(n²))。總 O(n²),比對撞的 O(n³) 更快 —— 「求數量」比「列組合」能用更強的雜湊技巧。
