題目簡介// THE QUESTION

QUESTION

給一個字元陣列 s,原地反轉它。必須就地修改、只用 O(1) 額外記憶體。 例:['h','e','l','l','o'] → ['o','l','l','e','h']。

這是雙指針(Two Pointers)章的開門題,示範最基本的一種:對撞指針(converging / opposite-direction) —— 一個從頭、一個從尾,往中間收。「O(1) 空間」這個限制正是雙指針的招牌:不開新陣列,靠交換就地完成。

2

為什麼「對撞雙指針」// WHY

△ OK · 遞迴但吃棧
遞迴交換兩端

遞迴反轉 [l+1, r-1]。就地沒錯,但遞迴深度 O(n) 佔用呼叫棧,不是真 O(1)。

IN-PLACE · ADOPT ↓
● ADOPT · two pointers
l 從頭、r 從尾,交換後往中間收

l = 0、r = n−1。while (l < r):交換 s[l]↔s[r],然後 l++、r--。 兩指針對撞,交換 n/2 次就反轉完畢。O(n) 時間、O(1) 額外空間,迴圈版不吃棧。 (std::reverse 內部做的正是這個。)

3

核心策略 · 三步驟// STRATEGY

  1. 兩端起手 —— l = 0、r = s.size() − 1。
  2. 交換 + 內收 —— while (l < r):swap(s[l], s[r]); l++; r--;。
  3. 自然停 —— l >= r 即完成。偶數長度時兩指針交錯而過,奇數長度時停在正中間那格(不需動)。
4

動畫演示// CONVERGING POINTERS

"hello":交換 s[0]↔s[4]、s[1]↔s[3],l 與 r 對撞到中間 index 2 停。中間那格('l')奇數長度不用動。結果 "olleh"。

5

對撞指針模式// THE PATTERN

兩端往中間收 —— 一種掃描一半、卻覆蓋全體的骨架

對撞指針(opposite-direction / converging)的骨架長這樣:

　l = 0, r = n − 1
　while (l < r) {
　　… 用 s[l] 與 s[r] 做事(交換 / 比較 / 計算）…
　　依情況 l++ 或 r-- (或兩者)
　}

反轉字串是「交換兩端」;換成「比較兩端」就是回文檢查(125); 換成「看兩端和太大就縮右、太小就進左」就是已排序陣列的 Two Sum(167)、裝最多水(11)。 同一個「兩端內收」骨架,換掉中間那一步,就解一整族題 —— 這就是為什麼它是雙指針章的第一課。

6

複雜度// COMPLEXITY

T = O(n)　　S = O(1)

兩指針各走一半,合計掃過 n 個位置、交換 ⌊n/2⌋ 次。就地交換,只用兩個索引變數,不開新陣列、迴圈版不吃呼叫棧。

面向成本說明
掃描 + 交換O(n)交換 ⌊n/2⌋ 次
每步O(1)一次 swap
額外空間O(1)l、r 兩個索引
奇數長度中間不動l == r 時停
7

一句洞察// INSIGHT

要「原地、O(1)」處理陣列兩端 → 對撞雙指針:l 從頭、r 從尾,while (l < r) 交換後內收。 把中間那步的「交換」換成「比較 / 求和」,同一骨架就變成回文、Two Sum、裝水。這是雙指針的入門原型。
