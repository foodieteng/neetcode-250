完整實作 · 對撞雙指針// IN-PLACE

p344.cpp C++17 
class Solution {
public:
    void reverseString(vector<char>& s) {
        int l = 0, r = s.size() - 1;
        while (l < r) {
            swap(s[l], s[r]);       // 交換兩端
            l++; r--;                 // 往中間收
        }
    }
};
迴圈條件 l < r 同時處理奇偶:偶數長度時 l、r 交錯而過;奇數長度時停在 l == r(正中那格,本來就在對的位置,不用動)。 這是雙指針章要建立的肌肉記憶:兩端內收、while (l < r)。

一行版 · std::reverse// LIBRARY

// 標準庫一行搞定 —— 內部做的就是上面的對撞交換 class Solution { public: void reverseString(vector<char>& s) { reverse(s.begin(), s.end()); } }; 同樣 O(n)/O(1);std::reverse 就是「兩端 swap、往中間收」的封裝

實務上直接 std::reverse 最省事,結果與手寫雙指針完全相同。但面試若考「反轉」通常是要你展示雙指針的手寫版(證明你懂原地交換的機制),所以第 1 節的手寫版才是這題的學習重點。兩者本質是同一個演算法。

邊界與陷阱// PITFALLS

陷阱症狀正解
迴圈用 l <= r 奇數長度時中間那格和自己 swap(無害但多做);更糟是若寫錯還交換到越界 while (l < r)
只動一個指針 忘了 l++ 或 r-- → 無窮迴圈 / 重複交換 兩個都要動
開新陣列複製 O(n) 空間,違反 O(1) 要求 原地交換
r 初值用 size() s[size()] 越界 r = size() − 1
空 / 單字元 怕出錯而特判 l < r 天然不進迴圈,安全

細節走讀// WALKTHROUGH

// s = ['h','e','l','l','o'](n=5) l=0 r=4 : l<r → swap(s[0],s[4]) ['o','e','l','l','h'] l=1 r=3 l=1 r=3 : l<r → swap(s[1],s[3]) ['o','l','l','e','h'] l=2 r=2 l=2 r=2 : l<r? 否 → 停(中間 'l' 不動) "olleh"
// 偶數長度 s = ['a','b','c','d'](n=4) l=0 r=3 : swap → ['d','b','c','a'] l=1 r=2 l=1 r=2 : swap → ['d','c','b','a'] l=2 r=1 l=2 r=1 : l<r? 否 → 停(l 與 r 交錯而過) "dcba"
奇數(n=5)停在 l == r、偶數(n=4)停在 l > r —— 兩種都被 while (l < r) 一句話涵蓋,不必分開處理。 這正是「兩端內收」骨架的優雅:條件只有一個,奇偶自動正確。
