重點回顧// RECAP

關鍵一句話
模式對撞雙指針:l 從頭、r 從尾,往中間收
骨架while (l < r) { swap; l++; r--; }
原地交換不開新陣列 → O(1) 額外空間
奇偶l < r 一句涵蓋:奇停 l==r、偶停 l>r
庫函式std::reverse = 同演算法的封裝
可遷移把「交換」換成「比較 / 求和」→ 回文 / TwoSum / 裝水
複雜度O(n) 時間 / O(1) 空間
2

觀念題// CONCEPT Q

Q1

為什麼迴圈條件是 l < r 而不是 l <= r?奇數長度的中間那格會怎樣?

Q2

這題要求 O(1) 空間。開一個反轉後的新陣列(O(n))和遞迴反轉,各違反了什麼?

Q3

「對撞雙指針」的骨架不變,只換掉中間那一步,能解哪些題?各舉一個。

Q4

本題可以用 std::reverse 一行解。為什麼面試通常還是要你手寫雙指針版?

Q5

雙指針有「對撞(兩端往中間)」和「快慢(同向不同速)」兩大類。反轉字串是哪一類?另一類典型用在什麼題?

3

範例 Trace// 骨架換一步 → 回文檢查

// 同骨架:把「交換」換成「比較」→ 判斷回文 "abcba" l=0 r=4 : s[0]=='a' 與 s[4]=='a' 相等 → l++ r-- l=1 r=3 : s[1]=='b' 與 s[3]=='b' 相等 → l++ r-- l=2 r=2 : l<r? 否 → 全部相等,是回文 true(反轉是「swap 兩端」,回文是「compare 兩端」,骨架相同)
看清楚:反轉和回文用的是完全一樣的 while (l < r)、l++、r--,只有中間那一句從 swap 變成 if (s[l] != s[r]) return false。 學會反轉字串,等於學會了半個雙指針章 —— 骨架記住了,剩下的是換「中間那一步」。

應用題// APPLY

App 1

LC 345 只反轉母音:只把字串中的母音字母反轉、其餘不動(如 "leetcode" → "leotcede")。對撞雙指針要怎麼改?(提示:兩個指針各自跳過非母音)

App 2

LC 151 反轉句子中的單字:"the sky" → "sky the"。一個常見技巧是「先整體反轉、再各單字反轉」。這裡怎麼複用反轉?

App 3

陣列旋轉 LC 189:把陣列右旋 k 位。經典 O(1) 解法是「反轉全部、反轉前 k、反轉後 n−k」。為什麼三次反轉就等於旋轉?

// ANSWERS
A1

l < r 表示「還有兩個不同的位置要交換」。奇數長度時走到 l == r(正中那格),此時 l < r 為假、停 —— 中間字元和自己是一對,不需要動。用 l <= r 會多做一次「中間格和自己 swap」(無害但多餘),而且容易讓人誤以為要處理中間,反而添亂。l < r 最乾淨。

A2

新陣列:倒序複製到另一個 O(n) 大小的陣列再貼回,違反「O(1) 額外記憶體」。遞迴:雖然「就地交換」不開資料陣列,但遞迴深度 O(n) 佔用呼叫棧,那也是 O(n) 額外空間(隱性的),嚴格來說不算真 O(1)。迴圈版雙指針才是名副其實的 O(1)。

A3

同一個「兩端內收」骨架,換中間那步:
‧ 交換 → 反轉字串(本題)。
‧ 比較 s[l]==s[r] → 回文檢查(LC 125)。
‧ 看兩端和:太大 r--、太小 l++ → 已排序 Two Sum(LC 167)。
‧ 算面積 min(h[l],h[r])×(r−l),移動較矮那邊 → 裝最多水(LC 11)。
骨架都是 while (l < r) + 內收,差別只在中間邏輯與「移哪個指針」。

A4

因為面試考「反轉」的目的是驗證你懂原地交換的機制,不是考你會不會查標準庫。std::reverse 一行雖然對,但沒展示任何演算法思考;手寫雙指針能證明你掌握「兩端內收、O(1) 就地」。而且很多變體(只反轉母音、反轉區段、旋轉)沒有現成庫函式,得靠手寫骨架。實務用庫,面試手寫 —— 兩者不衝突,學的是機制。

A5

反轉字串是對撞型(opposite-direction / converging):兩指針從兩端往中間收,常用在「陣列兩端有對稱關係」的題(反轉、回文、已排序找和、裝水)。另一類快慢型(fast-slow / same-direction):兩指針同向、不同速或不同角色,常用在「就地移除 / 去重 / 找環 / 滑窗」(如快慢指標偵測鏈結串列有環、原地移除元素、快指針探路慢指針寫入)。兩端對稱 → 對撞;同向掃描 → 快慢。

App 1

LC 345:一樣 l/r 對撞,但每步先讓 l 往右跳過非母音、r 往左跳過非母音,兩邊都停在母音時才 swap(s[l],s[r]) 再各內收。骨架不變,只是加了「跳過不關心的元素」這個常見雙指針技巧(回文忽略非字母數字也是同招)。

App 2

LC 151:先把整個字串反轉("the sky" → "yks eht"),這樣單字順序對了、但每個單字內部字母是反的;再逐一反轉每個單字("yks" → "sky"、"eht" → "the")得到 "sky the"。兩層都用本題的對撞反轉。核心:反轉是可組合的積木 —— 大反轉修順序、小反轉修內部。

App 3

LC 189:右旋 k = 把後 k 個搬到前面。三步反轉:①反轉全部(後 k 個跑到前面但順序反了)②反轉前 k 個(修正它們的順序)③反轉後 n−k 個(修正剩下的)。因為「反轉再反轉 = 還原順序」,先整體反轉把兩段位置對調,再各自反轉把段內順序救回來,淨效果就是旋轉。三次反轉都 O(1) 空間,總 O(n) —— 又是反轉當積木的漂亮應用。
