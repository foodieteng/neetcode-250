應用題// APPLY

App 1

最多刪 k 個字元:若容許刪除 k 次而非 1 次,這招要怎麼推廣?為什麼會從 O(n) 變貴?(提示:每次不符都可能二選一分叉)

App 2

回傳「要刪哪個」而非 true/false:若題目要回傳被刪字元的 index,程式要多記什麼?|| 短路還夠用嗎?

App 3

LC 1216 K 次修改成回文(可替換而非刪除):最少改幾個字元能成回文?這變成什麼問題?和本題「刪除」的差別在哪?

// ANSWERS
A1

因為走到第一個不符 s[l] != s[r] 時,外側 [0, l−1] 與 [r+1, n−1] 已經兩兩對稱(一路相符才走到這),那些位置沒有問題、不需要動。真正的衝突就是 s[l] 對不上 s[r],而唯一的刪除要嘛拿掉 s[l](讓 s[l+1] 頂上去對 s[r])、要嘛拿掉 s[r](讓 s[l] 對 s[r−1])。刪中間某個已對稱的字元只會破壞已有的對稱,毫無幫助。所以候選只有這兩個。

A2

因為分叉只發生一次。主對撞掃到不符是 O(n);分叉後呼叫兩個 isPali,各自 O(n) —— 但它們是不再容錯的普通檢查,內部不會再分叉、不會遞迴放大。所以總量 = O(n) + O(n) + O(n) = O(n)。相對地,「枚舉刪每一個 index 再檢查」是 n 次 O(n) = O(n²)。把「刪除點」鎖定在唯一衝突處,是從平方降到線性的關鍵。

A3

因為題目只允許刪一次。主流程遇到不符時已經花掉那次機會,子檢查代表「刪掉某字元之後剩下的串」,它必須是真回文(不能再刪)。若 isPali 也容錯,等於允許刪兩個,像 "acbca" 之類本該 false 的會被判 true。子檢查零容錯,正好對應「總共只刪一次」。

A4

"cbbcc":外層 c,c 相符,內層 b,c 不符。刪左(拿掉 'b')→ 檢查 "bc" → b != c → false;刪右(拿掉那個多的 'c')→ 檢查 "bb" → true。只試刪左會誤判 false,只試刪右才對。但你事前不知道該刪哪邊,所以必須 A || B 兩邊都試、任一成立即可。

A5

s.substr(l+1) 會複製出一個新子字串,長度可達 O(n) —— 分叉時最多切兩次,額外 O(n) 空間 + 複製成本。用 isPali(s, l+1, r)(傳引用 + 索引區間)則零複製,維持 O(1) 空間。功能一樣,但「原字串 + 索引」表達子問題是雙指針/分治省空間的通則,值得養成習慣。

App 1

刪 k 個:每遇到不符都可能「刪左或刪右」,而且刪完還剩 k−1 次容錯 → 變成遞迴 / DP:solve(l, r, k) = 相符就 solve(l+1, r-1, k),不符就 solve(l+1, r, k-1) || solve(l, r-1, k-1)。因為每次不符都可能二分叉、且容錯沒立刻用完,狀態變成 (l, r, 剩餘 k),用記憶化是 O(n²·k) 級。「只刪 1 次」讓分叉只一次,才有 O(n) 的特權;k 一大就回到 DP。

App 2

回傳 index:主流程遇到不符時,先試 isPali(l+1, r),成功就回傳「刪 l」;否則試 isPali(l, r-1),成功回傳「刪 r」;都失敗回傳「無解」。|| 短路仍夠用,只是把「回傳 bool」換成「哪一路成功就回報對應的 index」。若全程相符則回報「不需刪 / -1」。邏輯骨架不變,只是把成敗換成位置。

App 3

替換(而非刪除):能改字元時,「最少改幾個成回文」= 不匹配的對數。對撞比較 l/r,每遇到一對 s[l] != s[r] 就 count++(把其中一個改成另一個即可修好這一對),最後看 count <= k。它比刪除單純:替換不改變長度、對稱結構固定,所以不需分叉 —— 直接數不匹配對即可,O(n)。刪除會改變後續配對(所以要分叉),替換不會 —— 這是兩者的本質差別。
