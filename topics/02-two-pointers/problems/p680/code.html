完整實作 · 雙指針 + 分叉// O(n)/O(1)

p680.cpp C++17 
class Solution {
private:
    // 普通回文檢查(不再容錯),用引用避免複製字串
    bool isPalindrome(string& s, int l, int r) {
        while (l < r) {
            if (s[l] != s[r]) return false;
            l++; r--;
        }
        return true;
    }

public:
    bool validPalindrome(string s) {
        int l = 0, r = s.size() - 1;
        while (l < r) {
            if (s[l] == s[r]) {
                l++; r--;                        // 相符 → 內收
            } else {
                // 第一個不符:刪左 或 刪右,各驗一次
                return isPalindrome(s, l + 1, r)
                    || isPalindrome(s, l, r - 1);
            }
        }
        return true;                          // 全程相符,不需刪
    }
};
兩層迴圈都是同一副對撞骨架:外層 validPalindrome 是「有一次容錯」的版本,內層 isPalindrome 是「零容錯」的普通版本。 不符時的 || 短路:先試刪左,成功就直接 true、根本不跑刪右。return 在分叉處直接結束,所以分叉只此一次。

細節 · 傳引用 vs 傳值// PASS BY REF

isPalindrome(string& s, ...) 用引用接字串,只傳索引 l/r,不複製子字串 —— 這是保持 O(1) 空間、O(n) 時間的關鍵。 若寫成 isPalindrome(s.substr(l+1, ...)) 每次分叉都建一個新子字串,平白多 O(n) 空間與複製成本。

// 好 vs 壞:檢查子區間的兩種寫法 ✓ isPalindrome(s, l+1, r); // 傳引用 + 索引,零複製 ✗ isPalindrome(s.substr(l+1, r-l)); // 每次切出新字串,O(n) 複製 // validPalindrome 的參數 string s(傳值)其實也可以改成 const string&, // 但因為只在進入點複製一次、之後都用引用,影響不大。 用「原字串 + 索引區間」表達子問題,是雙指針省空間的通則

邊界與陷阱// PITFALLS

陷阱症狀正解
只試刪一邊 只查 isPali(l+1, r) → 漏掉「該刪右」的情形(如 "cbbcc") 兩邊都試:A || B
分叉子檢查又允許刪 isPali 再容錯 → 等於能刪兩次,答案錯 子檢查是零容錯的普通回文
切子字串比較 substr 每次複製 → 慢、耗空間 傳引用 + 索引區間
誤以為刪除點在中間 去枚舉中間位置 → 複雜化、易錯 只在第一個不符處二選一
全相符時回錯 本來就是回文(刪 0 個)也要 true 迴圈自然結束回 true

細節走讀// WALKTHROUGH

// s = "cbbcc"(只有刪右能成) l=0 r=4 : 'c'=='c' ✓ → l=1 r=3 l=1 r=3 : 'b' != 'c' → 分叉: 刪左 isPali(s, 2, 3) : "bc" → 'b'!='c' → false 刪右 isPali(s, 1, 2) : "bb" → 'b'=='b' → true false || true = true true(刪掉 index 3 的 'c' → "cbbc" 是回文)
// s = "abc"(刪哪個都不行) l=0 r=2 : 'a' != 'c' → 分叉: 刪左 isPali(s, 1, 2) : "bc" → false 刪右 isPali(s, 0, 1) : "ab" → false false || false = false false
"cbbcc" 展示了為何兩邊都要試:刪左的 "bc" 失敗,但刪右的 "bb" 成功。若只試一邊就會誤判 false。 || 短路讓「先成功的那路」直接定案,兩路最多各跑一次。
