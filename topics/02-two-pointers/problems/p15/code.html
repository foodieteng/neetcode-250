完整實作 · 排序 + 對撞 + 去重// O(n²)

p15.cpp C++17 
class Solution {
public:
    vector<vector<int>> threeSum(vector<int>& nums) {
        int n = nums.size();
        sort(nums.begin(), nums.end());            // ① 排序:相同值相鄰 + 單調性
        vector<vector<int>> ans;

        for (int i = 0; i < n; i++) {
            if (i > 0 && nums[i] == nums[i - 1]) continue;  // ② 外層去重

            int l = i + 1, r = n - 1;

            while (l < r) {
                int sum = nums[i] + nums[l] + nums[r];

                if (sum < 0) {
                    l++;                              // 太小 → 進左
                } else if (sum > 0) {
                    r--;                              // 太大 → 縮右
                } else {
                    ans.push_back({nums[i], nums[l], nums[r]});
                    l++; r--;                        // 收下後兩端一起收縮
                    while (l < r && nums[l] == nums[l - 1]) l++;  // ③ 跳重複左值
                    while (l < r && nums[r] == nums[r + 1]) r--;  // ③ 跳重複右值
                }
            }
        }

        return ans;
    }
};
三個 continue/while 全是去重(見 概念頁 §5)。命中分支先 l++、r-- 再跳重複,所以判斷用 nums[l-1](剛用過的左值)、nums[r+1](剛用過的右值)。 注意只在找到解後才跳 l/r —— sum<0/>0 的推進本來就不會造重複組。

實戰剪枝 · 提早收工// PRUNING

// 排序後可加兩個剪枝,常數級加速(不改變 O(n²) 上界) for (int i = 0; i < n; i++) { if (nums[i] > 0) break; // 最小的都 > 0 → 三個正數不可能和為 0 if (i > 0 && nums[i] == nums[i-1]) continue; ... } nums[i] > 0 時,後面(更大)都是正數,三數和必 > 0 → 直接 break 收工

因為已排序,一旦 nums[i] > 0,i 之後全是正數,任三個相加都 > 0,不可能湊出 0 → break。 這個剪枝在多正數的輸入上省很多,但最壞情況仍是 O(n²)(例如大量負數與零)。原始碼不加也正確,面試時提一句是加分。

邊界與陷阱// PITFALLS

陷阱症狀正解
忘了排序 對撞的單調性失效,去重也亂 一開始 sort
命中後不跳重複 相同左/右值 → 產生重複三元組 l++/r-- 後 while 跳相鄰相同值
外層不去重 相同 nums[i] → 整批重複組 i>0 && nums[i]==nums[i-1] 就 continue
跳重複用錯索引 比 nums[l+1] 而非 nums[l-1] 已先 l++,要比剛用過的 nums[l-1]
用 set 去重 可過但多一層 log 與空間 排序 + 跳相鄰,免 set

細節走讀// WALKTHROUGH

// nums = [-1,0,1,2,-1,-4] → 排序 [-4,-1,-1,0,1,2] i=0 (-4): l=1,r=5 sum=-3<0→l++ ... 一路 l++ 到 l==r,無解 i=1 (-1): l=2,r=5 -1-1+2=0 → 收下 [-1,-1,2]; l++,r-- l=3,r=4 -1+0+1=0 → 收下 [-1,0,1]; l++,r-- → l==r 結束 i=2 (-1): nums[2]==nums[1] → continue(外層去重) i=3 (0): l=4,r=5 0+1+2=3>0→r-- → l==r,無解 i≥4: 內層空,結束 ans = [[-1,-1,2], [-1,0,1]]
i=2 的第二個 -1 若不跳過,會再次得到 [-1,-1,2] 和 [-1,0,1] —— 正是重複組的來源。 外層那一行 continue 就是為了堵這個洞;命中後跳 l/r 則堵「同一個 i 內」的重複。兩層去重缺一不可。
