題目簡介// THE QUESTION

QUESTION

給整數陣列 nums,回傳所有不重複的三元組 [nums[i], nums[j], nums[k]],滿足三個索引互異且 nums[i]+nums[j]+nums[k]=0。 答案集不能有重複三元組。例:[-1,0,1,2,-1,-4] → [[-1,-1,2],[-1,0,1]]。

這題是 Two Sum II(167) 的直接升級 —— 把三數和化約成兩數和:先排序,外層固定一個數 nums[i],剩下就是「在右側有序子陣列裡找兩數 = -nums[i]」,正是對撞雙指針。 難點不在主邏輯,而在「不重複」的去重處理。

2

為什麼「排序 + 固定 + 對撞」// WHY

× REJECT · O(n³)
三層迴圈枚舉

試所有 (i,j,k)。n 到 3000 → 2.7×10¹⁰,超時;還要另用 set 去重,更慢。

△ REJECT · O(n²) 空間
固定 i + 雜湊找兩數

固定 i,內層用雜湊表找 -nums[i]-nums[j]。O(n²) 時間但用了雜湊、去重繁瑣,還吃額外空間。

SORT + TWO POINTERS · ADOPT ↓
● ADOPT · O(n²) / O(1)*
排序 → 固定 i → 對撞找兩數 → 去重

排序後固定 nums[i],在 [i+1, n-1] 用對撞雙指針找 nums[l]+nums[r] = -nums[i]:太小 l++、太大 r--、剛好收下。 排序讓去重變簡單:相同的值都相鄰,跳過相鄰重複即可。外層 O(n) × 內層 O(n) = O(n²),除輸出外只用常數空間。

* 不計答案輸出;排序可原地 O(1)(遞迴堆疊 O(log n))。

3

核心策略 · 四步驟// STRATEGY

  1. 排序 —— sort(nums),讓相同值相鄰、並賦予對撞所需的單調性。
  2. 固定 i(外層去重) —— i 從左掃;若 nums[i]==nums[i-1] 就 continue(同樣的 i 會產生同樣的組)。
  3. 內層對撞 —— l=i+1, r=n-1:sum<0→l++、sum>0→r--、sum==0 收下三元組。
  4. 命中後內層去重 —— l++、r-- 後,跳過與剛用過相同的 nums[l]、nums[r]。

去重的三個位置// THE DEDUP TRIO

排序把去重變成「跳過相鄰相同值」

排序後,同一個值的所有副本都連續相鄰。要避免重複三元組,只需在三個位置跳過相鄰重複:

① 外層 i:if (i>0 && nums[i]==nums[i-1]) continue; —— 同樣的 nums[i] 會生出同樣一批組,只需第一個 i 處理。
② 命中後的 l:while (l<r && nums[l]==nums[l-1]) l++; —— 跳過和剛收下那組相同的左值。
③ 命中後的 r:while (l<r && nums[r]==nums[r+1]) r--; —— 跳過相同的右值。

注意 ②③ 的判斷用 nums[l-1] / nums[r+1] —— 是「剛剛用過的那一格」,因為在此之前已經先 l++、r--。只在「找到解之後」才需要跳 l、r;沒找到時 l++/r-- 自然推進、不會產生重複。這三處合起來,保證答案集無重複,而且不必額外用 set。

複雜度// COMPLEXITY

T = O(n²)　　S = O(1)(不計輸出)

排序 O(n log n);主體是「外層 i 掃 n 次 × 內層對撞每次 O(n)」= O(n²),主導總時間。除答案陣列外只用兩個索引 → O(1) 額外空間。比 O(n³) 暴力快一個量級。

面向成本說明
排序O(n log n)一次,被 O(n²) 蓋過
外層 iO(n)固定每個起點
內層對撞O(n)l+r 合計走一趟
總時間 / 空間O(n²) / O(1)不計輸出

一句洞察// INSIGHT

kSum 降維:排序後固定一個數,把 3Sum 變成 2Sum → 對撞雙指針。排序同時讓去重只需「跳過相鄰相同值」(外層 i、命中後的 l、r 三處)。 這是 167 的直接應用,也是 4Sum、kSum 遞迴的範本。
