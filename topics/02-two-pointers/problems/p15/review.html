重點回顧// RECAP

關鍵一句話
降維3Sum = 固定一數 + 2Sum(對撞)
前置排序:給單調性 + 讓相同值相鄰
內層找 nums[l]+nums[r] = -nums[i]
去重①外層 nums[i]==nums[i-1] → continue
去重②③命中後跳 nums[l-1]、nums[r+1]
剪枝nums[i]>0 → break(可選)
複雜度O(n²) 時間 / O(1) 空間(不計輸出)

觀念題// CONCEPT Q

Q1

為什麼一定要先排序?排序在這題同時提供了哪兩個好處(對撞、去重)?

Q2

命中後跳重複,判斷為什麼用 nums[l-1] 和 nums[r+1],而不是 nums[l+1]、nums[r-1]?和「先 l++、r--」的順序有什麼關係?

Q3

為什麼只在找到解之後才跳 l/r?當 sum<0 做 l++ 時,不會產生重複三元組嗎?

Q4

外層去重寫成 if (nums[i]==nums[i+1]) continue;(比後一個)會怎樣?為什麼要比前一個 nums[i-1]?

Q5

整體是 O(n²)。排序的 O(n log n) 為什麼被吸收掉?內層對撞為什麼是 O(n) 而非 O(n²)?

範例 Trace// 全零的去重

// nums = [0,0,0,0]:唯一解 [0,0,0],不能重複 排序後 [0,0,0,0] i=0 (0): l=1,r=3 0+0+0=0 → 收下 [0,0,0]; l++,r-- 跳重複左: nums[2]==nums[1]=0 → l++ → l=3 此時 l==r(=3)→ 內層結束 i=1 (0): nums[1]==nums[0] → continue(外層去重) i=2,3: 同理 continue ans = [[0,0,0]](只有一組,去重成功)
全零是去重的極端測試:內層命中一次後,跳重複左值把 l 直接推到 r,避免再收 [0,0,0];外層去重再擋掉 i=1,2,3 這些重複的 0 起點。 兩層去重合力,才讓 [0,0,0,0] 只回一組。

// ANSWERS
A1

排序提供兩個關鍵好處。(1) 單調性 → 能對撞:排序後 nums[l] 是子陣列最小、nums[r] 最大,才能靠比較 sum 與目標決定「該縮哪端」(同 167 的消去論證)。(2) 相同值相鄰 → 去重簡單:所有重複值連成一段,只要「跳過相鄰相同值」就能避免重複三元組,不必用 set。沒排序,這兩件事都做不到。

A2

因為在跳重複之前已經先執行了 l++; r--;。此刻 nums[l-1] 是剛剛收進答案的那個左值,nums[r+1] 是剛用過的右值。我們要跳過的是「和剛用過的那組相同」的值,所以比 nums[l-1]/nums[r+1]。若比 nums[l+1] 就變成「和下一個比」,語意錯,會漏跳或多跳。順序是「先移動、再回頭比剛離開的格子」。

A3

因為重複三元組只可能在「收下一組解」時產生 —— 若下一步的 (nums[l], nums[r]) 和剛收的完全相同,就會重複。而 sum<0 做 l++、sum>0 做 r-- 時根本還沒收任何組,只是在逼近目標,不會生出重複。所以跳重複只需綁在「命中分支」內。把跳重複放在每次移動都做,是多餘且可能出錯的。

A4

比後一個 nums[i+1] 會錯過第一次出現:假設 [-1,-1,2],i=0 時 nums[0]==nums[1] 成立就 continue,結果連第一個 -1 都沒處理 → 漏解。正確是比前一個 nums[i-1]:讓每組重複值的第一個正常進內層計算,只跳過第二個之後的重複起點。「跳後面的重複、保留第一個」是所有排序去重的通則。

A5

排序是 O(n log n),主體是雙層迴圈 O(n²);因為 n log n 遠小於 n²,被吸收,總時間 O(n²)。內層對撞是 O(n) 而非 O(n²):對固定的 i,l 只增、r 只減,兩者相遇前最多走 n 步(不是巢狀掃描)。外層 n 個 i × 內層 O(n) = O(n²)。這正是「排序 + 對撞」把 O(n³) 暴力降到 O(n²) 的關鍵。

App 1

LC 18 4Sum:排序後再套一層 —— 外兩層固定 a、b(各自去重),內層對撞找 c+d = target-a-b。時間 O(n³)。去重在四個位置:兩個外層各一(比前一個)、命中後的 l 與 r 各一。要留意 target 可為任意值、且四數和可能溢位 int,用 long long 累加較安全。

App 2

LC 16:維護一個 best 記「目前最接近 target 的三數和」。對撞時:sum==target 直接回傳(最接近就是相等);sum<target → l++(想變大);sum>target → r--(想變小);每步用 abs(sum-target) 更新 best。不需要去重,因為只求一個數值、不列舉組合。移動方向的依據從「和 0 比」換成「和 target 比」,骨架不變。

App 3

kSum 遞迴:排序一次後,kSum(start, target, k) 固定一個數 nums[i](去重),遞迴呼叫 kSum(i+1, target-nums[i], k-1)。基底是 k==2 —— 退化成 2Sum 對撞雙指針(本題內層)。整體時間 O(n^{k-1})(3Sum 是 O(n²)、4Sum 是 O(n³))。所以 167(2Sum)是所有 kSum 的遞迴底層引擎,3Sum 只是 k=3 的一層。
