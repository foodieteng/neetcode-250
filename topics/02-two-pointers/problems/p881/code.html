class Solution {
public:
    int numRescueBoats(vector<int>& people, int limit) {
        int n = people.size();
        sort(people.begin(), people.end());       // 排序 → 才能最輕配最重
        int l = 0, r = n - 1, boats = 0;

        while (l <= r) {
            boats++;                                // 這一步一定開一艘船

            int sum = people[l] + people[r];

            if (sum > limit) {
                r--;                              // 載不下 → 最重的單獨
            } else {
                l++; r--;                        // 載得下 → 最輕 + 最重 同船
            }
        }

        return boats;
    }
};
迴圈條件 l <= r(含等號)—— 當 l == r 剩最後一個人時,仍要進迴圈給他一艘船。無論 sum 過不過 limit,結果都是「這一人一船」:過了走 r--、沒過走 l++, r--,兩者都讓 l > r 結束。每次迴圈 boats++:每艘船不是載兩人(l++, r--)就是載一人(r--),最重的 r 每步必上船。

為什麼只需看「最重能不能配最輕」// ONLY CHECK L

// 對最重的 r,只要判斷「能不能帶上最輕的 l」就決定了一切 people[l] + people[r] > limit : 最輕的都載不下 → r 配「任何還在的人」都超重(其他人 ≥ people[l]) → r 只能單獨,r-- people[l] + people[r] <= limit : 最輕的載得下 → 讓 r 帶走最輕的 l 最省(交換論證) → l++, r-- 因為 people[l] 是「還在的最輕」,它決定了 r 有沒有伴 —— 不必試其他人

關鍵:對最重的 r,只需要跟「還在的最輕者 l」比一次就夠。若連 l(最輕)都超重,那 r 配剩下任何人(都 ≥ l)必然超重,直接單獨 —— 不必逐一試。若 l 載得下,交換論證保證帶 l 最優。 所以每個 r 只做一次比較,配對決策 O(1),整體才線性(排序後)。
