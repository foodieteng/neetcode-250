題目簡介// THE QUESTION

QUESTION

people[i] 是第 i 人的體重。有無限艘船,每艘載重上限 limit、最多載 2 人(且兩人合重 ≤ limit)。求載完所有人的最少船數。 例:people=[3,2,2,1], limit=3 → 3({3}、{1,2}、{2})。

每艘船最多 2 人,所以策略是「盡量湊兩人一船」。核心貪心:排序後,讓最輕的人陪最重的人 —— 若最重的人連最輕的都載不下,他只能單獨一船;若載得下,兩人同船最省。 這是 對撞雙指針 的貪心變體:l 指最輕、r 指最重,往中間收。

2

為什麼「最輕配最重」// WHY

× 難且慢
枚舉所有配對

把人兩兩配對求最少船,是配對/匹配問題,枚舉指數級,對 5×10⁴ 人不可行。

△ 想錯方向
最重配次重

把重的人湊在一起 → 幾乎都超重、各自單獨,船數爆。應該讓重的人配最輕的,才騰得出空間。

SORT + TWO POINTERS · ADOPT ↓
● ADOPT · O(n log n) / O(1)
排序,l 最輕、r 最重,對撞

排序後 l=0, r=n-1。每艘船先讓最重的 r 上:若 people[l]+people[r] ≤ limit,最輕的 l 一起上(l++, r--);否則 r 單獨(r--)。每次迴圈 boats++。 排序 O(n log n) 主導,對撞 O(n)。貪心最優(下方證明)。

3

核心策略 · 三步驟// STRATEGY

  1. 排序 —— sort(people),l=0(最輕)、r=n-1(最重)。
  2. 每艘船 —— boats++;若 people[l]+people[r] ≤ limit → 兩人同船(l++, r--);否則最重單獨(r--)。
  3. 收斂 —— l ≤ r 期間重複。回傳 boats。

為什麼這個貪心是最優// EXCHANGE ARGUMENT

最重的人配「最輕的人」永遠不吃虧

最重的人 H 一定要上某艘船。他頂多再帶一個人。問:帶誰最好?答案是還在的最輕者 L。

情況一:H + L > limit —— 連最輕的都載不下,那 H 配任何人都超重,只能單獨。這一步無可爭議。
情況二:H + L ≤ limit —— 讓 H 帶 L。用交換論證:假設某個最優解裡 H 沒帶 L(帶了別人 X,或自己單獨,而 L 在別船)。把 L 換到 H 這艘來 —— L 是最輕的,H+L ≤ H+X ≤ limit 仍合法;而 L 原本那艘少了 L 只會更輕、更合法。換完船數不增。所以「讓 H 帶最輕的 L」至少和最優一樣好 → 貪心最優。

直覺:最輕的人最好安置(誰都塞得下),把他拿去陪最難安置的最重者,是「用最不占空間的人填最重者的空位」。若最重者連最輕者都容不下,那最輕者留給別人也一樣 —— 反正 H 得自己走。每一步都不浪費船的容量。

複雜度// COMPLEXITY

T = O(n log n)　　S = O(1)

排序 O(n log n) 是瓶頸;之後對撞掃一趟 O(n)(l 只增、r 只減,合計 n 步)。除排序外只用兩個索引 → O(1) 額外空間。

面向成本說明
排序O(n log n)瓶頸
對撞掃描O(n)l+r 合計走一趟
每步O(1)一次加法比較
空間O(1)兩個索引
7

一句洞察// INSIGHT

每船最多 2 人 → 排序後最輕配最重:配得下兩人同船、配不下最重者單獨。交換論證保證最優:最輕者最好安置,拿去填最重者的空位不吃虧。 這是 167 對撞的貪心表親,和 11 一樣「排序/有序 → 對撞 + 貪心」。O(n log n)。
