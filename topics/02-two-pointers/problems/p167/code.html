完整實作 · 對撞雙指針// O(n)/O(1)

p167.cpp C++17 
class Solution {
public:
    vector<int> twoSum(vector<int>& numbers, int target) {
        int n = numbers.size();
        int l = 0, r = n - 1;

        while (l < r) {
            int sum = numbers[l] + numbers[r];

            if (sum == target) {
                return {l + 1, r + 1};       // 1-indexed
            } else if (sum < target) {
                l++;                       // 太小 → 需要更大 → 進左
            } else {
                r--;                       // 太大 → 需要更小 → 縮右
            }
        }

        return {-1, -1};                 // 題目保證有解,不會走到
    }
};
三分支恰好對應「單調消去」:sum<target → numbers[l] 太小,跟誰配都不足 → 丟掉它(l++);sum>target → numbers[r] 太大,跟誰配都超 → 丟掉它(r--)。 迴圈條件 l<r 保證兩指針不重疊(不會把同一元素用兩次)。回傳 {l+1, r+1} 是因為題目要 1-indexed。

對照 · 無序的 Two Sum(LC 1)// WHY SORTED MATTERS

// 陣列「沒排序」時,對撞雙指針失效 → 改用雜湊表 O(n) 空間 vector<int> twoSum(vector<int>& nums, int target) { unordered_map<int, int> seen; // 值 → 索引 for (int i = 0; i < (int)nums.size(); i++) { int need = target - nums[i]; if (seen.count(need)) return {seen[need], i}; seen[nums[i]] = i; } return {}; } 無序 → 無「往哪邊移」的單調性 → 只能記下看過的值,用 O(n) 空間換 O(n) 時間

同一個「兩數之和」,有序 / 無序決定了武器:有序用對撞雙指針(O(1) 空間),無序用雜湊表(O(n) 空間)。 若題目給的是無序陣列、又要求常數空間,可以「先排序 O(n log n) 再對撞」—— 但排序會打亂原索引,若要回傳原位置就得先存 (值,原索引) 再排。這也是 3Sum 的做法:排序後固定一個數,剩下用對撞找兩數。

邊界與陷阱// PITFALLS

陷阱症狀正解
忘了 1-indexed 回傳 {l, r} → 差一 {l + 1, r + 1}
迴圈寫 l <= r l==r 時同一元素用兩次 while (l < r)
移錯方向 sum<target 卻 r-- → 越走越小,錯 不足進左 l++、太大縮右 r--
套雜湊表 可過,但用了 O(n) 空間,違反常數空間要求 善用已排序 → 對撞 O(1)
擔心 sum 溢位 本題值 ≤ 1000 不會溢;大值題要留意 必要時用 long long

細節走讀// WALKTHROUGH

// numbers = [1,3,4,5,7,11], target = 9 l=0,r=5: 1+11=12 > 9 → 太大 → r-- (消去 11) l=0,r=4: 1+ 7= 8 < 9 → 不足 → l++ (消去 1) l=1,r=4: 3+ 7=10 > 9 → 太大 → r-- (消去 7) l=1,r=3: 3+ 5= 8 < 9 → 不足 → l++ (消去 3) l=2,r=3: 4+ 5= 9 == 9 → 回傳 {l+1, r+1} = {3, 4} [3, 4](numbers[3-1]=4、numbers[4-1]=5,和為 9)
注意窗口 [l, r] 單調縮小、從不回頭:每一步不是 l++ 就是 r--,被消去的元素永遠不再考慮。 唯一的解 (4,5) 全程被夾在窗內,直到兩指針正好停在它身上 —— 這就是概念頁「安全消去」證明的實際演出。
