對撞雙指針不在意數值正負,只靠「已排序 → 兩端是當前最小/最大」這個單調性。 不管全正、全負、含零,只要非遞減,sum 與 target 的大小關係就能可靠地指出「該縮哪一端」。

應用題// APPLY

App 1

LC 15 3Sum:找三個數和為 0。怎麼把它化約成「排序 + 固定一個數 + 對撞找兩數」?去重要注意什麼?

App 2

無序但要常數空間:給的是無序陣列,又要 O(1) 空間。可以先排序再對撞嗎?代價是什麼?若還要回傳原始索引,會遇到什麼問題、怎麼解?

App 3

回傳「有幾對」而非一對:排序陣列中,和 < target 的數對總數怎麼用同一副對撞指針 O(n) 算出?(提示:sum<target 時 l 與 [l+1, r] 全部都成立)

// ANSWERS
A1

因為已排序,numbers[r] 是目前窗 [l, r] 內的最大值。要讓 numbers[r] 成為答案,它得配一個窗內元素使和為 target;但它能配的最小夥伴是 numbers[l],而此刻連 numbers[l]+numbers[r] 都已 > target。換言之 numbers[r] 和窗內任何元素相加都 > target → 它絕不可能是解的一員。既然如此,把它整個排除(r--)不會漏掉任何解。這就是「安全消去」。

A2

唯一前提是已排序(單調)。對撞的每一步都靠「numbers[l] 是窗內最小、numbers[r] 是窗內最大」來判斷該縮哪端;這個「最小/最大」的保證來自排序。若沒排序,sum > target 時 numbers[r] 未必是最大,右邊可能還有更小的值能湊出解 —— 貿然 r-- 就會漏解。所以無序時對撞的「消去」不再安全,只能改用雜湊表。

A3

題目要求 i < j、同一元素不能用兩次。while (l < r) 保證兩指針始終指向不同位置。若寫 l <= r,當 l == r 時會用 numbers[l]+numbers[l](同一元素加自己)去比對 target,可能造出一個非法的「解」。所以嚴格小於 < 對應「兩個相異索引」這個約束。

A4

LC 1(無序):時間 O(n)、空間 O(n),用雜湊表邊掃邊記「看過的值 → 索引」,找 target - x。本題(有序):時間 O(n)、空間 O(1),用對撞雙指針。省掉雜湊表的關鍵就是「已排序」—— 排序提供了單調性,讓我們能靠比較 sum 與 target 決定移動方向,不必記住看過的值。一句話:有序把「查找」換成了「往哪走」。

A5

因為 l 只會增加、r 只會減少,每一次迴圈至少讓其中一個往中間移一步 → r - l 每步至少減 1。從 r - l = n - 1 縮到兩指針相遇,最多 n - 1 步,每步 O(1) → 全程 O(n)。它不是巢狀掃描(那才 O(n²)),而是「兩個指針各單向走一趟」的線性過程。

App 1

LC 15:先排序。外層固定第一個數 nums[i],問題就變成「在 nums[i+1..n-1] 這段有序子陣列裡,找兩數和為 -nums[i]」—— 正是本題的對撞雙指針。整體 O(n²)。去重要三處小心:外層 i 若和前一個相同就跳過;找到一組解後,l、r 要跳過相同值再繼續,避免重複三元組。核心引擎就是 167。

App 2

可以「先排序 O(n log n) 再對撞 O(n)」,總時間 O(n log n)、空間 O(1)(若允許原地排序)。代價:破壞了原始順序。若題目要回傳原始索引,排序後索引就亂了 —— 解法是先把 (值, 原索引) 一起打包成 pair 陣列再排序,對撞找到後回傳存下的原索引。這時空間又回到 O(n)(存 pair),所以「常數空間 + 回傳原索引 + 無序」三者不可兼得,通常直接用雜湊表最省事。

App 3

數對計數:排序後 l=0, r=n-1。若 numbers[l]+numbers[r] < target,那麼 numbers[l] 配上 [l+1, r] 之間任何一個都 < target(因為它們都 ≤ numbers[r])→ 一次加上 r - l 對,然後 l++;否則 r--。這樣每步 O(1)、總 O(n) 就數完所有 < target 的數對。這是對撞指針的另一個經典變體 —— 不只找一對,而是一次結算一整排。
