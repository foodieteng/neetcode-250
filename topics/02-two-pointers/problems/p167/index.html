題目簡介// THE QUESTION

QUESTION

給一個非遞減排序的 1-indexed 陣列 numbers 和 target,找兩個數 numbers[i]+numbers[j]=target(i<j),回傳 [i, j](1-indexed)。 保證恰有一組解,同一元素不可用兩次,且只能用常數額外空間。例:[2,7,11,15], target=9 → [1,2]。

這是對撞雙指針的代表題。Two Sum(LC 1) 因為無序只能用雜湊表(O(n) 空間);這題多了「已排序」這個條件, 就能改用一頭一尾兩指針對撞,把空間壓到 O(1) —— 排序把「查找」變成了「往哪邊移」的單調決策。

存看過的值找 target-x。O(n) 時間但用了 O(n) 額外空間 —— 題目明令常數空間,而且浪費了已排序這個資訊。

TWO POINTERS · ADOPT ↓
● ADOPT · O(n) / O(1)
l 頭、r 尾,依 sum 對撞收縮

l=0、r=n-1,算 sum = numbers[l]+numbers[r]:==target 就回傳;<target 說明左端太小 → l++;>target 說明右端太大 → r--。 每次移動安全消去一整排不可能的配對,兩指針最多各走 n 步 → O(n) 時間、O(1) 空間。這正是「排序 → 對撞」的威力。

3

核心策略 · 三步驟// STRATEGY

  1. 兩端起手 —— l=0、r=n-1,窗涵蓋整個陣列。
  2. 比 sum —— sum=numbers[l]+numbers[r]:等於 target → 回傳 {l+1, r+1}。
  3. 單調收縮 —— sum<target → l++(需要更大);sum>target → r--(需要更小)。l<r 期間重複。

為什麼移動指針不會漏解// THE ELIMINATION PROOF

每次移動 = 安全消去一整排不可能的配對

假設現在 sum = numbers[l]+numbers[r] > target。因為已排序,numbers[r] 是目前窗內最大。它和窗內任何元素 numbers[l..r-1] 相加,都 ≥ numbers[l]+numbers[r] > target —— 也就是說 numbers[r] 跟誰配都超過 target。既然它不可能是答案的一員,就能整個消去 r(r--),不會漏掉任何解。

對稱地,sum < target 時 numbers[l] 是窗內最小,和任何元素相加都 < target → 跟誰配都不足 → 安全消去 l(l++)。

每一步都排除掉「一個確定不在答案裡的元素」,而唯一的解始終被夾在 [l, r] 之間。窗口單調縮小,最多 n 步後 l、r 必定落在那組解上 —— 這就是對撞雙指針正確且 O(n) 的理由。「已排序」是這個消去論證成立的唯一前提。

複雜度// COMPLEXITY

T = O(n)　　S = O(1)

l 只增、r 只減,兩者之間的距離每步至少縮 1,合計最多 n 步。只用兩個索引 → O(1) 空間,滿足題目「常數額外空間」的硬性要求。

面向成本說明
掃描O(n)l+r 合計最多走 n 步
每步O(1)一次加法 + 一次比較
空間O(1)兩個索引,無雜湊表
vs LC 1O(n) 空間無序只能雜湊表

一句洞察// INSIGHT

已排序 → 對撞雙指針:一頭一尾,sum 太大縮右、不足進左。每次移動都「安全消去一整排不可能的配對」,所以 O(n)/O(1)。 這是 344/125/680 那一族「對撞型」的邏輯核心,也是 3Sum 的內層引擎。
