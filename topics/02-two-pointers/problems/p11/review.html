重點回顧// RECAP

關鍵一句話
面積公式min(兩牆高) × 寬度
起手l、r 從兩端(最寬)
貪心每步移動較矮的一端
為何安全丟掉的更窄且仍被矮牆封頂 → 不更優
平手移哪端都行(對稱)
易混≠ 接雨水(42);本題只兩壁
複雜度O(n) 時間 / O(1) 空間

觀念題// CONCEPT Q

Q1

為什麼面積用 min(height[l], height[r]) 而不是 max?從「水從哪溢出」解釋。

Q2

核心貪心是「移較矮牆」。證明:為什麼移它不會漏掉最優解?移較高牆會出什麼問題?

Q3

為什麼要先算面積、再移指針?順序反過來會漏掉什麼?

Q4

當 height[l] == height[r](平手),移左或移右有差嗎?為什麼?

Q5

本題和 接雨水(42) 都用雙指針、都跟「較矮側」有關,但問的東西不同。一句話說出兩者的根本差異。

範例 Trace// 平手時任移一端

// height = [4,3,2,1,4]:兩端等高 l=0,r=4: min(4,4)×4 = 16; 4<4? 否 → r-- maxArea=16 ← 最佳 l=0,r=3: min(4,1)×3 = 3 ; 否 → r-- maxArea=16 l=0,r=2: min(4,2)×2 = 4 ; 否 → r-- maxArea=16 l=0,r=1: min(4,3)×1 = 3 ; 否 → r-- maxArea=16 回傳 16;首步兩端等高 4、寬 4 就是最佳(平手歸 else 移右)
[4,3,2,1,4] 兩端都是 4、最寬,首步就是最優 16。平手時程式走 else 移右,但移左也會得到同樣答案 —— 因為兩端等高時,無論丟哪一端,被丟的那一端配上中間任何一條都更窄且高度 ≤ 4,不可能更好。平手兩端對稱,是這個貪心「移哪都安全」的特例。

要回傳是哪兩條線:除了最大面積,還要輸出兩線的索引。掃描時多記什麼?

// ANSWERS
A1

因為水會從較矮的那面牆溢出。容器能裝的水位頂多到「較矮牆的高度」;較高牆多出來的部分留不住水。所以有效高度是 min(height[l], height[r]),不是 max。面積 = 有效高度 × 兩牆間距 = min(...) × (r-l)。用 max 會高估 —— 那些超過矮牆的水根本裝不住。

A2

設 height[l] < height[r],移 l(l++)等於放棄「l 配 l+1..r 之間任一條」的所有容器。這些配對 (l, j):寬度 j-l < r-l(更窄)、高度 min(height[l], height[j]) ≤ height[l](仍被矮牆 l 封頂)→ 面積都 ≤ 當前面積(已算過),不可能更優,安全丟。反之移較高牆 r:新的 min 仍 ≤ height[l](矮牆沒動)、寬又縮 → 只會更差,而且可能把真正最優的 (l, 更左的高牆) 提早丟掉。所以必移較矮端。

A3

因為「當前這對 (l, r)」本身可能就是最優解(例如兩端最寬又夠高)。若先移再算,就跳過了 (l, r) 這對、直接算下一對,把它漏掉。正確順序是先用當前 (l, r) 算面積、更新 maxArea,再決定移哪端。[4,3,2,1,4] 就是首對即最優的例子,先算才抓得到。

A4

沒差,兩者都對。當 height[l] == height[r],兩端一樣高。無論丟哪一端,被丟的那端 e 配上中間任一條 j:寬度更窄、高度 min(height[e], height[j]) ≤ height[e](= 當前高度)→ 面積 ≤ 當前,不會更優。所以兩端對稱,丟哪個都不會錯過最優。程式碼把平手歸到 else(移右)只是實作選擇;寫成移左結果一樣。

A5

本題(11):只挑兩條線當容器的左右壁,中間的柱子不擋水,求那個單一最大矩形的面積。接雨水(42):整條是地形,每個位置頭上能積的水 = min(它左邊最高, 它右邊最高) − height[i],中間柱子會分隔水,求所有位置積水的總和。一句話:11 找「兩壁圍的最大單一矩形」,42 求「整片地形的積水總量」 —— 一個是 max 單體、一個是 sum 全部。

App 1

LC 42 雙指針:l=0, r=n-1,維護 leftMax、rightMax。每步比 height[l] 與 height[r]:較矮的那側可以確定它的水位 —— 若 height[l] < height[r],則位置 l 的水位由 leftMax 決定(因為右邊必有 ≥ height[r] > height[l] 的牆擋著),積水 += leftMax - height[l],l++;否則對稱處理右側。核心是「較矮側的水位已能確定,先結算它」——和本題「移較矮牆」形似但目的不同(42 是逐格累加積水,11 是找最大矩形)。

App 2

對拍:寫一個 O(n²) 暴力(雙迴圈試所有 (i,j) 取 min(h[i],h[j])×(j-i) 的最大),再用大量隨機小陣列比對雙指針解與暴力解是否一致(本卡的 C++ 已這樣驗過 30 萬筆、0 誤)。意義:貪心的正確性常常「看起來對但難證」,對拍能在幾秒內用實例把錯誤逼出來。面試臨場若對貪心沒把握,先講「我會用暴力對拍確認」是很好的工程直覺,也讓面試官看到你重視正確性。

App 3

多記最佳那對的索引:每次 curArea 刷新 maxArea 時,記下當前的 (l, r) 為 bestL, bestR。掃完回傳這兩個索引(和面積)。仍是 O(n)/O(1),只是把「最大值」升級成「最大值 + 產生它的一對」。這是所有「求極值」題想回溯出取得極值的具體選擇時的通用做法。
