完整實作 · 對撞貪心// O(n)/O(1)

p11.cpp C++17 
class Solution {
public:
    int maxArea(vector<int>& height) {
        int maxArea = 0, l = 0, r = height.size() - 1;

        while (l < r) {
            int curArea = min(height[l], height[r]) * (r - l);  // 較矮 × 寬
            maxArea = max(maxArea, curArea);

            if (height[l] < height[r]) {
                l++;                                   // 左牆較矮 → 移左
            } else {
                r--;                                   // 右牆較矮或平手 → 移右
            }
        }

        return maxArea;
    }
};
精髓在 if (height[l] < height[r]) l++ else r-- —— 移動較矮的那一端。移它才有機會讓「較矮牆」這個瓶頸變高;移較高牆則 min 不變、寬又縮,絕不可能更好。 平手(height[l] == height[r])歸到 else 移右,任移一端都對(對稱)。注意先算面積再移,才不會漏掉當前這對。

別搞混 · 和「接雨水」不同// VS TRAPPING RAIN

// 兩題長得像,但問的完全不同 本題 (11 盛水容器):只用「兩條」線當容器壁,面積 = min(兩線) × 寬 → 對撞找最大「單一矩形」 接雨水 (42 Trapping Rain Water):每個位置能積的水 = min(左最高, 右最高) - height[i] → 對所有位置的積水「加總」,牆之間的柱子會擋水 11 是「挑兩壁圍最大矩形」;42 是「所有凹槽積水總和」—— 別把兩者的解法互套

兩題都用雙指針、都跟「較矮的一側」有關,很容易混。本題(11)只選兩條線,中間的柱子不擋水(它們不是壁),求單一最大矩形。 接雨水(42)是地形,中間柱子會把水分隔,求每格積水的總和。42 也能雙指針,但邏輯是「較矮側的最高牆決定該格水位」,和這裡的貪心不一樣。

陷阱症狀正解
移動較高牆 面積不可能變大,還可能漏掉最優 永遠移較矮的那端
先移再算面積 漏掉當前這對的面積 先算面積、更新 max,再移
高度用成寬度 面積寫成 max(h)×寬 水由較矮側溢出 → min
套接雨水的解 把中間柱子當擋水牆 本題只有兩壁,中間不擋
l <= r 同一條線當兩壁,寬 0 while (l < r)
