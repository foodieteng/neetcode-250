題目簡介// THE QUESTION

QUESTION

height[i] 是第 i 條垂直線的高。任選兩條線與 x 軸圍成容器,求能裝最多水的面積。 面積 = 兩線較矮者 × 兩線間距 = min(height[i], height[j]) × (j - i)。例:[1,8,6,2,5,4,8,3,7] → 49。

面積由兩個因子決定:寬度(兩線間距)與高度(較矮的那條線,因為水會從矮的一側溢出)。 從最寬(兩端)開始對撞,寬度只會越來越小;所以想要更大面積,唯一的希望是「讓較矮的那面牆變高」—— 這就導出「移動較矮牆」的貪心。

● ADOPT · O(n) / O(1)
l、r 對撞,每步移動較矮的一端

l=0, r=n-1,每步算面積並更新最大;然後移動較矮的那一端(較高的留著,或許還能配到更高的另一邊)。 兩指針各走一趟、共 n 步 → O(n),O(1) 空間。移較矮牆是安全的貪心(下方證明)。

3

核心策略 · 三步驟// STRATEGY

  1. 兩端起手 —— l=0, r=n-1(最寬)。
  2. 算面積 —— area = min(height[l], height[r]) × (r-l),更新 maxArea。
  3. 移較矮牆 —— 若 height[l] < height[r] 則 l++,否則 r--。l<r 期間重複。

為什麼「移較矮牆」不會漏掉最優解// THE GREEDY PROOF

移較矮牆時,丟掉的那一整批配對都不可能更好

設此刻 height[l] < height[r](左牆較矮),當前面積 = height[l] × (r-l)。我們要 l++,等於永遠放棄「左牆 l 配上 l+1 .. r 之間任何一條」的所有容器。

看這些被放棄的配對 (l, j)(j < r):它們的寬度 j-l < r-l(更窄),而高度 min(height[l], height[j]) ≤ height[l](仍被較矮的 l 卡住,甚至更矮)。所以它們的面積都 ≤ height[l] × (r-l) —— 不會超過「我們剛剛已經算過」的當前面積。既然沒有一個能更好,安全丟掉。

對稱地,若右牆較矮就 r--。所以每一步都只排除「保證不更優」的一批,最優解始終留在 [l, r] 內。反之若移較高牆:min 仍被較矮牆封頂、寬度又縮,可能把真正的最優解丟掉。這就是為什麼一定移較矮那端。平手時移哪端都行(兩者對稱),程式碼歸到 else 移右。

6

複雜度// COMPLEXITY

T = O(n)　　S = O(1)

l 只增、r 只減,兩者相遇前合計最多走 n 步,每步 O(1) → O(n)。只用兩個索引 → O(1)。從 O(n²) 暴力降到 O(n),靠的是「移較矮牆」這個安全貪心一次砍掉一整排無用配對。

面向成本說明
掃描O(n)l+r 合計走一趟
每步O(1)算面積 + 比較 + 移一端
空間O(1)兩個索引
vs 暴力O(n²)枚舉每一對
7

一句洞察// INSIGHT

面積 = min(兩牆) × 寬。從最寬對撞、每步移較矮牆 —— 因為移它才可能讓瓶頸變高,而丟掉的那批(更窄且仍被矮牆封頂)保證不更優。 這是 167 對撞的貪心變體:不是二分消去,而是「瓶頸在哪就動哪」。O(n)。
