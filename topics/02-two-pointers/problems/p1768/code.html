完整實作 · 平行雙指針// O(m+n)

p1768.cpp C++17 
class Solution {
public:
    string mergeAlternately(string word1, string word2) {
        int m = word1.size(), n = word2.size();
        int l = 0, r = 0;
        string res = "";

        while (l < m && r < n) {          // 兩串都還有 → 交替取
            res = res + word1[l] + word2[r];
            l++; r++;
        }

        while (l < m) { res += word1.substr(l); break; }  // word1 有剩 → 接尾
        while (r < n) { res += word2.substr(r); break; }  // word2 有剩 → 接尾

        return res;
    }
};
主 while 交替各取一字元;結束時至多一方還有剩(不可能兩方都有剩)。那兩個 while … break 其實是「if」的寫法 —— substr(l) 一次把 word1 從 l 到底整段接上,break 讓它只做一次。改寫成 if (l < m) res += word1.substr(l); 語意更直白。

更簡潔 · 單迴圈 + reserve// TIDIER

// 一個索引跑到 max(m,n),各自越界就跳過;push_back 避免反覆配置 string mergeAlternately(string word1, string word2) { int m = word1.size(), n = word2.size(); string res; res.reserve(m + n); // 預留,避免多次重配置 for (int i = 0; i < max(m, n); i++) { if (i < m) res.push_back(word1[i]); // word1 還有就取 if (i < n) res.push_back(word2[i]); // word2 還有就取 } return res; } 同 O(m+n),但少了「接尾巴」的兩段,靠 i<m / i<n 自然處理長短不齊

單迴圈版把「交替」和「接尾巴」統一成 i < m / i < n 的條件 —— 較長的那串在後段自然只有它一個在追加。 reserve(m+n) + push_back 比反覆 res = res + … 更省(後者每次可能重新配置整個字串)。功能一樣,更短更穩。

邊界與陷阱// PITFALLS

陷阱症狀正解
只接一邊的尾巴 只處理 word1 剩餘 → word2 較長時漏字 兩個尾巴都檢查(至多一個非空)
順序取反 先取 word2 → 從 word2 開始,不符題意 先 word1[l] 再 word2[r]
用 res = res + … 大量拼接 反覆重配置,長字串偏慢 reserve + push_back / +=
誤以為兩尾都可能有 多寫互斥判斷 主迴圈到一方耗盡,只剩一方有尾
substr 越界 substr(l) 當 l==m 回空字串(安全) C++ substr(size()) 合法回 ""
