題目簡介// THE QUESTION

QUESTION

字串 s,把大寫轉小寫、移除所有非字母數字後,若正著讀和反著讀相同就是回文,回傳 true。 字母數字包含字母與數字。例:"A man, a plan, a canal: Panama" → true;"race a car" → false。

這是 344 反轉字串的雙生題:同一個「對撞雙指針」骨架,只是中間那步從交換換成比較。 再加兩個小處理:跳過非字母數字、忽略大小寫。不需要真的建出過濾後的字串,雙指針原地就能判斷 → O(1) 空間。

2

為什麼「原地雙指針」// WHY

△ WORKS · O(n) SPACE
先過濾成乾淨字串再判斷

掃一遍取出 alnum、轉小寫成新字串 t,再比 t == reverse(t)。對、好懂,但多用 O(n) 空間存 t。

△ WORKS · 兩趟
過濾後再跑雙指針

過濾一趟、雙指針一趟。省了反轉,但仍存了過濾字串,空間還是 O(n)。

IN-PLACE · ADOPT ↓
● ADOPT · O(1) SPACE
對撞雙指針,邊走邊跳過非 alnum

l 從頭、r 從尾。非字母數字就跳過(l++ 或 r--);兩邊都是 alnum 時比 tolower(s[l]) 與 tolower(s[r]): 不等 → false;相等 → 內收繼續。掃到 l ≥ r 沒失敗 → true。不建新字串,O(n) 時間、O(1) 空間。

3

核心策略 · 三步驟// STRATEGY

  1. 跳過非 alnum —— l 指的不是字母數字就 l++;r 指的不是就 r--。
  2. 忽略大小寫比較 —— 兩邊都 alnum 時,比 tolower(s[l]) 與 tolower(s[r]),不等就 return false。
  3. 內收 —— 相等則 l++、r--,重複到 l ≥ r;全程沒失敗回 true。

動畫演示// COMPARE + SKIP

"A b,a"(過濾成 "aba"):比 A↔a(忽略大小寫 ✓)、跳過空格與逗號、停在中間 'b'。全程相符 → true。

和「反轉字串」同一副骨架// SAME SKELETON

只換「中間那一步」:swap → compare(+ 跳過 + 小寫)

　// 反轉字串(344)
　while (l < r) { swap(s[l], s[r]); l++; r--; }

　// 回文檢查(125)
　while (l < r) {
　　if (!alnum(s[l])) { l++; continue; }
　　if (!alnum(s[r])) { r--; continue; }
　　if (lower(s[l]) != lower(s[r])) return false;
　　l++; r--;
　}

外殼完全相同:while (l < r) + 內收。差別只在中間 —— 反轉是交換兩端,回文是比較兩端,外加「跳過不關心的字元」與「大小寫正規化」。 學會 344 的骨架,125 只是換一個中間動作。這就是雙指針章「一副骨架、換內容」的威力(見 344 的模式討論)。

6

複雜度// COMPLEXITY

T = O(n)　　S = O(1)

每個字元最多被 l 或 r 經過一次(不是 alnum 就跳過、是就比較後內收),兩指針合計走 n 步。不建過濾字串,只用兩個索引 → O(1)。「先過濾成新字串」的寫法雖然直觀,但多花 O(n) 空間。

面向成本說明
掃描O(n)每字元至多碰一次
每步O(1)isalnum + tolower + 比較
額外空間O(1)l、r 兩索引,不建新字串
vs 過濾法O(n) 空間存過濾後字串

一句洞察// INSIGHT

回文 = 反轉骨架把「交換」換成「比較」+ 跳過非 alnum + 小寫。while (l < r) 兩端內收,不建過濾字串 → O(1) 空間。 不等即刻 false,撐到 l ≥ r 就是回文。骨架和 344 一模一樣。
