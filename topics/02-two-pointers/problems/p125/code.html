完整實作 · 雙指針 + 跳過// O(n)/O(1)

p125.cpp C++17 
class Solution {
public:
    bool isPalindrome(string s) {
        int l = 0, r = s.size() - 1;

        while (l < r) {
            if (!isalnum(s[l])) { l++; continue; }   // 左端非 alnum → 跳
            if (!isalnum(s[r])) { r--; continue; }   // 右端非 alnum → 跳

            if (tolower(s[l]) == tolower(s[r])) {
                l++; r--; continue;          // 相符 → 內收
            }
            return false;                          // 不符 → 不是回文
        }
        return true;
    }
};
三個 if 的順序是關鍵:先各自跳過非 alnum,兩邊都確定是 alnum 之後才比較。這樣 tolower 比的一定是「該比的字元」。 每個 continue 只動一個指針、重跑迴圈,直到兩端都對齊到 alnum。撐到 l ≥ r 都沒 return false → 回文。

對照寫法 · 先過濾再判斷// O(n) SPACE

// 直觀但多用 O(n) 空間:先建乾淨字串,再雙指針/反轉比較 bool isPalindrome(string s) { string t; for (char c : s) if (isalnum((unsigned char)c)) t += tolower((unsigned char)c); // 過濾 + 小寫 int l = 0, r = t.size() - 1; while (l < r) if (t[l++] != t[r--]) return false; // 已乾淨,直接比 return true; } 邏輯更好想(過濾後就是純字母數字),但存了 t → O(n) 空間;原地雙指針省下這塊

過濾法把「跳過 / 小寫」這些雜事先一次處理乾淨,剩下的比較很單純,適合快速寫對。原地雙指針把過濾融進雙指針的移動裡,不存中間字串 → O(1) 空間,是這題想考的進階寫法。
