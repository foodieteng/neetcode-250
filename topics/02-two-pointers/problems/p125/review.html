重點回顧// RECAP

關鍵一句話
骨架對撞雙指針 while (l < r),同 344
中間那步從「交換」換成「比較」
跳過非字母數字 → l++ / r--,continue

範例 Trace// 全標點 → true

// s = ".,"(過濾後是空字串,空字串算回文) l=0 r=1 : s[0]='.' 非 alnum → l++ l=1 r=1 l=1 r=1 : l<r? 否 → 停(從沒比較過任何字元) true
指針一路跳過所有非 alnum,l 追上 r 卻一次比較都沒做 —— 直接回 true。這正確:過濾後是空字串,空字串是回文。 這種「全跳過」的邊界,while (l < r) 天然涵蓋,不必特判空字串或全標點。

應用題// APPLY

App 1

LC 680 驗證回文 II:最多可刪一個字元使其成為回文。雙指針遇到不等時,怎麼用「試刪左 or 試刪右」各檢查一次?

App 2

LC 234 回文鏈結串列:單向鏈結串列沒有隨機存取,不能直接 l/r。O(1) 空間怎麼判回文?(提示:快慢指標找中點 + 反轉後半)

App 3

最長回文子字串(LC 5):找最長的回文子字串。「中心擴展」也是雙指針從中間往兩端擴,和本題往中間收方向相反。兩者關係是什麼?

// ANSWERS
A1

差在「中間那一步」:344 是 swap(s[l], s[r])(交換兩端),125 是 if (lower(s[l]) != lower(s[r])) return false(比較兩端),外加「非 alnum 就跳過」。外殼 while (l < r) + l++、r-- 完全一樣 —— 兩指針從兩端往中間收。所以說是同一副骨架:反轉「動手改」、回文「動眼看」。

A2

過濾法要先建一個只含小寫字母數字的新字串 t,那是 O(n) 額外空間。原地雙指針把「跳過非 alnum、轉小寫」融進指針移動與比較裡,不存任何中間字串,只用兩個索引 → O(1) 空間。兩者時間都 O(n),差在空間。這題(尤其面試)想看的就是 O(1) 的原地寫法。

A3

跳左、跳右必須在比較之前。順序是「先確定兩端都是 alnum,再比較」。若把比較放前面,可能拿非 alnum 字元(標點/空格)去比 —— 例如 "a,a" 會拿 ',' 和字母比而誤判。而跳左、跳右兩者之間的順序無所謂(先跳哪邊都行),因為每次只跳一個、continue 重來,最終都會把兩端都推到 alnum 才進比較。

A4

盲點是把「字母數字」誤記成「只有字母」。'0' 是數字,屬於 alnum,不跳過;'P' 是字母也不跳過。於是要比 tolower('0')='0' 與 tolower('P')='p' —— 數字和字母當然不等 → false。答錯的人通常以為 '0' 或大寫會被特別處理掉。記住:alnum = 字母 + 數字,兩者都參與比較。

A5

「跳過不關心的元素」是雙指針的通用配件。例:LC 345 只反轉母音(兩指針各自跳過非母音,只在母音處交換);合併兩個陣列時跳過某類元素;比較兩個字串是否相等但忽略某些字元(如 LC 844 退格字串比較,跳過被退格刪掉的字元)。核心:指針前進的條件裡加一句「不符就繼續走、不動作」,就能在掃描中內建過濾。

App 1

LC 680:一樣對撞雙指針。遇到 s[l] != s[r] 時,答案取決於「刪掉 l 或刪掉 r 之後,剩下的區間是不是回文」。所以寫一個輔助 isPali(l, r) 檢查子區間,主流程在第一個不等處回傳 isPali(l+1, r) || isPali(l, r-1)(試刪左端 or 試刪右端)。因為只允許刪一次,這個分叉只發生一次,總時間仍 O(n)。

App 2

LC 234:鏈結串列不能 r--(沒有反向指標)。O(1) 空間解法:快慢指標找中點(慢走一步、快走兩步,快到底時慢在中間)→ 反轉後半段 → 從頭和從反轉後的後半逐一比較。比完(可選)把後半反轉回去復原。核心思想仍是「兩端往中間比」,只是鏈結串列要先動手把後半掉頭,才能同向比較。

App 3

LC 5:本題是「已知是不是回文」→ 從兩端往中間收、驗證。最長回文子字串是「找回文」→ 從每個中心往兩端擴展,只要兩端相等就繼續擴。兩者是同一對稱關係的正反向:驗證是「收斂確認」,尋找是「擴展生成」。都靠「s[l] == s[r] 的對稱比較」,只是移動方向相反 —— 這也是雙指針靈活之處。
