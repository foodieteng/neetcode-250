// LC 27 移除所有 val=3:保留條件變成「num != val」 nums=[3,2,2,3,4], val=3 i=0 num=3: 3==3 → 跳過 num=2: 2!=3 → nums[0]=2, i=1 num=2: 2!=3 → nums[1]=2, i=2 num=3: 3==3 → 跳過 num=4: 4!=3 → nums[2]=4, i=3 回傳 i=3;前 3 個 [2,2,4] k=3, nums[0..2]=[2,2,4]
同一副「慢=寫入位、快=掃描」的骨架,把保留條件從「和上一個保留值不同」換成「不等於 val」,就從「去重」變「移除元素」。 這就是寫入指針的威力 —— 骨架固定,只換「哪些該寫」的判斷,一招通吃一族原地篩選題。

// ANSWERS
A1

比 nums[i-1](上一個已保留的唯一值)是因為:已排序 → 同一個值的所有重複都連續出現。所以「當前值是不是重複」等價於「它是否等於最近保留的那個值」。若比「原陣列前一個元素」,在覆寫發生後那個位置的值已被改動,語意會亂;而 nums[i-1] 永遠是「我上一次決定保留的值」,穩定可靠。已排序是這個簡潔判斷成立的前提 —— 沒排序就不能只比上一個。

A2

寫入位 i 只在「保留」時 +1,而快指針每步都 +1,所以 i 永遠 ≤ 快指針位置。當我們寫 nums[i] = num,i 這個位置≤ 快指針正在讀的位置 —— 也就是「已經讀過(或正在讀)的格子」。既然那格的資料已經被讀取/處理過,覆寫它不會丟失還沒讀的資訊。這和 88 的 i ≥ l 是同一類「寫入位不追上未讀讀取位」的位置論證。

A3

拿掉 i > 0:處理第一個元素時,num == nums[i-1] 會存取 nums[-1] → 越界(未定義行為)。加上 i > 0 保證只有「已經寫過至少一個」時才去比上一個保留值。第一個元素一定是唯一的開頭(前面沒有任何保留值可比),所以直接寫入即可。i > 0 && … 的短路正好處理這個開頭特例。

A4

只改判斷那行:if (i >= 2 && num == nums[i-2]) continue;。道理:要「每個值最多 2 個」,那麼一個 num 能不能再放,取決於往前數第 2 個保留值 nums[i-2] 是不是也等於它 —— 若相等,代表這個值已經放了 2 個(nums[i-1] 和 nums[i-2]),不能再放。若不等,還沒放滿 2 個,可以再放。保留 k 個就比 nums[i-k]。骨架不變,只換比較的偏移量。

A5

最大不同在指針的移動方式。對撞:兩指針相向而行(一頭一尾往中間),每步至少一個在動,靠「相遇」結束。快慢(本題):兩指針同向(都往右),快指針主導掃描、慢指針按條件落後前進,靠「快到底」結束。對撞處理「兩端對稱/配對」,快慢處理「原地篩選/覆寫」。同叫雙指針,一個是「兩端收斂」,一個是「一前一後同向」。

App 1

LC 283:寫入指針 i,快指針掃;保留條件是「非零」 —— if (num != 0) nums[i++] = num;。掃完後 [0, i) 是所有非零(保持原序),把 [i, n) 全填 0 即可(那些是被覆寫剩下的、要清成零)。或用「交換」版:遇非零就和 nums[i] 交換再 i++,省去補零。核心仍是「非零才保留 / 前移」的寫入指針。

App 2

沒排序就失效。本題靠「重複相鄰」才能只比「上一個保留值」;沒排序時,同一個值可能散在各處,比上一個保留值抓不到遠處的重複。此時要嘛先排序(O(n log n),但會改變相對順序),要嘛用雜湊集合記已見過的值(O(n) 時間、O(n) 空間,能保序)。「比上一個」是排序賦予的特權,無序就得付出空間或排序成本。

App 3

不完全一樣,是同名不同用。本題的「快慢」是陣列上的寫入指針:兩者都往右、每步快 +1、慢按條件 +1,重點是「慢當寫入位」。鏈結串列的「快慢指標」(Floyd)是速度不同:快每步走 2、慢走 1,用來找中點或偵測環(快追上慢 = 有環)。共通的抽象是「兩個同向指針、進度不同」,但一個用條件差(寫入指針)、一個用速度差(找環/中點)。兩種都算 fast-slow,場景不同。
