題目簡介// THE QUESTION

QUESTION

非遞減陣列 nums,原地去重,讓每個唯一元素只出現一次、保持相對順序。 回傳唯一元素個數 k;前 k 個要是排好序的唯一值,後面的可忽略。例:[0,0,1,1,2] → k=3, [0,1,2]。

前面幾題的雙指針是「兩端對撞」或「兩序列平行」;這題是第三種面貌:快慢指針(fast-slow / 同向)。 一個慢指針當「寫入位」(標記下一個要保留的位置),一個快指針掃描。慢指針只在遇到該保留的元素時才前進 —— 這是「原地篩選 / 覆寫」的萬用骨架。

2

為什麼「快慢寫入指針」// WHY

× REJECT · O(n) SPACE
另開陣列 / set 收唯一值

掃一遍把唯一值收進新陣列或 set 再寫回。對,但用了 O(n) 額外空間 —— 題目要原地。

△ 想太多
真的「刪除」元素(erase)

用 erase 一個個刪重複,每次搬移 O(n) → 最壞 O(n²)。其實不用刪,只要把唯一值覆寫到前面。

WRITE POINTER · ADOPT ↓
● ADOPT · O(n)/O(1)
慢=寫入位 i,快掃描,新值才寫

i 指「下一個唯一值要放的位置」(也是目前唯一數)。快指針逐一讀:若和上一個保留的 nums[i-1] 相同 → 跳過;不同 → nums[i++] = 這個值。 因為已排序,重複必相鄰,比對「上一個保留值」就夠。掃完 i 就是 k。原地 O(n)、O(1) 空間。

3

核心策略 · 三步驟// STRATEGY

  1. 寫入指針起點 —— i = 0(下一個唯一值要寫的位置 / 目前唯一數)。
  2. 快指針掃描 —— 對每個 num:若 i > 0 且 num == nums[i-1](和上一個保留值相同)→ 跳過。
  3. 寫新值 —— 否則 nums[i++] = num。掃完回傳 i(= 唯一元素個數 k)。

雙指針三大類 · 補齊快慢型// THE THIRD FLAVOR

對撞 · 平行 · 快慢(寫入指針)

‧ 對撞(一序列兩端往中間):344 反轉、125 回文、680 刪一字。
‧ 平行(兩序列各走一個):1768 交替合併、88 合併有序。
‧ 快慢 / 寫入指針(同一序列、同向、慢當寫入位):本題、去重、原地移除。

快慢型的心法:快指針決定「讀什麼」、慢指針決定「保留什麼、寫到哪」。慢只在「該保留」時前進,所以慢永遠 ≤ 快,覆寫的都是已讀過的位置,不會丟資料。 把「該保留」的條件換一下,同骨架就解一整族:去重(和上一個保留值不同)、移除某值(不等於 target,LC 27)、移動零(非零才寫,LC 283)、去重保留 2 個(和倒數第 2 個保留值不同,LC 80)。

複雜度// COMPLEXITY

T = O(n)　　S = O(1)

快指針掃一趟,每個元素常數處理;慢指針最多前進 n 次。原地覆寫,只用一個索引 → O(1)。比「erase 逐一刪」的 O(n²)、「另開陣列」的 O(n) 空間都好。

面向成本說明
掃描O(n)快指針走一趟
每步O(1)一次比較 + 可能一次寫
空間O(1)寫入指針 i,原地
vs eraseO(n²)逐一刪要搬移

一句洞察// INSIGHT

原地篩選/去重 → 快慢寫入指針:快掃描、慢當「下一個保留位」,只在該保留時 nums[i++]=值。 慢 ≤ 快 → 覆寫的都是已讀位置,不丟資料。換「保留條件」就解 27 / 283 / 80。這是雙指針的第三大類。
