完整實作 · 堆疊配對// O(n)

p20.cpp C++17 
class Solution {
public:
    bool isValid(string s) {
        stack<int> st;

        for (char c : s) {
            if (c == ')') {
                if (st.empty() || st.top() != '(') return false;
                st.pop();
            } else if (c == '}') {
                if (st.empty() || st.top() != '{') return false;
                st.pop();
            } else if (c == ']') {
                if (st.empty() || st.top() != '[') return false;
                st.pop();
            } else {
                st.push(c);                          // 左括號 → push
            }
        }

        return st.empty();                          // 剩左括號就不合法
    }
};
每個右括號都做同一件事:先確認堆疊非空、且頂端是對應的左括號,才 pop。st.empty() 擋「右括號太多」,top() != 對應 擋「類型/順序錯」。左括號一律 push。最後 return st.empty() 擋「左括號沒閉合完」。三個檢查、覆蓋全部不合法情況。

更精簡 · 用映射表去掉重複// PAIR MAP

// 三個 else-if 幾乎一樣 → 用「右→左」映射表合成一段 bool isValid(string s) { unordered_map<char, char> pair = {{')','('}, {']','['}, {'}','{'}}; stack<char> st; for (char c : s) { if (pair.count(c)) { // c 是右括號 if (st.empty() || st.top() != pair[c]) return false; st.pop(); } else { st.push(c); // 左括號 } } return st.empty(); } pair[c] = c 對應的左括號;一段邏輯處理三種右括號,不必寫三次

原版三個 else if 結構相同、只差括號字元。用一張 右括號 → 對應左括號 的映射表,就能把三段合成一段:pair.count(c) 判斷 c 是不是右括號,是就比 top() == pair[c]。更短、更好維護,加新括號種類只要改表。兩版都 O(n),邏輯完全一致。

邊界與陷阱// PITFALLS

陷阱症狀正解
忘了檢查空堆疊 右括號來時堆疊空,top() 越界(UB) 先 st.empty() 再 top()
只比數量 "([)]" 數量對卻順序錯 比對頂端類型,不是計數
忘了最後檢查空 "((" 沒閉合卻回 true return st.empty()
比錯方向 把右括號 push、拿左括號配 左 push、右來配頂端
短路順序寫反 top()!=x || empty() 先取 top → UB empty() || top()!=x(先判空)

細節走讀// WALKTHROUGH

// 合法 "([])" 與 不合法 "([)]" 對照 "([])" 合法: '(' push → [(] '[' push → [( [] ']' top='[' 匹配 pop → [(] ')' top='(' 匹配 pop → [] 掃完空 → true "([)]" 不合法: '(' push → [(] '[' push → [( [] ')' top='[' ≠ '(' → return false(順序錯,①擋下) 同樣的括號、不同順序,靠「比頂端類型」分辨合不合法
"([)]" 是最能說明「為什麼要堆疊」的反例:左右括號各兩個、數量完全正確,但交叉巢狀不合法。掃到 ) 時,還沒閉合的最近左括號是 [(在頂端)—— ) 對應的是 ( 不是 [,不匹配立刻判 false。只有堆疊「後進先出」能抓到這種順序錯,計數法抓不到。
