題目簡介// THE QUESTION

QUESTION

字串只含 ()[]{} 六種括號,判斷是否合法配對:每個左括號要被同類型的右括號閉合、且順序正確(正確巢狀)、每個右括號都有對應的左括號。 例:"()[]{}" → true;"(]" → false;"([)]" → false。

這是堆疊的招牌應用:括號匹配。關鍵觀察:合法時,最近出現、還沒閉合的左括號,一定最先被閉合(想像巢狀 ([]):內層的 [] 先閉,才輪到外層 ())。 這種「後進先出」正是堆疊(LIFO):左括號 push,右括號來就檢查堆疊頂端是不是它對應的左括號。

2

為什麼用堆疊// WHY

× 做不對
只數各括號的數量

數左右括號數量相等 → 不夠。"([)]" 數量都對卻順序錯。無法只靠計數判斷巢狀正確性。

△ 慢且繁
反覆刪除 "()" "[]" "{}"

一直把相鄰的成對括號消掉直到不能消。可行但每輪掃全串 → O(n²)。

STACK · ADOPT ↓
● ADOPT · O(n) / O(n)
左括號 push,右括號配頂端

掃一遍:遇左括號 → push;遇右括號 → 堆疊頂端必須是對應的左括號,是就 pop(配對成功),否則(堆疊空、或頂端不匹配)立刻回 false。 掃完後堆疊必須為空(沒有未閉合的左括號)才合法。一趟 O(n)。

3

核心策略 · 三步驟// STRATEGY

  1. 左括號 —— ( [ { 直接 push 進堆疊。
  2. 右括號 —— 若堆疊空 或 頂端不是對應左括號 → 回 false;否則 pop(配對掉一對)。
  3. 收尾 —— 掃完回傳 堆疊是否為空(空 = 全部配對 = 合法)。

三種不合法 · 對應三個檢查// THREE FAILURES

堆疊的三個檢查點,恰好擋住三種錯

合法的三個條件,對應程式的三個檢查:

① 類型錯("(]")—— 右括號來時頂端不是對應的左括號 → top() != 對應 擋下。
② 右括號多("())" 或 "]")—— 右括號來時堆疊是空的(沒有左括號可配)→ st.empty() 擋下。
③ 左括號多("(("、"([]")—— 掃完後堆疊還有殘留(左括號沒被閉合)→ 最後 return st.empty() 擋下。

順序錯的 "([)]" 也被 ① 擋:掃到 ) 時頂端是 [(不是 ()→ 不匹配。「檢查頂端 + 最後檢查空」這兩件事,合起來就完整覆蓋了「類型、順序、數量」三種合法性 —— 這正是堆疊優雅的地方:巢狀結構的正確性,被「後進先出」自動保證。

複雜度// COMPLEXITY

T = O(n)　　S = O(n)

掃字串一趟,每個字元一次 O(1) 的 push/pop/top。最壞情況(全是左括號 "(((((")堆疊裝到 n 個 → O(n) 空間。

一句洞察// INSIGHT

巢狀括號 → 最近的左括號最先閉合 → 堆疊(LIFO)。左括號 push、右括號配頂端 pop,最後堆疊要空。 「檢查頂端匹配」+「最後檢查空」自動涵蓋類型/順序/數量三種合法性。這是堆疊最經典的應用,O(n)。
