題目簡介// THE QUESTION

QUESTION

只用佇列(queue)的標準操作(尾端 push、前端 peek/pop、size、empty),實作一個堆疊(LIFO):push、pop、top、empty。 追問:能不能只用一個佇列?

矛盾點:佇列是先進先出(FIFO) —— 只能從前端拿;堆疊要後進先出(LIFO) —— 要拿最後加入的。 關鍵技巧:push 時,把新元素「旋轉」到佇列最前面 —— 加到尾端後,再把「舊的那些元素」一個個從前端搬到尾端繞一圈。於是佇列的 front 永遠是最後 push 的 = 堆疊 top,pop/top 就變成 O(1) 的前端操作。這正是追問「只用一個佇列」的答案。

2

為什麼「push 時旋轉」// WHY

△ 兩佇列
用兩個佇列搬來搬去

push 到 q2,把 q1 全倒進 q2,再交換名字 —— 效果一樣是「新元素在前」。可行,但要兩個佇列、程式較長。

× pop-heavy
push O(1)、pop 時才旋轉

push 直接進尾端;pop 時把前面 size-1 個搬到尾端、再取最後一個。也對,只是把成本挪到 pop。

ONE QUEUE · PUSH-HEAVY · ADOPT ↓
● ADOPT · push O(n) / 其餘 O(1)
單佇列,push 後旋轉到 front

push(x):先記下舊 size,q.push(x),再旋轉 size 次(q.push(q.front()); q.pop();)把舊元素繞到後面 → x 到最前。 pop/top 直接取 front(O(1))、empty 直接問。一個佇列、最短。

3

核心策略 · 四方法// STRATEGY

  1. push(x) —— sz = q.size();q.push(x);旋轉 sz 次:q.push(q.front()); q.pop();。
  2. pop() —— 取 q.front()、q.pop(),回傳它(O(1))。
  3. top() —— 回 q.front()(O(1))。
  4. empty() —— 回 q.empty()。

把 O(n) 成本放在哪一端// WHERE THE COST GOES

FIFO 與 LIFO 的落差,總得有一端付 O(n)

佇列只能「前端出、尾端進」,而堆疊要「最後進的先出」—— 兩者順序相反。要用佇列假裝堆疊,一定得在某處把順序「翻轉」一次,那次翻轉就是 O(n) 的旋轉,躲不掉。差別只在放在 push 還是 pop:

‧ push-heavy(本題):push 旋轉,讓 front 恆為 top → push O(n)、pop/top O(1)。
‧ pop-heavy:push 直接進尾端,pop 時才旋轉 size-1 個取最後 → push O(1)、pop/top O(n)。

選哪種看哪個操作比較頻繁:多 push 就用 pop-heavy、多 pop/top 就用 push-heavy。本題用 push-heavy,好處是 top() 這種只讀操作極快。「兩個 O(1) 資料結構湊出對方,必然在某個操作付出 O(n)」 —— 這是這類「用 A 實作 B」題的通則(反過來 用堆疊實作佇列(232) 也一樣,用兩個堆疊 + 攤還 O(1))。

複雜度// COMPLEXITY

push O(n)　pop / top / empty O(1)　S = O(n)

push 要旋轉當前 size 個元素 → O(n);pop/top/empty 只碰 front 或問狀態 → O(1)。一個佇列存 n 個元素 → O(n) 空間。

方法時間說明
pushO(n)旋轉 size 個到 front
pop / topO(1)直接取 front

空間O(n)一個佇列

一句洞察// INSIGHT

用 FIFO 假裝 LIFO:push 時把新元素旋轉到 front,front 就永遠是堆疊 top。順序翻轉的 O(n) 成本躲不掉,只能選放 push(本題)或 pop。 理解「兩個資料結構互相實作,總有一端付 O(n)」是這類題的核心。

// ONE-LINE TAKEAWAY
FIFO 假裝 LIFO:push(x) 後旋轉舊 size 個到後面 → 新元素到 front = top。pop/top 取 front O(1)、push O(n)。翻轉成本躲不掉,選放 push 或 pop。單佇列即可。
