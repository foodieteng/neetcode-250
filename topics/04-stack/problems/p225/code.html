完整實作 · 單佇列旋轉// push O(n)

p225.cpp C++17 
class MyStack {
    queue<int> q;
public:
    MyStack() {}

    void push(int x) {
        int sz = q.size();                    // 舊元素個數
        q.push(x);                          // x 先到尾端
        for (int i = 0; i < sz; i++) {       // 旋轉 sz 次
            q.push(q.front()); q.pop();       // 前端搬到尾端
        }                                     // → x 轉到最前(= top)
    }

    int pop() {
        int v = q.front(); q.pop();       // 取 front = top,O(1)
        return v;
    }

    int top() { return q.front(); }     // front = top,O(1)
    bool empty() { return q.empty(); }
};
push 的三行是靈魂:記舊 sz(在 push x 之前的大小)、把 x 加到尾端、再旋轉 sz 次把舊元素繞到後面。旋轉完 x 就在最前。之後 pop/top 只碰 front → O(1)。全部辛苦都集中在 push。用一個佇列就夠(這正是追問「只用一個佇列」的解)。

對照 · 兩佇列版(push-heavy)// TWO QUEUES

// 題目原本的「兩佇列」寫法:push 到 q2,把 q1 倒進去,再交換 queue<int> q1, q2; void push(int x) { q2.push(x); // 新元素先進 q2 while (!q1.empty()) { q2.push(q1.front()); q1.pop(); } // 把舊的全倒進 q2(接在後面) swap(q1, q2); // q1 現在是「新元素在前」的順序 } int pop() { int v = q1.front(); q1.pop(); return v; } int top() { return q1.front(); } bool empty(){ return q1.empty(); } 效果同單佇列:新元素排到最前;只是多用一個 q2 中轉,swap 後 q1 即結果

題目標題是「兩佇列」,標準寫法就是這版:新元素進 q2,把 q1 的舊元素全接在它後面,再 swap 讓 q1 變成結果。和單佇列邏輯完全一樣(都是讓新元素排最前),只是單佇列用「自己繞自己」省掉第二個佇列 —— 追問要的正是那個。另一個方向是 pop-heavy:push 直接進尾端(O(1)),pop 時把前 size-1 個搬走再取最後(O(n))。

邊界與陷阱// PITFALLS

陷阱症狀正解
先 push 再取 size 旋轉次數多一次 → x 又被繞走 先記 sz,再 push(x)
旋轉 sz+1 次 把 x 也繞到後面 → front 錯 旋轉恰好 sz 次(舊元素數)
用非標準操作 直接 q.back() 取尾 → 違反題意 只用 front/push/pop/size/empty
兩佇列忘了 swap 結果留在 q2、q1 空 → 全錯 倒完 swap(q1, q2)
誤以為要兩佇列 標題寫兩個,但一個就夠 單佇列旋轉即追問解

// push 1、push 2、push 3、pop、top(單佇列,front 在左) push(1): sz=0; [1] front=1 push(2): sz=1; [1,2] → 旋轉1次(1→後) [2,1] front=2 push(3): sz=2; [2,1,3] → 旋轉2次 [1,3,2][3,2,1] front=3 top() : front = 3 pop() : 取 front 3,[2,1] front=2 堆疊語意:push 順序 1,2,3 → top 是 3、pop 得 3 → 完全 LIFO
看 push(3):先 [2,1,3](3 在尾),旋轉 sz=2 次把 2、1 繞到後面 → [3,2,1],3 到最前。 每次 push 都這樣「把新元素頂到 front」,佇列就從插入順序的相反排列([3,2,1])—— front 是最後進的,正好是堆疊 top。FIFO 的容器,靠 push 旋轉,對外表現成 LIFO。
