題目簡介// THE QUESTION

QUESTION

一連串操作維護一份「紀錄」:整數 x → 記一筆分數 x;+ → 記「前兩筆之和」;D → 記「前一筆的兩倍」;C → 作廢前一筆。回傳所有分數總和。 例:["5","2","C","D","+"] → 30(紀錄變成 [5,10,15])。

這是第 04 章堆疊(Stack)的入門題。關鍵觀察:所有操作都只跟「最近 1~2 筆」互動 —— C 刪最後一筆、D 看最後一筆、+ 看最後兩筆。 這種「後進先出、只碰尾端」的存取模式,正是堆疊(LIFO)的定義。用一個堆疊當紀錄,照規則 push/pop,最後加總即可。

核心策略 · 四規則// STRATEGY

  1. 整數 x —— push(x)。
  2. D —— push(2 × top)(前一筆翻倍)。
  3. + —— push(top + 第二頂)(前兩筆之和)。
  4. C —— pop()(作廢前一筆)。掃完加總整個堆疊。

堆疊 = 只碰尾端的 LIFO// THE PRIMITIVE

後進先出 · 三個 O(1) 操作

堆疊(Stack)是「後進先出(LIFO)」的容器,只有三個核心操作,全是 O(1):push(x)(放到頂端)、pop()(移除頂端)、top()(看頂端)。像一疊盤子:只能從最上面拿放。

什麼題目適合堆疊?當你只需要跟「最近加入的那幾筆」互動、且後來的先被處理掉 —— 本題的 C/D/+ 全是這種。之後這一章會看到它更強的用法:括號匹配(20,最近的左括號先配對)、單調堆疊(739 下一個更大元素)、運算式求值。本題是最單純的「照規則 push/pop」暖身。

複雜度// COMPLEXITY

T = O(n)　　S = O(n)

掃一趟 n 個操作,每個是 O(1) 的 push/pop/top;最後加總堆疊 O(n)。堆疊最多存 n 筆 → O(n) 空間。

面向成本說明
掃描操作O(n)每個操作一次
push/pop/topO(1)只碰堆疊頂端
最後加總O(n)累加堆疊所有值
空間O(n)堆疊存紀錄
