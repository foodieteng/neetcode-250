完整實作 · stack<int>// O(n)

p682.cpp C++17 
class Solution {
public:
    int calPoints(vector<string>& operations) {
        stack<int> st;

        for (string op : operations) {
            if (op == "C") {
                st.pop();                         // 作廢前一筆
            } else if (op == "+") {
                int b = st.top(); st.pop();       // 取頂端 b
                int a = st.top();                 // 看第二頂 a
                st.push(b);                       // b 放回
                st.push(a + b);                   // push 前兩筆之和
            } else if (op == "D") {
                st.push(2 * st.top());            // 前一筆翻倍
            } else {
                st.push(stoi(op));              // 整數 → push
            }
        }

        int ans = 0;
        while (!st.empty()) { ans += st.top(); st.pop(); }  // 加總
        return ans;
    }
};
+ 需要看第二頂,但 std::stack 只能看 top() —— 所以先 pop 出 b、看新頂 a、再把 b 放回去、最後 push a+b。這個「彈出再放回」是用純堆疊看第二層的標準小技巧。其餘操作都只碰頂端,O(1)。

更乾淨 · vector 當堆疊// VECTOR AS STACK

// 用 vector 當堆疊 → 能直接索引第二頂,省掉「彈出再放回」 int calPoints(vector<string>& operations) { vector<int> st; for (auto& op : operations) { if (op == "C") st.pop_back(); else if (op == "D") st.push_back(2 * st.back()); else if (op == "+") st.push_back(st.back() + st[st.size() - 2]); // 直接取第二頂 else st.push_back(stoi(op)); } return accumulate(st.begin(), st.end(), 0); // 一行加總 } vector 支援 st[size-2],+ 不必彈出再放回;accumulate 一行求和,更簡潔

std::stack 只給 top(),看不到第二層,所以 + 要「彈出再放回」。改用 vector 當堆疊(push_back/pop_back/back() 就是 push/pop/top),還能直接索引 st[size-2] 拿第二頂,+ 一行寫完;最後 accumulate 一行加總。兩版都 O(n),vector 版更短更直觀,面試常用。

邊界與陷阱// PITFALLS

陷阱症狀正解
+ 只看一層 std::stack 看不到第二頂 → 拿不到 a 彈出再放回,或用 vector 索引
負數整數判斷 用 isdigit 判整數 → 漏掉負號 -2 先排除 C/D/+,其餘一律 stoi
把 D 當 + D 是兩倍(一筆),+ 是和(兩筆) D: 2×top;+: top+2nd
C 之後接著算 作廢後 top 已變 → 用舊值 pop 後 top 自動指向新的前一筆
忘了最後加總 回傳 top 或個數 累加整個堆疊

// operations = ["5","2","C","D","+"] "5": push 5 紀錄 [5] "2": push 2 紀錄 [5,2] "C": pop(移除 2) 紀錄 [5] "D": push 2×5 = 10 紀錄 [5,10] "+": push 5+10 = 15 紀錄 [5,10,15] 加總 = 5 + 10 + 15 = 30 回傳 30
看 "D" 那步:此時 top 是 5(因為 2 剛被 C 彈掉),所以 D 記的是 2×5=10 —— C 改變了「前一筆」是誰,後面的 D/+ 自動看到更新後的頂端。 這正是堆疊的好處:「前一筆」永遠等於 top(),不必自己追蹤是哪個位置。
