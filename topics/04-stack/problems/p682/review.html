範例 Trace// 含負數 + 連續 C

// ["5","-2","4","C","D","9","+","+"] → 27 "5": push 5 [5] "-2": push -2 [5,-2] "4": push 4 [5,-2,4] "C": pop(移除 4) [5,-2] "D": push 2×(-2) = -4 [5,-2,-4] "9": push 9 [5,-2,-4,9] "+": push -4+9 = 5 [5,-2,-4,9,5] "+": push 9+5 = 14 [5,-2,-4,9,5,14] 加總 = 5-2-4+9+5+14 = 27 回傳 27(負數、翻倍、連續 + 都照規則)
這個 case 蓋到負數(-2、D 出 -4)和連續 +(每個 + 都看當下的最後兩筆)。 注意 "-2" 是整數不是操作 —— 判斷「是不是整數」不能只看第一個字元是否為數字(負號會被誤判),要先排除 C/D/+,其餘一律當整數。

// ANSWERS
A1

四種操作都只碰紀錄的尾端(最近的):整數 push 在尾端加一筆;C 刪最後一筆;D 看最後一筆;+ 看最後兩筆。從來不需要存取「中間」或「最前面」的分數,也不需要按加入順序取出(那才是佇列)。這種「只在一端加入/移除、後進先出」正是堆疊(LIFO)的存取模式,所以堆疊天生契合,push/pop/top 都 O(1)。

A2

用純 std::stack:先把頂端 b 彈出,此時新頂端就是第二頂 a,讀到 a 後把 b 放回,再 push a+b(「彈出再放回」)。用 vector 當堆疊更方便:push_back/pop_back/back() 就是 push/pop/top,而且能直接索引 st[st.size()-2] 拿第二頂,+ 一行寫完,不必彈出再放回。所以需要「看第二層」時,vector 版更乾淨。

A3

最穩的判斷:先比對 C、D、+ 這三個固定字串,都不是就當整數(stoi)。若用 isdigit(op[0]) 判「是不是數字」,遇到負整數 "-2" 時第一個字元是 '-'、不是數字,會被誤判成非整數而漏掉。題目整數範圍含負數,所以要用「排除法」(不是那三個操作 → 整數),不要用「第一個字元是否為數字」。

A4

因為「前一筆」的定義就是「堆疊頂端 top()」,而 C 的 pop() 會讓 top 自動指向再前一筆。你不必用變數去追蹤「現在有效的最後一筆是第幾個位置」—— 堆疊的結構已經幫你維護了這個「最近」的語意。加一筆就 push、刪一筆就 pop,top 永遠是當下的最後一筆。用對資料結構,語意就免費對齊,這是堆疊在這題最舒服的地方。

A5

堆疊三核心,皆 O(1):push(x) 放到頂端、pop() 移除頂端、top() 看頂端(後進先出 LIFO)。直覺訊號:當題目出現「最近的、最後一個、往回找、配對、巢狀、後進先出」這類語言 —— 例如「最近未匹配的左括號」「往左第一個比我大的」「巢狀結構」—— 就該想到堆疊。本題的訊號是「作廢前一筆、看前一/兩筆」,全指向「最近」→ 堆疊。
